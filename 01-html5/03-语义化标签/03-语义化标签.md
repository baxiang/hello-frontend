# HTML5 语义化标签

## 学习目标
- 理解语义化的重要性
- 掌握 HTML5 新增的语义化标签
- 能够构建结构清晰、无障碍的网页

---

## 3.1 什么是语义化

### 概念

**HTML 语义化**是指使用恰当的 HTML 标签来表达内容的含义和结构，让代码对人类和机器都易于理解。

### 为什么要语义化？

```html
<!-- 非语义化 -->
<div id="header">
    <div class="nav">
        <div class="item"><a href="/">首页</a></div>
    </div>
</div>

<!-- 语义化 -->
<header>
    <nav>
        <ul>
            <li><a href="/">首页</a></li>
        </ul>
    </nav>
</header>
```

### 语义化的好处

1. **SEO 优化** - 搜索引擎更好地理解内容结构
2. **可访问性** - 屏幕阅读器准确朗读页面结构
3. **代码可读** - 开发者更容易理解和维护
4. **设备兼容** - 不同设备（手机、平板）更好解析

## 3.2 页面结构标签

### 经典布局结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>语义化页面</title>
</head>
<body>
    <header>
        <!-- 页眉/网站标题/主导航 -->
    </header>

    <nav>
        <!-- 主要导航 -->
    </nav>

    <main>
        <!-- 主要内容区域（每个页面只有一个） -->
        <article>
            <!-- 独立文章 -->
        </article>

        <section>
            <!-- 内容区块 -->
        </section>

        <aside>
            <!-- 侧边栏/相关内容 -->
        </aside>
    </main>

    <footer>
        <!-- 页脚 -->
    </footer>
</body>
</html>
```

### 语义化标签关系图

```
<body>
├── <header>          ← 页面头部
│   └── <nav>         ← 主导航
├── <main>            ← 主要内容（唯一）
│   ├── <article>     ← 独立文章
│   │   ├── <header>  ← 文章头部
│   │   ├── <section> ← 文章章节
│   │   └── <footer>  ← 文章底部
│   └── <aside>       ← 相关内容/侧边栏
└── <footer>          ← 页面底部
```

## 3.3 header 标签

### 页面头部

```html
<header>
    <!-- Logo -->
    <a href="/" class="logo">
        <img src="logo.png" alt="网站 Logo">
    </a>

    <!-- 主导航 -->
    <nav aria-label="主导航">
        <ul>
            <li><a href="/" aria-current="page">首页</a></li>
            <li><a href="/products">产品</a></li>
            <li><a href="/about">关于</a></li>
            <li><a href="/contact">联系</a></li>
        </ul>
    </nav>

    <!-- 搜索框 -->
    <form role="search">
        <input type="search" placeholder="搜索...">
        <button type="submit">搜索</button>
    </form>
</header>
```

### 文章/区块头部

```html
<article>
    <header>
        <h1>HTML5 语义化标签详解</h1>
        <p>
            作者：<a href="/authors/zhangsan">张三</a> |
            <time datetime="2024-01-15">2024 年 1 月 15 日</time> |
            阅读时间：5 分钟
        </p>
    </header>

    <!-- 文章内容 -->
</article>
```

### 注意事项

```html
<!-- 正确：header 内可以有多个元素 -->
<header>
    <h1>网站标题</h1>
    <p>网站副标题</p>
    <nav>...</nav>
</header>

<!-- 错误：header 内不应包含 footer、address -->
<header>
    <h1>标题</h1>
    <footer>页脚内容</footer>  <!-- ❌ -->
</header>

<!-- header 不是 sectioning 元素，不会创建新的大纲 -->
```

## 3.4 nav 标签

### 主导航

```html
<!-- 页面主导航 -->
<nav aria-label="主导航">
    <ul>
        <li><a href="/">首页</a></li>
        <li><a href="/products">产品</a></li>
        <li><a href="/about">关于</a></li>
    </ul>
</nav>

<!-- 面包屑导航 -->
<nav aria-label="面包屑">
    <ol>
        <li><a href="/">首页</a></li>
        <li><a href="/products">产品</a></li>
        <li>当前页面</li>
    </ol>
</nav>

<!-- 页脚导航 -->
<footer>
    <nav aria-label="页脚导航">
        <a href="/privacy">隐私政策</a>
        <a href="/terms">使用条款</a>
        <a href="/sitemap">网站地图</a>
    </nav>
</footer>
```

### 多个导航

```html
<!-- 当页面有多个 nav 时，使用 aria-label 区分 -->
<nav aria-label="主导航">
    <!-- 主导航链接 -->
</nav>

<aside>
    <nav aria-label="目录导航">
        <!-- 文章目录 -->
    </nav>
</aside>
```

### 注意

```html
<!-- 不是所有的链接组都要用 nav -->
<!-- nav 用于主要的导航区块 -->

<!-- ❌ 不推荐 -->
<nav class="social-links">
    <a href="#">微博</a>
    <a href="#">微信</a>
</nav>

<!-- ✅ 推荐 -->
<div class="social-links">
    <a href="#">微博</a>
    <a href="#">微信</a>
</div>
```

## 3.5 main 标签

### 主要内容区域

```html
<body>
    <header>...</header>

    <nav>...</nav>

    <main id="main-content">
        <!-- 页面的主要内容 -->
        <article>...</article>
    </main>

    <aside>...</aside>

    <footer>...</footer>
</body>
```

### 重要规则

```html
<!-- ✅ 正确：每个页面只有一个 main -->
<main>
    <article>...</article>
</main>

<!-- ❌ 错误：不要将 main 嵌套 -->
<main>
    <main>  <!-- 不允许 -->
        ...
    </main>
</main>

<!-- ❌ 错误：main 不应是其他元素的子元素 -->
<div>
    <main>  <!-- 不建议 -->
        ...
    </main>
</div>

<!-- ❌ 错误：main 内不应包含 header、nav、footer 等 -->
<main>
    <header>  <!-- 页面级的 header 不应该在这里 -->
        <nav> <!-- 主导航也不应该在这里 -->
        </nav>
    </header>
</main>
```

### 跳过导航

```html
<!-- 为无障碍访问提供"跳过导航"链接 -->
<a href="#main-content" class="skip-link">跳到主要内容</a>

<nav>
    <!-- 导航链接 -->
</nav>

<main id="main-content">
    <!-- 主要内容 -->
</main>

<style>
.skip-link {
    position: absolute;
    top: -40px;
    left: 0;
    padding: 8px;
    background: #000;
    color: #fff;
    z-index: 100;
}

.skip-link:focus {
    top: 0;
}
</style>
```

## 3.6 article 标签

### 独立文章

```html
<!-- 博客文章 -->
<article>
    <header>
        <h1>文章标题</h1>
        <time datetime="2024-01-15">2024 年 1 月 15 日</time>
    </header>

    <p>文章内容...</p>

    <footer>
        <p>标签：<a href="#">HTML</a> <a href="#">CSS</a></p>
    </footer>
</article>

<!-- 新闻文章 -->
<article>
    <h2>今日头条：前端技术大会召开</h2>
    <img src="news.jpg" alt="大会现场">
    <p>来自全国的前端开发者齐聚一堂...</p>
    <time datetime="2024-01-15">1 小时前</time>
</article>

<!-- 用户评论 -->
<article>
    <header>
        <img src="avatar.jpg" alt="用户头像">
        <span>张三</span>
        <time datetime="2024-01-15">2024 年 1 月 15 日</time>
    </header>
    <p>这篇文章写得很好！</p>
</article>
```

### 嵌套 article

```html
<!-- 文章和评论 -->
<article>
    <h1>主文章标题</h1>
    <p>主文章内容...</p>

    <section>
        <h2>评论</h2>

        <article>
            <p>评论 1 内容...</p>
        </article>

        <article>
            <p>评论 2 内容...</p>
        </article>
    </section>
</article>
```

### article vs section

```html
<!-- ✅ article: 内容独立，可单独分发 -->
<article>
    <h2>博客文章</h2>
    <p>可以单独发布的内容</p>
</article>

<!-- ✅ section: 内容的逻辑分组 -->
<section>
    <h2>章节标题</h2>
    <p>属于同一主题的内容</p>
</section>

<!-- 判断标准：内容是否可以独立存在 -->
<!-- article 可以单独拿出来，section 通常需要上下文 -->
```

## 3.7 section 标签

### 内容分节

```html
<!-- 文章章节 -->
<article>
    <h1>学习 TypeScript</h1>

    <section>
        <h2>第一章：基础类型</h2>
        <p>TypeScript 支持多种基础类型...</p>
    </section>

    <section>
        <h2>第二章：接口</h2>
        <p>接口是 TypeScript 的核心概念...</p>
    </section>

    <section>
        <h2>第三章：泛型</h2>
        <p>泛型提供了创建可复用组件的能力...</p>
    </section>
</article>
```

### 页面分节

```html
<main>
    <section aria-labelledby="products-heading">
        <h2 id="products-heading">我们的产品</h2>
        <p>产品介绍内容...</p>
        <a href="/products">查看更多</a>
    </section>

    <section aria-labelledby="features-heading">
        <h2 id="features-heading">产品特性</h2>
        <ul>
            <li>特性 1</li>
            <li>特性 2</li>
            <li>特性 3</li>
        </ul>
    </section>

    <section aria-labelledby="testimonials-heading">
        <h2 id="testimonials-heading">用户评价</h2>
        <!-- 用户评价内容 -->
    </section>
</main>
```

### 使用规则

```html
<!-- ✅ 正确：section 应该有标题 -->
<section>
    <h2>标题</h2>
    <p>内容</p>
</section>

<!-- ❌ 不推荐：没有标题的 section -->
<section>
    <p>一些内容</p>
</section>

<!-- ❌ 不推荐：仅为样式使用 section -->
<section class="wrapper">  <!-- 应该用 div -->
    <p>内容</p>
</section>

<!-- 记住：如果只是为了容器，用 div；如果要表达章节，用 section -->
```

## 3.8 aside 标签

### 侧边栏

```html
<div class="layout">
    <main>
        <article>...</article>
    </main>

    <aside>
        <!-- 相关内容 -->
        <nav aria-label="文章目录">
            <h3>目录</h3>
            <ul>
                <li><a href="#section1">第一章</a></li>
                <li><a href="#section2">第二章</a></li>
            </ul>
        </nav>

        <!-- 推荐内容 -->
        <section>
            <h3>推荐阅读</h3>
            <ul>
                <li><a href="#">相关文章 1</a></li>
                <li><a href="#">相关文章 2</a></li>
            </ul>
        </section>

        <!-- 广告 -->
        <div class="ad">
            <p>广告内容</p>
        </div>
    </aside>
</div>
```

### 行内 aside

```html
<article>
    <h1>前端发展史</h1>

    <p>2009 年，Node.js 发布，JavaScript 开始走向后端。</p>

    <aside>
        <p><strong>小知识：</strong>Node.js 的创始人是 Ryan Dahl，他在 2009 年的 JSConf EU 上首次展示了 Node.js。</p>
    </aside>

    <p>这一事件彻底改变了前端开发的格局。</p>
</article>
```

### 相关文章组

```html
<article>
    <h1>React Hooks 详解</h1>
    <p>正文内容...</p>

    <aside>
        <h3>相关文章</h3>
        <ul>
            <li><a href="#">useState 完全指南</a></li>
            <li><a href="#">useEffect 最佳实践</a></li>
            <li><a href="#">自定义 Hooks 教程</a></li>
        </ul>
    </aside>
</article>
```

## 3.10 其他语义化标签

### time 标签

```html
<!-- 机器可读的时间 -->
<time datetime="2024-01-15">2024 年 1 月 15 日</time>
<time datetime="2024-01-15T14:30">1 月 15 日下午 2:30</time>
<time datetime="2024-01-15">昨天</time>
<time datetime="P2H">2 小时</time>  <!-- 时间段 -->

<!-- 实际应用 -->
<article>
    <p>
        发布于 <time datetime="2024-01-15">2024 年 1 月 15 日</time>
        更新于 <time datetime="2024-01-16">1 月 16 日</time>
    </p>
    <p>阅读时间：<time datetime="PT5M">5 分钟</time></p>
</article>
```

### address 标签

```html
<!-- 联系信息 -->
<address>
    作者：<a href="mailto:zhangsan@example.com">张三</a><br>
    地址：北京市朝阳区 xxx 路<br>
    电话：<a href="tel:+8613800138000">138-0013-8000</a>
</address>

<!-- 在 article 中表示作者信息 -->
<article>
    <h1>文章标题</h1>
    <p>文章内容...</p>

    <address>
        作者：<a href="/authors/zhangsan">张三</a>
    </address>
</article>
```

### figure 和 figcaption

```html
<!-- 带说明的图片 -->
<figure>
    <img src="chart.png" alt="2024 年销售数据图表">
    <figcaption>图 1:2024 年各季度销售额对比</figcaption>
</figure>

<figure>
    <pre><code>function hello() {

```

### mark 标签

```html
<!-- 高亮显示 -->
<p>这是搜索结果的摘要，<mark>关键词</mark>会被高亮显示。</p>

    <h1><mark>React</mark> Hooks 教程</h1>
```

### dialog 标签

```html

</script>

<style>
dialog::backdrop {
    background: rgba(0, 0, 0, 0.5);
}
</style>
```

## 3.11 无障碍增强

### ARIA 角色

```html
<!-- 明确区域角色 -->
<header role="banner">...</header>
<nav role="navigation">...</nav>
<main role="main">...</main>
<article role="article">...</article>
<aside role="complementary">...</aside>
<footer role="contentinfo">...</footer>

<!-- 语义化标签通常不需要额外的 role -->
```

### ARIA 标签

```html
<!-- 描述复杂区域 -->
<section aria-labelledby="team-heading">
    <h2 id="team-heading">我们的团队</h2>
    <p aria-label="团队成员数量">共 50 人</p>
</section>

<!-- 动态内容通知 -->
<div role="status" aria-live="polite">
    <!-- 内容更新时会自动通知屏幕阅读器 -->
</div>

<div role="alert" aria-live="assertive">
    <!-- 重要警告，立即通知 -->
</div>
```

### 标题层级

```html
<!-- ✅ 正确：标题层级有序 -->
<h1>页面标题</h1>
  <h2>章节</h2>
    <h3>小节</h3>
    <h3>小节</h3>
  <h2>章节</h2>

<!-- ❌ 错误：跳级 -->
<h1>页面标题</h1>
<h3>直接跳到 h3</h3>  <!-- 不应该 -->
```

## 3.12 实战案例

### 完整博客页面

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
```

## 3.13 常见问题

### Q1: div 和 section 如何选择？

**答：** 如果只是为了样式或布局容器，用 div；如果内容属于同一主题的逻辑分组，用 section。

### Q2: article 和 section 的区别？

**答：**
- article：内容独立完整，可单独分发（如博客文章、新闻、评论）
- section：内容的章节分组，通常不能独立存在

### Q3: 什么时候用 aside？

**答：** 当内容与主要内容相关但可以独立存在时，如侧边栏、相关文章、广告等。

### Q4: 一个页面可以有多个 header 吗？

**答：** 可以。可以有页面级的 header，也可以有 article 或 section 的 header。
