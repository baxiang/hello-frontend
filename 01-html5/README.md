## 生活化比喻：HTML是什么？

**HTML 就像建筑的"骨架"**——定义网页的结构和内容。

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│  建筑骨架                    HTML 文档                │
│  ─────────                  ─────────                │
│  地基                        <!DOCTYPE html>         │
│  整体框架                    <html>                  │
│  设计图纸                    <head>（元信息）         │
│  室内空间                    <body>（可见内容）       │
│  房间                        <section>/<article>     │
│  墙壁                        <div>                   │
│  装饰品                      <span>                  │
└──────────────────────────────────────────────────────┘

- 没有骨架，建筑无法站立 → 没有 HTML，网页无法显示
- 骨架决定房间布局 → HTML 决定内容结构
- 骨架不负责美观 → HTML 不负责样式（那是 CSS）
```

## 📖 学习建议

### ✅ 应该做的

| 建议 | 原因 |
|------|------|
| 边学边写代码 | HTML 只有动手写才能记住 |
| 关注"为什么用这个标签" | 理解语义，不只是记忆 |
| 用浏览器查看效果 | 即时反馈，加深理解 |
| 对照规范检查代码 | 培养规范意识 |

### ❌ 应该避免的

| 避免 | 原因 |
|------|------|
| 只看不练 | 看懂≠会写，必须动手 |
| 全用 div | 失去语义化，不利于 SEO |
| 忽略 alt 属性 | 无障碍访问问题 |
| 跳级学习 | 基础不牢，后面更难 |

## 常见问题

### Q1：HTML 和 CSS、JavaScript 有什么区别？

**比喻：** HTML 是骨架，CSS 是装修（颜色、样式），JavaScript 是电器（交互功能）。

| 技术 | 作用 | 比喻 |
|------|------|------|
| HTML | 结构和内容 | 建筑骨架 |
| CSS | 样式和外观 | 装修设计 |
| JavaScript | 行为和交互 | 智能电器 |

### Q2：为什么 HTML5 叫"5"？

**答案：** HTML5 是 HTML 的第五个重大版本，增加了视频、音频、画布、语义化标签等新特性。

### Q3：学完 HTML 能做什么？

**答案：** 能做出网页的结构，但还需要 CSS（美化）和 JavaScript（交互）才能做出完整的网页。
