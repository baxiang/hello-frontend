## 1.1 npm 基础

### 常用命令

```bash
# 初始化项目
npm init -y

# 安装依赖
npm install lodash              # 生产依赖
npm install -D eslint           # 开发依赖
npm install -g http-server      # 全局安装

# 快捷命令
npm i lodash                    # install 缩写
npm add lodash                  # 同 install
npm remove lodash               # 卸载

# 查看依赖
npm list --depth=0              # 查看已安装的包
npm outdated                    # 查看过期的包
npm audit                       # 安全审计

# 脚本命令
npm run dev                     # 运行 dev 脚本
npm run build                   # 运行 build 脚本
npm test                        # 运行测试
npm run lint                    # 运行代码检查
```

## 1.2 package.json 配置

```json
{
  "name": "my-project",
  "version": "1.0.0",
  "description": "项目描述",
  "main": "src/index.js",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview",
    "lint": "eslint src --ext .js,.vue",
    "lint:fix": "eslint src --ext .js,.vue --fix",
    "test": "vitest",
    "test:coverage": "vitest --coverage"
  },
  "keywords": ["frontend", "vue"],
  "author": "张三",
  "license": "MIT",
  "dependencies": {
    "vue": "^3.3.0",
    "vue-router": "^4.2.0"
  },
  "devDependencies": {
    "@vitejs/plugin-vue": "^4.0.0",
    "eslint": "^8.40.0",
    "vite": "^4.3.0",
    "vitest": "^0.31.0"
  },
  "engines": {
    "node": ">=18.0.0",
    "npm": ">=9.0.0"
  },
  "browserslist": [
    "> 1%",
    "last 2 versions",
    "not dead"
  ]
}
```

### 版本号规则

```
^1.2.3  →  >=1.2.3 <2.0.0  （允许次版本更新）
~1.2.3  →  >=1.2.3 <1.3.0  （只允许补丁更新）
1.2.3   →  精确版本
*       →  任意版本
```

## 1.3 yarn 和 pnpm

### yarn 命令

```bash
# 安装
npm install -g yarn

# 常用命令
yarn add lodash
yarn add -D eslint
yarn remove lodash
yarn install
yarn run dev
```

### pnpm 命令

```bash
# 安装
npm install -g pnpm

# 常用命令
pnpm add lodash
pnpm add -D eslint
pnpm remove lodash
pnpm install
pnpm run dev
```

### 三者对比

| 特性 | npm | yarn | pnpm |
|------|-----|------|------|
| 安装速度 | 较慢 | 快 | 最快 |
| 磁盘空间 | 较大 | 较大 | 最小（硬链接） |
| 幽灵依赖 | 有 | 有 | 无 |
| 锁文件 | package-lock.json | yarn.lock | pnpm-lock.yaml |

## 1.4 常见问题

### Q1: 依赖安装失败怎么办？

```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install
```

### Q2: 如何查看包的版本？

```bash
npm view lodash versions
npm view lodash version
```

### Q3: 如何更新所有依赖？

```bash
# 查看过期依赖
npm outdated

# 更新到允许的最新版本
npm update

# 更新到最新版本（需要安装 npm-check-updates）
npx npm-check-updates -u
npm install
```
