## 第一阶段：入门基础（0-3 个月）

### 学习目标

达到 **L1 理解层** 能力：
- 理解语法结构和基本概念
- 能写出可运行的代码
- 能读懂基础文档

### 学习模块

```
第 1-4 周：HTML5
├── 01-基础语法     → 理解文档结构、常用标签
├── 02-表单详解     → 理解表单元素、基本验证
└── 03-语义化标签   → 理解语义化概念

第 5-8 周：CSS3
├── 01-基础语法     → 理解选择器、盒模型
├── 02-布局相关     → 理解 Flexbox、Grid（会用）
├── 03-动画与过渡   → 理解 transition、animation
└── 04-响应式设计   → 理解媒体查询

第 9-12 周：JavaScript 基础
├── 01-基础语法     → 理解变量、类型、函数
├── 02-DOM与BOM     → 理解 DOM 操作、事件绑定
└── 03-异步编程     → 理解 Promise、async/await（会用）
```

### 能力检验标准

| 模块 | L1 入门标准 | 检验方式 |
|------|-------------|---------|
| HTML5 | 能独立写出一个完整的页面结构 | 手写一个个人简介页面 |
| CSS3 | 能用 Flexbox/Grid 实现常见布局 | 实现一个导航栏 + 卡片布局 |
| JavaScript | 能实现简单的交互逻辑（点击、表单验证） | 实现一个计数器 + 表单验证 |

### 学习方法

**L1 层学习原则：**

```
┌──────────────────────────────────────────────────────┐
│  L1 学习流程                                          │
│                                                       │
│  1. 看语法结构图 → 理解"长什么样"                      │
│  2. 看最简示例 → 理解"最小用法"                        │
│  3. 看详细示例 → 理解"实际应用"                        │
│  4. 手写代码    → 验证"我真能写出来"                   │
│                                                       │
│  不要跳过：语法结构图是理解的关键                       │
└──────────────────────────────────────────────────────┘
```

**检验清单（每个概念）：**
- [ ] 我能画出/描述语法结构吗？
- [ ] 我能写出最简示例吗？
- [ ] 我能举出一个实际应用场景吗？

## 第二阶段：工程师能力（3-6 个月）

### 学习目标

达到 **L2 实践层** 能力：
- 能写出符合最佳实践的代码
- 能避免常见陷阱和反模式
- 能独立完成小型项目

### 学习模块

```
第 13-16 周：ES6+
├── 01-变量与作用域   → L2：避免 var陷阱，正确使用 let/const
├── 02-函数进阶       → L2：箭头函数适用场景
├── 03-对象与数组     → L2：解构最佳实践，Class 设计
├── 04-异步编程       → L2：Promise 链式调用最佳实践
└── 05-模块与元编程   → L2：模块化组织方式

第 17-20 周：TypeScript
├── 00-快速入门       → L1：理解类型系统概念
├── 01-类型基础       → L2：正确使用类型注解
├── 02-接口与对象     → L2：interface vs type 选择
├── 03-函数类型       → L2：函数签名最佳实践
├── 04-泛型编程       → L2：何时用泛型
└── 05-08 高级类型    → L2：实际场景应用

第 21-24 周：JavaScript 高级特性
├── 04-高级特性       → L2：闭包使用场景、避免内存泄漏
├── 06-作用域与闭包   → L2：正确理解作用域链
├── 07-对象详解       → L2：原型链理解、继承方式选择
└── 08-数组详解       → L2：数组方法选择（map vs forEach）
```

### 能力检验标准

| 模块 | L2 进阶标准 | 检验方式 |
|------|-------------|---------|
| ES6+ | 能识别反模式，写出符合规范的代码 | 代码审查：找出一段代码中的问题 |
| TypeScript | 能为复杂函数添加完整类型 | 为一个 API 请求函数添加类型 |
| JS 高级特性 | 能解释闭包、原型链的实际应用 | 实现一个简单的事件发射器 |

### 学习方法

**L2 层学习原则：**

```
┌──────────────────────────────────────────────────────┐
│  L2 学习流程                                          │
│                                                       │
│  1. 看最佳实践表格 → 知道"推荐做法"                    │
│  2. 看反模式对比   → 知道"为什么错误做法有问题"        │
│  3. 看适用场景     → 知道"何时用它"                    │
│  4. 对比自己的代码 → 找出差距                          │
│  5. 重写代码       → 应用最佳实践                      │
│                                                       │
│  核心：反模式对比让你从"会写"到"写得对"                │
└──────────────────────────────────────────────────────┘
```

**检验清单（每个核心概念）：**
- [ ] 我知道这个概念的最佳实践吗？
- [ ] 我能识别常见的反模式吗？
- [ ] 我知道何时用、何时不用吗？

## 第三阶段：技术专家（6-12 个月）

### 学习目标

达到 **L3 专家层** 能力：
- 理解底层原理和设计动机
- 能进行性能优化
- 能解决疑难问题
- 能做出技术选型决策

### 学习模块

```
第 25-28 周：Node.js + 工程化
├── 01-Node.js 基础      → L1-L2：会用、能写 API
├── 02-HTTP 与 Web 开发  → L2：Express 最佳实践
├── 03-数据库集成        → L2：数据库操作规范
├── 07-engineering       → L2-L3：构建原理、性能优化

第 29-36 周：框架学习（React 或 Vue）
├── 01-快速入门          → L1：理解框架概念
├── 02-核心概念          → L2：组件设计最佳实践
├── 03-Hooks/API 详解    → L2：正确使用 Hooks/Composition API
├── 04-组件模式          → L2：组件复用模式
├── 05-路由管理          → L2：路由设计
├── 06-状态管理          → L2-L3：状态管理原理、性能考量
├── 08-性能优化          → L3：虚拟 DOM、渲染优化
└── 10-实战项目          → 综合：完整项目开发
```

### 能力检验标准

| 模块 | L3 专家标准 | 检验方式 |
|------|-------------|---------|
| 工程化 | 能解释 Webpack/Vite 构建流程，能优化构建性能 | 分析一个项目的构建配置，提出优化方案 |
| 框架原理 | 能解释 Virtual DOM/响应式系统原理 | 手写一个简单的响应式系统 demo |
| 性能优化 | 能识别性能瓶颈，提出优化方案 | 分析一个慢页面，给出优化建议 |

### 学习方法

**L3 层学习原则：**

```
┌──────────────────────────────────────────────────────┐
│  L3 学习流程                                          │
│                                                       │
│  1. 看底层原理图 → 理解"它是怎么工作的"                │
│  2. 看设计动机   → 理解"为什么这样设计"                │
│  3. 看性能考量   → 理解"性能影响是什么"                │
│  4. 看知识关联   → 理解"它和什么相关"                  │
│  5. 动手验证     → 写代码演示原理                      │
```

## 2026年必备三大技术

> TypeScript + React/Vue + Next.js/Nuxt.js 是 2026 年前端开发的标配组合

### 技术重要性

```
┌─────────────────────────────────────────────────────────────────┐
│                   2026 前端技术栈金字塔                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                     ┌─────────────┐                             │
│                     │ Next.js     │  ⭐⭐⭐ 生产必备             │
│                     │ Nuxt.js     │  SSR/SSG、全栈开发           │
│                     └─────────────┘                             │
│                   ┌─────────────────┐                           │
│                   │  React / Vue    │  ⭐⭐⭐ 框架标配           │
│                   │                 │  组件开发核心              │
│                   └─────────────────┘                           │
│                 ┌─────────────────────┐                         │
│                 │   TypeScript        │  ⭐⭐⭐ 必须掌握         │
│                 │                     │  类型安全、项目标配      │
│                 └─────────────────────┘                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### TypeScript：项目标配

| 方面 | 说明 | 学习建议 |
|------|------|---------|
| **为什么必须学** | 2026年 90%+ 前端项目使用 TypeScript | 在学习 JavaScript 后立即学习 |
| **核心能力** | 类型注解、接口、泛型、类型推断 | 重点掌握类型思维 |
| **学习时机** | 第 17-20 周（L2 阶段） | 与 ES6+ 结合学习 |
| **检验标准** | 能为复杂项目添加完整类型定义 | 手写一个类型化的 API 模块 |

### React/Vue：框架选择

| 方面 | React | Vue |
|------|-------|-----|
| **特点** | 灵活、生态大、大型项目 | 简单、上手快、中小项目 |
| **学习建议** | 选择一个深入学习 | 两个都了解，精通一个 |
| **学习时机** | 第 29-36 周（L3 阶段） | 需要先掌握 TypeScript |
| **检验标准** | 能独立开发完整应用 | 包含路由、状态管理、性能优化 |

### Next.js/Nuxt.js：元框架（2026 新增重点）

| 方面 | Next.js (React) | Nuxt.js (Vue) |
|------|-----------------|---------------|
| **为什么必须学** | React 官方推荐，生产项目首选 | Vue 官方推荐，生产项目首选 |
| **核心能力** | SSR、SSG、App Router、API Routes | SSR、SSG、Auto-imports、Nitro |
| **学习时机** | 框架学习后（第五阶段） | ⭐ **新增 2-3 周专门学习** |
| **检验标准** | 能开发 SEO 优化的全栈应用 | 包含服务端渲染、部署 |

### 元框架学习重点

```
Next.js / Nuxt.js 学习路径：

第 1 周：渲染模式理解
├── SPA vs SSR vs SSG 区别
├── 为什么需要 SSR/SSG（SEO、首屏性能）
└── 何时选择哪种模式

第 2 周：核心功能
├── 文件路由系统（自动生成路由）
├── 数据获取方式（getServerSideProps 等）
├── API Routes / Nitro（后端 API）
└── 部署流程（Vercel、Netlify）

第 3 周：实战项目
├── 完整全栈应用开发
├── SEO 优化实践
├── 性能监控与优化
└── 真实项目部署

检验项目：开发一个博客/电商前台（含 SSR）
```

## 学习心态

### L1 阶段：接受"不会"

```
正常现象：
- 很多概念不理解原理 → 没关系，先学会用
- 代码写得很丑 → 没关系，能运行就行
- 不知道最佳实践 → 没关系，后面会学

不要：
- 追求完美 → 这是 L3 才有的能力
- 畏惧犯错 → 错误是学习的机会
```

### L2 阶段：接受"重构"

```
正常现象：
- 发现自己以前写的代码有问题 → 正常，这是进步
- 需要重写代码 → 正常，应用最佳实践
- 知道很多"不应该" → 正常，这是经验积累

核心心态：
- 对比反模式，找出差距
- 主动重构，应用最佳实践
```

### L3 阶段：接受"深挖"

```
正常现象：
- 需要看源码、原理图 → 正常，这是深入理解
- 需要做实验验证 → 正常，原理需要验证
- 需要长时间思考一个问题 → 正常，深入需要时间

核心心态：
- 不满足于"会用"
- 理解"为什么这样设计"
- 能解释给别人听
```

## 总结

三层内容架构规范服务于能力成长目标：

| 规范要素 | 服务于 |
|---------|--------|
| L1 语法结构图 + 代码示例 | 入门阶段：快速理解"长什么样" |
| L2 最佳实践 + 反模式对比 | 进阶阶段：从"会写"到"写得对" |
| L3 底层原理 + 性能考量 | 专家阶段：从"写得对"到"理解为什么对" |

**核心原则：循序渐进，不跳跃。**

每个概念先达到 L1，核心概念再达到 L2，关键概念最终达到 L3。
