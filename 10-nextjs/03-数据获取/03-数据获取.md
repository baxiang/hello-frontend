## 生活化比喻

**数据获取就像"餐厅的备菜系统"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                  餐厅备菜系统                         │
├──────────────────────────────────────────────────────┤
│                                                      │
│    默认缓存 = 预制菜                                  │
│    ─────────────                                     │
│    提前做好的菜放冰箱，客人点了直接端                │

│                                                      │
│    no-store = 现点现做                                │
│    ─────────────                                     │
│    每次客人点了才做，保证最新鲜                      │
│    cache: 'no-store' → 每次都重新请求               │
│    慢！但是实时的                                    │
│                                                      │
│    revalidate = 定时换菜                             │
│    ─────────────                                     │
│    预制菜放 60 秒，到期重新做                        │
│    revalidate: 60 → 60 秒后重新请求                 │
│    平衡了速度和新鲜度                                │
│                                                      │
│    Server Actions = 厨房直通电话                      │
│    ─────────────                                     │
│    客人不用跑到厨房，直接打电话下单                  │
│    表单提交 → 服务端直接处理                        │
│    不用额外写 API 路由                              │
│                                                      │
│    流式渲染 = 上菜顺序                                │
│    ─────────────                                     │
│    先上能上的菜（首屏），后上慢的菜                  │
│    Suspense → 慢的部分先显示加载状态                │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

### fetch 缓存策略

**最简示例（1-3行）：**

```tsx
export default async function Page() {
  const data = await fetch('https://api.example.com/data')
  const json = await data.json()
  return <pre>{JSON.stringify(json)}</pre>
}
```

**详细示例：**

```tsx
// 默认：缓存（类似 SSG）— 第一次请求后缓存结果
export default async function Page() {
  const data = await fetch('https://api.example.com/data')
  // 数据会被缓存，下次请求直接使用
  const json = await data.json()
  return <pre>{JSON.stringify(json)}</pre>
}

// 不缓存（类似 SSR）— 每次都重新请求
export default async function Page() {
  const data = await fetch('https://api.example.com/data', {
    cache: 'no-store'  // 每次都是最新的
  })
  const json = await data.json()
  return <pre>{JSON.stringify(json)}</pre>
}

// 定时重新验证（ISR）— 缓存但定期更新
export default async function Page() {
  const data = await fetch('https://api.example.com/data', {
    next: { revalidate: 60 }  // 60 秒后重新验证
  })
  const json = await data.json()
  return <pre>{JSON.stringify(json)}</pre>
}
```

### Server Actions

```tsx
// app/actions.ts
'use server'
export async function createPost(formData: FormData) {
  const title = formData.get('title') as string
  // 数据库操作...
  return { success: true }
}

// app/page.tsx
'use client'
import { createPost } from './actions'

export default function Page() {
  return (
    <form action={createPost}>
      <input name="title" />
      <button type="submit">提交</button>
    </form>
  )
}
```

## L2 实践层：用好

### 数据获取策略

| 策略 | 配置 | 适用场景 | 速度 | 实时性 |
|------|------|---------|:---:|:---:|
| 缓存（默认） | `fetch(url)` | 不经常变化的数据 | 快 | 低 |
| 不缓存 | `{ cache: 'no-store' }` | 实时数据（股票、天气） | 慢 | 高 |
| 定时更新 | `{ next: { revalidate: 60 } }` | 定期更新的内容 | 中 | 中 |

### 反模式：不要这样做

```tsx
// ❌ 错误：在 Client Component 中直接 fetch（失去 SSR 优势）
'use client'
export default function Page() {
  const [data, setData] = useState(null)
  useEffect(() => {
    fetch('/api/data').then(r => r.json()).then(setData)
  }, [])
  return <pre>{JSON.stringify(data)}</pre>
}

// ✅ 正确：在 Server Component 中 fetch
export default async function Page() {
  const data = await fetch('https://api.example.com/data')
  return <pre>{JSON.stringify(await data.json())}</pre>
}
```

```tsx
// ❌ 错误：所有数据都用 no-store（性能差）
const data = await fetch('/api/data', { cache: 'no-store' })

// ✅ 正确：根据数据特性选择策略
const posts = await fetch('/api/posts', { next: { revalidate: 60 } })  // 定时更新
const user = await fetch('/api/user', { cache: 'no-store' })           // 实时
```

### 适用场景

| 场景 | 推荐策略 | 原因 |
|------|---------|------|
| 博客文章 | 缓存或 revalidate | 内容不经常变化 |
| 用户信息 | no-store | 需要实时 |
| 商品列表 | revalidate: 60 | 定期更新 |
| 表单提交 | Server Actions | 无需额外 API |
| 长页面 | 流式渲染 | 首屏优先 |

## L3 专家层：深入

### 缓存工作原理

```
Next.js fetch 缓存流程：

首次请求 → fetch(url)
    ↓
响应存入 Data Cache
    ↓
后续请求 → 直接从缓存读取
    ↓
revalidate 到期 → 重新请求并更新缓存
    ↓
缓存未命中 → 重新请求
```

### 知识关联

```
数据获取关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  fetch 缓存 │────→│  Server     │────→│  流式渲染   │
│  cache/     │     │  Actions    │     │  Suspense/  │
│  revalidate │     │  'use       │     │  流式 SSR   │
│             │     │  server'    │     └─────────────┘
└─────────────┘     └─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **Data Cache** | Next.js 的数据缓存层 | `fetch(url)` 自动缓存 |
| **revalidate** | 定时重新验证缓存 | `{ next: { revalidate: 60 } }` |
| **Server Actions** | 服务端直接处理的函数 | `'use server'` |
| **no-store** | 不缓存，每次重新请求 | `{ cache: 'no-store' }` |
| **流式渲染** | 分块发送 HTML，首屏优先 | `Suspense` |

// 练习 1：实现带加载状态的 Server Action
'use server'
export async function submitForm(formData: FormData) {
  const title = formData.get('title') as string
  if (!title) return { error: '标题不能为空' }

  await new Promise(resolve => setTimeout(resolve, 1000))
  return { success: true, id: Date.now() }

  const posts = await fetch('https://api.example.com/posts', {

  }).then(r => r.json())

  return (
    <ul>
      {posts.map(post => (

      ))}
    </ul>
  )
}

## 常见问题

### Q1：缓存和 no-store 怎么选？

**内容不经常变用缓存，需要实时数据用 no-store：**

```tsx
// 博客文章 — 缓存
const posts = await fetch('/api/posts')

// 用户信息 — 不缓存
const user = await fetch('/api/user', { cache: 'no-store' })
```

### Q2：revalidate 和 no-store 有什么区别？

**revalidate 是定时更新缓存，no-store 是每次都不缓存：**

```tsx
fetch('/api/posts', { next: { revalidate: 60 } })  // 60 秒后更新
fetch('/api/user', { cache: 'no-store' })           // 每次都重新请求
```

### Q3：Server Actions 和 API Routes 选哪个？

**表单提交用 Server Actions（更简洁），外部 API 用 Route Handlers：**

```tsx
// Server Actions — 表单提交
'use server'
export async function createPost(formData: FormData) { }

// Route Handlers — 外部 API
export async function GET() {
  return Response.json({ posts: [] })
}
```
