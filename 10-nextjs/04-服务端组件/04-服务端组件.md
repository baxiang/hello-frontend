## 生活化比喻

**Server/Client 组件就像"前厅和后厨"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                  餐厅前厅与后厨                       │
├──────────────────────────────────────────────────────┤
│                                                      │
│    Server Component = 后厨                            │
│    ─────────────                                     │
│    在后厨做好菜直接端出来                            │
│    客人看不到后厨的操作                              │
│    可以直接用冰箱里的食材（数据库）                  │
│    但后厨不能有互动按钮（不能用 useState）           │
│                                                      │
│    Client Component = 前厅                            │
│    ─────────────                                     │
│    前厅可以和客人互动（useState、onClick）           │
│    但不能直接去冰箱拿食材（不能直接查数据库）        │
│    需要后厨把菜端过来（从 Server 传 props）          │
│                                                      │
│    'use client' = "这是前厅区域"的牌子               │
│    ─────────────                                     │
│    挂上这个牌子，这个组件变成前厅                    │
│    可以用所有 React Hooks                            │
│    但会增加客户端 JS 体积                            │
│                                                      │
│    Server → Client 传递 = 上菜                       │
│    ─────────────                                     │
│    后厨做好菜，端到前厅                              │
│    Server Component 把数据作为 props 传给 Client    │
│                                                      │
│    children 模式 = 前厅装饰                          │
│    ─────────────                                     │
│    后厨把菜放在前厅的展示台上                        │
│    Client Layout 包裹 Server 内容                   │
│    只有交互部分需要客户端 JS                        │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

### Server Component（默认）

**最简示例（1-3行）：**

```tsx
// Server Component（默认）
export default async function Page() {
  const data = await fetchData()
  return <ClientComponent data={data} />
}
```

**详细示例：**

```tsx
// Server Component — 可以 async，可以直接查询数据库
import db from './db'

export default async function Page() {
  const users = await db.user.findMany()
  return <UserList users={users} />
}
```

---

### Client Component

```tsx
// Client Component — 需要 'use client' 标记
'use client'
import { useState } from 'react'

export default function UserList({ users }: { users: { id: number; name: string }[] }) {
  const [filter, setFilter] = useState('')
  const filtered = users.filter(u => u.name.includes(filter))
  return (
    <div>
      <input value={filter} onChange={e => setFilter(e.target.value)} />
      {filtered.map(u => <div key={u.id}>{u.name}</div>)}
    </div>
  )
}
```

## L2 实践层：用好

### Server vs Client 组件对比

| 特性 | Server Component | Client Component |
|------|:---:|:---:|
| 服务端渲染 | ✅ | ✅ |
| 直接访问后端 | ✅ | ❌ |
| useState/useEffect | ❌ | ✅ |
| 事件处理（onClick） | ❌ | ✅ |
| 减少客户端 JS | ✅ | ❌ |
| 访问环境变量 | ✅（所有） | ❌（仅 NEXT_PUBLIC_） |

### 组件模式

```tsx
// 模式 1：Server 包裹 Client — 数据从服务端获取，客户端处理交互
// app/page.tsx (Server)
export default async function Page() {
  const data = await fetchData()
  return <ClientComponent initialData={data} />
}

// 模式 2：通过 children 传递 — 只有交互部分需要客户端 JS
// app/layout.tsx (Client)
'use client'
export default function Layout({ children }: { children: React.ReactNode }) {
  const [open, setOpen] = useState(false)
  return (
    <div>
      <nav>
        <button onClick={() => setOpen(!open)}>菜单</button>
      </nav>
      <main>{children}</main>  {/* Server Component 内容，不增加客户端 JS */}
    </div>
  )
}
```

### 反模式：不要这样做

```tsx
// ❌ 错误：整个页面都标 'use client'（失去 Server Component 优势）
'use client'
export default function Page() {
  const [data, setData] = useState(null)
  useEffect(() => { fetch('/api/data').then(r => r.json()).then(setData) }, [])
  return <div>{JSON.stringify(data)}</div>
}

// ✅ 正确：只在需要交互的组件用 'use client'
// page.tsx (Server)
export default async function Page() {
  const data = await fetchData()  // 服务端直接获取
  return <InteractivePart initialData={data} />
}

// interactive-part.tsx (Client)
'use client'
export default function InteractivePart({ initialData }: { initialData: Data }) {
  const [data, setData] = useState(initialData)
  return <button onClick={() => setData({ ...data })}>更新</button>
}
```

### 适用场景

| 场景 | 推荐 | 原因 |
|------|------|------|
| 数据获取 | Server | 直接查数据库，减少客户端请求 |
| SEO 页面 | Server | 服务端渲染 HTML |
| 表单交互 | Client | 需要 useState/useEffect |
| 动画效果 | Client | 需要客户端 JS |
| 布局框架 | Client | 需要交互（导航、侧边栏） |
| 页面内容 | Server | 减少客户端 JS 体积 |

## L3 专家层：深入

### 组件树渲染原理

```
Server → Client 边界：

Server Component (Page)
    ↓ 获取数据
Server Component (DataProcessor)
    ↓ 传递数据
Client Component (InteractivePart)  ← 'use client' 边界
    ↓ 只能传递可序列化的 props
Client Component (Button)

规则：
- Server 可以 import Client
- Client 不能 import Server（但可以通过 children 包裹）
- props 必须是可序列化的（不能传函数）
```

### 知识关联

```
组件类型关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Server     │────→│  数据传递   │────→│  Client     │
│  Component  │     │  props/     │     │  Component  │
│  数据获取   │     │  children   │     │  交互处理   │
└─────────────┘     └─────────────┘     └─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **Server Component** | 在服务端渲染的组件（默认） | `async function Page()` |
| **Client Component** | 在客户端渲染的组件 | `'use client'` |
| **RSC** | React Server Components | Next.js App Router 默认 |
| **Serialization** | 数据序列化传输 | Server → Client 只能传可序列化数据 |
| **组件边界** | Server 和 Client 的分界线 | `'use client'` 标记处 |

## 实践练习

```tsx
// 练习：实现 Server → Client 数据传递
// app/page.tsx (Server)
import db from '@/lib/db'
import ClientChart from './client-chart'

export default async function Page() {
  const sales = await db.sales.findMany()
  return (
    <div>
      <h1>销售数据</h1>
      <ClientChart data={sales} />
    </div>
  )
}

// app/client-chart.tsx (Client)
'use client'
import { useState } from 'react'

export default function ClientChart({ data }: { data: { month: string; revenue: number }[] }) {
  const [filter, setFilter] = useState('all')
  const filtered = filter === 'all' ? data : data.filter(d => d.month.includes(filter))
  return (
    <div>
      <input value={filter} onChange={e => setFilter(e.target.value)} placeholder="过滤月份" />
      <Chart data={filtered} />
    </div>
  )
}
```

## 常见问题

### Q1：什么时候用 Server，什么时候用 Client？

**默认用 Server，需要交互时才用 Client：**

```tsx
// Server — 数据获取、SEO、减少客户端 JS
export default async function Page() {
  const data = await fetchData()
  return <ClientPart data={data} />
}

// Client — 交互、动画、Hooks
'use client'
export default function ClientPart({ data }) {
  const [state, setState] = useState(data)
  return <button onClick={() => setState({ ...state })}>更新</button>
}
```

### Q2：Server Component 能用 useState 吗？

**不能。Server Component 在服务端渲染，不能使用 React Hooks。需要交互时用 Client Component。**

### Q3：Server Component 怎么传函数给 Client？

**不能直接传函数。用事件处理或 Server Actions 替代：**

```tsx
// ❌ 错误：Server 传函数给 Client
<Child onUpdate={() => { }} />  // 不能传函数

// ✅ 正确：用 Server Actions
'use server'
export async function updateData() { }

<Child action={updateData} />
```
