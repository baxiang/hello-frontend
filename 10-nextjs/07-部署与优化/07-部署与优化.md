## L1 理解层：会用

### 构建与部署

**最简示例（1-3行）：**

```bash
npm run build    # 构建
npm start        # 启动生产服务器
npx vercel       # 部署到 Vercel
```

**详细示例：**

```bash
# 本地构建测试
npm run build
npm start

# 部署到 Vercel
npm i -g vercel
vercel          # 预览部署
vercel --prod   # 生产部署
```

## L2 实践层：用好

### 部署选项对比

| 平台 | 说明 | 适合 | 成本 |
|------|------|------|------|
| **Vercel** | Next.js 官方平台，零配置 | 首选 | 免费额度充足 |
| **Netlify** | 支持 Next.js | 已有 Netlify 项目 | 免费额度充足 |
| **Docker** | 自托管 | 需要私有部署 | 服务器费用 |
| **Node.js** | 自建服务器 | 完全控制 | 服务器费用 |

### 性能优化

```tsx
// 图片优化 — 自动压缩、WebP、懒加载
import Image from 'next/image'
<Image
  src="/photo.jpg"
  width={800}
  height={600}
  alt="照片"
  priority  // 首屏图片优先加载
/>

// 字体优化 — 零布局偏移
import { Inter } from 'next/font/google'
const inter = Inter({ subsets: ['latin'], display: 'swap' })

export default function RootLayout({ children }) {
  return <html lang="zh" className={inter.className}><body>{children}</body></html>
}

// 动态元数据
export async function generateMetadata({ params }) {
  const post = await fetchPost(params.slug)
  return {
    title: post.title,
    description: post.excerpt
  }
}
```

### 反模式：不要这样做

```tsx
// ❌ 错误：用 img 标签（不优化）
<img src="/photo.jpg" alt="照片" />

// ✅ 正确：用 next/image（自动优化）
import Image from 'next/image'
<Image src="/photo.jpg" width={800} height={600} alt="照片" />
```

```tsx
// ❌ 错误：用外部字体链接（布局偏移）
<link href="https://fonts.googleapis.com/css2?family=Inter" rel="stylesheet" />

// ✅ 正确：用 next/font（零布局偏移）
import { Inter } from 'next/font/google'
const inter = Inter({ subsets: ['latin'] })
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 个人项目 | Vercel | 免费、零配置 |
| 企业项目 | Vercel Pro / Docker | 支持、控制 |
| 首屏图片 | `priority` | 预加载 |
| 非首屏图片 | 默认 | 懒加载 |
| SEO | metadata API | 静态 + 动态元数据 |

## L3 专家层：深入

### 性能优化原理

```
Next.js 优化流程：

npm run build
    ↓
分析依赖树
    ↓
代码分割（按路由）
    ↓
压缩 JS/CSS/HTML
    ↓
生成静态文件 + Server 组件 payload
    ↓
部署到 CDN
    ↓
用户访问 → CDN 就近返回
```

```
部署与优化关联：
```

## 常见问题

### Q1：Vercel 和 Netlify 选哪个？

**Next.js 项目首选 Vercel（官方支持、功能最全），已有 Netlify 项目可继续用：**

```bash
# Vercel
npx vercel --prod

# Netlify
npx netlify deploy --prod
```

### Q2：如何部署到 Docker？

```dockerfile
# Dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
EXPOSE 3000
CMD ["node", "server.js"]
```
