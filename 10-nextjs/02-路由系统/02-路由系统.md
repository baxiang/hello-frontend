## L1 理解层：会用

### 文件路由

**最简示例（1-3行）：**

```
app/page.tsx           → /
app/about/page.tsx     → /about
app/posts/page.tsx     → /posts
```

**详细示例：**

```tsx
// app/layout.tsx — 根布局（所有页面共享）
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="zh">
      <body>
        <nav><Link href="/">首页</Link></nav>
        {children}
      </body>
    </html>
  )
}

// app/posts/layout.tsx — 嵌套布局（仅 /posts/* 共享）
export default function PostsLayout({ children }: { children: React.ReactNode }) {
  return <div className="posts-layout">{children}</div>
}

// app/posts/[id]/page.tsx — 动态路由
import { notFound } from 'next/navigation'

export default async function PostPage({ params }: { params: Promise<{ id: string }> }) {
  const { id } = await params
  const post = await fetch(`https://api.example.com/posts/${id}`).then(r => r.json())
  if (!post) notFound()
  return <article><h1>{post.title}</h1><p>{post.content}</p></article>
}
```

### 特殊文件

| 文件 | 用途 |
|------|------|
| `page.tsx` | 页面组件（必需） |
| `layout.tsx` | 布局组件（共享 UI） |
| `loading.tsx` | 加载状态 |
| `error.tsx` | 错误边界 |
| `not-found.tsx` | 404 页面 |

```tsx
// app/posts/loading.tsx
export default function Loading() {
  return <div className="animate-pulse">加载中...</div>
}

// app/posts/error.tsx
'use client'
export default function Error({ error, reset }: { error: Error; reset: () => void }) {
  return (
    <div>
      <h2>出错了：{error.message}</h2>
      <button onClick={reset}>重试</button>
    </div>
  )
}
```

## L2 实践层：用好

### 路由模式速查

| 模式 | 路径 | 匹配 |
|------|------|------|
| 静态 | `app/about/page.tsx` | `/about` |
| 动态 | `app/posts/[id]/page.tsx` | `/posts/1`, `/posts/abc` |
| 捕获所有 | `app/docs/[...slug]/page.tsx` | `/docs/a/b/c` |
| 可选捕获 | `app/docs/[[...slug]]/page.tsx` | `/docs`, `/docs/a/b` |

### 导航方法

| 方法 | 用途 | 示例 |
|------|------|------|
| `<Link>` | 声明式导航 | `<Link href="/about">` |
| `useRouter` | 编程式导航 | `router.push('/about')` |
| `useSearchParams` | 查询参数 | `searchParams.get('q')` |

```tsx
import Link from 'next/link'
import { useRouter } from 'next/navigation'

// 声明式导航（推荐）
<Link href="/posts/1">文章一</Link>

// 编程式导航
'use client'
const router = useRouter()
router.push('/posts/1')
router.replace('/posts/1')  // 不留历史
router.back()
```

### 反模式：不要这样做

```tsx
// ❌ 错误：用 a 标签导航（会全页刷新）
<a href="/about">关于</a>

// ✅ 正确：用 Link 组件（客户端导航）
<Link href="/about">关于</Link>
```

```tsx
// ❌ 错误：在 Server Component 中使用 useRouter
export default async function Page() {
  const router = useRouter()  // ❌ Server Component 不支持
}

// ✅ 正确：在 Client Component 中使用
'use client'
export default function Page() {
  const router = useRouter()  // ✅
}
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 静态页面 | `app/about/page.tsx` | 简单直接 |
| 详情页面 | `app/posts/[id]/page.tsx` | 动态参数 |
| 文档站点 | `app/docs/[...slug]/page.tsx` | 多级路径 |
| 加载状态 | `loading.tsx` | 自动显示/隐藏 |
| 错误处理 | `error.tsx` | 自动捕获错误 |

## L3 专家层：深入

### 路由渲染原理

```
Next.js App Router 渲染流程：

用户访问 /posts/1
    ↓
匹配 app/posts/[id]/page.tsx
    ↓
Server Component 渲染（服务端）
    ↓
生成 HTML + RSC Payload
    ↓
客户端水合（Hydration）
    ↓
Client Component 变为可交互
```

### 知识关联

```
路由系统关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  文件路由   │────→│  导航方法   │────→│  特殊文件   │
│  page/      │     │  Link/      │     │  loading/   │
│  layout/    │     │  useRouter  │     │  error/     │
│  [id]/      │     └─────────────┘     │  not-found  │
└─────────────┘                         └─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **文件路由** | 文件夹结构自动映射为 URL | `app/about/page.tsx` → `/about` |
| **动态路由** | 用 `[param]` 捕获 URL 参数 | `app/posts/[id]/page.tsx` |
| **Layout** | 共享的 UI 框架 | `layout.tsx` |

## 常见问题

### Q1：Link 和 a 标签有什么区别？

**Link 是客户端导航，预加载目标页面；a 标签会全页刷新：**

```tsx
<Link href="/about">关于</Link>  // ✅ 客户端导航，预加载
<a href="/about">关于</a>        // ❌ 全页刷新
```

### Q2：[id] 和 [...slug] 有什么区别？

**[id] 捕获一段路径，[...slug] 捕获多段：**

```
app/posts/[id]/page.tsx   → /posts/123（id = "123"）
app/docs/[...slug]/page.tsx → /docs/a/b/c（slug = ["a", "b", "c"]）
```

### Q3：如何处理 404 页面？

**创建 not-found.tsx，调用 notFound() 触发：**

```tsx
// app/not-found.tsx
export default function NotFound() {
  return <h1>页面不存在</h1>
}

// 在 page.tsx 中
import { notFound } from 'next/navigation'
if (!data) notFound()
```
