## 生活化比喻

**样式方案就像"装修方式"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                  装修方式                             │
├──────────────────────────────────────────────────────┤
│                                                      │
│    CSS Modules = 独立装修每个房间                     │
│    ─────────────                                     │
│    每个房间的装修互不影响                            │
│    page.module.css → 只有这个页面能用                │
│    不用担心样式冲突                                  │
│                                                      │
│    Tailwind = 标准化装修包                            │
│    ─────────────                                     │
│    不用自己设计，直接用现成的装修方案                │
│    p-4 bg-white rounded → 内边距+白底+圆角          │
│    快速、一致、不用写 CSS 文件                      │
│                                                      │
│    全局 CSS = 公共区域装修                            │
│    ─────────────                                     │
│    大厅、走廊等公共空间的装修                        │
│    所有页面共享                                      │
│    字体、颜色变量、重置样式                         │
│                                                      │
│    CSS-in-JS = 动态装修                               │
│    ─────────────                                     │
│    根据条件动态改变装修风格                          │
│    根据用户主题偏好切换颜色                          │
│    需要客户端 JS 支持                                │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

### CSS Modules

**最简示例（1-3行）：**

```tsx
import styles from './page.module.css'
export default function Page() {
  return <div className={styles.container}>内容</div>
}
```

**详细示例：**

```tsx
// app/page.tsx
import styles from './page.module.css'

export default function Page() {
  return (
    <div className={styles.container}>
      <h1 className={styles.title}>标题</h1>
      <p className={styles.text}>内容</p>
    </div>
  )
}
```

```css
/* app/page.module.css */
.container { padding: 2rem; }
.title { color: #333; font-size: 1.5rem; }
.text { color: #666; }

/* 组合类名 */
.container .title:hover { color: #3b82f6; }
```

---

### Tailwind CSS

```tsx
// Tailwind — Next.js 默认支持
export default function Page() {
  return (
    <div className="p-4 bg-white rounded-lg shadow hover:shadow-lg transition">
      <h1 className="text-2xl font-bold text-gray-800">标题</h1>
      <p className="text-gray-600 mt-2">内容</p>
    </div>
  )
}
```

```css
/* app/globals.css */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* 自定义组件类 */
@layer components {
  .btn-primary {
    @apply px-4 py-2 bg-blue-600 text-white rounded hover:bg-blue-700;
  }
}
```

## L2 实践层：用好

### 样式方案对比

| 方案 | 优点 | 缺点 | 适用 |
|------|------|------|------|
| CSS Modules | 作用域隔离，零配置 | 需要写 CSS 文件 | 通用 |
| Tailwind | 快速开发，一致性 | 类名长 | 新项目推荐 |
| 全局 CSS | 简单 | 样式冲突 | 重置、变量 |
| CSS-in-JS | 动态样式 | 需要客户端 JS | 主题切换 |

### 反模式：不要这样做

```tsx
// ❌ 错误：内联样式（难维护）
<div style={{ padding: '2rem', backgroundColor: 'white', borderRadius: '8px' }}>

// ✅ 正确：用 CSS Modules 或 Tailwind
<div className={styles.card}>  // CSS Modules
<div className="p-8 bg-white rounded-lg">  // Tailwind
```

```tsx
// ❌ 错误：全局 CSS 污染
/* globals.css */
.card { padding: 2rem; }  // 所有 .card 都受影响

// ✅ 正确：用 CSS Modules
/* card.module.css */
.card { padding: 2rem; }  // 只有导入这个文件的组件能用
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 组件样式 | CSS Modules | 作用域隔离 |
| 快速原型 | Tailwind | 不用写 CSS 文件 |
| 全局重置 | 全局 CSS | 所有页面共享 |
| 主题切换 | Tailwind + CSS 变量 | 动态切换 |
| 动画 | CSS Modules | 更好的控制 |

## L3 专家层：深入

### CSS 变量与主题

```css
/* app/globals.css */
:root {
  --primary: #3b82f6;
  --bg: #ffffff;
  --text: #1f2937;
}

[data-theme="dark"] {
  --primary: #60a5fa;
  --bg: #1f2937;
  --text: #f9fafb;
}
```

```tsx
// 使用 CSS 变量
export default function Page() {
  return (
    <div style={{ background: 'var(--bg)', color: 'var(--text)' }}>
      <button style={{ background: 'var(--primary)' }}>按钮</button>
    </div>
  )
}
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **CSS Modules** | 作用域隔离的 CSS | `import styles from './page.module.css'` |
| **Tailwind** | 实用优先的 CSS 框架 | `className="p-4 bg-white"` |
| **全局 CSS** | 所有页面共享的样式 | `app/globals.css` |
| **CSS 变量** | 可在 CSS 和 JS 中使用的变量 | `var(--primary)` |
| **@layer** | Tailwind 的层级控制 | `@layer components` |

## 实践练习

```tsx
// 练习 1：实现主题切换
// app/globals.css
:root {
  --bg: #ffffff;
  --text: #1f2937;
}
[data-theme="dark"] {
  --bg: #1f2937;
  --text: #f9fafb;
}

// app/page.tsx
'use client'
import { useState } from 'react'

export default function Page() {
  const [theme, setTheme] = useState('light')
  return (
    <div data-theme={theme} style={{ background: 'var(--bg)', color: 'var(--text)' }}>
      <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
        切换主题
      </button>
    </div>
  )
}
```

## 常见问题

### Q1：CSS Modules 和 Tailwind 选哪个？

**快速开发用 Tailwind，需要精细控制用 CSS Modules：**

```tsx
// Tailwind — 快速
<div className="p-4 bg-white rounded-lg">

// CSS Modules — 精细控制
<div className={styles.card}>
```

### Q2：如何在 Server Component 中使用 Tailwind？

**直接用，Tailwind 在服务端编译：**

```tsx
// Server Component
export default async function Page() {
  return <div className="p-4 bg-white">内容</div>  // ✅ 可以
}
```

### Q3：如何组织 CSS Modules 文件？

**和组件放在同一目录：**

```
app/
├── page.tsx
├── page.module.css
├── components/
│   ├── Header.tsx
│   └── Header.module.css
```
