## 13.0 通俗理解工具类型

### 什么是工具类型？

```
工具类型 = 别人已经写好的"类型函数"
         = 用来快速生成新类型的"工具"

就像：
- 工具箱里有各种工具
- 螺丝刀、锤子、扳手...
- 拿来就能用，不需要自己造
```

### 为什么要用工具类型？

```
不用工具类型：
interface User {
    id: number;
    name: string;
    email: string;
    age: number;
    createdAt: Date;
}

// 手动写一个"可选版本"
interface PartialUser {
    id?: number;
    name?: string;
    email?: string;
    age?: number;
    createdAt?: Date;
}

用工具类型：
type PartialUser = Partial<User>;  // 一行搞定！
```

### 常用工具类型一览

| 工具类型 | 作用 | 例子 |
|----------|------|------|
| Partial | 所有属性变可选 | `{ a?: string }` |
| Required | 所有属性变必填 | `{ a: string }` |
| Readonly | 所有属性变只读 | `{ readonly a: string }` |
| Pick | 选取部分属性 | `{ a: string }` |
| Omit | 排除部分属性 | `{ b: string }` |
| Record | 键值对类型 | `{ [key: string]: value }` |

## 13.1 Partial - 所有属性可选

### 用法

```typescript
interface User {
    id: number;
    name: string;
    email: string;
}

// 所有属性变为可选
type PartialUser = Partial<User>;

// 等同于
// { id?: number; name?: string; email?: string; }

const user: PartialUser = { name: 'Tom' };
```

### 实现原理

```typescript
type MyPartial<T> = {
    [P in keyof T]?: T[P];
};
```

## 13.3 Readonly - 所有属性只读

### 用法

```typescript
interface User {
    id: number;
    name: string;
}

// 所有属性变为只读
type ReadonlyUser = Readonly<User>;

// 等同于
// { readonly id: number; readonly name: string; }

const user: ReadonlyUser = { id: 1, name: 'Tom' };
// user.id = 2;  // ❌ 错误：只读
```

### 实现原理

```typescript
type MyReadonly<T> = {
    readonly [P in keyof T]: T[P];
};
```

## 13.4 Pick - 选取属性

### 用法

```typescript
interface User {
    id: number;
    name: string;
    email: string;
    password: string;
}

// 选取指定属性
type UserPreview = Pick<User, 'id' | 'name'>;

```

### 实现原理

```typescript
type MyPick<T, K extends keyof T> = {
    [P in K]: T[P];
};
```

## 13.5 Omit - 排除属性

### 用法

```typescript
interface User {
    id: number;
    name: string;
    email: string;
    password: string;
}

// 排除指定属性
type UserWithoutPassword = Omit<User, 'password'>;

```

### 实现原理

```typescript
type MyOmit<T, K extends keyof any> = {
    [P in Exclude<keyof T, K>]: T[P];
};
```

## 13.6 Record - 键值对类型

### 通俗理解

```
Record<键的类型, 值的类型>

= 定义一个"字典" / "映射"
= 类似 JavaScript 的对象
```

**简单理解：**
```
Record<Role, string[]> = "一个对象，键是 Role，值是 string 数组"
```

### 完整解释

```typescript
// 定义角色类型
type Role = 'admin' | 'user' | 'guest';

// Record<Role, string[]>
//   ↑      ↑
//   │      └── 值的类型：string 数组
//   └── 键的类型：Role（只能是 'admin' | 'user' | 'guest'）

const permissions: Record<Role, string[]> = {
    admin: ['read', 'write', 'delete'],    // ✅
    user: ['read', 'write'],               // ✅
    guest: ['read']                        // ✅
};
```

### 实际应用

```typescript
// 场景1：角色权限
type Role = 'admin' | 'user' | 'guest';

const rolePermissions: Record<Role, string[]> = {
    admin: ['read', 'write', 'delete'],
    user: ['read', 'write'],
    guest: ['read']
};

// 使用
rolePermissions.admin    // ['read', 'write', 'delete']
rolePermissions.user     // ['read', 'write']
rolePermissions.guest   // ['read']

// 场景2：用户信息
type UserId = number;

const userDatabase: Record<UserId, { name: string; age: number }> = {
    1: { name: 'Tom', age: 25 },
    2: { name: 'Jerry', age: 30 }
};

userDatabase[1]  // { name: 'Tom', age: 25 }

// 场景3：国家代码
type CountryCode = 'CN' | 'US' | 'JP';

const countryNames: Record<CountryCode, string> = {
    CN: '中国',
    US: '美国',
    JP: '日本'
};

countryNames.CN  // '中国'
```

### 对比：Record vs 普通对象

```typescript
// 不用 Record（一样，但不够类型安全）
const permissions1 = {
    admin: ['read', 'write'],
    user: ['read'],
    guest: ['read']
};

// 用 Record（类型安全）
const permissions2: Record<string, string[]> = {
    admin: ['read', 'write'],
    user: ['read'],
    guest: ['read']
};

// 区别：
// - Record 明确指定了键和值的类型
// - TypeScript 会检查键是否合法
```

### 实现原理

```typescript
type MyRecord<K extends keyof any, T> = {
    [P in K]: T;
};

// 展开：
// type Record<Role, string[]> = {
//     [P in 'admin' | 'user' | 'guest']: string[]
// }

// 等价于：
// {
//     admin: string[]
//     user: string[]
//     guest: string[]
// }
```

## 13.7 Exclude / Extract

### Exclude - 排除类型

```typescript
type T = 'a' | 'b' | 'c' | 'd';

// 排除 'a' 和 'c'
type Excluded = Exclude<T, 'a' | 'c'>;

// 结果：'b' | 'd'
```

### Extract - 提取类型

```typescript
type T = 'a' | 'b' | 'c' | 'd';

// 提取 'a' 和 'c'
type Extracted = Extract<T, 'a' | 'c'>;

// 结果：'a' | 'c'
```

---

## 13.8 NonNullable - 排除 null/undefined

```typescript
type T = string | null | undefined | number;

// 排除 null 和 undefined
type NonNull = NonNullable<T>;

// 结果：string | number
```

## 13.9 ReturnType / Parameters

### ReturnType - 获取返回类型

```typescript
function getUser() {
    return { id: 1, name: 'Tom' };
}

type User = ReturnType<typeof getUser>;
// { id: number; name: string; }
```

### Parameters - 获取参数类型

```typescript
function createUser(name: string, age: number) {
    return { name, age };
}

type Params = Parameters<typeof createUser>;
// [string, number]
```

## 13.10 自定义工具类型

### DeepPartial - 深度可选

```typescript
type DeepPartial<T> = {
    [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

interface Company {
    name: string;
    address: {
        city: string;
        street: string;
    };
}

type PartialCompany = DeepPartial<Company>;
// { name?: string; address?: { city?: string; street?: string; } }
```

### DeepReadonly - 深度只读

```typescript
type DeepReadonly<T> = {
    readonly [P in keyof T]: T[P] extends object ? DeepReadonly<T[P]> : T[P];
};

interface Company {
    name: string;
    address: {
        city: string;
    };
}

type ReadonlyCompany = DeepReadonly<Company>;
```

## 13.11 实践练习

### 练习 1：使用工具类型

```typescript
interface User {
    id: number;
    name: string;
    email: string;
    age: number;
}

// 1. Partial - 创建用户时，部分字段可选
type CreateUser = Partial<User>;

// 2. Pick - 只获取用户基本信息
type UserBasicInfo = Pick<User, 'id' | 'name'>;

// 3. Omit - 排除敏感信息
type PublicUser = Omit<User, 'email'>;
```

### 练习 2：自定义工具类型

```typescript
// 提取函数类型
type FnReturn<T> = T extends (...args: any[]) => infer R ? R : never;

function getData() {
    return { id: 1, name: 'Tom' };
}

type Data = FnReturn<typeof getData>;  // { id: number; name: string }
```

### 练习 3：Record 应用

```typescript
type UserRole = 'admin' | 'editor' | 'viewer';

const rolePermissions: Record<UserRole, { canEdit: boolean; canDelete: boolean }> = {
    admin: { canEdit: true, canDelete: true },
    editor: { canEdit: true, canDelete: false },
    viewer: { canEdit: false, canDelete: false }
};
```

## 本章小结

| 工具类型 | 作用 | 示例 |
|----------|------|------|
| Partial | 所有属性可选 | `Partial<User>` |
| Required | 所有属性必填 | `Required<User>` |
| Readonly | 所有属性只读 | `Readonly<User>` |
| Pick | 选取属性 | `Pick<User, 'id'>` |
| Omit | 排除属性 | `Omit<User, 'pwd'>` |
| Record | 键值对 | `Record<K, V>` |
| Exclude | 排除类型 | `Exclude<A, B>` |
| Extract | 提取类型 | `Extract<A, B>` |
| NonNullable | 排除空值 | `NonNullable<T>` |
| ReturnType | 返回类型 | `ReturnType<fn>` |
| Parameters | 参数类型 | `Parameters<fn>` |
