## 6.1 为什么要学类型守卫？

### 问题引入

```
场景：处理多种类型的数据

假设有一个函数，参数可能是数字或字符串：

function process(value: number | string) {
    // 怎么处理？
}

问题：
- 数字有 .toFixed() 方法
- 字符串有 .toUpperCase() 方法
- 混用会报错！
```

### 什么是类型守卫？

```
类型守卫 = 告诉 TypeScript "这个变量是什么类型"

就像：
- 安检门：检查你带的是什么类型的行李
- 分类垃圾桶：把不同类型的垃圾分开
- 机场安检：确认你是乘客还是工作人员

作用：
- 在运行时检查类型
- 告诉 TypeScript 变量的具体类型
- 让 TypeScript 在特定区域知道正确类型
```

### 什么是类型收窄？

```
类型收窄 = 从宽类型变成窄类型

例子：
- number | string（宽）
  ↓
  if (typeof x === 'string')
  ↓
  string（窄）

就像：
- 一箱水果 → 挑出苹果 → 只有苹果
- 一群人 → 筛选出成年人 → 只有成年人
```

### 类型收窄的方式一览

| 方式 | 说明 | 生活例子 |
|------|------|----------|
| `typeof` | 检查基本类型 | 检查行李是液体还是电子设备 |
| `instanceof` | 检查构造函数实例 | 检查是不是某品牌的汽车 |
| `in` | 检查属性是否存在 | 检查有没有钥匙 |
| 类型谓词 | 自定义类型检查 | 自定义安检标准 |
| 相等性检查 | 检查值是否相等 | 检查是不是同一个人 |
| 真值检查 | 检查真假值 | 检查灯是否开着 |

## 6.2 typeof 类型守卫 - 检查基本类型

### 什么是 typeof？

```
typeof = JavaScript 内置的类型检查工具

可以检查：
- 'string' - 字符串
- 'number' - 数字
- 'boolean' - 布尔值
- 'undefined' - 未定义
- 'function' - 函数
- 'object' - 对象（注意：数组也是 object）

就像：
- 质检员检查产品类型
- 医生检查血型
```

### 基本用法

```typescript
// 联合类型：可能是字符串或数字
function processValue(value: string | number): string {
    // 使用 typeof 检查类型
    if (typeof value === 'string') {
        // 在这个分支，TypeScript 知道 value 是 string
        return `字符串：${value.toUpperCase()}`;
    } else {
        // 在这个分支，TypeScript 知道 value 是 number
        return `数字：${value.toFixed(2)}`;
    }
}

// 测试
console.log(processValue('hello'));  // 字符串：HELLO
console.log(processValue(123.456));  // 数字：123.46
```

### 支持的类型

```typescript
typeof 'hello' === 'string'
typeof 123 === 'number'
typeof true === 'boolean'
typeof undefined === 'undefined'
typeof function() {} === 'function'
typeof [] === 'object'
```

## 6.3 instanceof 守卫 - 检查对象实例

### 什么是 instanceof？

```
instanceof = 检查对象是不是某个"构造函数"创建的

语法：object instanceof Constructor

就像：
- 检查这辆车是不是 BMW 生产的
- 检查这本书是不是某个出版社出版的

适用场景：
- 类（class）创建的对象
- 内置构造函数（Date, Array, Error 等）
```

### 基本用法

```typescript
// 定义两个类
class Bird {
    fly() {
        console.log('🐦 鸟在飞翔');
    }
}

class Fish {
    swim() {
        console.log('🐟 鱼在游泳');
    }
}

// 处理函数
function move(animal: Bird | Fish) {
    // 使用 instanceof 检查
    if (animal instanceof Bird) {
        animal.fly();  // TypeScript 知道是 Bird
    } else {
        animal.swim(); // TypeScript 知道是 Fish
    }
}

// 测试
const bird = new Bird();
const fish = new Fish();

move(bird);  // 🐦 鸟在飞翔
move(fish);  // 🐟 鱼在游泳
```

### instanceof vs typeof

```
typeof：
- 检查基本类型
- typeof 'hello' === 'string'
- typeof 123 === 'number'
- typeof [] === 'object'（数组也是 object！）

instanceof：
- 检查对象实例
- [] instanceof Array
- new Date() instanceof Date
```

### 实际应用：日期处理

```typescript
// 日期处理函数
function formatDate(value: Date | string | number): string {
    if (value instanceof Date) {
        // 已经是 Date 对象
        return value.toISOString();
    } else {
        // 字符串或数字，转换为 Date
        return new Date(value).toISOString();
    }
}
```

## 6.4 in 操作符守卫 - 检查属性是否存在

### 什么是 in？

```
in = 检查对象里有没有某个属性

语法：'属性名' in 对象

返回：true 或 false

就像：
- 检查房子里有没有钥匙
- 检查口袋里有没有手机

适用场景：
- 两个对象类型有不同属性
- 检查可选属性是否存在
```

### 基本用法

```typescript
// 定义两个接口
interface User {
    name: string;
    email: string;
}

interface Product {
    title: string;
    price: number;
}

// 处理函数
function describe(value: User | Product): string {
    // 使用 in 检查属性
    if ('name' in value) {
        // 有 name 属性，说明是 User
        return `用户：${value.name} (${value.email})`;
    } else {
        // 没有 name 属性，说明是 Product
        return `商品：${value.title} - ¥${value.price}`;
    }
}
```

### 为什么接口不能用 instanceof？

```typescript
// 接口在编译后会被删除，不存在于运行时
interface User {
    name: string;
}

// 编译后变成：
// interface 完全消失

// 所以不能用 instanceof
// user instanceof User  // ❌ 错误！

// 只能用 in 检查属性
'name' in user  // ✅ 可以
```

## 6.5 类型谓词 - 自定义类型守卫

### 什么是类型谓词？

```
类型谓词 = 自定义的"类型检查函数"

语法：function fn(x: unknown): x is Type

作用：
- 告诉 TypeScript "这个变量确实是某个类型"
- 可以封装复杂的类型检查逻辑

就像：
- 自定义安检标准
- 专业资格认证
```

### 基本语法

```typescript
// 定义类型谓词函数
function isString(value: unknown): value is string {
    return typeof value === 'string';
}

// 使用
function process(value: unknown) {
    if (isString(value)) {
        // TypeScript 知道 value 是 string
        console.log(value.toUpperCase());
    }
}
```

### 对比：类型谓词 vs 类型断言

```
类型谓词（推荐）：
function isString(x: unknown): x is string {
    return typeof x === 'string';
}

if (isString(value)) {
    value.toUpperCase();  // ✅ 安全
}

─────────────────────────────────────────

类型断言（谨慎使用）：
(value as string).toUpperCase();  // ⚠️ 可能出错

如果 value 不是字符串，运行时可能报错
```

### 实用类型守卫函数

```typescript
// 检查是否为数字
function isNumber(value: unknown): value is number {
    return typeof value === 'number' && !isNaN(value);
}

// 检查是否为数组
function isArray(value: unknown): value is unknown[] {
    return Array.isArray(value);
}

// 检查是否为对象（非数组）
function isObject(value: unknown): value is Record<string, unknown> {
    return typeof value === 'object' && value !== null && !Array.isArray(value);
}

// 检查是否为有效日期
function isValidDate(value: unknown): value is Date {
    return value instanceof Date && !isNaN(value.getTime());
}
```

## 6.6 可辨识联合 - 最佳实践

### 什么是可辨识联合？

```
可辨识联合 = 用一个"标签"属性区分不同类型

核心思想：
- 每个类型都有一个共同的"标识"属性（通常是 kind 或 type）
- 通过检查这个标识来区分类型

优点：
- 代码更清晰
- 易于扩展
- 配合 switch 语句更优雅
```

### 完整示例

```typescript
// 定义可辨识联合
type Shape =
    | { kind: 'circle'; radius: number }
    | { kind: 'square'; side: number }
    | { kind: 'triangle'; base: number; height: number };

// 计算面积
function getArea(shape: Shape): number {
    // 使用 switch 检查 kind 属性
    switch (shape.kind) {
        case 'circle':
            return Math.PI * shape.radius ** 2;
        case 'square':
            return shape.side ** 2;
        case 'triangle':
            return (shape.base * shape.height) / 2;
        default:
            // 完整性检查：确保处理了所有情况
            const _exhaustive: never = shape;
            return _exhaustive;
    }
}
```

## 6.7 完整性检查 - 确保处理所有情况

### 什么是完整性检查？

```
完整性检查 = 确保处理了所有可能的类型

目的：
- 防止遗漏处理某些情况
- 新增类型时，编译器会报错提醒

就像：
- 考试检查是否每道题都做了
- 清单检查是否每项都完成了
```

### 使用 never 进行完整性检查

```typescript
// 定义状态类型
type Status = 'pending' | 'loading' | 'success' | 'error';

// 处理函数
function handleStatus(status: Status): string {
    switch (status) {
        case 'pending':
            return '等待中';
        case 'loading':
            return '加载中';
        case 'success':
            return '成功';
        case 'error':
            return '错误';
        default:
            // 如果新增状态，这里会报错！
            const _exhaustive: never = status;
            return _exhaustive;
    }
}
```

### 如果新增类型会怎样？

```typescript
// 假设我们新增了 'cancelled' 状态
type StatusNew = 'pending' | 'loading' | 'success' | 'error' | 'cancelled';

function handleStatusNew(status: StatusNew): string {
    switch (status) {
        case 'pending':
            return '等待中';
        case 'loading':
            return '加载中';
        case 'success':
            return '成功';
        case 'error':
            return '错误';
        // ⚠️ 忘记处理 'cancelled'
        default:
            // ❌ 编译错误！
            const _exhaustive: never = status;
            return _exhaustive;
    }
}
```

## 6.8 实践练习

### 练习 1：实现类型守卫

```typescript
// 定义联合类型
type Animal =
    | { type: 'dog'; bark: () => void }
    | { type: 'cat'; meow: () => void }
    | { type: 'bird'; sing: () => void };

// 实现类型守卫
function isDog(animal: Animal): animal is { type: 'dog'; bark: () => void } {
    return animal.type === 'dog';
}

function isCat(animal: Animal): animal is { type: 'cat'; meow: () => void } {
    return animal.type === 'cat';
}

// 使用
function makeSound(animal: Animal): void {
    if (isDog(animal)) {
        animal.bark();
    } else if (isCat(animal)) {
        animal.meow();
    } else {
        animal.sing();
    }
}
```

### 练习 2：实现连接状态机

```typescript
// 定义状态
type ConnectionState =
    | { status: 'disconnected' }
    | { status: 'connecting'; progress: number }
    | { status: 'connected'; sessionId: string }
    | { status: 'error'; error: Error };

// 状态处理
function handleConnection(state: ConnectionState): string {
    switch (state.status) {
        case 'disconnected':
            return '已断开连接';
        case 'connecting':
            return `连接中 ${state.progress}%`;
        case 'connected':
            return `已连接，会话 ID: ${state.sessionId}`;
        case 'error':
            return `连接错误：${state.error.message}`;
        default:
            const _exhaustive: never = state;
            return _exhaustive;
    }
}
```

## 6.9 常见问答

### Q1: 类型守卫和类型断言有什么区别？

```typescript
// 类型守卫（安全）
if (typeof value === 'string') {
    console.log(value.length);  // ✅ TypeScript 知道是 string
}

// 类型断言（谨慎）
console.log((value as string).length);  // ⚠️ 假设 value 是 string，可能出错
```

### Q2: 什么时候用 typeof，什么时候用 instanceof？

```typescript
// 用 typeof：检查基本类型
if (typeof name === 'string') { }
if (typeof age === 'number') { }

// 用 instanceof：检查对象实例
if (date instanceof Date) { }
if (user instanceof User) { }
```

### Q3: 可辨识联合有什么优点？

```typescript
// 优点 1：代码清晰
type Shape =
    | { kind: 'circle'; radius: number }
    | { kind: 'square'; side: number };

// 优点 2：易于扩展
type Shape3D = Shape & { z: number };

// 优点 3：完整性检查
function getArea(shape: Shape): number {
    switch (shape.kind) {
        case 'circle': return Math.PI * shape.radius ** 2;
        case 'square': return shape.side ** 2;
        // 忘记处理会报错
    }
}
```

## 本章小结

| 守卫方式 | 语法 | 适用场景 |
|----------|------|----------|
| typeof | `typeof x === 'string'` | 原始类型 |
| instanceof | `x instanceof Class` | 类实例 |
| in | `'prop' in x` | 对象属性 |
| 类型谓词 | `x is Type` | 自定义判断 |
| 可辨识联合 | `switch (x.kind)` | 联合类型 |
