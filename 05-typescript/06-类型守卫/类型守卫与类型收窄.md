**类型守卫就像"安检门"**：

```
联合类型 = 一群混在一起的人：
┌─────────────────────────────┐
│  乘客、工作人员、安检员     │
│  type Person = Passenger | Staff | Security │

类型守卫 = 安检门：
┌─────────────────────────────┐

└─────────────────────────────┘
```

## 第一步：看看问题

假设你有一个函数，接收字符串或数字：

```typescript
function process(value: string | number) {
    value.toUpperCase();  // ❌ 报错：number 没有 toUpperCase
    value.toFixed(2);     // ❌ 报错：string 没有 toFixed
}
```

**问题：** TypeScript 不知道 `value` 到底是字符串还是数字，所以两个方法都不能调用。

**你需要告诉 TypeScript："现在是字符串"或"现在是数字"。**

---

## 第二步：typeof 类型守卫

最简单的类型守卫是 `typeof`：

```typescript
function process(value: string | number) {
    if (typeof value === 'string') {
        value.toUpperCase();  // ✅ TypeScript 知道这是 string
    } else {
        value.toFixed(2);     // ✅ TypeScript 知道这是 number
    }
}
```

`typeof value === 'string'` 就是一个**类型守卫**。它告诉 TypeScript："如果这个条件成立，value 就是 string。"

## 第三步：试试基础类型守卫

### typeof

适用于基本类型：

```typescript
function format(value: string | number | boolean) {
    if (typeof value === 'string') {
        return value.toUpperCase();
    } else if (typeof value === 'number') {
        return value.toFixed(2);
    } else {
        return value ? '是' : '否';
    }
}
```

### instanceof

适用于类实例：

```typescript
class Dog {
    bark() { return '汪汪'; }
}

class Cat {
    meow() { return '喵喵'; }
}

function sound(animal: Dog | Cat) {
    if (animal instanceof Dog) {
        animal.bark();  // ✅ TypeScript 知道这是 Dog
    } else {
        animal.meow();  // ✅ TypeScript 知道这是 Cat
    }
}
```

### in

适用于检查对象是否有某个属性：

```typescript
interface Fish {
    swim(): void;
    fins: number;
}

interface Bird {
    fly(): void;
    wings: number;
}

function move(animal: Fish | Bird) {
    if ('swim' in animal) {
        animal.swim();  // ✅ TypeScript 知道这是 Fish
    } else {
        animal.fly();   // ✅ TypeScript 知道这是 Bird
    }
}
```

## 第四步：类型谓词 — 自定义类型守卫

有时候内置的类型守卫不够用。你可以写自己的类型守卫函数。

```typescript
interface User {
    id: number;
    name: string;
}

function isUser(value: unknown): value is User {
    return (
        typeof value === 'object' &&
        value !== null &&
        'id' in value &&
        'name' in value
    );
}
```

看到 `value is User` 了吗？这就是**类型谓词**。它的意思是："如果这个函数返回 true，那 value 就是 User 类型。"

使用：

```typescript
function processValue(value: unknown) {
    if (isUser(value)) {
        console.log(value.name);  // ✅ TypeScript 知道这是 User
    }
}
```

## 第五步：可辨识联合的自动收窄

在上一节我们学了可辨识联合。TypeScript 会根据 `switch` 或 `if` 自动收窄类型：

```typescript
type Shape =
    | { kind: 'circle'; radius: number }
    | { kind: 'square'; side: number };

function area(shape: Shape): number {
    if (shape.kind === 'circle') {
        return Math.PI * shape.radius ** 2;  // ✅ shape 是 Circle
    } else {
        return shape.side ** 2;               // ✅ shape 是 Square
    }
}
```

当 TypeScript 看到 `shape.kind === 'circle'`，它会自动知道 `shape` 是 `Circle` 类型。这就是**类型收窄**。

## 第六步：常见错误和修复

| 错误 | 原因 | 修复 |
|------|------|------|
| `Property 'x' does not exist on type 'A \| B'` | 访问了非共有属性 | 先用类型守卫判断 |
| `This condition will always return 'false'` | 类型不可能满足条件 | 检查类型定义 |
| `Object is possibly 'null'` | 可能是 null | 先用 `if (obj)` 检查 |

## 第七步：什么时候用哪种守卫？

| 场景 | 推荐方案 | 示例 |
|------|---------|------|
| 基本类型 | `typeof` | `typeof x === 'string'` |
| 类实例 | `instanceof` | `x instanceof Dog` |
| 接口/对象 | `in` | `'name' in obj` |
| 复杂验证 | 类型谓词 | `function isUser(x): x is User` |
| 可辨识联合 | `switch` | `switch (shape.kind)` |

## 总结：速查表

| 守卫 | 适用 | 示例 |
|------|------|------|
| `typeof` | 基本类型 | `typeof x === 'string'` |
| `instanceof` | 类实例 | `x instanceof Dog` |
| `in` | 属性检查 | `'name' in obj` |
| 类型谓词 | 自定义检查 | `function isUser(x): x is User` |
| 字面量判断 | 可辨识联合 | `shape.kind === 'circle'` |

## 常见问题

### Q：类型谓词和类型断言有什么区别？

**类型谓词是运行时检查，类型断言是编译时告诉编译器"相信我"：**

```typescript
// 类型谓词 — 运行时检查
function isUser(x: unknown): x is User {
    return typeof x === 'object' && x !== null && 'id' in x;
}

// 类型断言 — 不检查，直接告诉编译器
const user = value as User;
```

### Q：为什么需要 never 类型做完整性检查？

**如果联合类型新增成员但忘记处理，never 会让编译器报错：**

```typescript
type Status = 'pending' | 'success';

function handle(s: Status) {
    switch (s) {
        case 'pending': break;
        // 漏了 'success'
        default:
            const _: never = s;  // ❌ 报错：Type '"success"' is not assignable to 'never'
    }
}
```
