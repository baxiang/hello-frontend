## 4.1 type 基础用法

### 4.1.1 什么是类型别名？

```
┌─────────────────────────────────────────────────────────────┐
│  类型别名（type alias）= 给类型起个"昵称"                  │
│                                                             │
│  作用：                                                     │
│  - 简化复杂类型的书写                                       │
│  - 让类型更有意义                                          │
│  - 方便复用                                                 │
│                                                             │
│  就像：                                                     │
│  - 给长名字起外号                                           │
│  - 给复杂地址起简称                                        │
└─────────────────────────────────────────────────────────────┘
```

### 4.1.2 基本用法

```typescript
// 基础类型别名
type Name = string;
type Age = number;
type IsDone = boolean;

// 使用
const name: Name = '张三';
const age: Age = 25;
const isDone: IsDone = true;

/** 用户 ID 类型 */
type UserID = string | number;
```

### 4.1.3 对象类型别名

```typescript
// 对象类型
type User = {
    id: number;
    name: string;
    email: string;
};

const user: User = {
    id: 1,
    name: '张三',
    email: 'zhangsan@example.com'
};

// 嵌套对象
type Address = {
    street: string;
    city: string;
    zipcode: string;
    country?: string;
};

type Person = {
    name: string;
    age: number;
    address: Address;
};

const person: Person = {
    name: '李四',
    age: 30,
    address: {
        street: '朝阳路',
        city: '北京',
        zipcode: '100000'
    }
};
```

## 4.2 联合类型

### 4.2.1 什么是联合类型？

```
┌─────────────────────────────────────────────────────────────┐
│  联合类型 = "或"关系 = 多种类型之一                         │
│                                                             │
│  语法：类型1 | 类型2 | 类型3                               │
│                                                             │
│  就像：                                                     │
│  - 成绩：优秀 | 良好 | 及格 | 不及格                        │
│  - 状态：pending | success | error                        │
│  - 性别：male | female | other                            │
└─────────────────────────────────────────────────────────────┘
```

### 4.2.2 基本用法

```typescript
// 字符串联合类型
type Status = 'pending' | 'success' | 'error';

const status1: Status = 'pending';
const status2: Status = 'success';
// const status3: Status = 'unknown';  // ❌ 错误

// 数字联合类型
type ResultCode = 200 | 201 | 400 | 401 | 404 | 500;

const code: ResultCode = 200;

// 混合联合类型
type ID = string | number;

const userId: ID = 'abc123';
const postId: ID = 12345;
```

### 4.2.3 联合类型的实际应用

```typescript
// 1. API 响应
interface ApiResponse {
    status: 'loading' | 'success' | 'error';
    data?: any;
    error?: string;
}

// 2. 配置值
type Theme = 'light' | 'dark' | 'auto';
const theme: Theme = 'dark';

// 3. 回调类型
type Callback = (error: Error | null, result?: any) => void;
```

## 4.3 交叉类型

### 4.3.1 什么是交叉类型？

```
┌─────────────────────────────────────────────────────────────┐
│  交叉类型 = "与"关系 = 多种类型都要满足                     │
│                                                             │
│  语法：类型1 & 类型2 & 类型3                                │
│                                                             │
│  就像：                                                     │
│  - 既是学生又是班干部                                      │
│  - 既是教师又是父亲                                        │
│  - 既有优点又有缺点                                        │
└─────────────────────────────────────────────────────────────┘
```

### 4.3.2 基本用法

```typescript
type Person = {
    name: string;
    age: number;
};

type Employee = {
    company: string;
    salary: number;
};

// 交叉类型：既是 Person 又是 Employee
type PersonEmployee = Person & Employee;

const person: PersonEmployee = {
    name: '张三',
    age: 30,
    company: 'ABC公司',
    salary: 20000
};
```

### 4.4.3 联合 vs 交叉

```typescript
// 联合类型：可以是 A 或 B（满足其一）
type Union = string | number;

// 交叉类型：既是 A 又是 B（全部满足）
type Intersection = { name: string } & { age: number };

const obj: Intersection = {
    name: '张三',
    age: 25
};
```

## 4.4 映射类型

### 4.4.1 什么是映射类型？

```
┌─────────────────────────────────────────────────────────────┐
│  映射类型 = 根据已有类型创建新类型                           │
│                                                             │
│  语法：{ [K in 已有类型]: 新类型 }                        │
│                                                             │
│  就像：                                                     │
│  - 复制粘贴后修改                                           │
│  - 批量处理                                                 │
│  - 模板复制                                                 │
└─────────────────────────────────────────────────────────────┘
```

### 4.4.2 基本用法

```typescript
// 将所有属性变为可选
type Partial<T> = {
    [K in keyof T]?: T[K];
};

interface User {
    id: number;
    name: string;
    email: string;
}

type PartialUser = Partial<User>;
// 等同于：
// {
//     id?: number;
//     name?: string;
//     email?: string;
// }

// 将所有属性变为只读
type Readonly<T> = {
    readonly [K in keyof T]: T[K];
};

type ReadonlyUser = Readonly<User>;
```

## 4.5 模板字面量类型

### 4.5.1 什么是模板字面量类型？

```
┌─────────────────────────────────────────────────────────────┐
│  模板字面量类型 = 用模板字符串语法定义类型                   │
│                                                             │
│  语法：`类型${类型}`                                       │
│                                                             │
│  就像：                                                     │
│  - onClick → onClick                                       │
│  - getUser → getUser                                      │
│  - setName → setName                                       │
└─────────────────────────────────────────────────────────────┘
```

### 4.5.2 基本用法

```typescript
// 事件名类型
type EventName = `on${string}`;
const event1: EventName = 'onClick';
const event2: EventName = 'onChange';
// const event3: EventName = 'click';  // ❌

// 方法名类型
type MethodName = `get${string}`;
type SetterName = `set${string}`;

const getter: MethodName = 'getUser';
const setter: SetterName = 'setName';

// 组合多个部分
type Handler = `on${'Click' | 'Change' | 'Focus'}`;
const handler1: Handler = 'onClick';
const handler2: Handler = 'onChange';
```

```typescript

type UserRole = 'admin' | 'user' | 'guest';

const user: User = {
```

```typescript
// 定义状态类型

interface ApiResponse<T> {

    message?: string;

// 使用
const response: ApiResponse<string[]> = {
    status: 'success',
    data: ['item1', 'item2']
```

## 4.7 常见问答

### Q1: type 和 interface 有什么区别？

```typescript
// 接口
interface User {
    name: string;
}

// 类型别名
type User = {
    name: string;
};

// 主要区别：
// - 接口可以声明合并，类型别名不行
// - 接口可以被类实现，类型别名不行（用于对象时）
// - 类型别名可以用于联合类型、交叉类型
```

### Q2: 联合类型和交叉类型有什么区别？

```typescript
// 联合类型：满足其一
type A = string | number;
const a: A = 'hello';  // OK
const b: A = 123;       // OK

// 交叉类型：全部满足
type B = { name: string } & { age: number };
const obj: B = { name: '张三', age: 25 };  // OK
// const obj2: B = { name: '张三' };  // ❌ 缺少 age
```
