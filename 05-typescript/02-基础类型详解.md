# TypeScript 基础类型详解

> 用生活中的例子理解每个类型概念

## 本章学习目标

- 掌握所有基础类型的使用
- 理解 any、unknown、never 的区别
- 学会类型推断和类型断言
- 理解可选链、空值合并等新特性

## 2.1 布尔类型（boolean）- 是非判断

### 什么是布尔类型？

**布尔类型只有两个值：true（真）或 false（假）**，用于判断是非。

```typescript
// 生活中的例子：
// - 灯：开/关
// - 门：开/关
// - 开关：on/off

// TypeScript 中：
let isLightOn: boolean = true;   // 灯开着
let isDoorOpen: boolean = false; // 门关着

// 实际应用场景
interface User {
    id: number;
    name: string;
    isActive: boolean;    // 用户是否激活
    isVip: boolean;       // 是否 VIP 用户
    isEmailVerified: boolean;  // 邮箱是否验证
}

const user: User = {
    id: 1,
    name: '张三',
    isActive: true,      // 已激活
    isVip: false,        // 不是 VIP
    isEmailVerified: true // 邮箱已验证
};
```

## 2.2 数字类型（number）- 所有数字

### TypeScript 只有一种数字类型

**不像 JavaScript 分整数和浮点数，TypeScript 的 number 可以表示所有数字。**

```typescript
// 整数
let age: number = 25;
let count: number = -10;

// 小数
let price: number = 99.99;
let pi: number = 3.14159;

// 特殊值
let infinityVal: number = Infinity;  // 无穷大
let notANumber: number = NaN;        // 不是数字（Not a Number）

// 实际应用场景
interface Product {
    id: number;
    name: string;
    price: number;      // 价格
    stock: number;      // 库存
    discount: number;   // 折扣（0-1 之间的小数）
    rating: number;     // 评分（0-5 之间）
}
```

### 数字分隔符 - 让大数字更易读

```typescript
// 没有分隔符 - 很难数清楚有几个 0
let salary = 1000000;

// 有分隔符 - 一眼看出是 100 万
let salary2 = 1_000_000;

// 分隔符不影响实际值
console.log(1000000 === 1_000_000);  // true

// 可以这样用
let million = 1_000_000;      // 100 万
let billion = 1_000_000_000;  // 10 亿
let phone = 138_0013_8000;    // 手机号，方便记忆
```

## 2.3 字符串类型（string）- 文本内容

### 什么是字符串？

**字符串就是文本内容**，用单引号、双引号或反引号包裹。

```typescript
// 单引号
let name1: string = '张三';

// 双引号
let name2: string = "李四";

// 反引号（模板字符串）- 可以嵌入变量
let name3: string = `王五`;

// 模板字符串的强大之处
const userName = '张三';
const age = 25;

// 普通拼接 - 麻烦
const message1 = '你好，我叫' + userName + '，今年' + age + '岁';

// 模板字符串 - 简洁
const message2 = `你好，我叫${userName}，今年${age}岁`;

console.log(message2);  // 你好，我叫张三，今年 25 岁
```

### 多行字符串

```typescript
// 普通字符串 - 多行很麻烦
const html1 = '<div>\n' +
    '  <h1>标题</h1>\n' +
    '  <p>内容</p>\n' +
    '</div>';

// 模板字符串 - 多行很简单
const html2 = `
<div>
  <h1>标题</h1>
  <p>内容</p>
</div>
`;
```

### 常用字符串方法

```typescript
const text = 'Hello, TypeScript!';

// 长度
text.length;  // 18

// 大小写转换
text.toUpperCase();  // 'HELLO, TYPESCRIPT!'
text.toLowerCase();  // 'hello, typescript!'

// 查找
text.includes('TypeScript');  // true
text.startsWith('Hello');     // true
text.endsWith('!');           // true

// 提取
text.slice(0, 5);    // 'Hello'
text.substring(0, 5); // 'Hello'

// 替换
text.replace('TypeScript', 'World');  // 'Hello, World!'

// 分割成数组
text.split(', ');  // ['Hello', 'TypeScript!']
```

## 2.4 数组类型 - 一组数据

### 什么是数组？

**数组就是存储一组数据的容器**，就像购物清单。

```typescript
// 生活例子：购物清单
// ┌─────────────────────────────┐
// │ 购物清单                     │
// │ 1. 苹果                      │
// │ 2. 香蕉                      │
// │ 3. 牛奶                      │
// │ 4. 面包                      │
// └─────────────────────────────┘

// TypeScript 中
const shoppingList: string[] = ['苹果', '香蕉', '牛奶', '面包'];

// 或者使用泛型语法
const shoppingList2: Array<string> = ['苹果', '香蕉', '牛奶', '面包'];
```

### 数组类型声明方式

```typescript
// 方式 1：类型 + 方括号（推荐）
let numbers: number[] = [1, 2, 3, 4, 5];
let names: string[] = ['张三', '李四', '王五'];

// 方式 2：泛型语法
let numbers2: Array<number> = [1, 2, 3, 4, 5];
let names2: Array<string> = ['张三', '李四', '王五'];

// 混合类型数组
let mixed: (number | string)[] = [1, 'hello', 2, 'world'];
```

### 数组常用方法

```typescript
const numbers = [1, 2, 3, 4, 5];

// 1. map - 对每个元素进行操作，返回新数组
const doubled = numbers.map(n => n * 2);
// [2, 4, 6, 8, 10]

// 2. filter - 过滤符合条件的元素
const greaterThan2 = numbers.filter(n => n > 2);
// [3, 4, 5]

// 3. find - 找到第一个符合条件的元素
const first = numbers.find(n => n > 2);
// 3

// 4. some - 是否有元素符合条件
const hasEven = numbers.some(n => n % 2 === 0);
// true

// 5. every - 是否所有元素都符合条件
const allPositive = numbers.every(n => n > 0);
// true

// 6. reduce - 累加/汇总
const sum = numbers.reduce((total, n) => total + n, 0);
// 15

// 7. forEach - 遍历每个元素
numbers.forEach(n => console.log(n));
```

## 2.5 元组类型（Tuple）- 固定长度的数组

### 什么是元组？

**元组就是固定长度、固定类型的数组**，就像坐标点 (x, y)。

```typescript
// 生活例子：坐标点
// (x, y) = (10, 20)
// 第一个数是 x 坐标，第二个数是 y 坐标
// 不能颠倒，不能多，不能少

// TypeScript 中
let point: [number, number] = [10, 20];

// 访问元素
let x: number = point[0];  // 10
let y: number = point[1];  // 20

// 解构
const [x2, y2] = point;
```

### 元组 vs 数组

```typescript
// 数组 - 长度不固定，类型相同
let numbers: number[] = [1, 2, 3];
numbers.push(4);  // ✅ 可以添加
numbers.push(5);  // ✅ 可以添加

// 元组 - 长度固定，类型可以不同
let tuple: [string, number] = ['张三', 25];
// tuple.push('李四');  // ❌ 错误：不能添加

// 元组每个位置有特定含义
let person: [string, number, boolean] = ['张三', 25, true];
// 位置 0：姓名（字符串）
// 位置 1：年龄（数字）
// 位置 2：是否激活（布尔值）
```

### 实际应用场景

```typescript
// 场景 1：RGB 颜色
type RGB = [number, number, number];
const color: RGB = [255, 128, 64];  // [红，绿，蓝]

// 场景 2：日期
type DateTuple = [number, number, number];
const date: DateTuple = [2024, 3, 15];  // [年，月，日]

// 场景 3：函数返回多个值
function getUserInfo(): [string, number] {
    return ['张三', 25];  // [姓名，年龄]
}

const [name, age] = getUserInfo();
```

### 可选元素

```typescript
// 第二个元素可选
let optional: [string, number?] = ['张三'];
// 也可以
let full: [string, number?] = ['张三', 25];

// 生活例子：
// 点餐：主食（必选）+ 饮料（可选）
type Order = [string, string?];
const order1: Order = ['米饭'];           // 只要主食
const order2: Order = ['米饭', '可乐'];   // 主食 + 饮料
```

### 剩余元素

```typescript
// 第一个是 string，后面可以有任意多个 number
let rest: [string, ...number[]] = ['分数', 90, 85, 88];

// 生活例子：
// 成绩单：姓名（固定）+ 多科成绩（任意数量）
type ScoreReport = [string, ...number[]];

const score1: ScoreReport = ['张三', 90, 85, 88];
const score2: ScoreReport = ['李四', 78, 92];
const score3: ScoreReport = ['王五', 100];
```

**剩余元素的理解：**
```
想象一个队伍：
- 队长站在最前面（固定）
- 队员可以任意多个站在后面（剩余）

[string, ...number[]]
   ↓        ↓
  队长     队员们（可以有 0 个或多个）
```

## 2.6 枚举类型（Enum）- 有限的选项

### 什么是枚举？

**枚举就是一组有限的、命名的常量**，就像星期一到星期日。

```typescript
// 生活例子：星期
// 只有 7 个选项：周一、周二、周三、周四、周五、周六、周日
// 不能是"周八"，不能是"星期 A"

// TypeScript 中
enum DayOfWeek {
    Monday,    // 0
    Tuesday,   // 1
    Wednesday, // 2
    Thursday,  // 3
    Friday,    // 4
    Saturday,  // 5
    Sunday     // 6
}

// 使用
let today: DayOfWeek = DayOfWeek.Monday;

// 数字枚举默认从 0 开始递增
console.log(DayOfWeek.Monday);  // 0
console.log(DayOfWeek.Tuesday); // 1
```

### 自定义起始值

```typescript
enum Status {
    Pending = 10,   // 10
    Processing = 20, // 20
    Completed = 30   // 30
}

// 或者每个都明确指定
enum Color {
    Red = 1,
    Green = 2,
    Blue = 3
}
```

### 字符串枚举

```typescript
// 更直观，推荐使用
enum Status {
    Pending = 'pending',
    Success = 'success',
    Error = 'error'
}

// 使用
const status1: Status = Status.Pending;
const status2: Status = Status.Success;

// 实际应用场景
enum OrderStatus {
    Pending = 'pending',      // 待支付
    Paid = 'paid',           // 已支付
    Shipped = 'shipped',     // 已发货
    Delivered = 'delivered', // 已送达
    Cancelled = 'cancelled'  // 已取消
}

interface Order {
    id: string;
    status: OrderStatus;
    amount: number;
}

const order: Order = {
    id: 'ORD-001',
    status: OrderStatus.Pending,
    amount: 100
};
```

### 枚举的好处

```typescript
// 不用枚举 - 容易写错
type Status = string;
const status1: Status = 'pendng';  // 拼写错误，但不会报错

// 用枚举 - 有提示，不会错
enum OrderStatus {
    Pending = 'pending',
    Success = 'success'
}
const status2: OrderStatus = OrderStatus.Pending;  // 有代码提示
```

## 2.7 any 类型 - 任意类型

### 什么是 any？

**any 表示"可以是任何类型"**，相当于关闭了类型检查。

```typescript
// 可以接受任何值
let value: any = 123;
value = 'hello';     // ✅
value = true;        // ✅
value = { a: 1 };    // ✅
value = [1, 2, 3];   // ✅

// 可以调用任何方法（不会报错，但可能运行时出错）
value.toFixed();
value.toUpperCase();
value.map(x => x);
```

### 为什么不推荐使用 any？

```typescript
// ❌ 问题：失去类型保护
function add(a: any, b: any): any {
    return a + b;
}

// 调用时不会检查类型
add(1, 2);        // 3 ✅
add('1', '2');    // '12' ❌ 可能不是你想要的
add(true, false); // 1 ❌ 完全没意义

// ✅ 正确做法：使用具体类型
function add2(a: number, b: number): number {
    return a + b;
}

add2(1, 2);        // 3 ✅
// add2('1', '2'); // ❌ 编译时就报错
```

### 什么时候可以使用 any？

```typescript
// 1. 第三方库没有类型定义
declare const lib: any;

// 2. 迁移旧代码时的过渡
function legacyFunction(data: any) {
    // TODO: 后续添加具体类型
}

// 3. 确实无法确定类型时（优先考虑 unknown）
```

## 2.8 unknown 类型 - 安全的 any

### 什么是 unknown？

**unknown 也表示"可以是任何类型"，但使用前必须检查类型**，是类型安全的 any。

```typescript
let value: unknown = 'hello';
value = 123;      // ✅
value = true;     // ✅
value = { a: 1 }; // ✅

// ❌ 不能直接使用
// let str: string = value;  // 错误

// ✅ 必须先检查类型
if (typeof value === 'string') {
    let str: string = value;  // 现在可以用了
    console.log(str.toUpperCase());
}
```

### any vs unknown 对比

```typescript
// any - 直接使用，不安全
let anyVal: any = 'hello';
console.log(anyVal.length);  // 可以直接访问

// unknown - 必须检查，安全
let unknownVal: unknown = 'hello';
// console.log(unknownVal.length);  // ❌ 错误

if (typeof unknownVal === 'string') {
    console.log(unknownVal.length);  // ✅ 检查后才能用
}
```

### 生活例子

```
any 就像"免检通道"：
- 什么东西都可以过
- 但可能有危险品混入

unknown 就像"安检通道"：
- 什么东西都可以带
- 但必须经过安检才能使用
```

## 2.9 void 类型 - 没有返回值

### 什么是 void？

**void 表示"没有返回值"**，用于函数不返回任何值的情况。

```typescript
// 函数没有返回值
function sayHello(): void {
    console.log('Hello!');
    // 没有 return 语句
}

// 或者省略返回类型（推荐）
function sayHello2() {
    console.log('Hello!');
}

// 箭头函数
const log = (): void => {
    console.log('日志');
};
```

### void vs undefined

```typescript
// void - 表示函数没有返回值
function log(): void {
    console.log('log');
    // 隐式返回 undefined
}

// undefined - 是一个具体的值
function getUndefined(): undefined {
    return undefined;
}

// 实际使用中，void 更常见
```

## 2.10 null 和 undefined - 空值

### 什么是 null 和 undefined？

```typescript
// null - 空值，表示"有意的空"
let empty: null = null;

// undefined - 未定义，表示"还没有值"
let notDefined: undefined = undefined;
```

### 严格空值检查

```typescript
// 开启 strictNullChecks 后
// let name: string = null;  // ❌ 错误

// 正确做法：明确声明可以是 null
let name: string | null = null;
let age: number | undefined = undefined;
```

### 可选链操作符（?.）- 安全访问属性

**什么是可选链？**

**可选链用于安全地访问嵌套属性，如果中间某个属性是 null 或 undefined，会直接返回 undefined，不会报错。**

```typescript
// 生活例子：
// 想象你要打开一个保险箱：
// 保险箱 → 抽屉 → 盒子 → 戒指
//
// 传统方式：每一步都要检查是否存在
// 可选链：如果中间任何一步没有，直接返回"没有戒指"

// 传统方式 - 很麻烦
interface User {
    name: string;
    address?: {
        city: string;
        street?: string;
    };
}

const user: User = { name: '张三' };

// 传统方式 - 要写很多判断
let city;
if (user.address) {
    city = user.address.city;
} else {
    city = undefined;
}

// 可选链方式 - 简洁
const city2 = user.address?.city;  // undefined

// 如果 address 存在
const user2: User = {
    name: '李四',
    address: {
        city: '北京',
        street: '朝阳路'
    }
};

const city3 = user2.address?.city;  // '北京'
```

**可选链的更多用法：**

```typescript
// 1. 访问嵌套属性
const city = user.address?.city;

// 2. 调用方法（如果方法存在）
const upperName = user.getName?.();

// 3. 访问数组元素
const firstTag = user.tags?.[0];

// 4. 链式调用
const value = obj?.prop?.method?.();
```

### 空值合并操作符（??）- 提供默认值

**什么是空值合并？**

**空值合并用于提供默认值，当左侧是 null 或 undefined 时，返回右侧的默认值。**

```typescript
// 传统方式 - 用 || 提供默认值
const name = userName || '默认用户';

// 问题：空字符串、0 也会被当作默认值
const count = userCount || 10;  // 如果 userCount 是 0，会得到 10 ❌

// 空值合并 - 只有 null/undefined 才用默认值
const count2 = userCount ?? 10;  // 如果 userCount 是 0，会得到 0 ✅

// 实际使用
interface Config {
    timeout?: number;
    retries?: number;
}

function createConfig(config: Config) {
    return {
        timeout: config.timeout ?? 5000,   // 默认 5000ms
        retries: config.retries ?? 3       // 默认 3 次
    };
}
```

### || vs ?? 对比

```typescript
// || - 只要左侧是"假值"就用右侧
'' || 'default'      // 'default'
0 || 10              // 10
false || true        // true
null ?? 'default'    // 'default'

// ?? - 只有左侧是 null/undefined 才用右侧
'' ?? 'default'      // ''
0 ?? 10              // 0
false ?? true        // false
null ?? 'default'    // 'default'
```

## 2.11 never 类型 - 永远不会发生

### 什么是 never？

**never 表示"永远不会发生的值"**，用于抛出错误或无限循环的函数。

```typescript
// 抛出错误的函数 - 永远不会正常返回
function error(message: string): never {
    throw new Error(message);
}

// 无限循环 - 永远不会结束
function infiniteLoop(): never {
    while (true) {}
}

// 使用场景：完整性检查
type Status = 'pending' | 'loading' | 'success' | 'error';

function handleStatus(status: Status): string {
    switch (status) {
        case 'pending':
            return '等待中';
        case 'loading':
            return '加载中';
        case 'success':
            return '成功';
        case 'error':
            return '错误';
        default:
            // 如果新增类型，这里会报错，提醒你需要处理
            const _exhaustive: never = status;
            return _exhaustive;
    }
}
```

### 生活例子

```
never 就像"死胡同"：
- 走进去了就出不来
- 要么抛错误（跳出来）
- 要么无限循环（永远走）
```

## 2.12 字面量类型 - 具体的值

### 什么是字面量类型？

**字面量类型表示"只能是某个具体的值"**，而不是一个类型范围。

```typescript
// 普通类型 - 可以是任何字符串
let name: string = '张三';  // 可以是任何字符串

// 字面量类型 - 只能是特定的值
type Direction = 'up' | 'down' | 'left' | 'right';

let dir: Direction = 'up';    // ✅
// let dir2: Direction = 'north';  // ❌ 错误：不在这四个值中

// 实际应用场景
type Status = 'pending' | 'loading' | 'success' | 'error';
type Method = 'GET' | 'POST' | 'PUT' | 'DELETE';
type Size = 'small' | 'medium' | 'large';
```

## 2.13 类型推断 - TypeScript 自动猜类型

### 什么是类型推断？

**类型推断是 TypeScript 自动猜测变量类型的功能**，不需要你显式声明。

```typescript
// TypeScript 会自动推断
let name = '张三';    // 推断为 string
let age = 25;         // 推断为 number
let isActive = true;  // 推断为 boolean

// 函数返回值也会推断
function add(a: number, b: number) {
    return a + b;  // 推断返回类型为 number
}

// 数组推断
const numbers = [1, 2, 3];  // 推断为 number[]
```

### 什么时候需要显式声明？

```typescript
// 1. 变量初始化为 any 时
let data: string[] = [];  // 需要声明

// 2. 函数参数必须声明
function greet(name: string) {  // 必须声明
    return `Hello, ${name}`;
}

// 3. 对象字面量
const user: { name: string; age: number } = {
    name: '张三',
    age: 25
};
```

## 2.14 类型断言 - 你比 TS 更懂

### 什么是类型断言？

**类型断言是告诉 TypeScript："相信我，我知道这个类型是什么"**。

```typescript
// 场景：TypeScript 不知道具体类型，但你知道
let value: unknown = 'hello';

// 方式 1：as 语法（推荐）
let str: string = value as string;

// 方式 2：尖括号语法（与 JSX 冲突）
let str2: string = <string>value;
```

### 实际应用场景

```typescript
// 1. DOM 元素
const canvas = document.getElementById('myCanvas') as HTMLCanvasElement;

// 2. JSON 解析
const data = JSON.parse(jsonString) as UserData;

// 3. 测试代码
const mockUser = { id: 1, name: '测试' } as User;
```

### as const - 常量断言

```typescript
// 普通对象 - 类型会被推断为宽泛的类型
const obj1 = { x: 1, y: 2 };
// obj1.x 的类型是 number

// as const - 推断为精确的字面量类型
const obj2 = { x: 1, y: 2 } as const;
// obj2.x 的类型是 1（字面量）

// 实际使用
const roles = ['admin', 'user'] as const;
type Role = typeof roles[number];  // 'admin' | 'user'
```

## 2.15 satisfies 操作符（TS 4.9+）- 验证类型

### 什么是 satisfies？

**satisfies 用于验证类型，同时保留字面量类型信息**。

```typescript
// 对比 as
const config1 = {
    port: 3000,
    host: 'localhost'
} as { port: number; host: string };

// config1.port 的类型是 number

// 使用 satisfies
const config2 = {
    port: 3000,
    host: 'localhost'
} satisfies { port: number; host: string };

// config2.port 的类型是 3000（字面量）
```

### 生活例子

```
as 就像"强制转换"：
- 把这个东西当成那个类型用
- 但会丢失原有信息

satisfies 就像"质量检查"：
- 检查是否符合标准
- 但保留原有特征
```

### 练习 2：类型守卫

```typescript
// 类型检查函数
function isString(value: unknown): value is string {
    return typeof value === 'string';
}

function isNumber(value: unknown): value is number {
    return typeof value === 'number' && !isNaN(value);
}

// 使用
function process(value: unknown) {
    if (isString(value)) {
        console.log(value.toUpperCase());
    } else if (isNumber(value)) {
        console.log(value.toFixed(2));
    }
}
```

## 2.17 常见问答

### Q1: any 和 unknown 到底有什么区别？

**答：**

| 特性 | any | unknown |
|------|-----|---------|
| 类型安全 | ❌ 不安全 | ✅ 安全 |
| 直接使用 | ✅ 可以 | ❌ 必须先检查 |
| 推荐度 | 不推荐 | 推荐 |

```typescript
// any - 危险
let anyVal: any = 'hello';
console.log(anyVal.length);  // 可以直接访问，但不安全

// unknown - 安全
let unknownVal: unknown = 'hello';
// console.log(unknownVal.length);  // ❌ 错误

if (typeof unknownVal === 'string') {
    console.log(unknownVal.length);  // ✅ 检查后才能用
}
```

### Q2: 可选链？.和空值合并？？什么时候用？

**答：**
- `?.` 用于**安全访问属性**（如果属性不存在，返回 undefined）
- `??` 用于**提供默认值**（只有 null/undefined 才用默认值）

```typescript
// 可选链
const city = user.address?.city;  // 如果 address 不存在，返回 undefined

// 空值合并
const timeout = config.timeout ?? 5000;  // 如果 timeout 是 null/undefined，用 5000
```

### Q3: 什么时候用 as const？

**答：** 当你需要保留字面量类型信息时。

```typescript
// 普通推断
const roles = ['admin', 'user'];
// roles 的类型是 string[]

// as const
const roles2 = ['admin', 'user'] as const;
// roles2 的类型是 readonly ['admin', 'user']
```
