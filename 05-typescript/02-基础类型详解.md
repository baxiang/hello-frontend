## 2.1 布尔类型（boolean）- 是非判断

### 2.1.1 什么是布尔类型？

```
┌─────────────────────────────────────────────────────────────┐
│  布尔类型 = 只有两个值的类型                                │
│                                                             │
│  - true（真/是/开）                                         │
│  - false（假/否/关）                                        │
│                                                             │
│  就像：                                                     │
│  - 电灯开关：开/关                                           │
│  - 门：开/关                                               │
│  - 开关：on/off                                            │
└─────────────────────────────────────────────────────────────┘
```

### 2.1.2 基本用法

```typescript
// 声明布尔类型
let isLightOn: boolean = true;   // 灯开着
let isDoorOpen: boolean = false; // 门关着

// 实际应用场景
interface User {
    id: number;
    name: string;
    isActive: boolean;        // 用户是否激活
    isVip: boolean;          // 是否 VIP 用户
    isEmailVerified: boolean; // 邮箱是否验证
}

const user: User = {
    id: 1,
    name: '张三',
    isActive: true,       // 已激活
    isVip: false,         // 不是 VIP
    isEmailVerified: true // 邮箱已验证
};
```

### 2.1.3 布尔运算

```typescript
// 与运算（&&）- 两个都为 true 才为 true
const a = true && true;   // true
const b = true && false;  // false

// 或运算（||）- 有一个为 true 就为 true
const c = true || false;  // true
const d = false || false; // false

// 非运算（!）- 取反
const e = !true;  // false
const f = !false; // true
```

## 2.2 数字类型（number）- 所有数字

### 2.2.1 TypeScript 只有一种数字类型

```
┌─────────────────────────────────────────────────────────────┐
│  JavaScript/TypeScript 的 number = 所有数字类型             │
│                                                             │
│  - 整数：1, 2, 100, -5                                      │
│  - 小数：3.14, -0.5, 0.001                                  │
│  - 正数/负数                                               │
│  - 特殊值：Infinity, NaN                                    │
│                                                             │
│  不像某些语言区分 int、float、double                       │
└─────────────────────────────────────────────────────────────┘
```

### 2.2.2 基本用法

```typescript
// 整数
let age: number = 25;
let count: number = -10;

// 小数
let price: number = 99.99;
let pi: number = 3.14159;

// 特殊值
let infinity: number = Infinity;  // 无穷大
let notANumber: number = NaN;     // 不是数字

// 实际应用场景
interface Product {
    id: number;
    name: string;
    price: number;      // 价格
    stock: number;      // 库存
    discount: number;  // 折扣（0-1 之间的小数）
    rating: number;     // 评分（0-5 之间）
}
```

### 2.2.3 数字分隔符

```typescript
// 没有分隔符 - 很难数清楚
let salary = 1000000;

// 有分隔符 - 一眼看出是 100 万
let salary2 = 1_000_000;

// 可以这样用
let million = 1_000_000;      // 100 万
let billion = 1_000_000_000;  // 10 亿
let phone = 138_0013_8000;    // 手机号

// 分隔符不影响实际值
console.log(1000000 === 1_000_000);  // true
```

## 2.3 字符串类型（string）- 文本内容

### 2.3.1 什么是字符串？

```
┌─────────────────────────────────────────────────────────────┐
│  字符串 = 文本内容 = 用引号包裹的字符序列                      │
│                                                             │
│  三种写法：                                                  │
│  - 单引号：'hello'                                          │
│  - 双引号："hello"                                          │
│  - 反引号：`hello`（模板字符串）                             │
│                                                             │
│  就像：                                                     │
│  - 一篇文章                                                 │
│  - 一句话                                                   │
│  - 一个名字                                                 │
└─────────────────────────────────────────────────────────────┘
```

### 2.3.2 基本用法

```typescript
// 单引号
let name1: string = '张三';

// 双引号
let name2: string = '李四';

// 反引号（模板字符串）
let name3: string = `王五`;

// 模板字符串 - 可以嵌入变量
const userName = '张三';
const age = 25;

// 普通拼接 - 麻烦
const message1 = '你好，我叫' + userName + '，今年' + age + '岁';

// 模板字符串 - 简洁
const message2 = `你好，我叫${userName}，今年${age}岁`;

console.log(message2);  // 你好，我叫张三，今年 25 岁
```

### 2.3.3 多行字符串

```typescript
// 普通字符串 - 多行很麻烦
const html1 = '<div>\n' +
    '  <h1>标题</h1>\n' +
    '  <p>内容</p>\n' +
    '</div>';

// 模板字符串 - 多行很简单
const html2 = `
<div>
  <h1>标题</h1>
  <p>内容</p>
</div>
`;
```

## 2.4 数组类型 - 一组数据

### 2.4.1 什么是数组？

```
┌─────────────────────────────────────────────────────────────┐
│  数组 = 存储一组有序数据的容器                              │
│                                                             │
│  特点：                                                     │
│  - 有序（按索引访问，0 开始）                              │
│  - 同类型（通常存放相同类型的数据）                          │
│  - 可变（可以添加、删除元素）                               │
│                                                             │
│  就像：                                                     │
│  - 购物清单                                                 │
│  - 学生名单                                                 │
│  - 成绩列表                                                 │
└─────────────────────────────────────────────────────────────┘
```

### 2.4.2 数组类型声明

```typescript
// 方式 1：类型 + 方括号
let numbers: number[] = [1, 2, 3, 4, 5];
let names: string[] = ['张三', '李四', '王五'];
let flags: boolean[] = [true, false, true];

// 方式 2：泛型 Array
let numbers2: Array<number> = [1, 2, 3];
let names2: Array<string> = ['张三', '李四'];

// 混合类型数组（不太推荐）
let mixed: (string | number)[] = [1, 'hello', 2, 'world'];
```

### 2.4.3 数组操作

```typescript
const arr: number[] = [1, 2, 3, 4, 5];

// 访问元素
arr[0];   // 1
arr[4];   // 5

// 添加元素
arr.push(6);      // 末尾添加
arr.unshift(0);   // 开头添加

// 删除元素
arr.pop();         // 末尾删除
arr.shift();       // 开头删除

// 数组长度
arr.length;  // 5
```

## 2.5 元组类型 - 固定长度数组

### 2.5.1 什么是元组？

```
┌─────────────────────────────────────────────────────────────┐
│  元组 = 固定长度 + 固定类型的数组                            │
│                                                             │
│  与数组的区别：                                             │
│  - 数组：长度可变，类型相同                                 │
│  - 元组：长度固定，类型可以不同                             │
│                                                             │
│  就像：                                                     │
│  - 数组：可伸缩的抽屉                                       │
│  - 元组：固定数量的抽屉，每个抽屉类型固定                    │
└─────────────────────────────────────────────────────────────┘
```

### 2.5.2 基本用法

```typescript
// 定义元组
let user: [string, number] = ['张三', 25];

// 访问
user[0];  // '张三'
user[1];  // 25

// 赋值
user[0] = '李四';
user[1] = 30;

// 实际应用场景
// 坐标
let point: [number, number] = [100, 200];

// RGB 颜色
let color: [number, number, number] = [255, 0, 0];

// 姓名和年龄
let person: [string, number] = ['张三', 25];
```

## 2.6 枚举类型 - 命名常量

### 2.6.1 什么是枚举？

```
┌─────────────────────────────────────────────────────────────┐
│  枚举 = 一组命名常量的集合                                  │
│                                                             │
│  作用：                                                     │
│  - 给数值起名字，更易读                                      │
│  - 限制取值范围                                             │
│  - 代码更清晰                                               │
│                                                             │
│  就像：                                                     │
│  - 星期几：Monday, Tuesday, ...                             │
│  - 颜色：Red, Green, Blue                                   │
│  - 状态：Pending, Success, Error                            │
└─────────────────────────────────────────────────────────────┘
```

### 2.6.2 基本用法

```typescript
// 数字枚举（默认从 0 开始）
enum Color {
    Red,    // 0
    Green,  // 1
    Blue   // 2
}

const c1: Color = Color.Red;  // 0
const c2: Color = Color.Green;  // 1

// 字符串枚举
enum Status {
    Pending = 'pending',
    Success = 'success',
    Error = 'error'
}

const s1: Status = Status.Pending;  // 'pending'

// 常量枚举（编译时内联）
const enum Direction {
    Up = 'UP',
    Down = 'DOWN',
    Left = 'LEFT',
    Right = 'RIGHT'
}

const d: Direction = Direction.Up;  // 'UP'
```

## 2.7 any 类型 - 任意类型

### 2.7.1 什么是 any？

```
┌─────────────────────────────────────────────────────────────┐
│  any = 任意类型 = 关闭类型检查                             │
│                                                             │
│  特点：                                                     │
│  - 可以赋值为任何类型                                       │
│  - 可以访问任何属性                                         │
│  - 可以调用任何方法                                         │
│  - 不会有编译错误                                           │
│                                                             │
│  警告：                                                     │
│  - 尽量少用 any，否则失去 TypeScript 的意义                │
│  - 只在无法确定类型时使用                                   │
└─────────────────────────────────────────────────────────────┘
```

### 2.7.2 基本用法

```typescript
// 声明为 any
let value: any;

value = 1;         // OK
value = 'hello';    // OK
value = true;       // OK
value = {};         // OK
value = [1, 2, 3];  // OK

// 可以做任何操作
value.foo;              // OK
value.bar();             // OK
value[0];               // OK
```

### 2.7.3 any 的使用场景

```typescript
// 场景 1：处理未知类型的数据
function processData(data: any): void {
    console.log(data.name);  // 不报错
    console.log(data.value); // 不报错
}

// 场景 2：逐步迁移 JavaScript 到 TypeScript
// 旧代码可能是 any
let oldCode: any = getOldCode();
oldCode.someMethod();

// 场景 3：第三方库没有类型定义
// import _ from 'some-vanilla-js-library';  // any
```

## 2.8 unknown 类型 - 安全的 any

### 2.8.1 什么是 unknown？

```
┌─────────────────────────────────────────────────────────────┐
│  unknown = 安全的 any                                      │
│                                                             │
│  与 any 的区别：                                            │
│  - any：可以随意操作                                       │
│  - unknown：需要类型检查后才能操作                          │
│                                                             │
│  就像：                                                     │
│  - any：不知道里面有什么，但可以直接用                       │
│  - unknown：不知道里面有什么，需要先检查                    │
└─────────────────────────────────────────────────────────────┘
```

### 2.8.2 基本用法

```typescript
let value: unknown;

value = 1;         // OK
value = 'hello';    // OK
value = true;       // OK

// 不能直接操作
// value.foo;      // ❌ 错误
// value();        // ❌ 错误

// 需要类型 narrowing（收窄）
if (typeof value === 'string') {
    console.log(value.toUpperCase());  // OK
}

if (value instanceof Array) {
    console.log(value.length);  // OK
}
```

## 2.9 void 类型 - 空值

### 2.9.1 什么是 void？

```
┌─────────────────────────────────────────────────────────────┐
│  void = 没有返回值                                         │
│                                                             │
│  使用场景：                                                  │
│  - 函数没有返回值                                           │
│  - 只有副作用（如 console.log）                             │
│                                                             │
│  就像：                                                     │
│  - 打印机：执行打印，但不返回东西                           │
│  - 事件处理：执行操作，但不返回结果                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.9.2 基本用法

```typescript
// 函数返回 void
function greet(name: string): void {
    console.log(`Hello, ${name}!`);
    // 没有 return 语句，或者 return;
}

// 变量声明为 void（很少用）
let useless: void = undefined;
```

## 2.10 never 类型 - 从不返回

### 2.10.1 什么是 never？

```
┌─────────────────────────────────────────────────────────────┐
│  never = 从不返回 = 永远不返回                             │
│                                                             │
│  使用场景：                                                  │
│  - 抛出异常的函数                                            │
│  - 无限循环的函数                                            │
│  - 类型守卫中永远为 false 的分支                             │
│                                                             │
│  就像：                                                     │
│  - 永远到不了的终点                                         │
│  - 永远不会执行的代码                                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.10.2 基本用法

```typescript
// 抛出异常的函数
function error(message: string): never {
    throw new Error(message);
}

// 无限循环
function infiniteLoop(): never {
    while (true) {
        // 永远不返回
    }
}

// 类型守卫
function process(value: string | number) {
    if (typeof value === 'string') {
        // value 是 string
    } else if (typeof value === 'number') {
        // value 是 number
    } else {
        // value 是 never，永远不会执行到这里
        const _exhaustive: never = value;
    }
}
```

## 2.11 null 和 undefined

### 2.11.1 基本概念

```typescript
// TypeScript 中，null 和 undefined 是两种类型

// null - 有意的"空"
let user: string | null = null;  // 用户不存在
user = '张三';  // 用户存在

// undefined - 未定义
let value: string | undefined;  // 还没赋值
value = 'hello';
value = undefined;
```

### 2.11.2 strictNullChecks

```typescript
// tsconfig.json 中 strictNullChecks 的影响

// strictNullChecks: false（默认）
let name1: string = undefined;  // OK
let name2: string = null;       // OK

// strictNullChecks: true
let name3: string = undefined;  // ❌
let name4: string = null;        // ❌

// 正确写法
let name5: string | undefined = undefined;
let name6: string | null = null;
let name7: string | undefined | null = undefined;
```

## 2.13 常见问答

### Q1: any 和 unknown 有什么区别？

```typescript
// any：可以随意操作
let a: any = 'hello';
a.toUpperCase();  // OK

// unknown：需要先检查
let b: unknown = 'hello';
// b.toUpperCase();  // ❌ 错误
if (typeof b === 'string') {
    b.toUpperCase();  // OK
}
```

### Q2: 什么时候用 void，什么时候用 never？

```typescript
// void：函数正常返回（即使没有 return）
function log(msg: string): void {
    console.log(msg);
}

// never：函数永不返回
function throwError(msg: string): never {
    throw new Error(msg);
}
```
