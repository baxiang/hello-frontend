## 6.1 接口基础

### 什么是接口

接口用于定义对象的结构：

```typescript
interface User {
    id: number;
    name: string;
    email: string;
}

// 使用接口
const user: User = {
    id: 1,
    name: 'Tom',
    email: 'tom@example.com'
};
```

### 接口 vs 类型别名

```typescript
// 接口
interface User {
    name: string;
    age: number;
}

// 类型别名
type User = {
    name: string;
    age: number;
};
```

两者功能相似，接口可以继承和实现。

## 6.2 可选属性

### 使用 ?

```typescript
interface User {
    id: number;
    name: string;
    email?: string;  // 可选属性
}

// 两种都合法
const user1: User = { id: 1, name: 'Tom' };
const user2: User = { id: 1, name: 'Tom', email: 'tom@example.com' };
```

### 实际应用

```typescript
interface SearchParams {
    keyword: string;
    page?: number;
    limit?: number;
}

function search(params: SearchParams) {
    const page = params.page || 1;
    const limit = params.limit || 10;
    // ...
}

search({ keyword: 'TypeScript' });
search({ keyword: 'TypeScript', page: 2 });
```

## 6.3 只读属性

### 使用 readonly

```typescript
interface User {
    readonly id: number;
    name: string;
}

const user: User = { id: 1, name: 'Tom' };
user.id = 2;      // ❌ 错误：id 是只读的
user.name = 'Jerry';  // ✅
```

### 实际应用

```typescript
interface Config {
    readonly apiUrl: string;
    readonly apiKey: string;
}

const config: Config = {
    apiUrl: 'https://api.example.com',
    apiKey: 'secret-key'
};

// 防止配置被修改
```

## 6.4 接口继承

### 单继承

```typescript
interface Person {
    name: string;
    age: number;
}

interface User extends Person {
    email: string;
}

const user: User = {
    name: 'Tom',
    age: 25,
    email: 'tom@example.com'
};
```

### 多继承

```typescript
interface A {
    a: string;
}

interface B {
    b: number;
}

interface C extends A, B {
    c: boolean;
}

const value: C = {
    a: 'hello',
    b: 123,
    c: true
};
```

## 6.5 实现接口

### 类实现接口

```typescript
interface Printable {
    print(): void;
}

class Document implements Printable {
    print() {
        console.log('Printing document...');
    }
}

class Image implements Printable {
    print() {
        console.log('Printing image...');
    }
}
```

### 多接口实现

```typescript
interface Flyable {
    fly(): void;
}

interface Swimmable {
    swim(): void;
}

class Duck implements Flyable, Swimmable {
    fly() {
        console.log('Duck flying');
    }
    swim() {
        console.log('Duck swimming');
    }
}
```

## 6.6 索引签名

### 什么是索引签名

允许动态添加属性：

```typescript
interface StringMap {
    [key: string]: string;
}

const map: StringMap = {
    name: 'Tom',
    city: 'Beijing'
};
```

### 实际应用

```typescript
interface UserScores {
    [subject: string]: number;
}

const scores: UserScores = {
    math: 90,
    english: 85,
    chinese: 92
};
```

### 混合索引签名

```typescript
interface User {
    name: string;
    [key: string]: string | number;
}

const user: User = {
    name: 'Tom',
    age: 25,
    city: 'Beijing'
};
```

## 6.7 函数类型接口

### 接口定义函数

```typescript
interface AddFn {
    (a: number, b: number): number;
}

const add: AddFn = (a, b) => a + b;
```

### 完整示例

```typescript
interface Callback {
    (error: Error | null, result?: string): void;
}

function fetchData(callback: Callback) {
    // 模拟异步操作
    callback(null, 'Data loaded');
}

fetchData((err, result) => {
    if (err) {
        console.error(err);
    } else {
        console.log(result);
    }
});
```

## 6.8 泛型接口

### 基本用法

```typescript
interface Response<T> {
    code: number;
    message: string;
    data: T;
}

interface User {
    id: number;
    name: string;
}

const userResponse: Response<User> = {
    code: 200,
    message: 'success',
    data: { id: 1, name: 'Tom' }
};
```

### 实际应用

```typescript
interface ApiResponse<T> {
    data: T;
    status: number;
    timestamp: Date;
}

interface Product {
    id: number;
    name: string;
    price: number;
}

const productResponse: ApiResponse<Product[]> = {
    data: [
        { id: 1, name: 'Apple', price: 5 },
        { id: 2, name: 'Banana', price: 3 }
    ],
    status: 200,
    timestamp: new Date()
};
```

## 6.9 实践练习

### 练习 1：定义用户接口

```typescript
interface User {
    id: number;
    name: string;
    email: string;
    age?: number;        // 可选
    readonly createdAt: Date;  // 只读
}

const user: User = {
    id: 1,
    name: 'Tom',
    email: 'tom@example.com',
    createdAt: new Date()
};
```

### 练习 2：接口继承

```typescript
interface Person {
    name: string;
    age: number;
}

interface Employee extends Person {
    company: string;
    salary: number;
}

const employee: Employee = {
    name: 'Tom',
    age: 25,
    company: 'ABC',
    salary: 10000
};
```

### 练习 3：实现接口

```typescript
interface Logger {
    log(message: string): void;
    error(message: string): void;
}

class ConsoleLogger implements Logger {
    log(message: string) {
        console.log(`[LOG] ${message}`);
    }
    error(message: string) {
        console.error(`[ERROR] ${message}`);
    }
}
```

### 练习 4：索引签名

```typescript
interface StringDictionary {
    [key: string]: string;
}

const dict: StringDictionary = {
    hello: '你好',
    world: '世界',
    typescript: 'TypeScript'
};
```

## 本章小结

| 特性 | 语法 | 说明 |
|------|------|------|
| 可选属性 | `prop?: type` | 属性可能不存在 |
| 只读属性 | `readonly prop: type` | 创建后不能修改 |
| 接口继承 | `extends` | 继承其他接口 |
| 实现接口 | `implements` | 类实现接口 |
| 索引签名 | `[key: string]: type` | 动态属性 |
| 泛型接口 | `interface<T>` | 支持多种类型 |
