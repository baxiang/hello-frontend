## 12.0 通俗理解 Type Guard（类型守卫）

### 为什么叫"守卫"？

```
Guard = 守卫/检查/把关

类型守卫 = 在代码中"检查"类型 = 告诉 TypeScript 当前是什么类型
```

### 通俗比喻

```
场景：你在管理一个通道，只允许特定类型的人通过

类型守卫 = 门口的检查员
- 检查"你是谁"
- 决定"你能做什么"

代码中：
- if (typeof value === 'string')  = 检查 value 是字符串
- 检查通过后，TypeScript 就知道"这是字符串"
- 可以安全地调用字符串的方法（toUpperCase 等）
```

### 什么是类型收窄？

```
收窄 = 范围变小

类型收窄 = 从"宽类型"变成"窄类型"

举例：
string | number  →  string（检查后）
     ↑              ↑
   宽类型         窄类型
```

### 实际例子

```typescript
// 原始类型：string | number（可以是字符串或数字）
function process(value: string | number) {
    // 类型守卫：检查
    if (typeof value === 'string') {
        // 收窄后：value 变成 string
        return value.toUpperCase();  // ✅ 可以用字符串方法
    }
    // 收窄后：value 变成 number
    return value.toFixed(2);  // ✅ 可以用数字方法
}
```

### 守卫的种类

| 守卫方式 | 作用 | 例子 |
|----------|------|------|
| typeof | 检查原始类型 | `typeof x === 'string'` |
| instanceof | 检查类实例 | `x instanceof Array` |
| in | 检查对象属性 | `'name' in obj` |
| 自定义 | 自定义检查逻辑 | `isString(x)` |

## 12.1 typeof 类型守卫

### 基本用法

```typescript
function process(value: string | number): string | number {
    if (typeof value === 'string') {
        // value 在这里是 string 类型
        return value.toUpperCase();
    }
    // value 在这里是 number 类型
    return value.toFixed(2);
}
```

### 支持的类型

```typescript
typeof 'hello' === 'string'
typeof 123 === 'number'
typeof true === 'boolean'
typeof undefined === 'undefined'
typeof function() {} === 'function'
typeof [] === 'object'
```

## 12.2 instanceof 守卫

### 解决什么问题？

```
场景：区分不同的类（class）

问题：typeof 只能区分原始类型，无法区分类
解决：使用 instanceof 检查实例
```

### 通俗理解

```
instanceof = "是谁的实例"

Dog instanceof Dog = "这条狗是 Dog 类的实例吗？"
```

### 基本用法

```typescript
class Dog {
    bark(): void {
        console.log('Woof!');
    }
}

class Cat {
    meow(): void {
        console.log('Meow!');
    }
}

function makeSound(animal: Dog | Cat): void {
    // 检查 animal 是否是 Dog 的实例
    if (animal instanceof Dog) {
        animal.bark();  // ✅ TypeScript 知道这是 Dog
    } else {
        animal.meow();  // ✅ TypeScript 知道这是 Cat
    }
}
```

### 对比：typeof vs instanceof

```typescript
// typeof：只能区分原始类型
typeof 'hello' === 'string'  // ✅
typeof 123 === 'number'     // ✅

// instanceof：区分类（class）
class Dog {}
const dog = new Dog();
dog instanceof Dog   // ✅
dog instanceof Cat   // false
```

### 实际应用场景

```typescript
// 场景：处理不同类型的错误
class ValidationError extends Error {
    field: string;
}

class NetworkError extends Error {
    statusCode: number;
}

function handleError(error: ValidationError | NetworkError) {
    if (error instanceof ValidationError) {
        // 收窄为 ValidationError
        console.log('验证错误:', error.field);
    } else if (error instanceof NetworkError) {
        // 收窄为 NetworkError
        console.log('网络错误:', error.statusCode);
    }
}
```

## 12.3 in 操作符守卫

### 解决什么问题？

```
场景：区分不同的接口（interface）

问题：接口在编译后会被删除，无法使用 instanceof
解决：使用 in 检查属性是否存在
```

### 通俗理解

```
in = "有没有这个属性"

'swim' in fish = "鱼有没有 swim 这个属性？"
```

### 基本用法

```typescript
interface Fish {
    swim(): void;
}

interface Bird {
    fly(): void;
}

function move(animal: Fish | Bird): void {
    // 检查 animal 是否有 swim 属性
    if ('swim' in animal) {
        animal.swim();  // ✅ TypeScript 知道这是 Fish
    } else {
        animal.fly();   // ✅ TypeScript 知道这是 Bird
    }
}
```

### 实际应用场景

```typescript
// 场景：API 返回不同形状的数据
interface SuccessResponse {
    status: 'success';
    data: string[];
}

interface ErrorResponse {
    status: 'error';
    message: string;
}

function handleResponse(response: SuccessResponse | ErrorResponse) {
    if ('data' in response) {
        // 收窄为 SuccessResponse
        console.log('数据:', response.data);
    } else {
        // 收窄为 ErrorResponse
        console.log('错误:', response.message);
    }
}
```

### 对比：接口 vs 类

| 检查方式 | 用于 | 例子 |
|----------|------|------|
| instanceof | 类（class） | `dog instanceof Dog` |
| in | 接口（interface） | `'swim' in animal` |

### 为什么接口不能用 instanceof？

```typescript
// 接口在编译后会被删除，不存在于运行时
interface User {
    name: string;
}

// 编译后变成：
// interface 完全消失

// 所以不能用 instanceof
// user instanceof User  // ❌ 错误！

// 只能用 in 检查属性
'name' in user  // ✅ 可以
```

## 12.4 类型谓词（is）

### 什么是类型谓词

返回类型是 `x is Type` 的函数：

```typescript
interface Fish {
    swim(): void;
}

interface Bird {
    fly(): void;
}

function isFish(animal: Fish | Bird): animal is Fish {
    return (animal as Fish).swim !== undefined;
}

function move(animal: Fish | Bird): void {
    if (isFish(animal)) {
        animal.swim();
    } else {
        animal.fly();
    }
}
```

### 实际应用

```typescript
interface User {
    type: 'user';
    name: string;
}

interface Admin {
    type: 'admin';
    permissions: string[];
}

function isAdmin(person: User | Admin): person is Admin {
    return person.type === 'admin';
}

function handle(person: User | Admin) {
    if (isAdmin(person)) {
        console.log('Admin permissions:', person.permissions);
    } else {
        console.log('User name:', person.name);
    }
}
```

## 12.5 可辨识联合

### 完整示例

```typescript
interface Square {
    kind: 'square';
    size: number;
}

interface Circle {
    kind: 'circle';
    radius: number;
}

interface Triangle {
    kind: 'triangle';
    base: number;
    height: number;
}

type Shape = Square | Circle | Triangle;

function getArea(shape: Shape): number {
    switch (shape.kind) {
        case 'square':
            return shape.size * shape.size;
        case 'circle':
            return Math.PI * shape.radius * shape.radius;
        case 'triangle':
            return 0.5 * shape.base * shape.height;
    }
}
```

## 12.6 完整性检查

### 使用 never

```typescript
type Shape = 
    | { kind: 'circle'; radius: number }
    | { kind: 'square'; size: number }
    | { kind: 'triangle'; base: number; height: number };

function getArea(shape: Shape): number {
    switch (shape.kind) {
        case 'circle':
            return Math.PI * shape.radius ** 2;
        case 'square':
            return shape.size ** 2;
        case 'triangle':
            return 0.5 * shape.base * shape.height;
        default:
            // 完整性检查：如果漏掉类型，TS 会报错
            const _exhaustive: never = shape;
            throw new Error('Unknown shape');
    }
}
```

### 添加新类型时的提示

如果添加新的 Shape 类型但忘记处理，编译时会报错：

```typescript
// 添加新类型
interface Pentagon {
    kind: 'pentagon';
    side: number;
}

type Shape = Square | Circle | Triangle | Pentagon;

// 编译错误：存在未处理的类型
function getArea(shape: Shape): number { ... }
```

## 12.7 实践练习

### 练习 1：typeof 守卫

```typescript
function format(value: string | number | boolean): string {
    if (typeof value === 'string') {
        return value.toUpperCase();
    } else if (typeof value === 'number') {
        return value.toFixed(2);
    } else {
        return value ? 'Yes' : 'No';
    }
}
```

### 练习 2：自定义类型守卫

```typescript
interface Person {
    name: string;
    age: number;
}

interface Company {
    name: string;
    employees: number;
}

function isPerson(obj: Person | Company): obj is Person {
    return 'age' in obj;
}

function describe(value: Person | Company): string {
    if (isPerson(value)) {
        return `${value.name} is ${value.age} years old`;
    } else {
        return `${value.name} has ${value.employees} employees`;
    }
}
```

### 练习 3：可辨识联合

```typescript
interface Loading {
    status: 'loading';
}

interface Success {
    status: 'success';
    data: string[];
}

interface Error {
    status: 'error';
    message: string;
}

type AsyncData = Loading | Success | Error;

function handleAsyncData(data: AsyncData): void {
    switch (data.status) {
        case 'loading':
            console.log('Loading...');
            break;
        case 'success':
            console.log('Data:', data.data);
            break;
        case 'error':
            console.log('Error:', data.message);
            break;
    }
}
```

|----------|------|----------|
