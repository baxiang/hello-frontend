```

│                  容器形状                             │
├──────────────────────────────────────────────────────┤
│                                                      │
│    string = 长条形容器                                │
│    ─────────────                                     │
│    只能装文字："Hello"、"张三"                       │
│                                                      │
│    number = 数字容器                                  │
│    ─────────────                                     │
│    只能装数字：42、3.14、-1                          │
│                                                      │
│    boolean = 开关容器                                 │
│    ─────────────                                     │
│    只能装两个值：true 或 false                       │
│                                                      │
│    any = 万能容器                                     │
│    ─────────────                                     │
│    什么都能装，但失去了容器的意义                    │
│    （不推荐用）                                      │
│                                                      │
│    unknown = 密封容器                                 │
│    ─────────────                                     │
│    能装任何东西，但打开前必须先检查里面是什么        │
│    （比 any 安全）                                   │
│                                                      │
│    never = 空容器                                     │
│    ─────────────                                     │

│                                                      │
│    tuple = 组合餐盒                                   │
│    ─────────────                                     │
│    第一格必须放汉堡，第二格放薯条                    │

│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

### 基础类型

**最简示例（1-3行）：**

```typescript
let name: string = '张三';
let age: number = 25;
let active: boolean = true;
```

**详细示例：**

```typescript
// string
let greeting: string = 'Hello';
let template: string = `Hello, ${name}!`;

// number（整数、小数、二进制、十六进制）
let integer: number = 42;
let float: number = 3.14;
let hex: number = 0xf00d;
let binary: number = 0b1010;
let big: bigint = 9007199254740991n;

// boolean
let isActive: boolean = true;
let isEmpty: boolean = false;

// null 和 undefined
let empty: null = null;
let notDefined: undefined = undefined;

// 数组
let numbers: number[] = [1, 2, 3];
let names: Array<string> = ['Tom', 'Jerry'];

// 元组（固定长度和类型）
let tuple: [string, number] = ['Tom', 25];
tuple[0].toUpperCase();  // ✅ string 方法
tuple[1].toFixed(2);     // ✅ number 方法

// 枚举
enum Direction { Up, Down, Left, Right }
let dir: Direction = Direction.Up;  // 0

// 字符串枚举（推荐，更易读）
enum Status {
    Pending = 'pending',
    Success = 'success',
    Error = 'error'
}
```

---

### any vs unknown vs never

```typescript
// any — 放弃类型检查（不推荐）
let data: any = fetchData();
data.foo();  // ✅ 编译通过，运行时报错

// unknown — 安全的 any
let data: unknown = fetchData();
// data.foo();  // ❌ 编译错误
if (typeof data === 'object' && data !== null) {
    // 必须先检查类型才能使用
}

// never — 永不返回
function throwError(msg: string): never {
    throw new Error(msg);
}

function infiniteLoop(): never {
    while (true) {}
}
```

## L2 实践层：用好

### 类型选择指南

| 类型 | 用途 | 示例 |
|------|------|------|
| `string` | 文本 | `let name: string` |
| `number` | 数字 | `let age: number` |
| `boolean` | 真假 | `let active: boolean` |
| `any` | 不推荐 | 避免使用 |
| `unknown` | 不确定类型 | API 返回值 |
| `never` | 永不返回 | 错误处理函数 |
| `void` | 无返回值 | `function fn(): void` |
| `tuple` | 固定长度数组 | `[string, number]` |

### 反模式

```typescript
// ❌ 错误：用 null 表示默认值
let name: string = null;

// ✅ 正确：用 undefined 或可选属性
let name: string | undefined;
```

```typescript
// ❌ 错误：元组越界访问
const pair: [string, number] = ['age', 25];
pair[2];  // 编译错误

// ✅ 正确：在元组长度内访问
pair[0];  // 'age'
pair[1];  // 25
```

### 适用场景

| 场景 | 推荐类型 | 原因 |
|------|---------|------|
| API 返回值不确定 | `unknown` | 必须先检查 |
| 迁移 JS 代码 | `any`（临时） | 逐步替换 |
| 函数无返回 | `void` | 语义清晰 |
| 错误处理 | `never` | 表示永不返回 |
| 固定格式数据 | `tuple` | 类型安全 |

## L3 专家层：深入

### any vs unknown 对比

```typescript
// any — 关闭类型检查
let x: any = 'hello';
x.toUpperCase();       // ✅ 编译通过
x.nonExistent();       // ✅ 编译通过（运行时报错）

// unknown — 保持类型安全
let y: unknown = 'hello';
// y.toUpperCase();    // ❌ 编译错误
if (typeof y === 'string') {
    y.toUpperCase();   // ✅ 检查后才能使用
}
```

### 枚举的底层原理

```typescript
// 数字枚举
enum Direction { Up = 1, Down, Left, Right }
// 编译后:
// var Direction;
// (function (Direction) {
//     Direction[Direction["Up"] = 1] = "Up";
//     Direction[Direction["Down"] = 2] = "Down";
//     ...
// })(Direction || (Direction = {}));

// 字符串枚举（推荐）
enum Status {
    Pending = 'pending',
    Success = 'success'
}
// 编译后更简洁，没有反向映射
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **类型注解** | 显式标注变量类型 | `let x: number` |
| **类型推断** | TypeScript 自动推断类型 | `let x = 5` → number |
| **类型断言** | 告诉编译器"相信我" | `x as string` |
| **元组** | 固定长度和类型的数组 | `[string, number]` |
| **枚举** | 一组有名字的常量 | `enum Color { Red, Green }` |
| **never** | 永不返回的类型 | `function err(): never` |

    return 'unknown';
}

## 常见问题

### Q1：any 和 unknown 有什么区别？

**any 关闭类型检查，unknown 保持类型安全。unknown 必须先检查类型才能使用。**

### Q2：什么时候用枚举，什么时候用联合类型？

**简单选项用联合类型（`'a' \| 'b'`），需要数值映射或复杂逻辑用枚举。**

### Q3：void 和 never 有什么区别？

**void 表示函数没有返回值（返回 undefined），never 表示函数永远不会返回（抛异常或死循环）。**
