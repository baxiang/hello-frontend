## 生活化比喻

**类型就像"不同形状的容器"**：

```
没有类型（JavaScript）：
┌─────────────────────────────┐
│  📦  📦  📦  📦            │
│  什么都能装，但不知道是什么 │
│  用的时候才发现拿错了       │
└─────────────────────────────┘

有类型（TypeScript）：
┌─────────────────────────────┐
│  📦文字 📦数字 📦开关 📦空 │
│  每个容器有固定形状         │
│  装错了在门口就被拦下       │
└─────────────────────────────┘
```

## 第一步：看看 JavaScript 的类型问题

先来看一段 JavaScript 代码：

```javascript
function calculatePrice(price, discount) {
    return price - discount;
}

calculatePrice(100, 20);    // 返回 80 ✅ 正常
calculatePrice('100', 20);  // 返回 NaN ⚠️ 字符串减数字？
calculatePrice(100, '20');  // 返回 80 ⚠️ 居然能算？（隐式转换）
```

**发现问题了吗？**

JavaScript 会在背后偷偷做类型转换，有时候能算，有时候算出 NaN。这种"有时候行有时候不行"的行为，是 bug 的温床。

再来看一个更常见的例子：

```javascript
function getUser(id) {
    // 假设从 API 获取用户
    return { name: '张三', age: 25 };
}

const user = getUser(1);
console.log(user.name);  // '张三' ✅
console.log(user.nmae);  // undefined ⚠️ 拼写错了，但 JS 不报错！
```

如果你拼错了属性名，JavaScript 不会提醒你，只有运行的时候才会发现 `undefined`。

---

## 第二步：TypeScript 怎么解决？

TypeScript 的做法是给每个值加上**类型标签**：

```typescript
function calculatePrice(price: number, discount: number): number {
    return price - discount;
}

calculatePrice(100, 20);    // ✅ 通过
calculatePrice('100', 20);  // ❌ 报错：类型"string"不能赋给类型"number"
```

现在，如果你传错了类型，TypeScript 在**编译时**就会告诉你，不用等到运行。

对于对象，TypeScript 会检查属性名：

```typescript
function getUser(id: number): { name: string; age: number } {
    return { name: '张三', age: 25 };
}

const user = getUser(1);
console.log(user.name);  // ✅ 通过
console.log(user.nmae);  // ❌ 报错：属性"nmae"不存在，你是不是想写"name"？
```

看到最后一个报错了吗？TypeScript 甚至会**猜测你想写什么**，帮你纠正拼写错误。

## 第三步：试试基础类型

TypeScript 有 7 种基础类型。让我们一个一个来看。

### string（字符串）

就是文字，用单引号、双引号或反引号包裹：

```typescript
let name: string = '张三';
let greeting: string = "Hello";
let template: string = `你好，${name}！`;  // 反引号支持插值
```

### number（数字）

**整数和小数都是 number**，这和 JavaScript 一样：

```typescript
let integer: number = 42;
let float: number = 3.14;
let negative: number = -1;

// 还支持二进制和十六进制
let binary: number = 0b1010;    // 10
let hex: number = 0xf00d;       // 61453
```

### boolean（布尔值）

只有两个值：`true` 或 `false`：

```typescript
let isActive: boolean = true;
let isDeleted: boolean = false;
```

## 第四步：数组和元组

### 数组

数组的类型注解有两种写法：

```typescript
// 写法 1：类型后面加 []（更常用）
let numbers: number[] = [1, 2, 3];

// 写法 2：Array<类型>
let names: Array<string> = ['Tom', 'Jerry'];
```

两种写法完全一样。TypeScript 会确保你只能往数组里放指定类型的值：

```typescript
numbers.push(4);      // ✅ 可以
numbers.push('hello'); // ❌ 报错：类型"string"不能赋给类型"number"
```

### 元组（Tuple）

元组是**固定长度和固定类型**的数组。就像组合餐盒，第一格必须放汉堡，第二格必须放薯条：

```typescript
let pair: [string, number] = ['年龄', 25];

pair[0].toUpperCase();  // ✅ 第一个是 string
pair[1].toFixed(2);     // ✅ 第二个是 number
```

元组适合表示**成对的数据**，比如键值对、坐标等：

```typescript
type Point = [number, number];
const point: Point = [10, 20];

type Entry = [string, number];
const entry: Entry = ['age', 25];
```

## 第五步：null、undefined 和 void

### null 和 undefined

`null` 表示"有意的空"，`undefined` 表示"未定义"：

```typescript
let empty: null = null;
let notDefined: undefined = undefined;
```

实际开发中，我们通常用**联合类型**来处理可能为空的值：

```typescript
let name: string | null = null;  // 可能是字符串，也可能是 null
name = '张三';  // ✅ 可以
name = null;    // ✅ 也可以
```

### void

`void` 表示函数没有返回值：

```typescript
function log(message: string): void {
    console.log(message);
    // 没有 return，或者 return undefined
}
```

## 第六步：三个特殊的类型

### any — 万能类型（不推荐）

`any` 意思是"什么类型都行"：

```typescript
let data: any = 'hello';
data = 42;       // ✅ 可以
data = true;     // ✅ 也可以
data.foo();      // ✅ 编译通过（但运行时报错！）
```

**为什么不推荐？** 因为 `any` 等于关闭了类型检查：

```typescript
function process(data: any) {
    console.log(data.name);  // 编译通过，但运行时可能报错
}
```

**正确做法：定义具体类型**

```typescript
interface Data { name: string }
function process(data: Data) {
    console.log(data.name);  // 有完整的类型提示
}
```

> 💡 **建议**：学习阶段可以暂时用 `any`，但尽量早点改掉这个习惯。

### unknown — 安全的 any

`unknown` 也是"什么都能装"，但**打开前必须先检查**：

```typescript
let data: unknown = fetchData();

// ❌ 直接使用会报错
// data.toUpperCase();

// ✅ 先检查类型
if (typeof data === 'string') {
    data.toUpperCase();  // 现在可以用了
}
```

**any vs unknown 对比：**

| 特性 | any | unknown |
|------|-----|---------|
| 能装任何类型 | ✅ | ✅ |
| 直接使用 | ✅（危险） | ❌ 必须先检查 |
| 类型安全 | ❌ | ✅ |

**记住：能用 unknown 就不要用 any。**

### never — 永不返回

`never` 表示"不可能发生"的情况：

```typescript
// 情况 1：函数永远不返回（抛异常）
function throwError(msg: string): never {
    throw new Error(msg);
}

// 情况 2：死循环
function infiniteLoop(): never {
    while (true) {}
}
```

`never` 最常见的用途是**穷尽检查**——确保所有情况都被处理了：

```typescript
type Status = 'pending' | 'success' | 'error';

function handleStatus(status: Status) {
    switch (status) {
        case 'pending': return '等待中';
        case 'success': return '成功';
        case 'error': return '失败';
        default:
            // 如果 Status 新增了状态，这里会报错
            const _exhaustive: never = status;
            return _exhaustive;
    }
}
```

## 第七步：枚举（Enum）

枚举是一组**有名字的常量**。比如方向只有上下左右，状态只有成功失败：

```typescript
// 数字枚举（默认从 0 开始）
enum Direction {
    Up,      // 0
    Down,    // 1
    Left,    // 2
    Right    // 3
}

let dir: Direction = Direction.Up;
```

**更推荐字符串枚举**，因为更易读：

```typescript
enum Status {
    Pending = 'pending',
    Success = 'success',
    Error = 'error'
}

function showStatus(status: Status) {
    if (status === Status.Success) {
        console.log('操作成功');
    }
}
```

枚举的好处是**限制取值范围**：

```typescript
function setStatus(status: Status) { }

setStatus(Status.Success);  // ✅ 可以
setStatus('unknown');       // ❌ 报错：类型"unknown"不能赋给类型"Status"
```

## 总结：类型速查表

| 类型 | 含义 | 示例 |
|------|------|------|
| `string` | 文字 | `let name: string = '张三'` |
| `number` | 数字（整数和小数） | `let age: number = 25` |
| `boolean` | 真假 | `let active: boolean = true` |
| `null` | 有意的空 | `let empty: null = null` |
| `undefined` | 未定义 | `let notDefined: undefined` |
| `void` | 无返回值 | `function fn(): void` |
| `any` | 任何类型（不推荐） | `let x: any` |
| `unknown` | 安全的 any | `let x: unknown` |
| `never` | 永不返回 | `function err(): never` |
| 数组 | 同类型值的集合 | `let nums: number[] = [1, 2]` |
| 元组 | 固定长度和类型的数组 | `let pair: [string, number]` |
| 枚举 | 有名字的常量组 | `enum Status { Success, Error }` |

**记住：**
- 优先用具体类型，少用 `any`
- 不确定的类型用 `unknown`，不用 `any`
- 枚举用来限制取值范围

## 常见问题

### Q：any 和 unknown 到底选哪个？

**永远优先用 unknown。** any 是"闭着眼睛拿东西"，unknown 是"打开包裹前先检查"：

```typescript
// ❌ any — 闭着眼睛拿
let data: any = fetchData();
data.foo();  // 编译通过，运行时报错

// ✅ unknown — 先检查再拿
let data: unknown = fetchData();
if (typeof data === 'object') {
    // 现在可以安全使用
}
```

### Q：什么时候用枚举，什么时候用联合类型？

**简单选项用联合类型，需要数值映射或复杂逻辑用枚举：**

```typescript
// 联合类型 — 简单选项
type Role = 'admin' | 'user' | 'guest';

// 枚举 — 需要数值或更复杂的场景
enum HttpStatus {
    OK = 200,
    NotFound = 404,
    ServerError = 500
}
```

### Q：void 和 never 有什么区别？

**void 表示函数执行完了但没返回值，never 表示函数永远不会执行完：**

```typescript
// void — 执行完了，只是没返回值
function log(msg: string): void {
    console.log(msg);  // 执行完就结束
}

// never — 永远不会正常结束
function throwError(): never {
    throw new Error();  // 抛异常，不会执行到函数末尾
}
```
