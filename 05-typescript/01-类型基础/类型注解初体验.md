## L1 理解层：会用

### 类型注解语法

**最简示例（1-3行）：**

```typescript
let name: string = '张三';
let age: number = 25;
let active: boolean = true;
```

**详细示例：**

```typescript
// 基础类型
let message: string = 'Hello';
let count: number = 42;
let isActive: boolean = true;

// 数组
let numbers: number[] = [1, 2, 3];
let names: Array<string> = ['Tom', 'Jerry'];

// 对象
let user: { name: string; age: number } = { name: '张三', age: 25 };

// 函数
function add(a: number, b: number): number {
    return a + b;
}

// 联合类型
let id: string | number = 'abc';
id = 123;  // ✅

// 任意类型（不推荐）
let anything: any = 'hello';
anything = 42;  // ✅ 但失去了类型检查
```

---

### 类型推断

```typescript
// TypeScript 自动推断类型
let name = 'Tom';      // 推断为 string
let age = 25;          // 推断为 number
let active = true;     // 推断为 boolean
let numbers = [1, 2];  // 推断为 number[]

// 函数返回值推断
function add(a: number, b: number) {
    return a + b;  // 推断返回 number
}

// 对象属性推断
const user = { name: 'Tom', age: 25 };
// 推断为 { name: string; age: number }
```

## L2 实践层：用好

### 常见编译错误

| 错误 | 原因 | 修复 |
|------|------|------|
| `Type 'string' is not assignable to type 'number'` | 类型不匹配 | 检查赋值类型 |
| `Property 'x' does not exist on type 'T'` | 访问了不存在的属性 | 检查类型定义 |
| `Parameter 'x' implicitly has an 'any' type` | 缺少类型标注 | 添加 `x: number` |
| `Type 'undefined' is not assignable to type 'string'` | 可能为 undefined | 用 `!` 或 `??` |

### 反模式

```typescript
// ❌ 错误：类型不匹配
let age: number = '25';

// ✅ 正确
let age: number = 25;
```

```typescript
// ❌ 错误：滥用 any
function process(data: any) {
    console.log(data.name);  // 没有类型提示
}

// ✅ 正确：定义具体类型
interface Data { name: string }
function process(data: Data) {
    console.log(data.name);  // 有类型提示
}
```

### 何时需要手动标注

```typescript
// ✅ 需要标注：函数参数
function greet(name: string) { }

// ✅ 需要标注：函数返回值（复杂时）
function getUser(): User | null { }

// ❌ 不需要：简单变量（可推断）
let count = 0;

// ✅ 需要：空数组/对象
let numbers: number[] = [];
let user: User = {} as User;
```

## L3 专家层：深入

### 类型推断规则

```
推断优先级：

1. 初始化值 → 推断具体类型
   let x = 5;          // number
   let y = 'hello';    // string

2. 上下文类型 → 从使用场景推断
   const fn: (x: number) => void = (x) => { };
   // x 推断为 number

3. 最佳公共类型 → 多种类型取并集
   let arr = [1, 'hello'];  // (string | number)[]
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **类型注解** | 显式标注变量或函数的类型 | `let x: number` |
| **类型推断** | TypeScript 自动推断类型 | `let x = 5` → number |
| **编译错误** | 类型检查不通过时的报错 | `Type 'string' is not assignable` |
| **联合类型** | 多个类型之一 | `string \| number` |

## 常见问题

### Q1：什么时候需要写类型注解？

**函数参数必须写，返回值复杂时建议写，简单变量可以不写（推断足够）：**

```typescript
// 必须写
function fn(name: string) { }

// 建议写（复杂时）
function fn(): User | null { }

// 可以不写
let count = 0;
```

### Q2：any 和 unknown 有什么区别？

**any 关闭类型检查，unknown 保持类型安全（必须先检查才能使用）：**

```typescript
let x: any = 'hello';
x.toUpperCase();  // ✅ 编译通过（运行时报错如果 x 不是 string）

let y: unknown = 'hello';
// y.toUpperCase();  // ❌ 编译错误
if (typeof y === 'string') y.toUpperCase();  // ✅
```
