## 学习目标

- 理解 TypeScript 的类型推断规则
- 掌握类型断言（as）的使用场景
- 学会使用 const 断言

---

## L1 理解层

**最简示例（1-3行）：**

```typescript
let x = 5;              // 推断为 number
let el = document.getElementById('app') as HTMLDivElement;
const config = { port: 3000 } as const;
```

**详细示例：**

```typescript
// 类型推断
let name = 'Tom';           // string
let age = 25;               // number
let arr = [1, 2, 3];        // number[]
let obj = { x: 1, y: 2 };   // { x: number; y: number }

// 类型断言
const input = document.getElementById('input') as HTMLInputElement;
input.value = 'Hello';

const data = JSON.parse('{"name": "Tom"}') as { name: string };

// const 断言（字面量类型）
const config = { port: 3000, host: 'localhost' } as const;
// config.port 的类型是 3000（不是 number）
```

## L2 实践层

### 推断 vs 断言选择

| 场景 | 推荐方案 | 示例 |
|------|---------|------|
| 简单赋值 | 类型推断 | `let x = 5` |
| 函数返回值 | 显式标注 | `function fn(): string` |
| DOM 元素 | 类型断言 | `el as HTMLInputElement` |
| JSON 解析 | 类型断言 | `JSON.parse(str) as User` |
| 不可变配置 | const 断言 | `{ a: 1 } as const` |

### 反模式

```typescript
// ❌ 错误：过度使用断言
const x = 'hello' as any as number;  // 绕过类型检查

// ✅ 正确：让推断工作
const x: number = Number('hello');
```

## L3 专家层

### 推断规则

```
推断优先级：

1. 初始化值 → 推断具体类型
   let x = 5;          // number
   let y = 'hello';    // string

2. 上下文类型 → 从使用场景推断
   const fn: (x: number) => void = (x) => { };  // x 推断为 number

3. 最佳公共类型 → 多种类型取交集
   let arr = [1, 'hello'];  // (string | number)[]
```

### const 断言原理

```typescript
// 普通对象
const obj = { port: 3000 };
// obj.port 类型: number（可以修改）

// const 断言
const obj = { port: 3000 } as const;
// obj.port 类型: 3000（只读，字面量类型）
// obj.port = 4000;  // ❌ 编译错误
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **类型推断** | TypeScript 自动推断变量类型 | `let x = 5` → number |
| **类型断言** | 告诉编译器"这是某个类型" | `value as string` |
| **const 断言** | 将值设为只读字面量类型 | `{ a: 1 } as const` |
| **字面量类型** | 具体的值作为类型 | `'success' \| 'error'` |

## 常见问题

### Q1：什么时候需要类型断言？

**当 TypeScript 无法推断具体类型时：DOM 元素、JSON 解析、unknown 类型转换。**

### Q2：const 断言和普通 const 有什么区别？

**const 是变量不可重新赋值，as const 是值本身不可变且类型为字面量。**
