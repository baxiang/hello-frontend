## L1 理解层：会用

### 类型推断

**最简示例（1-3行）：**

```typescript
let name = 'Tom';           // 推断为 string
let age = 25;               // 推断为 number
let arr = [1, 2, 3];        // 推断为 number[]
```

**详细示例：**

```typescript
// 变量推断
let count = 0;      // number
let text = 'hello'; // string

// 函数返回值推断
function add(a: number, b: number) {
    return a + b;  // 推断返回 number
}

// 对象属性推断
const user = { name: 'Tom', age: 25 };
// 推断为 { name: string; age: number }

// 上下文类型推断
window.addEventListener('click', (event) => {
    // event 推断为 MouseEvent
});

const numbers: number[] = [1, 2, 3];
// 1, 2, 3 推断为 number（因为数组类型是 number[]）
```

---

### 类型断言

**最简示例（1-3行）：**

```typescript
const input = document.getElementById('myInput') as HTMLInputElement;
input.value = 'Hello';
```

**详细示例：**

```typescript
// as 语法（推荐）
const value = unknownValue as string;

// 尖括号语法（不推荐，与 JSX 冲突）
// const value = <string>unknownValue;

// DOM 元素
const canvas = document.getElementById('game') as HTMLCanvasElement;
const ctx = canvas.getContext('2d')!;

// JSON 解析
interface User { id: number; name: string }
const user = JSON.parse(jsonString) as User;

// 事件对象
function handleClick(e: Event) {
    const target = e.target as HTMLButtonElement;
    console.log(target.value);
}
```

### const 断言

```typescript
// 普通对象
const config = { port: 3000 };
// config.port 类型: number（可以修改）

// const 断言
const config = { port: 3000 } as const;
// config.port 类型: 3000（只读，字面量类型）
// config.port = 4000;  // ❌ 编译错误

// 数组
const colors = ['red', 'green'] as const;
// 类型: readonly ['red', 'green']

// 路由配置
const routes = {
    home: '/',
    about: '/about',
    contact: '/contact'
} as const;

type Route = typeof routes[keyof typeof routes];
// '/' | '/about' | '/contact'
```

### 非空断言

```typescript
// ! 断言 — 告诉编译器这个值不会是 null/undefined
const el = document.getElementById('app')!;  // HTMLElement
el.innerHTML = 'Hello';

// 谨慎使用
function process(user: User | undefined) {
    user!.name;  // ❌ 如果 user 是 undefined，运行时报错
}

// ✅ 正确做法：先检查
function process(user: User | undefined) {
    if (user) {
        user.name;  // ✅ 安全
    }
}
```

## L2 实践层：用好

### 推断 vs 断言选择

| 场景 | 推荐方案 | 示例 |
|------|---------|------|
| 简单赋值 | 类型推断 | `let x = 5` |
| DOM 元素 | 类型断言 | `el as HTMLInputElement` |
| JSON 解析 | 类型断言 | `JSON.parse(str) as User` |
| 不可变配置 | const 断言 | `{ a: 1 } as const` |
| 确定非空 | 先检查再用 | `if (x) x.prop` |

### 反模式

```typescript
// ❌ 错误：过度使用断言
const x = 'hello' as any as number;  // 绕过类型检查

// ✅ 正确：用类型转换
const x = Number('hello');
```

```typescript
// ❌ 错误：非空断言不安全
const el = document.getElementById('app')!;
el.innerHTML = 'Hello';  // 如果元素不存在，运行时报错

// ✅ 正确：先检查
const el = document.getElementById('app');
if (el) el.innerHTML = 'Hello';
```

### 推断的边界

```typescript
// 空数组推断为 never[]
let arr = [];  // never[]
arr.push(1);   // ❌ 报错

// ✅ 正确：显式标注类型
let arr: number[] = [];
arr.push(1);  // ✅

// 对象字面量推断
const config = { port: 3000, host: 'localhost' };
// 推断为 { port: number; host: string }

// 联合类型推断
let value = Math.random() > 0.5 ? 1 : 'hello';
// 推断为 number | string
```

## L3 专家层：深入

### 推断规则

```
推断优先级：

1. 初始化值 → 推断具体类型
   let x = 5;          // number
   let y = 'hello';    // string

2. 上下文类型 → 从使用场景推断
   const fn: (x: number) => void = (x) => { };
   // x 推断为 number

3. 最佳公共类型 → 多种类型取并集
   let arr = [1, 'hello'];  // (string | number)[]

4. 泛型推断 → 从参数推断泛型参数
   function first<T>(arr: T[]): T | undefined { }
   first([1, 2, 3]);  // T = number
```

### const 断言原理

```typescript
// 普通对象
const obj = { port: 3000 };
// 类型: { port: number }
// obj.port = 4000;  // ✅ 可以修改

// const 断言
const obj = { port: 3000 } as const;
// 类型: { readonly port: 3000 }
// obj.port = 4000;  // ❌ 不能修改

// 深层 const
const config = {
    server: { host: 'localhost', port: 3000 },
    debug: true
} as const;
// 所有属性都是 readonly，所有值都是字面量类型
```

### 知识关联

```
断言与推断关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  类型推断   │────→│  类型断言   │────→│  const 断言 │
│  自动识别/  │     │  as/        │     │  字面量/    │
│  上下文     │     │  非空       │     │  只读       │
└─────────────┘     └─────────────┘     └─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **类型推断** | TypeScript 自动推断变量类型 | `let x = 5` → number |
| **类型断言** | 告诉编译器"这是某个类型" | `value as string` |
| **const 断言** | 将值设为只读字面量类型 | `{ a: 1 } as const` |
| **非空断言** | 告诉编译器这个值不为空 | `value!` |
| **字面量类型** | 具体的值作为类型 | `3000`、`'hello'` |

## 常见问题

### Q1：什么时候需要类型断言？

**当 TypeScript 无法推断具体类型时：DOM 元素、JSON 解析、unknown 类型转换。**

### Q2：const 断言和普通 const 有什么区别？

**const 是变量不可重新赋值，as const 是值本身不可变且类型为字面量。**

### Q3：非空断言安全吗？

**不安全。它只告诉编译器"相信我"，但运行时如果值是 null/undefined 仍会报错。优先用可选链 `?.` 或先检查。**
