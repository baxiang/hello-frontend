# 类型推断与类型断言 ⭐⭐

> 理解 TypeScript 什么时候能猜对，什么时候需要你告诉它

---

## 学习目标

学完本节，你能：
- 理解 TypeScript 的类型推断机制
- 知道什么时候不需要写类型注解
- 掌握类型断言的使用场景
- 学会 const 断言和非空断言

## 生活化比喻

**类型推断就像"智能识别"，类型断言就像"你告诉系统"**：

```
类型推断 = 智能识别：
┌─────────────────────────────┐
│  系统看到 5 → 自动识别数字  │
│  系统看到 "hello" → 文字   │
│  你不用贴标签，系统自己知道 │
└─────────────────────────────┘

类型断言 = 你告诉系统：
┌─────────────────────────────┐
│  "相信我，这个是 canvas"    │
│  系统不再检查，直接放行     │
│  但如果信错了，运行时报错   │
└─────────────────────────────┘
```

## 第一步：TypeScript 其实很聪明

还记得我们在上一节写的类型注解吗？

```typescript
let name: string = '张三';
let age: number = 25;
let numbers: number[] = [1, 2, 3];
```

你有没有觉得有点啰嗦？`'张三'` 明明是字符串，为什么还要告诉 TypeScript 它是字符串？

**好消息是——TypeScript 能自己猜到：**

```typescript
let name = '张三';      // TypeScript 自动推断为 string
let age = 25;           // 自动推断为 number
let numbers = [1, 2, 3]; // 自动推断为 number[]
```

这就是**类型推断**。TypeScript 会根据你赋的值自动判断类型。

---

## 第二步：TypeScript 怎么推断？

让我们看看 TypeScript 在不同场景下的推断能力。

### 变量推断

```typescript
let count = 0;      // 推断为 number
let text = 'hello'; // 推断为 string
let active = true;  // 推断为 boolean
```

### 函数返回值推断

```typescript
function add(a: number, b: number) {
    return a + b;  // TypeScript 推断返回 number
}

function greet(name: string) {
    return `Hello, ${name}`;  // 推断返回 string
}
```

TypeScript 看到 `return a + b`，知道两个数字相加还是数字，所以推断返回值是 `number`。

### 对象属性推断

```typescript
const user = { name: 'Tom', age: 25 };
// 推断为 { name: string; age: number }

user.name;   // ✅ 有提示
user.age;    // ✅ 有提示
user.email;  // ❌ 报错：属性"email"不存在
```

### 上下文推断

TypeScript 甚至能根据你的使用场景推断类型：

```typescript
// 监听点击事件，event 自动推断为 MouseEvent
window.addEventListener('click', (event) => {
    console.log(event.clientX);  // ✅ 有提示
});

// 给 number 数组赋值，元素自动推断为 number
const numbers: number[] = [1, 2, 3];
```

## 第三步：什么时候 TypeScript 猜不到？

虽然 TypeScript 很聪明，但有些情况它真的猜不到。

### 情况 1：空数组

```typescript
let arr = [];  // 推断为 never[]（什么都不能放）

arr.push(1);   // ❌ 报错：类型"number"不能赋给类型"never"
```

TypeScript 看到 `[]`，不知道里面会放什么，所以推断为 `never[]`（空数组）。

**解决办法：手动标注类型**

```typescript
let arr: number[] = [];  // 明确告诉 TypeScript 这是数字数组
arr.push(1);             // ✅ 可以
```

### 情况 2：函数参数

```typescript
function greet(name) { }  // ❌ 报错：参数"name"隐式具有"any"类型
```

TypeScript 猜不到 `name` 是什么类型，所以报错。

**解决办法：加上类型注解**

```typescript
function greet(name: string) { }  // ✅ 明确是 string
```

### 情况 3：DOM 元素

```typescript
const input = document.getElementById('myInput');
// 推断为 HTMLElement | null

input.value;  // ❌ 报错：HTMLElement 上不存在属性"value"
```

`getElementById` 返回的是通用的 `HTMLElement`，但你需要的是更具体的 `HTMLInputElement`（有 `value` 属性）。

## 第四步：类型断言 — 你告诉 TypeScript

当 TypeScript 猜不到或者猜错了，你可以用**类型断言**告诉它正确的类型。

### as 语法

```typescript
const input = document.getElementById('myInput') as HTMLInputElement;
input.value = 'Hello';  // ✅ 现在可以用了
```

`as HTMLInputElement` 的意思是："相信我，这个元素是 `HTMLInputElement`。"

### 常见场景

**DOM 元素：**

```typescript
const canvas = document.getElementById('game') as HTMLCanvasElement;
const ctx = canvas.getContext('2d');
```

**JSON 解析：**

```typescript
interface User { id: number; name: string }

const jsonString = '{"id": 1, "name": "张三"}';
const user = JSON.parse(jsonString) as User;

console.log(user.name);  // '张三' ✅ 有类型提示
```

**事件对象：**

```typescript
function handleClick(e: Event) {
    const target = e.target as HTMLButtonElement;
    console.log(target.value);  // ✅ 有提示
}
```

### 断言的本质

类型断言**不会改变运行时的值**，它只是告诉编译器："按这个类型检查"。

```typescript
const value = someValue as string;
// 运行时 someValue 是什么还是什么
// 只是编译器按 string 检查
```

## 第五步：const 断言 — "这个值永不改变"

有时候你不仅想告诉 TypeScript 类型，还想告诉它"这个值不会变"。

### 普通 const vs const 断言

```typescript
// 普通 const — 变量不能重新赋值，但属性可以改
const config = { port: 3000 };
config.port = 4000;  // ✅ 可以修改属性

// const 断言 — 值和属性都不能改
const config = { port: 3000 } as const;
config.port = 4000;  // ❌ 报错：无法为只读属性赋值
```

### const 断言的威力

const 断言会把所有属性变成**字面量类型**（更精确）：

```typescript
// 普通对象
const routes = {
    home: '/',
    about: '/about'
};
// 类型：{ home: string; about: string }

// const 断言
const routes = {
    home: '/',
    about: '/about'
} as const;
// 类型：{ readonly home: '/'; readonly about: '/about' }
```

看到区别了吗？普通对象的 `home` 是 `string`，const 断言后变成了 `'/'`（具体的值）。

### 实际用途

const 断言最常见的用途是**路由配置**和**枚举替代**：

```typescript
const API = {
    users: '/api/users',
    posts: '/api/posts',
    comments: '/api/comments'
} as const;

// 现在可以提取所有 API 路径的类型
type APIPath = typeof API[keyof typeof API];
// '/api/users' | '/api/posts' | '/api/comments'

function fetchAPI(path: APIPath) { }

fetchAPI('/api/users');   // ✅ 可以
fetchAPI('/api/unknown'); // ❌ 报错：不在允许的路径中
```

## 第六步：非空断言 — "这个值一定有"

有时候你知道一个值不会是 `null` 或 `undefined`，但 TypeScript 不确定。

### ! 语法

```typescript
const el = document.getElementById('app');
// 类型：HTMLElement | null

el.innerHTML = 'Hello';  // ❌ 报错：可能是 null

// 非空断言
const el = document.getElementById('app')!;
// 类型：HTMLElement（去掉了 null）

el.innerHTML = 'Hello';  // ✅ 通过
```

`!` 的意思是："相信我，这个值不会是 null。"

### ⚠️ 警告：非空断言不安全

非空断言只是告诉编译器"相信我"，但**如果运行时真的是 null，还是会报错**：

```typescript
const el = document.getElementById('nonexistent')!;
el.innerHTML = 'Hello';  // 运行时：Cannot set properties of null
```

**更安全的做法：先检查**

```typescript
const el = document.getElementById('app');
if (el) {
    el.innerHTML = 'Hello';  // ✅ 安全
}

// 或者用可选链
el?.innerHTML = 'Hello';  // ✅ 如果是 null，什么都不做
```

## 第七步：推断的边界

让我们看看 TypeScript 推断的一些边界情况。

### 最佳公共类型

当你往数组里放不同类型的值时，TypeScript 会取**所有类型的并集**：

```typescript
let arr = [1, 'hello', true];
// 推断为 (string | number | boolean)[]
```

### 泛型推断

TypeScript 能从参数推断泛型类型：

```typescript
function first<T>(arr: T[]): T | undefined {
    return arr[0];
}

first([1, 2, 3]);  // T 推断为 number，返回 number | undefined
first(['a', 'b']); // T 推断为 string，返回 string | undefined
```

### 推断规则总结

| 场景 | TypeScript 怎么做 | 示例 |
|------|------------------|------|
| 有初始值 | 推断具体类型 | `let x = 5` → number |
| 函数返回值 | 从 return 推断 | `return a + b` → number |
| 上下文类型 | 从使用场景推断 | `addEventListener` → MouseEvent |
| 空数组 | 推断为 never[] | `let arr = []` → never[] |
| 函数参数 | **无法推断** | `function fn(name)` → 报错 |

---

## 总结：推断 vs 断言速查表

| 场景 | 推荐方案 | 示例 |
|------|---------|------|
| 有初始值 | 让 TypeScript 推断 | `let x = 5` |
| 空数组 | 手动标注类型 | `let arr: number[] = []` |
| 函数参数 | 手动标注类型 | `function fn(name: string)` |
| DOM 元素 | 类型断言 | `el as HTMLInputElement` |
| JSON 解析 | 类型断言 | `JSON.parse(str) as User` |
| 不可变配置 | const 断言 | `{ port: 3000 } as const` |
| 确定非空 | 先检查再用 | `if (el) el.innerHTML` |

**记住：**
- TypeScript 能推断的时候，不用写类型注解
- TypeScript 猜不到时，用类型断言告诉它
- 非空断言要小心，优先用先检查

## 常见问题

### Q：什么时候需要类型断言？

**当 TypeScript 无法推断具体类型时：DOM 元素、JSON 解析、从 unknown 转换：**

```typescript
// DOM
const input = document.getElementById('myInput') as HTMLInputElement;

// JSON
const user = JSON.parse(json) as User;
```

### Q：const 断言和普通 const 有什么区别？

**const 是变量不能重新赋值，as const 是值本身不可变且类型更精确：**

```typescript
const obj = { port: 3000 };
obj.port = 4000;  // ✅ 可以改属性

const obj = { port: 3000 } as const;
obj.port = 4000;  // ❌ 不能改
```

### Q：非空断言安全吗？

**不安全。它只告诉编译器"相信我"，但运行时如果真的是 null 仍会报错。优先用可选链 `?.` 或先检查：**

```typescript
// ❌ 不安全
el!.innerHTML = 'Hello';

// ✅ 安全
el?.innerHTML = 'Hello';
```
