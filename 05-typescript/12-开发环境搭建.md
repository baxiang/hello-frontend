## 2.1 安装 Node.js

### 什么是 Node.js

Node.js 是一个 JavaScript 运行环境，让我们可以在服务器端运行 JavaScript。

TypeScript 需要 Node.js 来安装和运行。

### 检查是否已安装

```bash
# 打开终端，输入以下命令
node --version

# 如果显示版本号，说明已安装
# 例如：v20.10.0
```

### 下载安装

如果没有安装，访问官网下载：https://nodejs.org/

```
推荐下载 LTS（长期支持）版本
```

### 使用 nvm 管理 Node.js（推荐）

如果你需要同时使用多个 Node.js 版本，推荐使用 nvm：

```bash
# macOS / Linux
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Windows
# 下载：https://github.com/coreybutler/nvm-windows/releases
```

### nvm 使用示例

```bash
# 查看已安装的 Node 版本
nvm list

# 安装指定版本
nvm install 20

# 切换到指定版本
nvm use 20

# 设置默认版本
nvm alias default 20
```

## 2.2 什么是 node_modules？

### 通俗理解

想象一下你要做一个蛋糕：

```
传统方式（不用 node_modules）：
- 你需要自己种小麦、磨面粉、养鸡取蛋...
- 每次做蛋糕都要从零开始

使用 node_modules：
- 超市已经帮你准备好了面粉、鸡蛋、奶油...
- 你只需要拿来做蛋糕就行
```

**node_modules 就是这个"超市"，里面存放了别人已经写好的代码（包），我们可以直接拿来用。**

### 官方定义

```
node_modules 是 Node.js 项目的依赖目录
用于存放项目运行所需的所有第三方代码包
```

### 实际例子

当你安装一个包（比如 `lodash`）时：

```bash
pnpm add lodash
```

会在 node_modules 中创建：

```
node_modules/
└── lodash/
    ├── dist/
    │   └── lodash.min.js    # 编译后的代码
    ├── package.json         # 包的信息
    └── README.md            # 说明文档
```

现在你可以在代码中使用 lodash：

```typescript
import _ from 'lodash';

const arr = [1, 2, 3];
console.log(_.sum(arr));  // 6
```

### node_modules 的结构

一个典型的 node_modules 目录：

```
node_modules/
├── .bin/                    # 可执行命令
│   ├── tsc -> ../typescript/bin/tsc
│   └── vite -> ../vite/bin/vite.js
├── lodash/                  # lodash 库
├── vue/                     # Vue 框架
├── axios/                   # HTTP 库
├── typescript/              # TypeScript
├── @types/                  # 类型定义
│   ├── node/
│   └── react/
└── ...（几百个文件夹）
```

### node_modules 从哪里来？

当你运行以下命令时：

```bash
pnpm install
# 或
npm install
```

会根据 `package.json` 中的依赖列表，自动下载所有包到 `node_modules`。

### package.json - 依赖清单

```json
{
    "name": "my-project",
    "dependencies": {
        "vue": "^3.0.0",      // 生产依赖
        "axios": "^1.0.0"
    },
    "devDependencies": {
        "typescript": "^5.0.0",  // 开发依赖
        "vite": "^5.0.0"
    }
}
```

### 什么时候需要 node_modules？

```
┌─────────────────────────────────────────────────────────┐
│  场景 1：运行项目                                        │
│  pnpm install   →  安装依赖到 node_modules              │
│  pnpm dev       →  读取 node_modules 启动项目           │
├─────────────────────────────────────────────────────────┤
│  场景2：部署项目（生产环境）                              │
│  pnpm install --production   →  只安装 dependencies     │
│  （不安装 devDependencies）                              │
├─────────────────────────────────────────────────────────┤
│  场景3：分享代码                                         │
│  上传代码时通常不包含 node_modules                       │
│  他人克隆后需要运行 pnpm install                         │
└─────────────────────────────────────────────────────────┘
```

### 注意事项

```
⚠️  node_modules 通常很大（几百MB到几GB）
⚠️  不应该提交到 Git（已加入 .gitignore）
⚠️  删除后需要重新 pnpm install
```

### .gitignore 示例

```
# 不提交 node_modules
node_modules/
```

## 2.3 包管理器介绍

### npm（Node Package Manager）

npm 是 Node.js 官方默认的包管理器，随 Node.js 一起安装。

```bash
# 查看 npm 版本
npm --version
```

### pnpm（Performant npm）

pnpm 是一个更高效、更节省磁盘空间的包管理器。

```bash
# 安装 pnpm
npm install -g pnpm

# 验证安装
pnpm --version
```

### npm vs pnpm 对比

| 特性 | npm | pnpm |
|------|-----|------|
| 磁盘占用 | 较大（每个项目独立） | 较小（全局存储） |
| 安装速度 | 较慢 | 较快 |
| node_modules | 扁平结构 | 符号链接 |
| 依赖管理 | 可能出现依赖重复 | 避免重复依赖 |
| 兼容性 | 高 | 高 |

### 为什么推荐 pnpm

```
npm 的 node_modules 结构：
node_modules/
├── package-a/
├── package-b/
└── package-c/
    └── node_modules/
        └── package-a/   # 重复安装

pnpm 的 node_modules 结构：
node_modules/
├── .pnpm/
│   ├── package-a@1.0.0/
│   └── package-b@2.0.0/
└── package-a -> .pnpm/package-a@1.0.0/
└── package-b -> .pnpm/package-b@2.0.0/
```

**pnpm 优势：**
- 节省磁盘空间（相同依赖只存储一份）
- 安装速度更快
- 避免依赖提升（deduping）问题
- 更安全的依赖管理

### 工程实践推荐

```bash
# 现代前端项目推荐使用 pnpm
# 但如果团队统一使用 npm 也可以

# 推荐配置
npm install -g pnpm  # 全局安装 pnpm
pnpm --version       # 验证
```

## 2.4 npm/pnpm 参数详解

### npm install 参数

```bash
# 完整参数形式
npm install <package> --save-dev

# 简写形式
npm install <package> -D
```

| 参数 | 简写 | 说明 | 写入位置 |
|------|------|------|----------|
| `--save` | `-S` | 保存为生产依赖（默认） | dependencies |
| `--save-dev` | `-D` | 保存为开发依赖 | devDependencies |
| `--save-optional` | `-O` | 保存为可选依赖 | optionalDependencies |
| `--save-exact` | `-E` | 保存精确版本 | dependencies |

### 依赖类型说明

```json
{
    "dependencies": {
        "vue": "^3.0.0",           // 生产环境需要
        "axios": "^1.0.0"
    },
    "devDependencies": {
        "typescript": "^5.0.0",    // 开发环境需要
        "eslint": "^8.0.0",
        "vite": "^5.0.0"
    },
    "optionalDependencies": {
        "fsevents": "^2.0.0"       // 可选依赖
    }
}
```

### 何时使用什么依赖

```
┌─────────────────────────────────────────────────────────┐
│  dependencies（生产依赖）                                │
│  - 项目运行时必须依赖的包                                │
│  - 例如：vue、react、axios、lodash                      │
├─────────────────────────────────────────────────────────┤
│  devDependencies（开发依赖）                             │
│  - 仅在开发时使用的包                                    │
│  - 例如：typescript、eslint、vite、jest                 │
├─────────────────────────────────────────────────────────┤
│  optionalDependencies（可选依赖）                       │
│  - 缺失不会导致项目失败的包                              │
│  - 例如：平台特定的原生模块                              │
└─────────────────────────────────────────────────────────┘
```

### 全局安装 vs 项目安装

```bash
# 全局安装（-g）
npm install -g typescript
# 或
pnpm add -g typescript

# 特点：
# - 安装到系统目录，所有项目可用
# - 命令行工具推荐全局安装
# - 版本统一，不受项目影响

# 项目安装
npm install --save-dev typescript
# 或
pnpm add -D typescript

# 特点：
# - 安装到当前项目的 node_modules
# - 每个项目独立版本
# - 团队协作更一致
```

### 全局安装 vs 项目安装 对比

| 特性 | 全局安装 (-g) | 项目安装 |
|------|---------------|----------|
| 安装位置 | 系统目录 | 项目 node_modules |
| 作用范围 | 所有项目 | 当前项目 |
| 版本控制 | 统一版本 | 项目独立版本 |
| 适用场景 | 命令行工具 | 库/框架依赖 |
| 推荐程度 | 谨慎使用 | 推荐 |

### 工程实践推荐

```bash
# ✅ 推荐：项目内安装开发依赖
pnpm add -D typescript
pnpm add -D vite
pnpm add -D eslint

# ✅ 推荐：项目内安装生产依赖
pnpm add vue
pnpm add axios

# ⚠️ 谨慎：全局安装命令行工具
# 仅在需要命令行使用时全局安装
npm install -g typescript  # 可以
pnpm add -g typescript     # 可以

# ❌ 避免：全局安装项目依赖
npm install -g vue         # 错误！
```

## 2.4 安装 TypeScript

### 方式一：全局安装（不推荐用于项目）

```bash
# npm
npm install -g typescript

# pnpm
pnpm add -g typescript

# 验证安装
tsc --version

# 应该显示：Version 5.x.x
```

### 方式二：项目内安装（推荐）

```bash
# 创建项目文件夹
mkdir my-ts-project
cd my-ts-project

# 初始化项目
npm init -y
# 或
pnpm init

# 安装 TypeScript（开发依赖）
npm install --save-dev typescript
# 或
pnpm add -D typescript

# 验证安装
npx tsc --version
# 或
pnpm tsc --version
```

### 常用命令对比

| 命令 | npm | pnpm | 说明 |
|------|-----|------|------|
| 安装依赖 | `npm install` | `pnpm install` | 安装所有依赖 |
| 添加依赖 | `npm i <pkg>` | `pnpm add <pkg>` | 添加生产依赖 |
| 添加开发依赖 | `npm i -D <pkg>` | `pnpm add -D <pkg>` | 添加开发依赖 |
| 全局安装 | `npm i -g <pkg>` | `pnpm add -g <pkg>` | 全局安装 |
| 运行脚本 | `npm run <script>` | `pnpm <script>` | 运行 package.json 中的脚本 |
| 删除依赖 | `npm uninstall <pkg>` | `pnpm remove <pkg>` | 删除依赖 |

## 2.5 第一个 TypeScript 程序

### 创建项目

```bash
# 使用 pnpm（推荐）
mkdir hello-typescript
cd hello-typescript
pnpm init

# 安装 TypeScript
pnpm add -D typescript ts-node @types/node
```

### 编写代码

创建文件 `src/index.ts`：

```typescript
// src/index.ts
const message: string = 'Hello, TypeScript!';
console.log(message);
```

### 编译运行

```bash
# 编译 TypeScript
npx tsc src/index.ts

# 运行编译后的 JavaScript
node src/index.js

# 输出：Hello, TypeScript!
```

### 目录结构

```
hello-typescript/
├── src/
│   ├── index.ts      # TypeScript 源码
│   └── index.js      # 编译后的 JavaScript
├── package.json
├── pnpm-lock.yaml   # pnpm 锁文件
└── node_modules/
```

| `pnpm add -D <pkg>` | 添加开发依赖 |

## 2.7 VS Code 配置

### 安装 VS Code

访问 https://code.visualstudio.com/ 下载安装。

### TypeScript 支持

VS Code 内置了 TypeScript 支持，安装后自动启用。

### 推荐配置

打开 VS Code 设置（`Cmd + ,` 或 `Ctrl + ,`），添加以下配置：

```json
{
    // 保存时自动格式化
    "editor.formatOnSave": true,

    // 保存时自动修复
    "editor.codeActionsOnSave": {
        "source.fixAll.eslint": "explicit",
        "source.organizeImports": "explicit"
    },

    // 使用单引号
    "javascript.preferences.quoteStyle": "single",
    "typescript.preferences.quoteStyle": "single",

    // Tab 改为 2 个空格
    "editor.tabSize": 2,

    // 自动删除末尾空格
    "editor.trimTrailingWhitespace": true,

    // 文件自动保存
    "files.autoSave": "afterDelay",
    "files.autoSaveDelay": 1000,

    // TypeScript 配置
    "typescript.tsdk": "node_modules/typescript/lib",
    "typescript.preferences.importModuleSpecifier": "relative"
}
```

## 2.9 实践：搭建开发环境

### 步骤 1：创建项目

```bash
# 使用 pnpm（推荐）
mkdir ts-project
cd ts-project
pnpm init

# 安装 TypeScript 和相关工具
pnpm add -D typescript ts-node @types/node
```

### 步骤 2：创建 tsconfig.json

```bash
npx tsc --init
```

修改 `tsconfig.json`：

```json
{
    "compilerOptions": {
        "target": "ES2022",
        "module": "ESNext",
        "strict": true,
        "esModuleInterop": true,
        "skipLibCheck": true,
        "forceConsistentCasingInFileNames": true,
        "outDir": "./dist",
        "rootDir": "./src"
    },
    "include": ["src/**/*"],
    "exclude": ["node_modules", "dist"]
}
```

### 步骤 3：编写代码

创建 `src/index.ts`：

```typescript
// 问候函数
function greet(name: string): string {
    return `Hello, ${name}!`;
}

// 输出
const result = greet('TypeScript');
console.log(result);
```

### 步骤 4：运行

```bash
# 方式一：编译后运行
npx tsc
node dist/index.js

# 方式二：直接运行
npx ts-node src/index.ts
```
