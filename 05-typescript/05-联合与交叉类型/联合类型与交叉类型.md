## L1 理解层：会用

### 联合类型

**最简示例（1-3行）：**

```typescript
type ID = string | number;
function format(id: ID) { return String(id); }
format('abc');   // ✅
format(123);     // ✅
```

**详细示例：**

```typescript
// 基础联合
type Status = 'pending' | 'success' | 'error';
type Result = { data: string } | { error: string };

// 函数中使用
function getStatusText(status: Status): string {
    switch (status) {
        case 'pending': return '等待中';
        case 'success': return '成功';
        case 'error': return '失败';
    }
}

// 联合类型的属性访问（只能访问共有属性）
interface Dog { name: string; bark(): void }
interface Cat { name: string; meow(): void }
type Pet = Dog | Cat;

function getPetName(pet: Pet): string {
    return pet.name;  // ✅ name 是共有属性
}
// pet.bark();  // ❌ Cat 没有 bark
```

---

### 交叉类型

**最简示例：**

```typescript
type WithId = { id: number };
type WithName = { name: string };
type User = WithId & WithName;  // { id: number; name: string }
```

**详细示例：**

```typescript
// 合并多个类型
interface HasId { id: number }
interface HasName { name: string }
interface HasEmail { email: string }

type User = HasId & HasName & HasEmail;
// { id: number; name: string; email: string }

// 混合类型
type StyledButton = React.ButtonHTMLAttributes<HTMLButtonElement> & {
    variant: 'primary' | 'secondary';
    size: 'sm' | 'md' | 'lg';
};

// 交叉冲突（never）
type Impossible = { x: number } & { x: string };
// x 的类型是 never（不可能同时是 number 和 string）
```

---

### 可辨识联合

**最简示例：**

```typescript
interface Circle { kind: 'circle'; radius: number }
interface Square { kind: 'square'; side: number }
type Shape = Circle | Square;

function area(shape: Shape): number {
    switch (shape.kind) {
        case 'circle': return Math.PI * shape.radius ** 2;
        case 'square': return shape.side ** 2;
    }
}
```

**详细示例：**

```typescript
// API 响应 — 成功/失败
interface SuccessResponse<T> {
    ok: true;
    data: T;
}

interface ErrorResponse {
    ok: false;
    error: string;
    code: number;
}

type ApiResponse<T> = SuccessResponse<T> | ErrorResponse;

function handleResponse<T>(response: ApiResponse<T>): T | never {
    if (response.ok) {
        return response.data;  // TypeScript 知道这是 SuccessResponse
    }
    throw new Error(`Error ${response.code}: ${response.error}`);
}
```

## L2 实践层：用好

### 联合 vs 交叉

| 操作符 | 含义 | 结果 | 记忆 |
|--------|------|------|------|
| `\|` | 联合（或） | 满足任一类型即可 | "你是 A **或** B" |
| `&` | 交叉（且） | 必须满足所有类型 | "你是 A **且** B" |

### 反模式：不要这样做

```typescript
// ❌ 错误：联合类型访问非共有属性
type Result = { data: string } | { error: string };
function handle(r: Result) {
    console.log(r.data);  // ❌ error 分支没有 data
}

// ✅ 正确：先判断再访问
function handle(r: Result) {
    if ('data' in r) console.log(r.data);
    else console.log(r.error);
}
```

```typescript
// ❌ 错误：可辨识联合缺少判别字段
type Shape = { type: 'circle'; radius: number } | { side: number };
// 第二个类型没有 type 字段，无法区分

// ✅ 正确：所有类型都有判别字段
type Shape = { type: 'circle'; radius: number } | { type: 'square'; side: number };
```

### 适用场景

| 场景 | 推荐方案 | 示例 |
|------|---------|------|
| 多种可能值 | 联合类型 | `type Status = 'ok' \| 'error'` |
| 合并接口属性 | 交叉类型 | `type A = B & C` |
| 状态机/协议 | 可辨识联合 | `{ type: 'success', data } \| { type: 'error' }` |
| 精确字符串值 | 字面量联合 | `type Role = 'admin' \| 'user'` |

## L3 专家层：深入

### 穷尽性检查（Exhaustive Check）

```typescript
type Event = 'click' | 'hover' | 'scroll';

function handleEvent(event: Event) {
    switch (event) {
        case 'click': break;
        case 'hover': break;
        case 'scroll': break;
        default:
            const _exhaustive: never = event;  // 如果漏了 case，这里会报错
    }
}

// 如果新增了 Event 类型但忘记处理：
// type Event = 'click' | 'hover' | 'scroll' | 'touch';
// default 分支会报错：Type '"touch"' is not assignable to type 'never'
```

```
联合与交叉关联：
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **联合类型** | 多个类型之一 | `string \| number` |
| **交叉类型** | 合并所有类型 | `A & B` |
| **字面量类型** | 具体值作为类型 | `'success' \| 'error'` |
| **可辨识联合** | 通过共同字段区分的联合 | `{ type: 'A' } \| { type: 'B' }` |
| **穷尽性检查** | 确保所有分支都被处理 | `const _: never = value` |

## 实践练习

```typescript
// 练习 1：类型安全的状态机
type State =
    | { status: 'idle' }
    | { status: 'loading' }
    | { status: 'success'; data: string }
    | { status: 'error'; message: string };

function render(state: State): string {
    switch (state.status) {
        case 'idle': return '等待中...';
        case 'loading': return '加载中...';
        case 'success': return `数据：${state.data}`;
        case 'error': return `错误：${state.message}`;
    }
}

// 练习 2：实现 Nullable
type Nullable<T> = T | null | undefined;

// 练习 3：合并配置
type BaseConfig = { timeout: number; retries: number };
type AuthConfig = { token: string; refreshToken: string };
type FullConfig = BaseConfig & AuthConfig;
```

## 常见问题

### Q1：联合类型和交叉类型有什么区别？

**联合是"或"（满足任一），交叉是"且"（满足所有）：**

```typescript
type A = { x: number } | { y: string };   // 有 x 或 有 y
type B = { x: number } & { y: string };   // 同时有 x 和 y
```

### Q2：为什么联合类型只能访问共有属性？

**因为 TypeScript 不知道当前值具体是哪种类型，只能保证所有类型都有的属性是安全的。**

### Q3：可辨识联合的"可辨识"是什么意思？

**通过一个共同字段（通常是 `type` 或 `kind`）来区分联合中的具体类型，让 TypeScript 在 switch/if 中自动收窄类型。**
