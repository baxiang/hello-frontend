## 4.1 什么是类型注解

### 语法

```typescript
// 变量名: 类型 = 值
let name: string = 'Tom';
let age: number = 25;
let isStudent: boolean = true;
```

### 基础语法示例

```typescript
// 字符串
let message: string = 'Hello';

// 数字
let count: number = 42;
let price: number = 19.99;

// 布尔
let isActive: boolean = true;

// 数组
let numbers: number[] = [1, 2, 3];
let names: Array<string> = ['Tom', 'Jerry'];
```

## 4.2 类型推断

### 什么是类型推断

TypeScript 会根据赋值自动推断类型：

```typescript
// TS 自动推断为 string
let name = 'Tom';  // 等同于 let name: string = 'Tom'

// TS 自动推断为 number
let count = 42;    // 等同于 let count: number = 42

// TS 自动推断为 boolean
let isActive = true;  // 等同于 let isActive: boolean = true
```

### 推断规则

```typescript
// 赋值时推断
let x = 'hello';  // string

// 函数返回推断
function add(a, b) {
    return a + b;  // 推断为 number（因为 + 操作符）
}
```

### 赋值后类型确定

```typescript
let name = 'Tom';
name = 123;  // ❌ 错误：number 不能赋值给 string
name = 'Jerry';  // ✅ 正确
```

## 4.3 认识编译错误

### 错误示例

```typescript
let name: string = 'Tom';
name = 123;  // ❌ 编译错误
```

错误信息：
```
Type 'number' is not assignable to type 'string'.
```

### 常见错误类型

| 错误 | 说明 |
|------|------|
| Type 'X' is not assignable to type 'Y' | 类型不匹配 |
| Property 'X' does not exist on type 'Y' | 属性不存在 |
| Argument of type 'X' is not assignable to parameter of type 'Y' | 参数类型不匹配 |
| Object is possibly 'null' | 对象可能为 null |

### VS Code 中的错误提示

```
红色波浪线：编译错误
黄色波浪线：警告
```

## 4.4 基础类型注解

### 4.4.1 字符串

```typescript
let name: string = 'Tom';
let greeting: string = `Hello, ${name}!`;
```

### 4.4.2 数字

```typescript
let age: number = 25;
let price: number = 19.99;
let binary: number = 0b1010;  // 二进制
let hex: number = 0xff;      // 十六进制
```

### 4.4.3 布尔

```typescript
let isActive: boolean = true;
let isComplete: boolean = false;
```

### 4.4.4 数组

```typescript
// 两种写法等价
let numbers: number[] = [1, 2, 3];
let names: Array<string> = ['Tom', 'Jerry'];
```

### 4.4.5 对象

```typescript
let user: { name: string; age: number } = {
    name: 'Tom',
    age: 25
};
```

## 4.5 函数类型注解

### 参数类型

```typescript
function add(a: number, b: number): number {
    return a + b;
}

add(1, 2);      // ✅
add('1', '2');  // ❌ 编译错误
```

### 返回值类型

```typescript
function greet(name: string): string {
    return `Hello, ${name}!`;
}

// TypeScript 会推断返回类型，但显式标注更好
```

### 无返回值

```typescript
function log(message: string): void {
    console.log(message);
}
```

## 4.7 类型注解 vs 类型推断

### 什么时候用类型注解

```typescript
// 1. 函数参数必须标注
function greet(name: string): string {
    return `Hello, ${name}!`;
}

// 2. 变量初始化时不确定类型
let data: unknown = fetchData();

// 3. 想要明确表达意图
const status: 'loading' | 'success' | 'error' = 'loading';
```

### 什么时候用类型推断

```typescript
// 大部分情况下可以让 TS 推断
let count = 0;           // number
let name = 'Tom';        // string
let isActive = true;     // boolean

// 函数返回类型通常可以推断
function add(a: number, b: number) {
    return a + b;        // 推断返回 number
}
```

### 最佳实践

```typescript
// ✅ 推荐：参数类型注解，返回类型推断
function greet(name: string) {
    return `Hello, ${name}!`;
}

// ✅ 也可以：都显式标注
function greet(name: string): string {
    return `Hello, ${name}!`;
}
```

| 概念 | 说明 |
|------|------|
| 类型注解 | `let x: type = value` |
