## L1 理解层：会用

### 条件类型

**最简示例（1-3行）：**

```typescript
type IsString<T> = T extends string ? true : false;
type A = IsString<'hello'>;  // true
type B = IsString<42>;       // false
```

**详细示例：**

```typescript
// 基础条件类型
type NonNullable<T> = T extends null | undefined ? never : T;
type A = NonNullable<string | null>;  // string

// 嵌套条件
type DeepReadonly<T> = T extends object
    ? { readonly [K in keyof T]: DeepReadonly<T[K]> }
    : T;

// Flatten — 展平数组
type Flatten<T> = T extends any[] ? T[number] : T;
type A = Flatten<number[]>;       // number
type B = Flatten<string>;         // string

// Unwrap — 解包 Promise
type Unwrap<T> = T extends Promise<infer U> ? U : T;
type A = Unwrap<Promise<string>>;  // string
type B = Unwrap<number>;           // number
```

---

### infer 推断

```typescript
// 提取函数返回值
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;
type A = ReturnType<() => string>;  // string

// 提取函数参数
type Parameters<T> = T extends (...args: infer P) => any ? P : never;
type B = Parameters<(name: string, age: number) => void>;  // [string, number]

// 提取数组元素
type ElementType<T> = T extends (infer U)[] ? U : T;
type C = ElementType<number[]>;  // number

// 提取 Promise 值
type Awaited<T> = T extends Promise<infer U> ? Awaited<U> : T;
type D = Awaited<Promise<Promise<string>>>;  // string
```

---

### 映射类型

```typescript
// Partial — 所有属性可选
type MyPartial<T> = { [K in keyof T]?: T[K] };

// Required — 所有属性必填
type MyRequired<T> = { [K in keyof T]-?: T[K] };

// Readonly — 所有属性只读
type MyReadonly<T> = { readonly [K in keyof T]: T[K] };

// Record — 键值映射
type MyRecord<K extends string, V> = { [P in K]: V };

// Pick — 挑选属性
type MyPick<T, K extends keyof T> = { [P in K]: T[P] };

// Omit — 排除属性
type MyOmit<T, K extends keyof T> = { [P in Exclude<keyof T, K>]: T[P] };
```

## L2 实践层：用好

### 常用条件类型速查

| 类型 | 说明 | 示例 |
|------|------|------|
| `T extends U ? X : Y` | 条件类型 | `string extends string ? 'yes' : 'no'` |
| `infer R` | 推断类型 | `T extends Promise<infer R> ? R : never` |
| `keyof T` | 键的联合类型 | `keyof User` → `'id' \| 'name'` |
| `T[K]` | 索引访问 | `User['name']` → `string` |

### 键重映射

```typescript
// 添加前缀
type WithPrefix<T, Prefix extends string> = {
    [K in keyof T as `${Prefix}${Capitalize<string & K>}`]: T[K];
};

type Result = WithPrefix<{ name: string; age: number }, 'get'>;
// { getName: string; getAge: number }

// 过滤属性
type Filter<T, V> = {
    [K in keyof T as T[K] extends V ? K : never]: T[K];
};

type StringProps = Filter<{ name: string; age: number; email: string }, string>;
// { name: string; email: string }
```

### 反模式

```typescript
// ❌ 错误：条件类型过度嵌套（可读性差）
type Complex<T> = T extends string
    ? T extends 'a' ? 1 : T extends 'b' ? 2 : 3
    : never;

// ✅ 正确：拆分为多个类型
type StringCode<T extends string> = T extends 'a' ? 1 : T extends 'b' ? 2 : 3;
type Complex<T> = T extends string ? StringCode<T> : never;
```

## L3 专家层：深入

### 分布式条件类型

```typescript
// 条件类型在联合类型上会自动"分发"
type ToArray<T> = T extends any ? T[] : never;

type A = ToArray<string | number>;  // string[] | number[]
// 不是 (string | number)[]

// 如果不希望分发，用方括号包裹
type ToArrayNonDist<T> = [T] extends [any] ? T[] : never;
type B = ToArrayNonDist<string | number>;  // (string | number)[]
```

### 模板字面量类型

```typescript
// 事件名称生成
type EventName<T extends string> = `on${Capitalize<T>}`;
type ClickEvent = EventName<'click'>;  // 'onClick'

// CSS 属性组合
type Alignment = 'top' | 'bottom' | 'left' | 'right';
type Position = `${Alignment}-${Alignment}`;
// 'top-top' | 'top-bottom' | 'top-left' | ...

// 路径类型
type Route = `/${string}`;
type APIRoute = `/api/${string}`;
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **条件类型** | 根据条件选择类型 | `T extends U ? X : Y` |
| **infer** | 在条件类型中推断未知类型 | `T extends Promise<infer U>` |
| **映射类型** | 通过遍历键构造新类型 | `{ [K in keyof T]: T[K] }` |
| **键重映射** | 在映射时修改键名 | `[K in keyof T as \`get${K}\`]` |
| **分布式条件** | 条件类型在联合类型上自动分发 | `ToArray<string \| number>` |

## 实践练习

```typescript
// 练习 1：实现 DeepPartial
type DeepPartial<T> = T extends object
    ? { [K in keyof T]?: DeepPartial<T[K]> }
    : T;

type Nested = { user: { name: string; address: { city: string } } };
type Result = DeepPartial<Nested>;
// { user?: { name?: string; address?: { city?: string } } }

// 练习 2：实现函数柯里化类型
type Curry<T extends (...args: any[]) => any> =
    T extends (arg: infer A) => infer R
        ? (arg: A) => R
        : T extends (arg1: infer A1, arg2: infer A2, ...rest: infer Rest) => infer R
            ? (arg1: A1) => Curry<(arg2: A2, ...rest: Rest) => R>
            : T;

// 练习 3：实现类型安全的 event bus
type EventBusEvents = {
    'user:login': { userId: number };
    'user:logout': { userId: number };
    'error': { message: string };
};

type EventListener<K extends keyof EventBusEvents> = (
    data: EventBusEvents[K]
) => void;
```

## 常见问题

### Q1：infer 和 keyof 有什么区别？

**keyof 获取已知对象的键，infer 推断未知的类型：**

```typescript
keyof User;           // 已知 User，获取键 'id' | 'name'
T extends { id: infer I } ? I : never;  // 未知 T，推断 id 的类型
```

### Q2：条件类型什么时候会分发？

**当条件类型作用于联合类型时会自动分发。用方括号包裹可以阻止分发：**

```typescript
type Wrap<T> = T extends any ? T[] : never;
Wrap<string | number>;  // string[] | number[]（分发）

type Wrap2<T> = [T] extends [any] ? T[] : never;
Wrap2<string | number>;  // (string | number)[]（不分发）
```
