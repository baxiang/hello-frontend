# 第一部分：条件类型

---

## 1.1 什么是条件类型？

```
条件类型 = 类型的"if-else"

就像 JavaScript 的三元运算符：
condition ? value1 : value2

条件类型：
T extends U ? X : Y
```

### 通俗比喻

```
场景：判断一个人是不是学生

JavaScript 三元运算符：
isStudent ? "是学生" : "不是学生"

条件类型：
T extends Student ? true : false
```

### 实际例子

```typescript
// 判断 T 是不是 string
type IsString<T> = T extends string ? true : false;

// 使用
type A = IsString<string>;  // true
type B = IsString<number>;  // false
type C = IsString<'hello'>; // true（'hello' 是字符串字面量）
```

---

## 1.2 条件类型基础

### 语法

```typescript
T extends U ? X : Y
```

**解读：**
- `T extends U` - 如果 T 是 U 的子类型
- `? X` - 返回 X
- `: Y` - 否则返回 Y

### 示例

```typescript
// 基础用法
type IsString<T> = T extends string ? true : false;

type A = IsString<string>;  // true
type B = IsString<number>;  // false

// 判断是否是 any
type IsAny<T> = 0 extends (T extends any ? 1 : 0) ? true : false;

// 判断是否是 never
type IsNever<T> = [T] extends [never] ? true : false;
```

### 实际应用

```typescript
// 场景：去除数组类型中的 void
type RemoveVoid<T> = T extends void ? never : T;

type Result = RemoveVoid<string | void | number>;
// 结果：string | number

// 场景：判断函数
type IsFunction<T> = T extends (...args: any[]) => any ? true : false;

type A = IsFunction<() => void>;  // true
type B = IsFunction<string>;      // false
```

## 1.3 infer 推断（重点！）

### 什么是 infer？

```
infer = "推断" = 从已知类型中"提取"部分类型

就像：
- 你知道口袋里有一封信
- infer 就像把手伸进去"拿出来"
```

### 从最简单的开始

#### 1. 函数类型怎么写？

```typescript
// 一个简单的函数
function add(a: number, b: number): number {
    return a + b;
}

// 这个函数的"类型"怎么写？
// (参数类型) => 返回类型

type AddFn = (a: number, b: number) => number;
```

#### 2. `(...args: any[]) => any` 是什么？

```
(...args: any[]) => any
│    │         │    │
│    │         │    └── 返回类型：any（任意类型）
│    │         └─────── 箭头
│    └───────────────── 参数：任意类型任意数量
└─────────────────────── 函数
```

**大白话：**
```
(...args: any[]) => any = 一个函数，参数任意，返回值任意 = "任何函数"
```

#### 3. `T extends (...args: any[]) => infer R` 完整解释

```typescript
// 完整写法
T extends (...args: any[]) => infer R ? R : never

// 拆开看：
// T extends (...args: any[]) => infer R
// │  │                              │
// │  │                              └── 用 R 表示推断出的返回类型
// │  └── "如果 T 是一个函数"
// └── "T 是"

// ? R : never
// ? R      = 那么返回 R（推断出的类型）
// : never  = 否则返回 never
```

**通俗翻译：**
```
"如果 T 是一个函数，就提取它的返回类型；
如果不是函数，就返回 never"
```

### 完整例子

```typescript
// 定义一个工具类型：提取函数的返回类型
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

// ====== 测试 ======

// 场景1：传入一个返回数字的函数
function getNumber() {
    return 123;
}

type Num = MyReturnType<typeof getNumber>;
// 结果：number

// 场景2：传入一个返回对象的函数
function getUser() {
    return { name: 'Tom', age: 25 };
}

type User = MyReturnType<typeof getUser>;
// 结果：{ name: string; age: number }

// 场景3：传入一个字符串（不是函数）
type Err = MyReturnType<string>;
// 结果：never（因为 string 不是函数）
```

### 更多 infer 应用

#### 提取参数类型

```typescript
// 提取函数第一个参数的类型
type FirstParam<T> = T extends (first: infer P, ...args: any[]) => any ? P : never;

function greet(name: string, age: number) {}

type P = FirstParam<typeof greet>;
// 结果：string
```

#### 提取 Promise 内部类型

```typescript
// 提取 Promise<T> 中的 T
type UnwrapPromise<T> = T extends Promise<infer R> ? R : T;

type A = UnwrapPromise<Promise<string>>;  // string
type B = UnwrapPromise<number>;            // number（不是 Promise）
```

### 一句话总结

```
infer R = "我帮你推断出返回类型，用 R 表示"
?= "如果成功，返回推断出的类型"
: never = "如果失败，返回 never"
```

## 1.4 分布式条件类型

### 什么是分布式？

```
分布式 = 联合类型会自动"分发"处理
```

### 通俗理解

```typescript
// 场景：把每个类型都变成数组

type ToArray<T> = T extends any ? T[] : never;

type Result = ToArray<string | number>;
// 分布式后：string[] | number[]
// 不是 (string | number)[]
```

### 为什么需要分布式？

```typescript
// 不用分布式
type Bad<T> = T[];  // 整个联合变成数组

type A = Bad<string | number>;
// 结果：(string | number)[]  - 错误！

// 用分布式
type Good<T> = T extends any ? T[] : never;

type B = Good<string | number>;
// 结果：string[] | number[]  - 正确！
```

### 实际应用

```typescript
// 场景：排除 null 和 undefined
type NonNullable<T> = T extends null | undefined ? never : T;

type A = NonNullable<string | null | undefined>;
// 结果：string
```

# 第二部分：映射类型

---

## 2.1 什么是映射类型？

```
映射类型 = 批量处理对象的属性
        = 循环遍历属性并转换

就像：
- 有一个清单（键的列表）
- 对每个项目做同样的操作
```

### 通俗比喻

```
场景：把所有人的名字都改成大写

不用映射类型：
interface User1 {
    name: string;
    nickname: string;
    email: string;
}

interface User2 {
    name: string;      // 手动写
    nickname: string;  // 手动写
    email: string;     // 手动写
}

用映射类型：
type UppercaseKeys<T> = {
    [P in keyof T]: string;  // 把所有属性变成 string
};
```

---

## 2.2 映射类型基础

### 语法

```typescript
{ [P in K]: T }
```

**解读：**
- `P` - 属性名（自己取的变量名）
- `K` - 键的联合类型
- `T` - 值的类型

### 简单例子

```typescript
// 定义要处理的键
type Keys = 'name' | 'age';

// 映射：把每个键映射到 string 类型
type Mapped = {
    [P in Keys]: string;
};

// 结果：
// {
//     name: string;
//     age: string;
// }
```

### 用 keyof 遍历所有属性

```typescript
interface User {
    id: number;
    name: string;
    email: string;
}

// keyof User = 'id' | 'name' | 'email'
// 遍历每个属性，类型保持不变

type MappedUser = {
    [P in keyof User]: User[P];
};

// 结果：
// {
//     id: number;
//     name: string;
//     email: string;
// }
```

---

## 2.3 映射修饰符

### readonly - 添加只读

```typescript
interface User {
    name: string;
    age: number;
}

// 添加 readonly
type Readonly<T> = {
    readonly [P in keyof T]: T[P];
};

type ReadonlyUser = Readonly<User>;

// 结果：
// {
//     readonly name: string;
//     readonly age: number;
// }
```

### ? - 添加可选

```typescript
interface User {
    name: string;
    age: number;
}

// 添加 ?
type Optional<T> = {
    [P in keyof T]?: T[P];
};

type OptionalUser = Optional<User>;

// 结果：
// {
//     name?: string;
//     age?: number;
// }
```

### - 移除修饰符

```typescript
interface User {
    readonly name?: string;
}

// 移除 readonly 和 ?
type Mutable<T> = {
    -readonly [P in keyof T]-?: T[P];
};

type MutableUser = Mutable<User>;

// 结果：
// {
//     name: string;
// }
```

## 2.4 键的重映射

### as 子句

```
as = "重新映射" = 改变键名
```

### 例子：把键名改成大写

```typescript
interface User {
    id: number;
    name: string;
    email: string;
}

// 把所有键变成大写
type UppercaseKeys<T> = {
    [P in keyof T as Uppercase<string & P>]: T[P];
};

type UppercaseUser = UppercaseKeys<User>;

// 结果：
// {
//     ID: number;
//     NAME: string;
//     EMAIL: string;
// }
```

### 例子：添加前缀

```typescript
interface Config {
    port: number;
    host: string;
}

// 给键添加前缀
type PrefixedConfig<T> = {
    [P in keyof T as `get${Capitalize<string & P>}`]: T[P];
};

type Prefixed = PrefixedConfig<Config>;

// 结果：
// {
//     getPort: number;
//     getHost: string;
// }
```

### 例子：过滤属性

```typescript
interface User {
    id: number;
    name: string;
    password: string;
    email: string;
}

// 排除 password
type SafeUser<T> = {
    [P in keyof T as P extends 'password' ? never : P]: T[P];
};

type Safe = SafeUser<User>;

// 结果：
// {
//     id: number;
//     name: string;
//     email: string;
// }
```

## 2.5 模板字面量类型

### 什么是模板字面量？

```
模板字面量 = 用反引号 `` 拼接字符串类型
```

### 例子

```typescript
// 基础
type Greeting = `Hello, ${string}`;
// 只能是 "Hello, xxx"

type Method = 'get' | 'post' | 'put' | 'delete';

// 组合
type API = `${Method}/${string}`;
// 'get/user' | 'post/user' | 'put/user' | 'delete/user'
```

### 在映射类型中使用

```typescript
interface Handler {
    onClick: () => void;
    onChange: () => void;
    onSubmit: () => void;
}

// 把 onXxx 变成 handleXxx
type HandlerToHandle<T> = {
    [P in keyof T as P extends `on${infer R}` ? `handle${R}` : P]: T[P];
};

type Result = HandlerToHandle<Handler>;

// 结果：
// {
//     handleClick: () => void;
//     handleChange: () => void;
//     handleSubmit: () => void;
// }
```

## 2.6 完整示例

```typescript
// 综合示例：实现 DeepPartial

type DeepPartial<T> = {
    [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

// 测试
interface Company {
    name: string;
    address: {
        city: string;
        street: string;
    };
}

type PartialCompany = DeepPartial<Company>;

// 结果：
// {
//     name?: string;
//     address?: {
//         city?: string;
//         street?: string;
//     };
// }
```

## 本章小结

### 条件类型

| 概念 | 语法 | 说明 |
|------|------|------|
| 条件类型 | `T extends U ? X : Y` | 类型的 if-else |
| infer | `infer R` | 推断并提取类型 |
| 分布式 | 联合类型分发 | 自动处理联合类型 |

### 映射类型

| 概念 | 语法 | 说明 |
|------|------|------|
| 映射类型 | `[P in K]: T` | 批量处理属性 |
| keyof | 获取所有键 | `'a' \| 'b'` |
| readonly | 添加只读 | `readonly [P in K]` |
| ? | 添加可选 | `[P in K]?` |
| as | 键重映射 | 改变键名 |
| 模板字面量 | `` `${string}` `` | 拼接字符串 |
