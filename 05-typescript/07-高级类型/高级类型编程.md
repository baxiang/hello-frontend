## 第一步：看看问题

在上一节我们看到了 Partial 的实现：

```typescript
type Partial<T> = { [K in keyof T]?: T[K] };
```

但如果对象有嵌套属性呢？

```typescript
interface Config {
    database: { host: string; port: number };
    cache: { enabled: boolean; ttl: number };
}

type PartialConfig = Partial<Config>;
// { database?: { host: string; port: number }; cache?: { ... } }
```

`Partial` 只让第一层属性变可选，嵌套的属性还是必填的。

**怎么让所有层级都变可选？**

---

## 第二步：递归类型 — 类型定义中引用自身

答案是**递归**——让类型定义引用自身：

```typescript
type DeepPartial<T> = T extends object
    ? { [K in keyof T]?: DeepPartial<T[K]> }
    : T;
```

`DeepPartial` 的意思是：
- 如果 T 是对象，遍历所有属性，每个属性都调用 `DeepPartial`
- 如果 T 不是对象，直接返回 T

现在：

```typescript
type DeepConfig = DeepPartial<Config>;
// {
//     database?: { host?: string; port?: number };
//     cache?: { enabled?: boolean; ttl?: number };
// }
```

所有层级的属性都变成可选了！

## 第三步：试试递归类型

### DeepRequired

和 DeepPartial 相反，让所有层级都必填：

```typescript
type DeepRequired<T> = T extends object
    ? { [K in keyof T]-?: DeepRequired<T[K]> }
    : T;
```

`-?` 的意思是"去掉可选标记"。

### DeepReadonly

让所有层级都只读：

```typescript
type DeepReadonly<T> = T extends object
    ? { readonly [K in keyof T]: DeepReadonly<T[K]> }
    : T;
```

## 第四步：模板字面量类型 — 用字符串模板构造类型

TypeScript 4.1 引入了模板字面量类型。你可以用字符串模板来构造新类型：

```typescript
type EventName<T extends string> = `on${Capitalize<T>}`;

type ClickEvent = EventName<'click'>;   // 'onClick'
type HoverEvent = EventName<'hover'>;   // 'onHover'
```

### CSS BEM 命名

```typescript
type BEM<B extends string, E extends string, M extends string> =
    `${B}__${E}--${M}`;

type ButtonPrimary = BEM<'btn', 'icon', 'primary'>;
// 'btn__icon--primary'
```

### API 路径组合

```typescript
type APIVersion = 'v1' | 'v2';
type Endpoint = 'users' | 'posts' | 'comments';

type APIPath = `/api/${APIVersion}/${Endpoint}`;
// '/api/v1/users' | '/api/v1/posts' | '/api/v1/comments' |
// '/api/v2/users' | '/api/v2/posts' | '/api/v2/comments'
```

## 第五步：infer — 自动推断未知类型

`infer` 是 TypeScript 的关键字，用来在条件类型中推断未知类型。

### 提取函数返回值

```typescript
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

type Result = ReturnType<() => string>;  // string
```

`infer R` 的意思是："如果 T 是一个函数，把它的返回值类型推断为 R。"

### 提取函数参数

```typescript
type Parameters<T> = T extends (...args: infer P) => any ? P : never;

type Params = Parameters<(name: string, age: number) => void>;
// [name: string, age: number]
```

### 提取数组元素

```typescript
type ElementType<T> = T extends (infer U)[] ? U : T;

type Item = ElementType<number[]>;  // number
```

## 第六步：类型编程挑战

### Trim — 去除字符串首尾空格

```typescript
type Trim<S extends string> = S extends ` ${infer R}` ? Trim<R>
    : S extends `${infer L} ` ? Trim<L>
    : S;

type Result = Trim<'  hello  '>;  // 'hello'
```

### CamelCase — 转驼峰命名

```typescript
type CamelCase<S extends string> = S extends `${infer First}-${infer Rest}`
    ? `${First}${Capitalize<CamelCase<Rest>>}`
    : S;

type Result = CamelCase<'background-color'>;  // 'backgroundColor'
```

## 第七步：实际应用场景

类型编程最常见的用途是**生成类型安全的 API 客户端**：

```typescript
// API 路由定义
type Routes = {
    '/users': { GET: { response: User[] }; POST: { body: CreateUser; response: User } };
    '/posts': { GET: { response: Post[] }; POST: { body: CreatePost; response: Post } };
};

// 类型安全的 API 客户端
type APIClient = {
    [K in keyof Routes]: {
        [M in keyof Routes[K]]: (
            body?: Routes[K][M] extends { body: infer B } ? B : never
        ) => Promise<Routes[K][M]['response']>;
    };
};

// 使用
const api: APIClient = {
    '/users': {
        GET: () => fetch('/users').then(r => r.json()),
        POST: (body) => fetch('/users', { method: 'POST', body: JSON.stringify(body) }).then(r => r.json())
    }
};
```

## 总结：速查表

| 概念 | 语法 | 示例 |
|------|------|------|
| 递归类型 | 类型引用自身 | `DeepPartial<T>` |
| 模板字面量 | `` `模板${T}` `` | `` `on${Capitalize<T>}` `` |
| infer | 推断未知类型 | `T extends Promise<infer U>` |
| 条件类型 | `T extends U ? X : Y` | `T extends object ? X : Y` |
| 映射类型 | `[K in keyof T]` | `{ [K in keyof T]?: T[K] }` |

### 练习 1：实现 SnakeCase

```typescript
// 把驼峰命名转蛇形命名
// 'camelCase' → 'camel_case'

type SnakeCase<S extends string> = S extends `${infer First}${infer Rest}`
    ? Rest extends Uncapitalize<Rest>
        ? `${Lowercase<First>}${SnakeCase<Rest>}`
        : `${Lowercase<First>}_${SnakeCase<Rest>}`
    : S;

type Result = SnakeCase<'camelCase'>;  // 'camel_case'
```

### 练习 2：实现 OptionalKeys

```typescript
// 提取对象中的可选键
// { id: number; name?: string } → 'name'

type OptionalKeys<T> = {
    [K in keyof T]: T extends Record<K, T[K]> ? never : K;
}[keyof T];

type User = { id: number; name?: string };
type Result = OptionalKeys<User>;  // 'name'
```

### 练习 3：实现 PromiseAll 类型

```typescript
// 提取 Promise 数组的值
// PromiseAll<[Promise<string>, Promise<number>]> → Promise<[string, number]>

type PromiseAll<T extends Promise<any>[]> =
    Promise<{ [K in keyof T]: T[K] extends Promise<infer U> ? U : T[K] }>;
```

## 常见问题

### Q：递归类型会导致编译变慢吗？

**会。超过 5 层嵌套时编译速度明显下降。建议限制递归深度或用其他方式替代。**

### Q：infer 和 keyof 有什么区别？

**keyof 获取已知对象的键，infer 推断未知的类型：**

```typescript
keyof User;           // 已知 User，获取键 'id' | 'name'
T extends { id: infer I } ? I : never;  // 未知 T，推断 id 的类型
```

### Q：模板字面量类型有什么实际用途？

**自动生成事件名称、API 路径、CSS 类名等，减少手动拼写错误：**

```typescript
type APIPath = `/api/v1/${'users' | 'posts'}`;
// '/api/v1/users' | '/api/v1/posts'
// 输入路径时自动提示，避免拼写错误
```
