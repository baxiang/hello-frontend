## L1 理解层：会用

### 递归类型

**最简示例（1-3行）：**

```typescript
type DeepReadonly<T> = T extends object ? { readonly [K in keyof T]: DeepReadonly<T[K]> } : T;
```

**概念解释：**

递归类型就是"类型定义中引用自身"。就像俄罗斯套娃，每一层都应用相同的规则。

**详细示例：**

```typescript
// 场景：深层嵌套的配置对象
interface NestedConfig {
    database: { host: string; port: number };
    cache: { enabled: boolean; ttl: number };
}

// DeepPartial — 所有层级都变成可选
type DeepPartial<T> = T extends object
    ? { [K in keyof T]?: DeepPartial<T[K]> }
    : T;

type ConfigUpdate = DeepPartial<NestedConfig>;
// {
//     database?: { host?: string; port?: number };
//     cache?: { enabled?: boolean; ttl?: number };
// }

// DeepRequired — 所有层级都变成必填
type DeepRequired<T> = T extends object
    ? { [K in keyof T]-?: DeepRequired<T[K]> }
    : T;
```

### 模板字面量类型

**最简示例（1-3行）：**

```typescript
type EventName<T extends string> = `on${Capitalize<T>}`;
type ClickEvent = EventName<'click'>;  // 'onClick'
```

**概念解释：**

模板字面量类型让你用字符串模板来构造新类型。输入一个字符串，自动按照模板规则生成新字符串类型。

**详细示例：**

```typescript
// 场景 1：事件名称自动生成
type EventName<T extends string> = `on${Capitalize<T>}`;
type ClickEvent = EventName<'click'>;    // 'onClick'
type HoverEvent = EventName<'hover'>;    // 'onHover'

// 场景 2：CSS BEM 命名
type BEM<B extends string, E extends string, M extends string> =
    `${B}__${E}--${M}`;

type ButtonPrimary = BEM<'btn', 'icon', 'primary'>;
// 'btn__icon--primary'

// 场景 3：API 路径组合
type APIVersion = 'v1' | 'v2';
type Endpoint = 'users' | 'posts' | 'comments';
type APIPath = `/api/${APIVersion}/${Endpoint}`;
// '/api/v1/users' | '/api/v1/posts' | '/api/v1/comments' | ...
```

### infer — 类型推断

```typescript
// 场景：从函数类型中提取返回值
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;
type Result = ReturnType<() => string>;  // string

// 场景：从 Promise 中提取值
type UnwrapPromise<T> = T extends Promise<infer U> ? U : T;
type Value = UnwrapPromise<Promise<string>>;  // string

// 场景：从数组中提取元素类型
type ElementType<T> = T extends (infer U)[] ? U : T;
type Item = ElementType<number[]>;  // number
```

## L2 实践层：用好

### 字符串工具类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `Uppercase<S>` | 转大写 | `Uppercase<'hello'>` → `'HELLO'` |
| `Lowercase<S>` | 转小写 | `Lowercase<'HELLO'>` → `'hello'` |
| `Capitalize<S>` | 首字母大写 | `Capitalize<'hello'>` → `'Hello'` |
| `Uncapitalize<S>` | 首字母小写 | `Uncapitalize<'Hello'>` → `'hello'` |

### 类型编程示例

```typescript
// Trim — 去除字符串首尾空格
type Trim<S extends string> = S extends ` ${infer R}` ? Trim<R>
    : S extends `${infer L} ` ? Trim<L>
    : S;

type Result = Trim<'  hello  '>;  // 'hello'

// CamelCase — 转驼峰命名
type CamelCase<S extends string> = S extends `${infer First}-${infer Rest}`
    ? `${First}${Capitalize<CamelCase<Rest>>}`
    : S;

type Result = CamelCase<'background-color'>;  // 'backgroundColor'

// TupleToUnion — 元组转联合类型
type TupleToUnion<T extends any[]> = T[number];
type Result = TupleToUnion<[1, 2, 3]>;  // 1 | 2 | 3
```

### 反模式：不要这样做

```typescript
// ❌ 错误：递归类型缺少终止条件（无限递归）
type BadRecursive<T> = { nested: BadRecursive<T> };  // 永远递归下去

// ✅ 正确：用条件类型设置终止条件
type GoodRecursive<T> = T extends object
    ? { [K in keyof T]: GoodRecursive<T[K]> }
    : T;  // 非对象类型时停止递归
```

```typescript
// ❌ 错误：模板字面量类型过于复杂（可读性差）
type BadTemplate<T> = T extends `${infer A}_${infer B}_${infer C}_${infer D}` ? ... : T;

// ✅ 正确：拆分为多个简单类型
type Prefix<T> = T extends `${infer A}_${infer Rest}` ? A : never;
type Suffix<T> = T extends `${infer A}_${infer Rest}` ? Rest : never;
```

### 适用场景

| 场景 | 推荐方案 | 示例 |
|------|---------|------|
| 深层嵌套对象 | 递归类型 | `DeepPartial<T>` |
| 事件/方法名生成 | 模板字面量 | `` `on${Capitalize<T>}` `` |
| CSS 类名生成 | 模板字面量 | `` `${Block}__${Element}` `` |
| API 路径组合 | 模板字面量 | `` `/api/${Version}/${Endpoint}` `` |
| 提取函数返回值 | infer | `ReturnType<T>` |

## L3 专家层：深入

### 类型编程技巧

```typescript
// 1. 提取元组首尾元素
type First<T extends any[]> = T extends [infer F, ...any[]] ? F : never;
type Last<T extends any[]> = T extends [...any[], infer L] ? L : never;

type F = First<[1, 2, 3]>;  // 1
type L = Last<[1, 2, 3]>;   // 3

// 2. 用条件类型过滤
type Filter<T, V> = T extends V ? T : never;
type Strings = Filter<string | number | boolean, string>;  // string

// 3. 用递归处理元组
type Reverse<T extends any[]> = T extends [infer F, ...infer R]
    ? [...Reverse<R>, F]
    : T;

type Result = Reverse<[1, 2, 3]>;  // [3, 2, 1]
```

### 性能考量

| 操作 | 性能影响 | 说明 |
|------|---------|------|
| 递归类型（浅层） | 低 | 2-3 层嵌套无影响 |
| 递归类型（深层） | 高 | 超过 5 层编译变慢 |
| 模板字面量 | 极低 | 编译时计算，无运行时开销 |
| infer 推断 | 低 | TypeScript 原生优化 |
| 复杂条件类型 | 中等 | 多个嵌套条件会降低 IDE 响应速度 |

## 实践练习

```typescript
// 练习 1：实现 SnakeCase（驼峰转蛇形）
type SnakeCase<S extends string> = S extends `${infer First}${infer Rest}`
    ? Rest extends Uncapitalize<Rest>
        ? `${Lowercase<First>}${SnakeCase<Rest>}`
        : `${Lowercase<First>}_${SnakeCase<Rest>}`
    : S;

type Result = SnakeCase<'camelCase'>;  // 'camel_case'

// 练习 2：实现 OptionalKeys（提取可选键）
type OptionalKeys<T> = {
    [K in keyof T]: T extends Record<K, T[K]> ? never : K;
}[keyof T];

type User = { id: number; name?: string };
type Result = OptionalKeys<User>;  // 'name'

// 练习 3：实现 PromiseAll 类型
type PromiseAll<T extends Promise<any>[]> =
    Promise<{ [K in keyof T]: T[K] extends Promise<infer U> ? U : T[K] }>;
```

## 常见问题

### Q1：递归类型会导致编译变慢吗？

**会。超过 5 层嵌套时编译速度明显下降。建议限制递归深度或用其他方式替代。**

### Q2：infer 和 keyof 有什么区别？

**keyof 获取已知对象的键，infer 推断未知的类型：**

```typescript
keyof User;           // 已知 User，获取键 'id' | 'name'
T extends { id: infer I } ? I : never;  // 未知 T，推断 id 的类型
```

### Q3：模板字面量类型有什么实际用途？

**自动生成事件名称、API 路径、CSS 类名等，减少手动拼写错误：**

```typescript
type APIPath = `/api/v1/${'users' | 'posts'}`;
// '/api/v1/users' | '/api/v1/posts'
// 输入路径时自动提示，避免拼写错误
```
