# TypeScript 开发环境搭建

> 安装 TypeScript、配置 VS Code、创建练习项目

---

## 学习目标

学完本节，你能：
- 在本地安装和配置 TypeScript 开发环境
- 用 VSCode 编写和运行 TypeScript 代码
- 快速验证各章节的"动手试试"练习

---

## 第一步：安装 Node.js 和 TypeScript

TypeScript 需要 Node.js 环境。

### 安装 Node.js

访问 [https://nodejs.org/](https://nodejs.org/) 下载 LTS 版本（推荐 v22+）。

安装后验证：

```bash
node --version   # v22+
npm --version    # 10+
```

### 安装 TypeScript

```bash
# 全局安装（用于命令行编译）
npm install -g typescript

# 验证
tsc --version    # 6.0.x
```

### 安装 ts-node（可选但推荐）

`ts-node` 可以直接运行 `.ts` 文件，不用先编译成 `.js`：

```bash
npm install -g ts-node
```

---

## 第二步：创建练习项目

为了方便验证各章节的"动手试试"，创建一个统一的练习项目：

```bash
# 创建项目目录
mkdir ts-practice && cd ts-practice

# 初始化项目
npm init -y

# 安装 TypeScript（本地）
npm install -D typescript

# 生成 tsconfig.json
npx tsc --init
```

### 推荐的目录结构

```
ts-practice/
├── src/
│   ├── 01-类型注解/
│   │   └── test.ts
│   ├── 02-基础类型/
│   │   └── test.ts
│   ├── 03-接口与对象/
│   │   └── test.ts
│   └── ...
├── tsconfig.json
└── package.json
```

每一章的"动手试试"，在对应的文件夹里创建 `.ts` 文件即可。

## 第三步：配置 VSCode

### 推荐插件

在 VSCode 插件市场搜索安装：

| 插件 | 用途 |
|------|------|
| **TypeScript and JavaScript Language Features** | VSCode 内置，提供类型提示、错误检查 |
| **Prettier** | 代码格式化 |
| **ESLint** | 代码规范检查 |

### 项目设置

创建 `.vscode/settings.json`：

```json
{
    "typescript.preferences.importModuleSpecifier": "relative",
    "editor.formatOnSave": true,
    "editor.defaultFormatter": "esbenp.prettier-vscode",
    "editor.codeActionsOnSave": {
        "source.fixAll.eslint": "explicit"
    }
}
```

---

## 第四步：配置 tsconfig.json

打开 `tsconfig.json`，修改核心配置：

```json
{
    "compilerOptions": {
        "target": "ES2022",
        "module": "NodeNext",
        "strict": true,
        "outDir": "./dist",
        "rootDir": "./src",
        "esModuleInterop": true,
        "skipLibCheck": true,
        "forceConsistentCasingInFileNames": true,
        "sourceMap": true
    },
    "include": ["src/**/*"],
    "exclude": ["node_modules", "dist"]
}
```

关键配置说明：

| 配置 | 含义 |
|------|------|
| `strict: true` | 开启所有严格检查（推荐） |
| `outDir` | 编译后的 `.js` 输出目录 |
| `rootDir` | TypeScript 源文件目录 |
| `sourceMap: true` | 生成 source map，方便调试 |

## 第五步：编写并运行第一个程序

### 创建文件

在 `src/01-类型注解/` 下创建 `test.ts`：

```typescript
// src/01-类型注解/test.ts
let name: string = '张三';
let age: number = 25;

console.log(`${name}, ${age}岁`);
```

### 方式 1：编译后运行（推荐）

```bash
# 编译整个项目
npx tsc

# 运行生成的 JS
node dist/01-类型注解/test.js
```

### 方式 2：用 ts-node 直接运行

```bash
npx ts-node src/01-类型注解/test.ts
# 输出：张三, 25岁
```

### 方式 3：监听模式（自动编译）

```bash
npx tsc --watch
# 保存 .ts 文件后自动编译，不用手动运行 tsc
```

## 第六步：配合各章节使用

每个章节的"动手试试"环节，按以下步骤验证：

1. **在对应目录创建 `.ts` 文件**

```bash
# 比如"基础类型详解"章节
mkdir -p src/02-基础类型
touch src/02-基础类型/test.ts
```

2. **把"动手试试"的代码复制进去**

3. **在 VSCode 中观察类型提示和错误**
   - 鼠标悬停变量看类型推断
   - 红色波浪线表示类型错误

4. **运行验证**

```bash
npx ts-node src/02-基础类型/test.ts
```

---

## 总结：常用命令速查

| 命令 | 用途 |
|------|------|
| `npx tsc` | 编译整个项目 |
| `npx tsc --watch` | 监听模式，自动编译 |
| `npx ts-node src/test.ts` | 直接运行 .ts 文件 |
| `node dist/test.js` | 运行编译后的 .js |

**记住：**
- 学习时优先用 `ts-node`，快速验证
- 提交代码前用 `tsc` 检查类型错误
- 保持 `strict: true`，不要关闭严格模式
