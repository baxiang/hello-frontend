## 生活化比喻

**泛型就像"万能充电器"**：

```
没有泛型（专用充电器）：
┌─────────────────────────────┐
│  iPhone 充电器 → 只能充 iPhone  │
│  Android 充电器 → 只能充 Android│
│  每个设备都要写一个函数      │
└─────────────────────────────┘

有泛型（万能充电器）：
┌─────────────────────────────┐
│  一个充电器，插什么都能充    │
│  而且知道插的是什么设备      │
│  function charge<T>(device: T): T │
└─────────────────────────────┘
```

## 第一步：看看没有泛型的问题

先来看一个场景：你需要写一个函数，获取数组的第一个元素。

```typescript
// 给数字数组写一个
function firstNumber(arr: number[]): number | undefined {
    return arr[0];
}

// 给字符串数组写一个
function firstString(arr: string[]): string | undefined {
    return arr[0];
}

// 给布尔数组写一个
function firstBoolean(arr: boolean[]): boolean | undefined {
    return arr[0];
}
```

**发现问题了吗？**

三个函数的逻辑完全一样，只是类型不同。如果要支持所有类型，你得写无数个函数。

有人可能会说："那用 `any` 不就行了？"

```typescript
function first(arr: any[]): any {
    return arr[0];
}

const result = first([1, 2, 3]);
// result 的类型是 any —— 失去了类型提示！
result.toFixed();  // 编译通过，但如果数组里不是数字，运行时报错
```

用 `any` 确实能解决问题，但**失去了类型安全**。

---

## 第二步：泛型怎么解决？

泛型的做法是——**把类型变成参数**：

```typescript
function first<T>(arr: T[]): T | undefined {
    return arr[0];
}
```

看到 `<T>` 了吗？`T` 就是一个**类型参数**。它的意思是："这个函数适用于任何类型，调用时再确定具体是什么类型。"

现在调用这个函数：

```typescript
const num = first([1, 2, 3]);       // T 推断为 number，返回 number | undefined
const str = first(['a', 'b']);      // T 推断为 string，返回 string | undefined
const bool = first([true, false]);  // T 推断为 boolean，返回 boolean | undefined
```

TypeScript 会根据你传的数组自动推断 `T` 是什么类型。而且返回值有完整的类型提示！

## 第三步：试试泛型

### 泛型函数

```typescript
function identity<T>(value: T): T {
    return value;
}

identity(42);       // T = number
identity('hello');  // T = string
```

### 泛型接口

```typescript
interface Repository<T> {
    findById(id: number): T | undefined;
    findAll(): T[];
    save(item: T): void;
}
```

`Repository<T>` 的意思是："这是一个仓库，但里面存什么类型，用的时候再确定。"

```typescript
// 用户仓库
const userRepository: Repository<User> = {
    findById(id) { /* ... */ },
    findAll() { /* ... */ },
    save(user) { /* ... */ }
};

// 产品仓库
const productRepository: Repository<Product> = {
    findById(id) { /* ... */ },
    findAll() { /* ... */ },
    save(product) { /* ... */ }
};
```

### 泛型类

```typescript
class Stack<T> {
    private items: T[] = [];

    push(item: T): void {
        this.items.push(item);
    }

    pop(): T | undefined {
        return this.items.pop();
    }

    peek(): T | undefined {
        return this.items[this.items.length - 1];
    }
}

// 数字栈
const numStack = new Stack<number>();
numStack.push(1);
numStack.push(2);

// 字符串栈
const strStack = new Stack<string>();
strStack.push('hello');
```

## 第四步：泛型约束 — 限制类型必须满足条件

有时候你希望泛型类型满足某些条件。比如，你想写一个函数打印值的 `length` 属性：

```typescript
function logLength<T>(value: T): void {
    console.log(value.length);  // ❌ 报错：T 上不存在属性"length"
}
```

TypeScript 不知道 `T` 有没有 `length` 属性。你需要告诉它："T 必须有 length"。

用 `extends` 约束：

```typescript
function logLength<T extends { length: number }>(value: T): void {
    console.log(value.length);  // ✅ 现在可以用了
}

logLength('hello');     // ✅ 5
logLength([1, 2, 3]);   // ✅ 3
logLength(42);          // ❌ number 没有 length 属性
```

`T extends { length: number }` 的意思是："T 必须是包含 `length: number` 的类型。"

## 第五步：keyof — 获取对象的键

`keyof` 用来获取对象所有键的联合类型：

```typescript
interface User {
    id: number;
    name: string;
    email: string;
}

type UserKeys = keyof User;  // 'id' | 'name' | 'email'
```

`keyof User` 的意思是："User 接口的所有键的联合类型。"

### 泛型 + keyof — 类型安全的属性访问

```typescript
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
    return obj[key];
}

const user = { id: 1, name: '张三', email: 'zhang@example.com' };

getProperty(user, 'name');   // ✅ 返回 string
getProperty(user, 'age');    // ❌ 报错：'age' 不在 'id' | 'name' | 'email' 中
```

`K extends keyof T` 保证了 `key` 必须是对象 `obj` 的某个键。

## 第六步：infer — 自动推断类型

`infer` 是 TypeScript 的一个关键字，用来在条件类型中**推断未知类型**。

### 提取函数返回值

```typescript
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

type Result = ReturnType<() => string>;  // string
```

`infer R` 的意思是："如果 T 是一个函数，把它的返回值类型推断为 R。"

### 提取函数参数

```typescript
type Parameters<T> = T extends (...args: infer P) => any ? P : never;

type Params = Parameters<(name: string, age: number) => void>;  // [string, number]
```

### 提取数组元素

```typescript
type ElementType<T> = T extends (infer U)[] ? U : T;

type Item = ElementType<number[]>;  // number
```

---

## 第七步：多类型参数

泛型可以有多个类型参数：

```typescript
function pair<A, B>(first: A, second: B): [A, B] {
    return [first, second];
}

pair(1, 'hello');       // [number, string]
pair(true, { name: '张三' });  // [boolean, { name: string }]
```

---

## 总结：泛型速查表

| 语法 | 含义 | 示例 |
|------|------|------|
| `<T>` | 类型参数 | `function fn<T>(x: T): T` |
| `<T, U>` | 多类型参数 | `function pair<T, U>(a: T, b: U)` |
| `extends` | 泛型约束 | `<T extends { length: number }>` |
| `keyof T` | 键的联合类型 | `keyof User` → `'id' \| 'name'` |
| `T[K]` | 索引访问类型 | `User['name']` → `string` |
| `infer R` | 推断未知类型 | `T extends Promise<infer R> ? R : never` |

**记住：**
- 泛型让代码可以"适用于任何类型"
- 用 `extends` 约束泛型必须满足的条件
- 用 `keyof` 获取对象的键
- 用 `infer` 推断未知类型

## 常见问题

### Q：泛型和 any 有什么区别？

**any 丢失类型信息，泛型保持类型信息：**

```typescript
// any — 编译器不知道返回什么
function first(arr: any[]): any { return arr[0]; }
const x = first([1, 2, 3]);  // x 是 any，没有提示

// 泛型 — 编译器推断出具体类型
function first<T>(arr: T[]): T | undefined { return arr[0]; }
const y = first([1, 2, 3]);  // y 是 number | undefined，有完整提示
```

### Q：什么时候需要显式指定泛型参数？

**大多数情况 TypeScript 能自动推断，以下情况需要显式指定：**

```typescript
// 自动推断
const result = first([1, 2, 3]);  // T = number

// 需要显式指定
const empty = first<number>([]);  // 空数组无法推断

// API 调用
const user = await fetchAPI<User>('/api/users/1');
```

### Q：泛型约束和类型断言选哪个？

**优先用泛型约束（安全），类型断言只在确定时使用：**

```typescript
// ✅ 泛型约束（编译器检查）
function getId<T extends { id: number }>(obj: T) {
    return obj.id;
}

// ❌ 类型断言（绕过检查）
function getIdUnsafe(obj: any) {
    return (obj as { id: number }).id;
}
```
