- 掌握接口的定义和使用

## 3.1 接口基础 - 什么是接口？

### 通俗理解

**接口（interface）就是一份"合同"或"设计图纸"**，它规定了对象应该有什么属性，但不关心具体值是什么。

```typescript
// 定义一份"用户合同"
interface User {
    id: number;      // 必须有 id，且是数字
    name: string;    // 必须有 name，且是字符串
    email: string;   // 必须有 email，且是字符串
}

// ✅ 符合合同
const user1: User = {
    id: 1,
    name: '张三',
    email: 'zhang@example.com'
};

// ✅ 也符合合同（值可以不同）
const user2: User = {
    id: 2,
    name: '李四',
    email: 'li@example.com'
};

// ❌ 不符合合同 - 缺少 email
// const user3: User = {
//     id: 3,
//     name: '王五'
// };

// ❌ 不符合合同 - 类型错误
// const user4: User = {
//     id: 'abc',     // 应该是 number
//     name: '赵六',
//     email: 'zhao@example.com'
// };
```

### 生活例子

```
想象你去办身份证，需要填写表格：

┌─────────────────────────────┐
│      身份证申请表            │
├─────────────────────────────┤
│ 姓名：________ (必填)       │
│ 性别：________ (必填)       │
│ 出生日期：____ (必填)       │
│ 身份证号：____ (必填)       │
└─────────────────────────────┘

这个表格就是"接口"，规定了：
- 必须填哪些信息（属性）
- 每类信息是什么格式（类型）

至于你填"张三"还是"李四"，不重要，只要符合格式就行。
```

## 3.2 可选属性与只读属性

### 可选属性 - 有些信息不是必须的

```typescript
interface Person {
    name: string;      // 必填
    age: number;       // 必填
    phone?: string;    // 可选（注意 ? 符号）
    address?: string;  // 可选
}

// ✅ 只提供必填项
const p1: Person = {
    name: '张三',
    age: 25
};

// ✅ 提供全部
const p2: Person = {
    name: '李四',
    age: 30,
    phone: '13800138000',
    address: '北京市朝阳区'
};

// ✅ 只提供部分可选
const p3: Person = {
    name: '王五',
    age: 28,
    phone: '13900139000'
    // address 可选，可以不提供
};
```

### 只读属性 - 一旦设定不能修改

```typescript
interface Product {
    readonly id: number;  // 只读属性
    name: string;
    price: number;
}

const product: Product = {
    id: 1,
    name: 'iPhone',
    price: 5999
};

// ❌ 错误：不能修改只读属性
// product.id = 2;

// ✅ 可以修改普通属性
product.name = 'iPhone 15';
product.price = 6999;
```

### 生活例子

```
只读属性就像身份证号码：
- 出生时就确定了
- 一辈子不能改
- 但你的姓名、地址可以改

interface Citizen {
    readonly idCard: string;  // 身份证号，不能改
    name: string;              // 姓名，可以改
    address: string;           // 地址，可以改
}
```

## 3.3 索引签名 - 属性名不固定的情况

### 问题场景

有时候我们不知道对象会有哪些属性，只知道属性的类型。

```typescript
// 场景：存储用户信息，但不知道有哪些字段
interface UserInfo {
    [key: string]: string | number;  // 索引签名
}

const user: UserInfo = {
    name: '张三',      // ✅ string
    age: 25,          // ✅ number
    city: '北京',      // ✅ string
    salary: 10000     // ✅ number
};

// 可以继续添加任意字段
user.gender = '男';
user.height = 175;
```

### 详细解释

```typescript
// 语法解析
interface UserInfo {
    //       ↓ 属性名是字符串
    [key: string]: string | number;  // ← 属性值是 string 或 number
}

// 意思是：
// "这个对象可以有任意多个属性，
//  属性名是字符串，
//  属性值可以是字符串或数字"
```

### 实际应用场景

**场景 1：CSS 样式对象**

```typescript
interface CSSStyles {
    [property: string]: string | number;
}

const styles: CSSStyles = {
    color: 'red',
    fontSize: 16,
    marginTop: '10px',
    display: 'flex',
    backgroundColor: '#fff'
    // 可以添加任意 CSS 属性
};
```

**场景 2：翻译字典**

```typescript
interface TranslationDict {
    [key: string]: string;
}

const zhCN: TranslationDict = {
    hello: '你好',
    goodbye: '再见',
    thank_you: '谢谢',
    welcome: '欢迎'
    // 可以继续添加
};

console.log(zhCN['hello']);  // '你好'
```

**场景 3：配置对象**

```typescript
interface AppConfig {
    [feature: string]: boolean;
}

const config: AppConfig = {
    enableDarkMode: true,
    enableNotifications: false,
    enableBetaFeatures: true,
    enableAnalytics: false
    // 可以随时添加新功能开关
};
```

### 混合使用：已知属性 + 索引签名

```typescript
interface User {
    // 已知的固定属性
    id: number;
    name: string;
    
    // 其他任意属性
    [key: string]: string | number | undefined;
}

const user: User = {
    id: 1,              // ✅ 固定属性
    name: '张三',        // ✅ 固定属性
    email: 'zhang@example.com',  // ✅ 任意属性
    age: 25,            // ✅ 任意属性
    city: '北京'         // ✅ 任意属性
};
```

### 生活例子

```
索引签名就像一个"万能表格"：

┌─────────────────────────────┐
│      用户信息表              │
├─────────────────────────────┤
│ 姓名：张三 (固定)           │
│ 年龄：25   (固定)           │
│ 其他信息：                  │
│   - 邮箱：zhang@example.com │
│   - 城市：北京              │
│   - 职业：工程师            │
│   - ... (可以任意添加)      │
└─────────────────────────────┘

固定字段必须填，其他字段可以随意添加。
```

## 3.4 函数类型接口 - 给函数定义规范

### 为什么需要函数类型接口？

有时候我们需要定义"什么样的函数可以传进来"。

```typescript
// 场景：一个函数接收另一个函数作为参数
interface CompareFunc {
    (a: number, b: number): number;
}

// 语法解释：
// (a: number, b: number)  → 函数有两个 number 参数
// : number                → 函数返回 number

// 符合规范的函数
const add: CompareFunc = (a, b) => a + b;
const subtract: CompareFunc = (a, b) => a - b;

// 使用
function calculate(a: number, b: number, fn: CompareFunc): number {
    return fn(a, b);
}

calculate(5, 3, add);       // 8
calculate(5, 3, subtract);  // 2
```

### 实际应用场景

**场景 1：回调函数**

```typescript
// 定义回调函数的规范
interface ClickHandler {
    (event: MouseEvent): void;
}

// 符合规范的函数
const handleClick: ClickHandler = (event) => {
    console.log('点击了', event.target);
};

// 使用
document.addEventListener('click', handleClick);
```

**场景 2：验证函数**

```typescript
// 定义验证函数的规范
interface Validator {
    (value: string): boolean;
}

// 各种验证函数
const isEmail: Validator = (value) => {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value);
};

const isPhone: Validator = (value) => {
    return /^1[3-9]\d{9}$/.test(value);
};

// 使用
function validate(value: string, validator: Validator): boolean {
    return validator(value);
}

validate('test@example.com', isEmail);  // true
validate('13800138000', isPhone);       // true
```

**场景 3：转换函数**

```typescript
// 定义转换函数的规范
interface Transformer<T, R> {
    (input: T): R;
}

// 各种转换函数
const toUpperCase: Transformer<string, string> = (str) => str.toUpperCase();
const parseNumber: Transformer<string, number> = (str) => Number(str);
const getUserAge: Transformer<{ age: number }, number> = (user) => user.age;

// 使用
function transform<T, R>(data: T, transformer: Transformer<T, R>): R {
    return transformer(data);
}

transform('hello', toUpperCase);  // 'HELLO'
transform('123', parseNumber);    // 123
```

### 生活例子

```
函数类型接口就像"插座标准"：

┌─────────────────────────────┐
│      三孔插座               │
│   ○ 火线 (L)                │
│   ○ 零线 (N)                │
│   ○ 地线 (E)                │
└─────────────────────────────┘

只要符合这个标准的电器（函数），
都可以插进来使用。

- 电视机符合标准 → 可以插
- 冰箱符合标准 → 可以插
- 洗衣机符合标准 → 可以插

接口不关心你是什么电器，只关心你的插头是否符合标准。
```

## 3.5 接口继承 - 扩展已有接口

### 基础继承

```typescript
// 基础接口
interface Animal {
    name: string;
    age: number;
}

// Dog 继承 Animal，添加自己的属性
interface Dog extends Animal {
    breed: string;  // 品种
}

// 使用
const dog: Dog = {
    name: '旺财',    // 来自 Animal
    age: 3,         // 来自 Animal
    breed: '哈士奇'  // Dog 自己的
};
```

### 多继承

```typescript
// 定义多个基础接口
interface Flyable {
    fly(): void;
}

interface Swimmable {
    swim(): void;
}

interface Runnable {
    run(): void;
}

// Duck 同时继承三个接口
interface Duck extends Flyable, Swimmable, Runnable {
    name: string;
}

// 实现
const duck: Duck = {
    name: '唐老鸭',
    fly() { console.log('飞'); },
    swim() { console.log('游'); },
    run() { console.log('跑'); }
};
```

### 生活例子

```
接口继承就像"手机型号升级"：

iPhone 14 基础款：
- A15 芯片
- 6.1 英寸屏幕
- 双摄像头

iPhone 14 Pro 继承基础款，额外增加：
- A16 芯片（升级）
- 120Hz 屏幕（新增）
- 三摄像头（新增）
- 灵动岛（新增）

Pro 款有基础款的所有功能，还多了新功能。
```

## 3.6 实现接口（implements）- class 遵守合同

### 基础实现

```typescript
// 定义合同
interface Animal {
    name: string;
    makeSound(): void;
}

// class 实现合同
class Dog implements Animal {
    name: string;
    breed: string;

    constructor(name: string, breed: string) {
        this.name = name;
        this.breed = breed;
    }

    // 必须实现接口中的方法
    makeSound(): void {
        console.log('汪汪汪');
    }
}

class Cat implements Animal {
    name: string;

    constructor(name: string) {
        this.name = name;
    }

    makeSound(): void {
        console.log('喵喵喵');
    }
}

// 使用
const dog = new Dog('旺财', '哈士奇');
const cat = new Cat('咪咪');

dog.makeSound();  // 汪汪汪
cat.makeSound();  // 喵喵喵
```

### 实现多个接口

```typescript
interface Flyable {
    fly(): void;
}

interface Swimmable {
    swim(): void;
}

// 同时实现两个接口
class Duck implements Flyable, Swimmable {
    fly() {
        console.log('鸭子在飞');
    }

    swim() {
        console.log('鸭子在游泳');
    }
}

const duck = new Duck();
duck.fly();   // 鸭子在飞
duck.swim();  // 鸭子在游泳
```

### 生活例子

```
implements 就像"考证书"：

interface 驾驶证 {
    开车 (): void;
}

interface 潜水证 {
    潜水 (): void;
}

// 一个人可以考多个证书
class Person implements 驾驶证，潜水证 {
    开车() {
        console.log('开车');
    }

    潜水() {
        console.log('潜水');
    }
}
```

## 3.7 接口 vs 类型别名 - 如何选择？

### 对比表

| 特性 | interface | type |
|------|-----------|------|
| 定义对象 | ✅ | ✅ |
| 联合类型 | ❌ | ✅ |
| 声明合并 | ✅ | ❌ |
| 被 class 实现 | ✅ | ❌ |
| 映射类型 | ❌ | ✅ |

### 相同点

```typescript
// 定义对象，两者都可以
interface User1 {
    name: string;
    age: number;
}

type User2 = {
    name: string;
    age: number;
};

// 使用效果一样
const u1: User1 = { name: '张三', age: 25 };
const u2: User2 = { name: '李四', age: 30 };
```

### 不同点

```typescript
// 1. 联合类型 - 只能用 type
type Status = 'pending' | 'success' | 'error';
type ID = string | number;

// 2. 声明合并 - 只能用 interface
interface Box {
    height: number;
}
interface Box {
    width: number;
}
// 自动合并为 { height: number; width: number; }

// 3. 被 class 实现 - 只能用 interface
interface Animal {
    name: string;
}
class Dog implements Animal {  // ✅
    name = 'Dog';
}

type Animal2 = { name: string };
// class Cat implements Animal2 {}  // ❌ 错误
```

### 选择建议

```
┌─────────────────────────────────────────┐
│  定义对象结构 → 优先用 interface         │
│                                         │
│  需要联合类型 → 用 type                  │
│  需要类型映射 → 用 type                  │
│  需要声明合并 → 用 interface             │
│  需要被 class 实现 → 用 interface        │
│                                         │
│  实在不确定 → 用 interface 不会错        │
└─────────────────────────────────────────┘
```

## 3.9 常见问答

### Q1: 索引签名的 [key: string] 是什么意思？

**答：** 表示"属性名可以是任意字符串"。

```typescript
interface Obj {
    [key: string]: number;
}

// 意思是：
// "这个对象可以有任意多个属性，
//  只要属性名是字符串，属性值是数字就行"

const obj: Obj = {
    a: 1,
    b: 2,
    c: 3,
    // 可以继续添加...
};
```

### Q2: 函数接口中的 (a: number, b: number): number 怎么读？

**答：** 从左到右读：
- `(a: number, b: number)` → 函数有两个参数，都是 number 类型
- `: number` → 函数返回 number 类型

```typescript
interface MathFunc {
    (a: number, b: number): number;
}

// 读作："一个函数，接收两个 number 参数，返回一个 number"
```

### Q3: interface 和 type 到底选哪个？

**答：** 记住这个原则：

```
定义对象结构 → 优先 interface
其他复杂类型 → 用 type
```
