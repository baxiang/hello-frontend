## 生活化比喻

**函数类型就像"工厂流水线的规格说明书"**：

```
没有规格（JavaScript）：
┌─────────────────────────────┐
│  输入：随便什么              │
│  输出：看心情                │
│  add(1, 2) → 3              │
│  add('1', '2') → '12'       │
│  add(1, 'hello') → '1hello' │
└─────────────────────────────┘

有规格（TypeScript）：
┌─────────────────────────────┐
│  输入：两个数字              │
│  输出：一个数字              │
│  add(1, 2) → 3 ✅           │
│  add('1', '2') → ❌ 拒绝   │
└─────────────────────────────┘
```

## 第一步：看看 JavaScript 函数的问题

先来看一段 JavaScript 代码：

```javascript
function calculatePrice(price, discount) {
    return price - discount;
}

calculatePrice(100, 20);    // 返回 80 ✅
calculatePrice('100', 20);  // 返回 NaN ⚠️
calculatePrice(100);        // 返回 NaN ⚠️（discount 是 undefined）
```

**发现问题了吗？**

JavaScript 函数不会告诉你：
- 参数应该是什么类型
- 需要传几个参数
- 返回值是什么类型

只有运行的时候才会暴露问题。

---

## 第二步：TypeScript 怎么解决？

TypeScript 的做法是给函数加上**类型签名**——明确告诉编译器参数和返回值的类型：

```typescript
function calculatePrice(price: number, discount: number): number {
    return price - discount;
}

calculatePrice(100, 20);    // ✅ 通过
calculatePrice('100', 20);  // ❌ 报错：类型"string"不能赋给类型"number"
calculatePrice(100);        // ❌ 报错：应有 2 个参数，但获得 1 个
```

现在，如果你传错了类型或少传了参数，TypeScript 在**编译时**就会告诉你。

## 第三步：试试基础函数类型

### 参数类型和返回值

```typescript
function greet(name: string): string {
    return `Hello, ${name}`;
}
```

语法很简单：
- `name: string` → 参数 name 必须是字符串
- `: string`（最后那个）→ 返回值也是字符串

### 箭头函数

箭头函数的类型注解一样：

```typescript
const greet = (name: string): string => `Hello, ${name}`;
```

### 函数类型表达式

有时候你需要定义一个"函数的类型"，比如给回调函数指定类型：

```typescript
type MathFn = (a: number, b: number) => number;

const add: MathFn = (a, b) => a + b;
const sub: MathFn = (a, b) => a - b;
```

`(a: number, b: number) => number` 的意思是：
- 接收两个数字参数
- 返回一个数字

## 第四步：可选参数、默认参数、剩余参数

### 可选参数

有些参数不是必须的。用 `?` 标记：

```typescript
function createUser(name: string, age?: number) {
    if (age) {
        return { name, age };
    }
    return { name };
}

createUser('张三');       // ✅ 可以不传 age
createUser('张三', 25);   // ✅ 也可以传
```

### 默认参数

给参数设默认值，不传就用默认的：

```typescript
function paginate(page: number, limit: number = 10) {
    return { page, limit };
}

paginate(1);       // { page: 1, limit: 10 }
paginate(1, 20);   // { page: 1, limit: 20 }
```

**注意：有默认值的参数自动变成可选的，不需要加 `?`。**

### 剩余参数

有时候你不知道会传几个参数。用 `...` 收集：

```typescript
function sum(...numbers: number[]): number {
    return numbers.reduce((total, n) => total + n, 0);
}

sum(1, 2, 3);       // 6
sum(10, 20, 30, 40); // 100
```

## 第五步：void 和 never

### void — 函数没有返回值

```typescript
function log(message: string): void {
    console.log(message);
    // 没有 return，或者 return undefined
}
```

`void` 的意思是"这个函数执行完了，但不返回值"。

### never — 函数永远不会返回

```typescript
function throwError(msg: string): never {
    throw new Error(msg);
}
```

`never` 表示"这个函数永远不会正常结束"——要么抛异常，要么死循环。

## 第六步：函数重载 — 同一个函数，不同签名

有时候一个函数需要根据不同的参数类型返回不同的结果。比如：

```javascript
// JavaScript：一个函数处理多种输入
function parse(value) {
    if (typeof value === 'string') return Number(value);
    if (typeof value === 'number') return String(value);
}

parse('123');   // 返回 123
parse(123);     // 返回 '123'
```

TypeScript 可以用**函数重载**来精确描述这种行为：

```typescript
// 重载签名（告诉 TypeScript 有哪些调用方式）
function parse(value: string): number;
function parse(value: number): string;

// 实现签名（真正的逻辑）
function parse(value: string | number): string | number {
    if (typeof value === 'string') return Number(value);
    return String(value);
}

// 使用
const num = parse('123');   // 类型是 number ✅
const str = parse(123);     // 类型是 string ✅
```

**规则：**
- 重载签名可以有多个，描述所有合法的调用方式
- 实现签名必须兼容所有重载签名
- 调用者只能看到重载签名，看不到实现签名

## 第七步：高阶函数 — 函数作为参数或返回值

高阶函数就是"接受函数作为参数"或"返回一个函数"的函数。

### 函数作为参数

```typescript
function forEach<T>(arr: T[], callback: (item: T, index: number) => void): void {
    for (let i = 0; i < arr.length; i++) {
        callback(arr[i], i);
    }
}

forEach([1, 2, 3], (item, index) => {
    console.log(`${index}: ${item}`);
});
```

### 返回函数

```typescript
function createMultiplier(factor: number): (n: number) => number {
    return (n: number) => n * factor;
}

const double = createMultiplier(2);
double(5);  // 10
```

## 总结：函数类型速查表

| 语法 | 含义 | 示例 |
|------|------|------|
| `参数: 类型` | 参数类型 | `function fn(name: string)` |
| `): 类型` | 返回值类型 | `function fn(): number` |
| `参数?: 类型` | 可选参数 | `function fn(name?: string)` |
| `参数 = 值` | 默认参数 | `function fn(limit = 10)` |
| `...args: 类型[]` | 剩余参数 | `function fn(...nums: number[])` |
| `(): void` | 无返回值 | `function fn(): void` |
| `(): never` | 永不返回 | `function fn(): never` |
| `(a: T) => R` | 函数类型 | `type Fn = (x: number) => string` |

**记住：**
- 函数参数必须写类型注解
- 返回值简单时可以省略（TypeScript 会推断）
- 可选参数用 `?`，默认参数用 `=`
- 函数重载用来描述多种调用方式

## 常见问题

### Q：可选参数和默认参数有什么区别？

**可选参数的值是 `类型 | undefined`，默认参数一定有值：**

```typescript
function fn1(name?: string) {
    console.log(name);  // string | undefined
}

function fn2(name: string = '匿名') {
    console.log(name);  // string（一定有值）
}
```

### Q：什么时候需要写返回值类型？

**简单时可以省略（TypeScript 会推断），复杂时建议写：**

```typescript
// 简单 → 可以省略
function add(a: number, b: number) {
    return a + b;  // 推断返回 number
}

// 复杂 → 建议写
function getUser(id: number): User | null {
    // ...
}
```

### Q：函数重载和联合类型有什么区别？

**重载用于"不同输入对应不同输出"，联合类型用于"可能返回多种类型之一"：**

```typescript
// 重载 — 编译器知道确切返回类型
function parse(x: string): number;
function parse(x: number): string;
const a = parse('123');  // 类型是 number

// 联合 — 编译器只知道可能是其中一种
function parse(x: string | number): string | number { }
const b = parse('123');  // 类型是 string | number
```
