## 生活化比喻

**接口就像"合同模板"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                  合同模板                             │
├──────────────────────────────────────────────────────┤
│                                                      │
│    interface = 合同模板                              │
│    ─────────────                                     │
│    规定了双方必须提供的信息                          │
│    甲方：姓名、电话、地址                            │
│    乙方：姓名、电话                                  │
│    不填全合同无效                                    │
│                                                      │
│    可选属性 = 选填项                                 │
│    ─────────────                                     │
│    传真号码可以填也可以不填                          │
│    fax?: string                                     │
│                                                      │
│    只读属性 = 盖章后不可修改                          │
│    ─────────────                                     │
│    合同编号一旦确定就不能改                          │
│    readonly id: number                              │
│                                                      │
│    索引签名 = 未知数量的额外条款                      │
│    ─────────────                                     │
│    除了标准条款，还可以附加任意数量的补充条款        │
│    [key: string]: any                               │
│                                                      │
│    接口继承 = 在旧合同基础上追加条款                  │
│    ─────────────                                     │
│    标准合同 + 附加条款 = VIP 合同                    │
│    interface VIP extends Standard { discount: number }│
│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

### 基础接口

**最简示例（1-3行）：**

```typescript
interface User { id: number; name: string }
const user: User = { id: 1, name: '张三' };
```

**详细示例：**

```typescript
// 基础定义
interface Product {
    id: number;
    name: string;
    price: number;
    description?: string;  // 可选属性
}

const product: Product = {
    id: 1,
    name: 'MacBook Pro',
    price: 14999
};

// 只读属性
interface Config {
    readonly apiKey: string;
    readonly version: string;
}

const config: Config = { apiKey: 'abc123', version: '1.0' };
// config.apiKey = 'new';  // ❌ 只读属性不能修改

// 函数类型接口
interface SearchFunc {
    (source: string, subString: string): boolean;
}

const mySearch: SearchFunc = (src, sub) => src.includes(sub);
```

### 索引签名

```typescript
// 字符串索引
interface StringMap {
    [key: string]: number;
}
const scores: StringMap = { math: 90, english: 85 };

// 数字索引（类似数组）
interface StringArray {
    [index: number]: string;
}
const colors: StringArray = ['red', 'green', 'blue'];

// 混合使用
interface Mixed {
    name: string;           // 具名属性
    [key: string]: string | number;  // 索引签名
    count: number;          // 具名属性必须兼容索引签名
}
```

### 接口继承

```typescript
// 单继承
interface Animal {
    name: string;
    eat(): void;
}

interface Dog extends Animal {
    breed: string;
    bark(): void;
}

const dog: Dog = {
    name: '旺财',
    breed: '金毛',
    eat() { console.log('吃骨头'); },
    bark() { console.log('汪汪'); }
};

// 多继承
interface HasId { id: number }
interface HasTimestamps { createdAt: Date; updatedAt: Date }

interface Post extends HasId, HasTimestamps {
    title: string;
    content: string;
}
```

## L2 实践层：用好

### interface vs type

| 特性 | interface | type |
|------|-----------|------|
| 定义对象 | ✅ | ✅ |
| 联合类型 | ❌ | ✅ `type A = B \| C` |
| 交叉类型 | ❌ | ✅ `type A = B & C` |
| 声明合并 | ✅ | ❌ |
| 计算属性 | ❌ | ✅ |
| extends | `extends` | `&` |

### 声明合并

```typescript
// 同名 interface 自动合并
interface User {
    id: number;
    name: string;
}

interface User {
    email: string;
}

// 等同于：
// interface User { id: number; name: string; email: string }

// 扩展第三方库类型
interface Window {
    myAppConfig: { theme: string };
}
```

### 反模式

```typescript
// ❌ 错误：用 any 定义对象
const user: any = { name: '张三' };

// ✅ 正确：用 interface
interface User { name: string }
const user: User = { name: '张三' };
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 定义对象形状 | interface | 可扩展、声明合并 |
| 联合/交叉类型 | type | interface 不支持 |
| 函数类型 | 两者均可 | type 更简洁 |
| 扩展第三方类型 | interface | 声明合并 |

## L3 专家层：深入

### interface 的底层原理

```
interface 在编译后完全消失，只是编译时的类型检查工具：

TypeScript:
    interface User { id: number; name: string }
    function greet(user: User) { }

编译后 JavaScript:
    function greet(user) { }
    // 没有 interface 的任何痕迹
```

### 知识关联

```
接口知识关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  基础接口   │────→│  索引签名   │────→│  接口继承   │
│  属性/      │     │  字符串/    │     │  单继承/    │
│  可选/      │     │  数字       │     │  多继承     │
│  只读       │     └─────────────┘     └─────────────┘
└─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **接口** | 定义对象结构的契约 | `interface User { name: string }` |
| **可选属性** | 可以缺少的属性 | `email?: string` |
| **只读属性** | 初始化后不可修改 | `readonly id: number` |
| **索引签名** | 动态键的类型定义 | `[key: string]: number` |
| **声明合并** | 同名 interface 自动合并 | 扩展第三方类型 |
| **接口继承** | 基于已有接口创建新接口 | `interface Dog extends Animal` |

## 常见问题

### Q1：interface 和 type 选哪个？

**优先用 interface（可扩展），需要联合/交叉类型时用 type：**

```typescript
// 对象定义 → interface
interface User { name: string }

// 联合类型 → type
type Status = 'pending' | 'success' | 'error';
```

### Q2：什么时候用索引签名？

**当对象有动态键时使用，比如字典、配置对象：**

```typescript
interface Translation {
    [key: string]: string;
}
const zh: Translation = { hello: '你好', bye: '再见' };
```
