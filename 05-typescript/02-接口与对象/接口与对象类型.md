# 接口与对象类型 ⭐⭐

> interface、type、索引签名、接口继承

---

## 学习目标

- 掌握 interface 定义对象类型
- 理解可选属性、只读属性
- 学会索引签名和接口继承

---

## L1 理解层

**最简示例（1-3行）：**

```typescript
interface User { id: number; name: string }
const user: User = { id: 1, name: 'Tom' };
```

**详细示例：**

```typescript
// 基础接口
interface User {
    id: number;
    name: string;
    email?: string;        // 可选属性
    readonly createdAt: Date;  // 只读
}

// 索引签名
interface Dictionary {
    [key: string]: number;
}
const scores: Dictionary = { math: 90, english: 85 };

// 接口继承
interface Animal { name: string }
interface Dog extends Animal { breed: string }
const dog: Dog = { name: '旺财', breed: '金毛' };

// type vs interface
type Point = { x: number; y: number };
interface Point2 { x: number; y: number }
```

## L2 实践层

### interface vs type

| 特性 | interface | type |
|------|-----------|------|
| 扩展 | `extends` | `&` |
| 联合类型 | ❌ | ✅ `type A = B \| C` |
| 声明合并 | ✅ | ❌ |
| 计算属性 | ❌ | ✅ |

### 反模式

```typescript
// ❌ 错误：用 any 代替接口
const user: any = { id: 1, name: 'Tom' };

// ✅ 正确：定义接口
interface User { id: number; name: string }
const user: User = { id: 1, name: 'Tom' };
```

## L3 专家层

### 声明合并

```typescript
interface Window {
    myCustomProp: string;
}
// 多个同名 interface 会自动合并
```

---

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **接口** | 定义对象结构的契约 | `interface User { id: number }` |
| **可选属性** | 可以缺少的属性 | `email?: string` |
| **只读属性** | 初始化后不可修改 | `readonly id: number` |
| **索引签名** | 动态键的类型 | `[key: string]: number` |
| **声明合并** | 同名 interface 自动合并 | 扩展第三方类型 |
