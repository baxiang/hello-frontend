## 2.1 接口基础 - 什么是接口？

### 什么是接口？

```
接口（interface）= 对象的"合同"或"设计图纸"

作用：
- 规定对象应该有什么属性
- 规定每个属性的类型
- 不关心具体的属性值

就像：
- 建筑图纸：规定房子有什么结构
- 合同：规定双方有什么权利和义务
- 菜单：规定餐厅提供什么菜品
```

### 基本用法

```typescript
// 定义接口（合同）
interface User {
    id: number;      // 必须有 id，且是数字
    name: string;    // 必须有 name，且是字符串
    email: string;   // 必须有 email，且是字符串
}

// 符合合同
const user1: User = {
    id: 1,
    name: '张三',
    email: 'zhang@example.com'
};

// 也符合合同（值可以不同）
const user2: User = {
    id: 2,
    name: '李四',
    email: 'li@example.com'
};
```

### 接口 vs 类型别名

```typescript
// 接口
interface User {
    name: string;
    age: number;
}

// 类型别名
type User = {
    name: string;
    age: number;
};

// 两者功能相似，接口可以继承和实现
```

## 2.2 可选属性与只读属性

### 可选属性 - 有些信息不是必须的

```
可选属性 = 可以有也可以没有的属性

语法：属性名 + 问号（?）

就像：
- 注册表单：手机号可选
- 用户资料：头像可选
- 订单信息：备注可选
```

```typescript
interface Person {
    name: string;      // 必填
    age: number;       // 必填
    phone?: string;    // 可选（注意 ? 符号）
    address?: string;  // 可选
}

// ✅ 只提供必填项
const p1: Person = {
    name: '张三',
    age: 25
};

// ✅ 提供全部
const p2: Person = {
    name: '李四',
    age: 30,
    phone: '13800138000',
    address: '北京市朝阳区'
};
```

### 只读属性 - 创建后不能修改

```
只读属性 = 创建后不能修改的属性

语法：readonly 属性名

就像：
- 身份证号：一旦确定就不能改
- 订单编号：系统生成，不能改
- 创建时间：记录后不能改
```

```typescript
interface User {
    readonly id: number;    // 只读
    readonly createdAt: Date; // 只读
    name: string;
}

const user: User = {
    id: 1,
    createdAt: new Date(),
    name: '张三'
};

user.name = '李四';  // OK

// user.id = 2;      // ❌ 错误！id 是只读的
// user.createdAt = new Date(); // ❌ 错误！
```

## 2.3 索引签名 - 动态属性

### 什么是索引签名？

```
索引签名 = 允许任意属性名的对象

语法：[key: 类型]: 值类型

就像：
- 字典：任意单词对应任意解释
- 成绩单：任意学生对应任意成绩
- 用户扩展属性：任意键对应任意值
```

### 基本用法

```typescript
// 字符串索引签名
interface StringMap {
    [key: string]: string;
}

const dict: StringMap = {
    hello: '你好',
    world: '世界',
    typescript: 'TypeScript'
};

// 数字索引签名
interface NumberArray {
    [index: number]: string;
}

const arr: NumberArray = ['a', 'b', 'c'];

// 混合使用
interface User {
    name: string;
    age: number;
    [key: string]: any;  // 允许任意额外属性
}

const user: User = {
    name: '张三',
    age: 25,
    email: 'zhang@example.com',  // 额外属性
    phone: '13800138000'          // 额外属性
};
```

### 索引签名的几种写法

```typescript
// 1. 字符串索引 - 键是字符串
interface StringDict {
    [key: string]: string;
}

// 2. 数字索引 - 键是数字
interface NumberArray {
    [index: number]: string;
}
const arr: NumberArray = ['a', 'b', 'c'];
console.log(arr[0]);  // 'a'

// 3. 混合固定属性和动态属性
interface User {
    name: string;           // 固定属性
    age: number;            // 固定属性
    [key: string]: any;    // 动态属性
}
```

### 注意事项

```typescript
// ❌ 错误：索引签名和其他属性类型冲突
interface Bad {
    [key: string]: string;
    name: number;  // ❌ 冲突：string vs number
}

// ✅ 正确：索引签名类型要兼容其他属性
interface Good {
    name: string;
    [key: string]: string | number;  // 包含 string
}
```

## 2.4 函数类型接口

### 接口定义函数

```
函数类型接口 = 规定函数的参数和返回值类型

语法：
interface 函数名 {
    (参数1: 类型1, 参数2: 类型2): 返回类型
}

就像：
- 计算器：输入两个数，输出一个数
- 处理器：输入数据，输出处理后的数据
```

```typescript
// 定义函数类型接口
interface AddFunc {
    (a: number, b: number): number;
}

// 使用接口
const add: AddFunc = (a, b) => a + b;

// 函数类型接口作为参数
interface Callback {
    (error: Error | null, result?: any): void;
}

function fetchData(callback: Callback): void {
    // 模拟异步操作
    setTimeout(() => {
        callback(null, { data: 'success' });
    }, 1000);
}
```

## 2.5 接口继承

### 接口继承接口

```
接口继承 = 子接口继承父接口的所有属性

语法：
interface 子接口 extends 父接口 { }

就像：
- 学生继承人的属性
- 员工继承人的属性
- 管理员继承用户的属性
```

```typescript
// 父接口
interface Person {
    name: string;
    age: number;
}

// 子接口继承父接口
interface Student extends Person {
    grade: string;
    school: string;
}

// 使用
const student: Student = {
    name: '张三',
    age: 18,
    grade: '高三',
    school: '第一中学'
};
```

### 多重继承

```typescript
interface A {
    a: string;
}

interface B {
    b: number;
}

// 继承多个接口
interface C extends A, B {
    c: boolean;
}

const obj: C = {
    a: 'hello',
    b: 123,
    c: true
};
```

## 2.6 接口实现

### 类实现接口

```
类实现接口 = 类必须遵守接口的"合同"

语法：
class 类名 implements 接口 { }

就像：
- 员工必须遵守岗位职责
- 会员卡必须符合会员规则
- 车辆必须符合交通法规
```

```typescript
// 定义接口
interface Printable {
    print(): void;
}

interface Loggable {
    log(): void;
}

// 类实现多个接口
class Document implements Printable, Loggable {
    content: string;

    constructor(content: string) {
        this.content = content;
    }

    print(): void {
        console.log(`打印: ${this.content}`);
    }

    log(): void {
        console.log(`记录: ${this.content}`);
    }
}

const doc = new Document('Hello TypeScript');
doc.print();  // 打印: Hello TypeScript
doc.log();    // 记录: Hello TypeScript
```

## 2.7 泛型接口

### 基本用法

```typescript
interface Response<T> {
    code: number;
    message: string;
    data: T;
```

## 2.8 综合示例：接口继承 + 可选属性

### 完整语法解析

```typescript
interface AuthRequest extends Request {
    user?: {
        id: number;
        name: string;
        role: string;
    };
}
```

这个语法包含三个知识点：

#### 1. `extends` - 接口继承

```
AuthRequest 继承 Request 的所有属性
```

#### 2. `?` - 可选属性

```
user? = user 可以不存在
```

#### 3. 嵌套对象类型

```
user?: { id: number; name: string; role: string }
```

### 实际应用场景

```typescript
// 场景：Express 中间件添加用户信息

// 1. 扩展 Request 接口
interface AuthRequest extends Request {
    user?: {
        id: number;
        name: string;
        role: string;
    };
}

// 2. 中间件设置用户
function authMiddleware(req: AuthRequest, res: Response, next: NextFunction) {
    const token = req.headers.authorization;

    if (token) {
        // 解析 token 后设置用户信息
        req.user = {
            id: 1,
            name: 'Tom',
            role: 'admin'
        };
    }

    next();
}

// 3. 路由中使用
app.get('/profile', authMiddleware, (req: AuthRequest, res: Response) => {
    // req.user 可能存在也可能不存在
    if (req.user) {
        res.json({
            message: `欢迎，${req.user.name}`,
            role: req.user.role
        });
    } else {
        res.json({ message: '未登录' });
    }
});
```

## 2.9 interface vs type vs class（深入理解）

### 什么是"编译后删除"？

```
TypeScript 的编译过程：
TypeScript (.ts)  →  编译  →  JavaScript (.js)

关键点：JavaScript 没有 interface 和 type！
```

```typescript
// TypeScript 源代码
interface User {
    name: string;
    age: number;
}

class UserClass {
    name: string;
    age: number;
}

type UserType = {
    name: string;
    age: number;
};
```

编译后：

```javascript
// JavaScript（编译后）
class UserClass {
    constructor() {
        // 保留
    }
}

// interface 和 type 完全消失了！
```

### 三者的区别

| 特性 | interface | type | class |
|------|-----------|------|-------|
| **定义对象** | ✅ | ✅ | ✅ |
| **编译后保留** | ❌ 删除 | ❌ 删除 | ✅ 保留 |
| **可实例化** | ❌ | ❌ | ✅ |
| **可继承** | extends | extends & 交叉 | extends |
| **可实现** | implements | ❌ | - |
| **声明合并** | ✅ | ❌ | ❌ |

## 2.11 常见问答

### Q1: 接口和类型别名有什么区别？

```typescript
// 接口
interface User {
    name: string;
}

// 类型别名
type User = {
    name: string;
};

// 主要区别：
// - 接口可以被类实现，类型别名不行（用于对象时）
// - 接口支持声明合并，类型别名不行
// - 类型别名可以用于联合类型、交叉类型等
```

### Q2: 什么时候用接口，什么时候用类型别名？

```typescript
// 用接口：定义对象结构、类实现
interface User {
    name: string;
    age: number;
}

// 用类型别名：联合类型、函数类型、元组
type ID = string | number;
type Callback = (data: any) => void;
type Point = [number, number];
```

## 本章小结

| 概念 | 语法 | 说明 |
|------|------|------|
| 接口定义 | `interface Name { }` | 定义对象结构 |
| 可选属性 | `name?: string` | 可以不提供 |
| 只读属性 | `readonly name: string` | 创建后不可修改 |
| 索引签名 | `[key: string]: any` | 动态属性 |
| 接口继承 | `interface B extends A` | 继承属性 |
| 接口实现 | `class C implements I` | 类遵守接口 |
