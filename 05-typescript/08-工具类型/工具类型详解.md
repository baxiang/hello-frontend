## 1. 什么是工具类型？

```
工具类型 = 别人已经写好的"类型函数"
         = 用来快速生成新类型的"工具"

就像：
- 工具箱里有各种工具
- 螺丝刀、锤子、扳手...
- 拿来就能用，不需要自己造
```

### 为什么要用工具类型？

```typescript
// 不用工具类型
interface User {
    id: number;
    name: string;
    email: string;
    age: number;
}

// 手动写一个"可选版本" - 太麻烦！
interface PartialUser {
    id?: number;
    name?: string;
    email?: string;
    age?: number;
}

// 用工具类型 - 一行搞定！
type PartialUser = Partial<User>;
```

### 常用工具类型一览

| 工具类型 | 作用 | 例子 |
|----------|------|------|
| Partial | 所有属性变可选 | `{ a?: string }` |
| Required | 所有属性变必填 | `{ a: string }` |
| Readonly | 所有属性变只读 | `{ readonly a: string }` |
| Pick | 选取部分属性 | `{ a: string }` |
| Omit | 排除部分属性 | `{ b: string }` |
| Record | 键值对类型 | `{ [key: string]: value }` |

## 2. Partial / Required / Readonly

### Partial<T> - 所有属性可选

```typescript
interface User {
    id: number;
    name: string;
    email: string;
}

// 所有属性变为可选
type PartialUser = Partial<User>;
// { id?: number; name?: string; email?: string; }

const user: PartialUser = { name: 'Tom' };  // ✅
```

**实现原理：**

```typescript
type MyPartial<T> = {
    [P in keyof T]?: T[P];
};
```

**实际应用：更新操作**

```typescript
interface Todo {
    id: number;
    title: string;
    completed: boolean;
}

function updateTodo(id: number, updates: Partial<Todo>) {
    // 只需要提供要更新的字段
}

updateTodo(1, { completed: true });  // ✅
```

### Required<T> - 所有属性必填

```typescript
interface User {
    id?: number;
    name?: string;
}

// 所有属性变为必填
type RequiredUser = Required<User>;
// { id: number; name: string; }
```

**实现原理：**

```typescript
type MyRequired<T> = {
    [P in keyof T]-?: T[P];  // -? 移除可选修饰符
};
```

### Readonly<T> - 所有属性只读

```typescript
interface User {
    id: number;
    name: string;
}

// 所有属性变为只读
type ReadonlyUser = Readonly<User>;
// { readonly id: number; readonly name: string; }

const user: ReadonlyUser = { id: 1, name: 'Tom' };
// user.id = 2;  // ❌ 错误：只读
```

**实现原理：**

```typescript
type MyReadonly<T> = {
    readonly [P in keyof T]: T[P];
};
```

## 3. Pick / Omit

### Pick<T, K> - 选取属性

```typescript
interface User {
    id: number;
    name: string;
    email: string;
    password: string;
}

// 选取指定属性
type UserPreview = Pick<User, 'id' | 'name'>;
// { id: number; name: string; }
```

**实现原理：**

```typescript
type MyPick<T, K extends keyof T> = {
    [P in K]: T[P];
};
```

**实际应用：API 响应**

```typescript
type UserResponse = Pick<User, 'id' | 'name' | 'email'>;
// 排除 password，不返回给前端
```

### Omit<T, K> - 排除属性

```typescript
interface User {
    id: number;
    name: string;
    email: string;
    password: string;
}

// 排除指定属性
type UserWithoutPassword = Omit<User, 'password'>;
// { id: number; name: string; email: string; }
```

**实现原理：**

```typescript
type MyOmit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>>;
```

**实际应用：表单数据**

```typescript
type UserCreate = Omit<User, 'id'>;  // 创建时不需要 id
type LoginForm = Omit<User, 'id' | 'email'>;  // 登录表单
```

## 4. Record<K, T> - 键值对类型

### 通俗理解

```
Record<键的类型, 值的类型>

= 定义一个"字典" / "映射"
= 类似 JavaScript 的对象
```

### 使用示例

```typescript
// 定义角色类型
type Role = 'admin' | 'user' | 'guest';

// Record<Role, string[]> = 键是 Role，值是 string 数组
const permissions: Record<Role, string[]> = {
    admin: ['read', 'write', 'delete'],
    user: ['read', 'write'],
    guest: ['read']
};

permissions.admin    // ['read', 'write', 'delete']
```

**实现原理：**

```typescript
type MyRecord<K extends keyof any, T> = {
    [P in K]: T;
};
```

**实际应用：**

```typescript
// 翻译字典
type TranslationKey = 'hello' | 'goodbye' | 'thank_you';

const translations: Record<TranslationKey, string> = {
    hello: '你好',
    goodbye: '再见',
    thank_you: '谢谢'
};

// 国家代码
type CountryCode = 'CN' | 'US' | 'JP';

const countryNames: Record<CountryCode, string> = {
    CN: '中国',
    US: '美国',
    JP: '日本'
};
```

## 5. Exclude / Extract / NonNullable

### Exclude<T, U> - 排除类型

```typescript
type T = 'a' | 'b' | 'c' | 'd';

// 排除 'a' 和 'c'
type Excluded = Exclude<T, 'a' | 'c'>;
// 结果：'b' | 'd'
```

**实现原理：**

```typescript
type MyExclude<T, U> = T extends U ? never : T;
```

### Extract<T, U> - 提取类型

```typescript
type T = 'a' | 'b' | 'c' | 'd';

// 提取 'a' 和 'c'
type Extracted = Extract<T, 'a' | 'c'>;
// 结果：'a' | 'c'
```

**实现原理：**

```typescript
type MyExtract<T, U> = T extends U ? T : never;
```

### NonNullable<T> - 排除 null/undefined

```typescript
type T = string | null | undefined | number;

// 排除 null 和 undefined
type NonNull = NonNullable<T>;
// 结果：string | number
```

**实现原理：**

```typescript
type MyNonNullable<T> = T extends null | undefined ? never : T;
```

## 6. ReturnType / Parameters

### ReturnType<T> - 获取返回类型

```typescript
function getUser() {
    return { id: 1, name: 'Tom' };
}

type User = ReturnType<typeof getUser>;
// { id: number; name: string; }
```

**实现原理：**

```typescript
type MyReturnType<T extends (...args: any) => any> =
    T extends (...args: any) => infer R ? R : any;
```

### Parameters<T> - 获取参数类型

```typescript
function createUser(name: string, age: number) {
    return { name, age };
}

type Params = Parameters<typeof createUser>;
// [string, number]
```

**实现原理：**

```typescript
type MyParameters<T extends (...args: any) => any> =
    T extends (...args: infer P) => any ? P : never;
```

## 7. 自定义工具类型

### DeepPartial - 深度可选

```typescript
type DeepPartial<T> = {
    [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

interface Company {
    name: string;
    address: {
        city: string;
        street: string;
    };
}

type PartialCompany = DeepPartial<Company>;
// { name?: string; address?: { city?: string; street?: string; } }
```

### DeepReadonly - 深度只读

```typescript
type DeepReadonly<T> = {
    readonly [P in keyof T]: T[P] extends object ? DeepReadonly<T[P]> : T[P];
};
```

### Mutable - 移除只读

```typescript
type Mutable<T> = {
    -readonly [P in keyof T]: T[P];
};

interface ReadonlyUser {
    readonly id: number;
    readonly name: string;
}

type User = Mutable<ReadonlyUser>;
// { id: number; name: string; }
```

### ValueOf - 获取值类型

```typescript
type ValueOf<T> = T[keyof T];

const colors = {
    red: '#ff0000',
    green: '#00ff00',
    blue: '#0000ff'
} as const;

type ColorValue = ValueOf<typeof colors>;
// '#ff0000' | '#00ff00' | '#0000ff'
```

### Nullable - 可空类型

```typescript
type Nullable<T> = T | null;

type T0 = Nullable<string>;  // string | null
```

## 8. 组合使用工具类型

```typescript
interface User {
    id: number;
    name: string;
    email: string;
    password: string;
}

// 组合使用
type UserUpdate = Partial<Omit<User, 'id' | 'password'>>;
// { name?: string; email?: string; }

type UserCreate = Required<Omit<User, 'id'>>;
// { name: string; email: string; password: string; }

type PublicUser = Readonly<Omit<User, 'password'>>;
// { readonly id: number; readonly name: string; readonly email: string; }
```

## 9. 实践练习

### 练习 1：使用工具类型

```typescript
interface Product {
    id: number;
    name: string;
    price: number;
    description: string;
    stock: number;
}

// 1. 创建商品时，id 可选
type CreateProduct = Omit<Product, 'id'>;

// 2. 更新商品时，所有字段可选
type UpdateProduct = Partial<Product>;

// 3. 商品列表项，只显示基本信息
type ProductListItem = Pick<Product, 'id' | 'name' | 'price'>;
```

### 练习 2：自定义工具类型

```typescript
// 实现 FilterByType - 按类型过滤属性
type FilterByType<T, U> = {
    [K in keyof T as T[K] extends U ? K : never]: T[K];
};

interface Mixed {
    name: string;
    age: number;
    email: string;
    active: boolean;
}

type StringFields = FilterByType<Mixed, string>;
// { name: string; email: string; }
```

### 练习 3：Record 应用

```typescript
type Status = 'pending' | 'processing' | 'completed' | 'cancelled';

const statusLabels: Record<Status, string> = {
    pending: '待处理',
    processing: '处理中',
    completed: '已完成',
    cancelled: '已取消'
};
```

## 本章小结

| 工具类型 | 作用 | 示例 |
|----------|------|------|
| Partial | 所有属性可选 | `Partial<User>` |
| Required | 所有属性必填 | `Required<User>` |
| Readonly | 所有属性只读 | `Readonly<User>` |
| Pick | 选取属性 | `Pick<User, 'id'>` |
| Omit | 排除属性 | `Omit<User, 'pwd'>` |
| Record | 键值对 | `Record<K, V>` |
| Exclude | 排除类型 | `Exclude<A, B>` |
| Extract | 提取类型 | `Extract<A, B>` |
| NonNullable | 排除空值 | `NonNullable<T>` |
| ReturnType | 返回类型 | `ReturnType<fn>` |
| Parameters | 参数类型 | `Parameters<fn>` |
