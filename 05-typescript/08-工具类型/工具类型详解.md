## 生活化比喻

**工具类型就像"瑞士军刀"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                  瑞士军刀                             │
├──────────────────────────────────────────────────────┤
│                                                      │
│    Partial = 把必填变成选填                           │
│    ─────────────                                     │
│    原本必须填所有字段，现在可以只填一部分              │
│    就像报名表从"全部必填"变成"选填"                  │
│                                                      │
│    Required = 把选填变成必填                          │
│    ─────────────                                     │
│    原本可选的字段现在必须填                            │
│                                                      │
│    Pick = 挑出需要的字段                              │
│    ─────────────                                     │
│    从完整表格中只挑出"姓名"和"邮箱"两列              │
│                                                      │
│    Omit = 排除不需要的字段                            │
│    ─────────────                                     │
│    从完整表格中去掉"密码"列                          │
│                                                      │
│    Record = 创建键值对映射                            │
│    ─────────────                                     │
│    创建一个字典，键是角色，值是权限列表              │
│                                                      │
│    ReturnType = 自动推断函数返回类型                  │
│    ─────────────                                     │
│    不用手动写返回类型，工具帮你推断                  │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

**最简示例（1-3行）：**

```typescript
type PartialUser = Partial<User>;       // 所有属性可选
type UserName = Pick<User, 'name'>;     // 只保留 name
type UserNoId = Omit<User, 'id'>;       // 排除 id
```

**详细示例：**

```typescript
interface User {
    id: number;
    name: string;
    email: string;
    age: number;
    role: 'admin' | 'user';
}

// Partial<T> — 所有属性变为可选
type UpdateUserInput = Partial<User>;
function updateUser(id: number, data: UpdateUserInput) { }
updateUser(1, { name: '新名字' });  // 只需传要更新的字段

// Required<T> — 所有属性变为必填
type StrictUser = Required<Partial<User>>;  // 等同于 User

// Pick<T, K> — 挑选指定属性
type UserBasicInfo = Pick<User, 'name' | 'email'>;
// { name: string; email: string }

// Omit<T, K> — 排除指定属性
type UserPublic = Omit<User, 'role'>;
// { id: number; name: string; email: string; age: number }

// Record<K, V> — 键值映射
type RolePermissions = Record<'admin' | 'user', string[]>;
const perms: RolePermissions = {
    admin: ['read', 'write', 'delete'],
    user: ['read']
};

// ReturnType<T> — 获取函数返回值类型
function createUser() { return { id: 1, name: '张三' }; }
type CreatedUser = ReturnType<typeof createUser>;
// { id: number; name: string }

// Parameters<T> — 获取函数参数类型
function greet(name: string, age: number) { }
type GreetParams = Parameters<typeof greet>;
// [name: string, age: number]
```

## L2 实践层：用好

### 工具类型速查

| 工具类型 | 说明 | 示例 |
|----------|------|------|
| `Partial<T>` | 所有属性可选 | `Partial<User>` |
| `Required<T>` | 所有属性必填 | `Required<Partial<User>>` |
| `Readonly<T>` | 所有属性只读 | `Readonly<User>` |
| `Pick<T, K>` | 挑选属性 | `Pick<User, 'name'>` |
| `Omit<T, K>` | 排除属性 | `Omit<User, 'password'>` |
| `Record<K, V>` | 键值映射 | `Record<string, number>` |
| `ReturnType<T>` | 函数返回值 | `ReturnType<() => string>` |
| `Parameters<T>` | 函数参数 | `Parameters<(x: number) => void>` |
| `Exclude<T, U>` | 从 T 排除 U | `Exclude<'a' \| 'b' \| 'c', 'a'>` |
| `Extract<T, U>` | 从 T 提取 U | `Extract<'a' \| 'b', 'a'>` |
| `NonNullable<T>` | 排除 null/undefined | `NonNullable<string \| null>` |

### 反模式：不要这样做

```typescript
// ❌ 错误：手动重复定义属性
interface CreateUserInput {
    name: string;
    email: string;
    // 和 User 重复，以后 User 改了这里也要改
}

// ✅ 正确：用 Omit 排除不需要的
type CreateUserInput = Omit<User, 'id'>;
```

### 适用场景

| 场景 | 推荐方案 | 示例 |
|------|---------|------|
| 表单更新 | `Partial<T>` | `updateUser(id, Partial<User>)` |
| 公开接口 | `Omit<T, K>` | `Omit<User, 'password'>` |
| 创建接口 | `Pick<T, K>` | `Pick<User, 'name' \| 'email'>` |
| 字典/映射 | `Record<K, V>` | `Record<Role, Permission[]>` |
| 提取类型 | `ReturnType` | `ReturnType<typeof fn>` |

## L3 专家层：深入

### 工具类型实现原理

```typescript
// Partial 实现
type MyPartial<T> = { [K in keyof T]?: T[K] };

// Required 实现
type MyRequired<T> = { [K in keyof T]-?: T[K] };

// Pick 实现
type MyPick<T, K extends keyof T> = { [P in K]: T[P] };

// Omit 实现
type MyOmit<T, K extends keyof T> = Pick<T, Exclude<keyof T, K>>;

// Record 实现
type MyRecord<K extends string | number | symbol, V> = { [P in K]: V };

// Readonly 实现
type MyReadonly<T> = { readonly [K in keyof T]: T[K] };
```

### 深层 Partial

```typescript
// 内置 Partial 只处理一层
type Nested = { user: { name: string; address: { city: string } } };
type P1 = Partial<Nested>;
// { user?: { name: string; address: { city: string } } }
// user 可选，但 user 内部的属性不可选

// 深层 Partial
type DeepPartial<T> = T extends object
    ? { [K in keyof T]?: DeepPartial<T[K]> }
    : T;

type P2 = DeepPartial<Nested>;
// { user?: { name?: string; address?: { city?: string } } }
```

### 知识关联

```
工具类型关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  属性变换   │────→│  属性挑选   │────→│  函数提取   │
│  Partial/   │     │  Pick/      │     │  ReturnType/│
│  Required/  │     │  Omit/      │     │  Parameters │
│  Readonly   │     │  Record     │     └─────────────┘
└─────────────┘     └─────────────┘
```

## 常见问题

### Q1：Partial 和 Omit 有什么区别？

**Partial 让所有属性可选，Omit 排除指定属性：**

```typescript
type A = Partial<User>;   // { id?: number; name?: string; ... }
type B = Omit<User, 'id'>; // { name: string; email: string; ... }（id 被移除）
```

### Q2：什么时候用 Pick，什么时候用 Omit？

**属性少时用 Pick，属性多时用 Omit：**

```typescript
// 只取 1-2 个属性 → Pick
type UserName = Pick<User, 'name'>;

// 排除 1-2 个属性 → Omit
type UserPublic = Omit<User, 'password' | 'token'>;
```
