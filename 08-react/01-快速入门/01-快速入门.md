## 1.1 React 简介

### 什么是 React？

React 是由 Facebook 开发的用于构建用户界面的 JavaScript 库。

```
┌─────────────────────────────────────────────────────────────┐
│  React 核心特点                                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  声明式      → 描述 UI 应该是什么样子                       │
│  组件化      → 可复用的独立 UI 单元                         │
│  单向数据流  → 数据从父组件流向子组件                       │
│  虚拟 DOM    → 高效的 UI 更新机制                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### React vs 传统开发

```javascript
// 传统方式：命令式编程
const button = document.createElement('button');
button.textContent = '点击我';
button.className = 'btn';
button.onclick = function() {
    alert('被点击了');
};
document.body.appendChild(button);

// React 方式：声明式编程
function App() {
    return (
        <button className="btn" onClick={() => alert('被点击了')}>
            点击我
        </button>
    );
}
```

## 1.2 环境搭建

### 使用 Vite 创建项目

```bash
# 创建 React 项目
npm create vite@latest my-react-app -- --template react

# 进入项目目录
cd my-react-app

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 项目结构

```
my-react-app/
├── public/
│   └── vite.svg
├── src/
│   ├── App.css
│   ├── App.jsx          # 主组件
│   ├── index.css
│   └── main.jsx         # 入口文件
├── index.html
├── package.json
├── vite.config.js
└── README.md
```

### 入口文件解析

```jsx

import './index.css'
```

## 1.3 第一个组件

### 函数组件

```jsx
// src/App.jsx
import './App.css'

function App() {
  return (
    <div className="App">
      <h1>Hello React!</h1>
      <p>这是我的第一个 React 组件</p>
    </div>
  )
}

export default App
```

### 组件的基本规则

```jsx
// ✅ 正确：组件名首字母大写
function MyComponent() {
  return <div>正确</div>
}

// ❌ 错误：组件名首字母小写
function myComponent() {
  return <div>错误</div>
}

// ✅ 正确：返回单个根元素
function GoodComponent() {
  return (
    <div>
      <h1>标题</h1>
      <p>内容</p>
    </div>
  )
}

// ❌ 错误：返回多个元素
function BadComponent() {
  return (
    <h1>标题</h1>
    <p>内容</p>
  )
}

// ✅ 正确：使用 Fragment 包裹多个元素
function FragmentComponent() {
  return (
    <>
      <h1>标题</h1>
      <p>内容</p>
    </>
  )
}
```

## 1.4 JSX 语法

### JSX 是什么？

JSX 是 JavaScript 的语法扩展，允许在 JavaScript 中编写类似 HTML 的代码。

```jsx
// JSX 语法
const element = <h1>Hello, React!</h1>;

// 编译后的 JavaScript
const element = React.createElement('h1', null, 'Hello, React!');
```

### JSX 表达式

```jsx
function App() {
  const name = '张三';
  const age = 25;

  return (
    <div>
      {/* 使用变量 */}
      <h1>Hello, {name}</h1>
      
      {/* 使用表达式 */}
      <p>年龄：{age} 岁</p>
      <p>明年：{age + 1} 岁</p>
      
      {/* 使用函数 */}
      <p>格式化：{name.toUpperCase()}</p>
      
      {/* 使用三元表达式 */}
      <p>状态：{age >= 18 ? '成年' : '未成年'}</p>
    </div>
  );
}
```

### JSX 属性

```jsx
function App() {
  const imageUrl = 'https://example.com/photo.jpg';
  const isActive = true;

  return (
    <div>
      {/* 字符串属性 */}
      <input type="text" placeholder="请输入" />
      
      {/* 动态属性 */}
      <img src={imageUrl} alt="照片" />
      
      {/* 布尔属性 */}
      <button disabled={false}>可点击</button>
      <input checked={isActive} readOnly />
      
      {/* class 使用 className */}
      <div className="container active">内容</div>
      
      {/* style 使用对象 */}
      <div style={{ color: 'red', fontSize: '16px' }}>
        红色文字
      </div>
    </div>
  );
}
```

## 1.5 条件渲染

```jsx
function App() {
  const isLoggedIn = true;
  const messages = ['消息1', '消息2'];

  return (
    <div>
      {/* if-else 使用三元表达式 */}
      {isLoggedIn ? <h1>欢迎回来</h1> : <h1>请先登录</h1>}
      
      {/* 只显示一种情况 */}
      {isLoggedIn && <button>退出</button>}
      
      {/* 列表长度判断 */}
      {messages.length > 0 ? (
        <p>你有 {messages.length} 条消息</p>
      ) : (
        <p>暂无消息</p>
      )}
    </div>
  );
}
```

## 1.6 列表渲染

```jsx
function App() {
  const fruits = [
    { id: 1, name: '苹果', price: 5 },
    { id: 2, name: '香蕉', price: 3 },
    { id: 3, name: '橙子', price: 4 }
  ];

  return (
    <ul>
      {/* 使用 map 渲染列表 */}
      {fruits.map(fruit => (
        <li key={fruit.id}>
          {fruit.name} - ¥{fruit.price}
        </li>
      ))}
    </ul>
  );
}
```

### key 的作用

```jsx
// ✅ 正确：使用唯一 id 作为 key
{items.map(item => <li key={item.id}>{item.name}</li>)}

// ❌ 错误：使用索引作为 key（可能导致问题）
{items.map((item, index) => <li key={index}>{item.name}</li>)}
```

## 1.7 事件处理

```jsx
function App() {
  // 事件处理函数
  const handleClick = () => {
    alert('按钮被点击了');
  };

  const handleMouseEnter = (e) => {
    console.log('鼠标进入', e.target);
  };

  return (
    <div>
      {/* 点击事件 */}
      <button onClick={handleClick}>点击我</button>
      
      {/* 鼠标事件 */}
      <div onMouseEnter={handleMouseEnter}>
        鼠标移入这里
      </div>
      
      {/* 传递参数 */}
      <button onClick={() => handleClick('参数')}>
        带参数点击
      </button>
      
      {/* 阻止默认行为 */}
      <form onSubmit={(e) => {
        e.preventDefault();
        console.log('表单提交');
      }}>
        <button type="submit">提交</button>
      </form>
    </div>
  );
}
```

## 1.8 常见问题

### Q1: 为什么组件名要首字母大写？

React 通过首字母大小写区分原生 HTML 元素和自定义组件。小写开头会被识别为 HTML 元素。

### Q2: 为什么需要 key？

key 帮助 React 识别列表中哪些元素发生了变化，从而高效更新 DOM。使用索引作为 key 在列表顺序变化时可能导致问题。

### Q3: JSX 和 HTML 有什么区别？

- `class` → `className`
- `for` → `htmlFor`
- `style` 接受对象而非字符串
- 事件名使用驼峰命名（`onclick` → `onClick`）
- 所有标签必须闭合
