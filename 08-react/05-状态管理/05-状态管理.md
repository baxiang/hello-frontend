## 生活化比喻

**状态管理就像"公司的信息传递系统"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                  公司信息传递                         │
├──────────────────────────────────────────────────────┤
│                                                      │
│    useState = 个人便签                                │
│    ─────────────                                     │
│    每个人自己记自己的事                              │
│    不跟别人分享                                      │
│                                                      │
│    Context = 公司公告板                              │
│    ─────────────                                     │
│    信息贴在公司大厅，所有员工都能看到                │
│    更新公告板，所有看公告的人自动看到新信息          │
│    缺点：公告板一变，所有人都要重新看（重新渲染）    │
│                                                      │
│    useReducer = 流程审批系统                          │
│    ─────────────                                     │
│    不能直接改状态，必须提交申请（dispatch action）   │
│    审批系统按规则处理申请，返回新状态                │
│    适合：复杂的状态变更逻辑                          │
│                                                      │
│    Zustand = 智能办公系统                            │
│    ─────────────                                     │
│    比 Context 更聪明                                 │
│    只通知关心这个数据的人（精准订阅）                │
│    代码简洁，无需 Provider 包裹                      │
│                                                      │
│    Redux = 大型企业 ERP 系统                          │
│    ─────────────                                     │
│    功能强大但配置复杂                                │
│    适合超大型团队                                   │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

### Context 基础用法

**最简示例（1-3行）：**

```tsx
const ThemeContext = createContext('light')
function App() {
  return <ThemeContext.Provider value="dark"><Child /></ThemeContext.Provider>
}
function Child() {
  const theme = useContext(ThemeContext)
  return <div className={theme}>内容</div>
}
```

**详细示例：**

```tsx
import { createContext, useContext, useState, ReactNode } from 'react'

// 1. 创建 Context — 适合简单全局状态
interface User { id: number; name: string }
interface UserContextType { user: User | null; login: (u: User) => void; logout: () => void }

const UserContext = createContext<UserContextType>({} as UserContextType)

// 2. 创建 Provider — 包裹整个应用
function UserProvider({ children }: { children: ReactNode }) {
  const [user, setUser] = useState<User | null>(null)
  const login = (u: User) => setUser(u)
  const logout = () => setUser(null)

  return <UserContext.Provider value={{ user, login, logout }}>{children}</UserContext.Provider>
}

// 3. 使用 — 任何子组件都能访问
function Profile() {
  const { user, logout } = useContext(UserContext)
  return user ? <span>欢迎，{user.name}</span> : <button onClick={() => login({ id: 1, name: '张三' })}>登录</button>
}

// 4. 应用入口
function App() {
  return (
    <UserProvider>
      <Profile />
    </UserProvider>
  )
}
```

### useReducer

```tsx
// 适合复杂状态逻辑 — 类似小型 Redux
type Action =
  | { type: 'increment' }
  | { type: 'decrement' }
  | { type: 'reset'; payload: number }

function counterReducer(state: number, action: Action): number {
  switch (action.type) {
    case 'increment': return state + 1
    case 'decrement': return state - 1
    case 'reset': return action.payload
    default: return state
  }
}

function Counter() {
  const [count, dispatch] = useReducer(counterReducer, 0)
  return (
    <div>
      <span>{count}</span>
      <button onClick={() => dispatch({ type: 'increment' })}>+1</button>
      <button onClick={() => dispatch({ type: 'reset', payload: 0 })}>重置</button>
    </div>
  )
}
```

## L2 实践层：用好

### 状态管理方案选择

| 方案 | 适用场景 | 复杂度 | 示例 |
|------|---------|:---:|------|
| `useState` | 组件内部状态 | 低 | 表单输入、开关 |
| `useReducer` | 复杂状态逻辑 | 中 | 购物车、表单验证 |
| `Context` | 简单全局状态 | 中 | 主题、语言、用户信息 |
| `Zustand` | 中型应用 | 低 | Todo、仪表盘 |
| `Redux Toolkit` | 大型应用 | 高 | 电商平台、社交应用 |

### Zustand 示例

```tsx
import { create } from 'zustand'

interface TodoStore {
  todos: { id: number; text: string; done: boolean }[]
  addTodo: (text: string) => void
  toggleTodo: (id: number) => void
  deleteTodo: (id: number) => void
}

// 创建 store — 无需 Provider
const useTodoStore = create<TodoStore>(set => ({
  todos: [],
  addTodo: (text) => set(state => ({
    todos: [...state.todos, { id: Date.now(), text, done: false }]
  })),
  toggleTodo: (id) => set(state => ({
    todos: state.todos.map(t => t.id === id ? { ...t, done: !t.done } : t)
  })),
  deleteTodo: (id) => set(state => ({
    todos: state.todos.filter(t => t.id !== id)
  }))
}))

// 使用 — 精准订阅，只关心 todos
function TodoList() {
  const todos = useTodoStore(s => s.todos)
  const toggleTodo = useTodoStore(s => s.toggleTodo)
  return todos.map(t => <div key={t.id} onClick={() => toggleTodo(t.id)}>{t.text}</div>)
}
```

### 反模式：不要这样做

```tsx
// ❌ 错误：Context 传递频繁变化的值（导致所有消费者重新渲染）
const [state, setState] = useState({ user, theme, settings })
<MyContext.Provider value={state}>  // 每次 setState 都创建新对象

// ✅ 正确：拆分 Context 或用 useMemo 稳定引用
const value = useMemo(() => ({ user, theme }), [user, theme])
<MyContext.Provider value={value}>
```

```tsx
// ❌ 错误：在 Context Provider 中直接放大量状态
const [user, setUser] = useState()
const [theme, setTheme] = useState()
const [todos, setTodos] = useState()
// 任何一个变化都会导致所有消费者重新渲染

// ✅ 正确：拆分多个 Context 或用 Zustand
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 主题切换 | Context | 全局共享，变化不频繁 |
| 用户信息 | Context | 登录/登出时才变化 |
| 购物车 | Zustand/useReducer | 频繁更新，需要精准订阅 |
| 表单状态 | useState/useReducer | 组件内部即可 |
| 大型应用 | Zustand/Redux | 模块化、DevTools |

## L3 专家层：深入

### Context 渲染原理

```
Context 更新流程：

Provider value 变化
    ↓
所有 useContext 的组件标记为"需要重新渲染"
    ↓
React 协调（Reconciliation）
    ↓
重新渲染所有消费者组件

问题：即使组件只用了 value 的一个字段，也会重新渲染
解决：拆分 Context 或用 Zustand（精准订阅）
```

### 知识关联

```
状态管理关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  组件状态   │────→│  全局状态   │────→│  状态库     │
│  useState/  │     │  Context/   │     │  Zustand/   │
│  useReducer │     │  Provider   │     │  Redux      │
└─────────────┘     └─────────────┘     └─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **Context** | React 提供的全局状态共享机制 | `createContext`, `useContext` |
| **Provider** | 提供 Context 值的组件 | `<UserContext.Provider>` |
| **useReducer** | 复杂状态管理的 Hook | `useReducer(reducer, initial)` |
| **Zustand** | 轻量级状态管理库 | `create(() => ({}))` |
| **精准订阅** | 只订阅状态的一部分，避免不必要渲染 | `useStore(s => s.todos)` |

## 实践练习

```tsx
// 练习 1：实现带持久化的主题管理
import { createContext, useContext, useState, useEffect } from 'react'

const ThemeContext = createContext<{ theme: string; toggle: () => void }>({} as any)

function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState(() => localStorage.getItem('theme') || 'light')

  useEffect(() => {
    localStorage.setItem('theme', theme)
    document.documentElement.className = theme
  }, [theme])

  const toggle = () => setTheme(t => t === 'light' ? 'dark' : 'light')

  return <ThemeContext.Provider value={{ theme, toggle }}>{children}</ThemeContext.Provider>
}

// 练习 2：用 Zustand 实现计数器
import { create } from 'zustand'

const useCounterStore = create<{ count: number; increment: () => void; decrement: () => void }>(set => ({
  count: 0,
  increment: () => set(s => ({ count: s.count + 1 })),
  decrement: () => set(s => ({ count: s.count - 1 }))
}))
```

## 常见问题

### Q1：Context 和 Zustand 选哪个？

**简单全局状态用 Context（无需额外依赖），频繁更新的状态用 Zustand（精准订阅，避免不必要渲染）：**

```tsx
// Context — 适合不频繁变化的状态
const ThemeContext = createContext('light')  // 主题切换不频繁

// Zustand — 适合频繁变化的状态
const useStore = create(() => ({ count: 0 }))  // 计数器频繁更新
```

### Q2：useReducer 和 useState 选哪个？

**简单状态用 useState，复杂状态逻辑（多个子值、下一个状态依赖前一个）用 useReducer：**

```tsx
// 简单状态 → useState
const [count, setCount] = useState(0)

// 复杂状态 → useReducer
const [state, dispatch] = useReducer(reducer, { count: 0, step: 1 })
```

### Q3：Context 会导致性能问题吗？

**会。Provider value 变化时，所有 useContext 的组件都会重新渲染。解决方法：拆分 Context、用 useMemo 稳定引用、或用 Zustand。**
