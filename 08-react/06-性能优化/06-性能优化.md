## L1 理解层：会用

### React.memo — 避免子组件重复渲染

**最简示例（1-3行）：**

```tsx
const ExpensiveList = React.memo(({ items }) => <ul>{items.map(i => <li key={i.id}>{i.name}</li>)}</ul>)
```

**详细示例：**

```tsx
import { memo, useState } from 'react'

// 子组件 — 用 memo 包裹，props 不变时不重新渲染
const TodoItem = memo(({ todo, onToggle }: { todo: Todo; onToggle: () => void }) => {
  console.log('TodoItem 渲染')  // 只在 todo 或 onToggle 变化时输出
  return <li onClick={onToggle}>{todo.text}</li>
})

// 父组件
function TodoList({ todos }: { todos: Todo[] }) {
  const [count, setCount] = useState(0)  // 这个状态变化不会导致 TodoItem 重新渲染
  return (
    <ul>
      {todos.map(todo => (
        <TodoItem key={todo.id} todo={todo} onToggle={() => {}} />
      ))}
    </ul>
  )
}
```

### useMemo — 缓存计算结果

```tsx
function TodoList({ todos, filter }: { todos: Todo[]; filter: string }) {
  // 只在 todos 或 filter 变化时重新计算
  const filtered = useMemo(
    () => todos.filter(t => t.text.includes(filter)),
    [todos, filter]
  )
  return <ul>{filtered.map(t => <li key={t.id}>{t.text}</li>)}</ul>
}
```

---

### useCallback — 缓存函数引用

```tsx
function TodoForm({ onAdd }: { onAdd: (text: string) => void }) {
  const [text, setText] = useState('')
  // 缓存函数引用，避免每次渲染都创建新函数
  const handleSubmit = useCallback((e: React.FormEvent) => {
    e.preventDefault()
    onAdd(text)
    setText('')
  }, [onAdd, text])

  return <form onSubmit={handleSubmit}><input value={text} onChange={e => setText(e.target.value)} /></form>
}
```

### 组件懒加载

```tsx
import { lazy, Suspense } from 'react'

// 按需加载 — 只有访问这个路由时才加载代码
```

## L2 实践层：用好

### 优化工具速查

| 工具 | 用途 | 何时用 | 何时不用 |
|------|------|--------|---------|
| `React.memo` | 避免子组件重复渲染 | 组件渲染频繁且 props 不变 | 简单组件、props 经常变化 |
| `useMemo` | 缓存计算结果 | 计算开销大（过滤、排序） | 简单计算（加法、字符串拼接） |
| `useCallback` | 缓存函数引用 | 传递给 memo 子组件 | 不传给子组件的函数 |
| `lazy` | 组件懒加载 | 大组件、非首屏组件 | 小组件、首屏必需组件 |

### 反模式：不要这样做

```tsx
// ❌ 错误：过度使用 memo（增加复杂度，收益微乎其微）
const SimpleText = memo(({ text }: { text: string }) => <span>{text}</span>)

// ✅ 正确：只在必要时使用
const ExpensiveList = memo(({ items }: { items: Item[] }) => <LongList items={items} />)
```

```tsx
// ❌ 错误：useMemo 缓存简单计算
const sum = useMemo(() => a + b, [a, b])  // 加法不值得缓存

// ✅ 正确：直接计算
const sum = a + b
```

```tsx
// ❌ 错误：useCallback 依赖项过多（失去了缓存意义）
const fn = useCallback(() => { ... }, [a, b, c, d, e])  // 频繁变化

// ✅ 正确：只依赖必要的值
const fn = useCallback(() => { ... }, [a])
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 长列表渲染 | React.memo + 虚拟列表 | 减少 DOM 节点 |
| 复杂计算 | useMemo | 避免重复计算 |
| 传递给子组件的回调 | useCallback | 避免子组件重新渲染 |
| 路由组件 | lazy + Suspense | 减少首屏体积 |
| 图片加载 | 懒加载 + 占位图 | 按需加载 |

## L3 专家层：深入

### React 渲染原理

```
渲染流程：

状态变化 → 标记组件为"脏"
    ↓
React 协调（Reconciliation）
    ↓
对比新旧 Virtual DOM
    ↓
只更新变化的部分（最小化 DOM 操作）
    ↓
浏览器重排/重绘

React.memo 的作用：
在"标记为脏"阶段跳过比较，直接复用上次渲染结果
```

### 性能考量

| 操作 | 开销 | 说明 |
|------|------|------|
| 组件渲染 | 低 | React 优化得很好 |
| DOM 操作 | 高 | 重排/重绘开销大 |
| useMemo 比较 | 极低 | 浅比较依赖数组 |
| React.memo 比较 | 低 | 浅比较 props |

### 知识关联

```
性能优化关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  渲染优化   │────→│  代码分割   │────→│  渲染优化   │
│  memo/      │     │  lazy/      │     │  虚拟列表/  │
│  useMemo/   │     │  Suspense   │     │  图片懒加载 │
│  useCallback│     └─────────────┘     └─────────────┘
└─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **React.memo** | 高阶组件，props 不变时跳过渲染 | `memo(Component)` |
| **useMemo** | 缓存计算结果 | `useMemo(() => fn(), [dep])` |
| **useCallback** | 缓存函数引用 | `useCallback(fn, [dep])` |
| **lazy** | 组件懒加载 | `lazy(() => import('./About'))` |
| **虚拟列表** | 只渲染可见区域的列表项 | `react-window` |

## 常见问题

### Q1：什么时候该用 memo，什么时候不该用？

**简单组件不用（React 渲染本身很快），列表项、图表等渲染开销大的组件用 memo：**

```tsx
// 不用 memo — 简单组件
function Label({ text }: { text: string }) { return <span>{text}</span> }

// 用 memo — 列表项、复杂组件
const ListItem = memo(({ item }: { item: Item }) => <ComplexCard item={item} />)
```

### Q2：useMemo 和 useCallback 有什么区别？

**useMemo 缓存值，useCallback 缓存函数（本质是 `useMemo(() => fn, deps)`）：**

```tsx
const value = useMemo(() => computeExpensive(a, b), [a, b])
const fn = useCallback(() => doSomething(a, b), [a, b])
```

### Q3：React 性能优化有银弹吗？

**没有。过度优化反而增加复杂度。正确做法：先用 React DevTools Profiler 找到瓶颈，再针对性优化。**
