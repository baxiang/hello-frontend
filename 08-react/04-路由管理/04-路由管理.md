## L1 理解层：会用

### 基础路由配置

**最简示例（1-3行）：**

```tsx
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom'

function App() {
  return (
    <BrowserRouter>
      <nav><Link to="/">首页</Link><Link to="/about">关于</Link></nav>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  )
}
```

**详细示例：**

```tsx
import { BrowserRouter, Routes, Route, Link, Outlet, useParams, useNavigate } from 'react-router-dom'

// 布局组件 — 所有子路由共享的框架
function Layout() {
  return (
    <div>
      <nav>
        <Link to="/">首页</Link>
        <Link to="/posts">文章</Link>
      </nav>
      <Outlet /> {/* 子路由渲染位置，类似"换乘站" */}
    </div>
  )
}

// 动态路由 — 路径中带参数
function PostDetail() {
  const { id } = useParams()  // 获取 URL 中的 :id
  return <h1>文章 {id}</h1>
}

// 嵌套路由配置
function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Layout />}>
          <Route index element={<Home />} />           {/* / */}
          <Route path="posts" element={<PostList />} /> {/* /posts */}
          <Route path="posts/:id" element={<PostDetail />} /> {/* /posts/1 */}
          <Route path="*" element={<NotFound />} />    {/* 404 */}
        </Route>
      </Routes>
    </BrowserRouter>
  )
}
```

### 导航与参数

```tsx
'use client'
import { useNavigate, useSearchParams } from 'react-router-dom'

function SearchPage() {
  const navigate = useNavigate()
  const [searchParams] = useSearchParams()
  const q = searchParams.get('q')  // 获取 ?q=xxx

  return (
    <div>
      <p>搜索：{q}</p>
      <button onClick={() => navigate('/results', { state: { query: q } })}>
        搜索
      </button>
    </div>
  )
}
```

## L2 实践层：用好

### 导航方法速查

| 方法 | 用途 | 示例 |
|------|------|------|
| `<Link>` | 声明式导航 | `<Link to="/about">` |
| `useNavigate` | 编程式导航 | `navigate('/about')` |
| `useParams` | 获取路由参数 | `const { id } = useParams()` |
| `useSearchParams` | 获取查询参数 | `const [search] = useSearchParams()` |

### 路由守卫

```tsx
// 认证守卫 — 未登录自动跳转到登录页
function ProtectedRoute({ children }: { children: React.ReactNode }) {
  const isAuthenticated = useAuth()
  const navigate = useNavigate()

  if (!isAuthenticated) {
    navigate('/login', { replace: true })
    return null
  }

  return <>{children}</>
}

// 使用
<Route path="/dashboard" element={
  <ProtectedRoute><Dashboard /></ProtectedRoute>
} />
```

### 反模式：不要这样做

```tsx
// ❌ 错误：用 a 标签导航（会全页刷新）
<a href="/about">关于</a>

// ✅ 正确：用 Link 组件（客户端导航，不刷新）
<Link to="/about">关于</Link>
```

```tsx
// ❌ 错误：在 useEffect 中手动修改 window.location
useEffect(() => {
  window.location.href = '/new-page'  // 全页刷新
}, [])

// ✅ 正确：用 useNavigate
const navigate = useNavigate()
useEffect(() => {
  navigate('/new-page')  // 客户端导航
}, [])
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 页面跳转 | `<Link>` | 客户端导航，不刷新 |
| 表单提交后跳转 | `useNavigate` | 编程式控制 |
| 多页面共享布局 | 嵌套路由 + `Outlet` | 避免重复代码 |
| 权限控制 | 路由守卫 | 未登录自动拦截 |
| 搜索/筛选 | `useSearchParams` | URL 参数传递状态 |

## L3 专家层：深入

### 路由渲染原理

```
React Router 渲染流程：

用户点击 Link → 阻止默认行为
    ↓
更新 history 状态（pushState）
    ↓
触发 URL 变化
    ↓
Routes 匹配新的 path
    ↓
渲染对应的 Route element
    ↓
不刷新页面，只更新变化的组件
```

### 知识关联

```
路由管理关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  路由配置   │────→│  导航与参数 │────→│  路由守卫   │
│  BrowserRouter│  │  Link/      │     │  权限/      │
│  Routes/    │     │  useNavigate│     │  拦截/      │
│  Route      │     │  useParams  │     │  重定向     │
└─────────────┘     └─────────────┘     └─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **BrowserRouter** | 基于 HTML5 history API 的路由器 | `<BrowserRouter>` |
| **嵌套路由** | 路由内部再嵌套子路由 | `<Route path="/" element={<Layout />}>` |
| **动态路由** | 路径中带参数的路由 | `path="posts/:id"` |
| **Outlet** | 嵌套路由的子组件渲染位置 | `<Outlet />` |
| **路由守卫** | 拦截路由访问的组件 | `ProtectedRoute` |

## 实践练习

```tsx
// 练习：带权限的博客路由系统
import { BrowserRouter, Routes, Route, Link, Navigate, Outlet } from 'react-router-dom'

// 模拟认证
const useAuth = () => ({ isAuthenticated: false, role: 'user' })

// 认证守卫
function AuthGuard() {
  const { isAuthenticated } = useAuth()
  return isAuthenticated ? <Outlet /> : <Navigate to="/login" replace />
}

// 管理员守卫
function AdminGuard() {
  const { isAuthenticated, role } = useAuth()
  if (!isAuthenticated) return <Navigate to="/login" replace />
  if (role !== 'admin') return <Navigate to="/dashboard" replace />
  return <Outlet />
}

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">首页</Link>
        <Link to="/dashboard">控制台</Link>
        <Link to="/admin">管理</Link>
      </nav>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route element={<AuthGuard />}>
          <Route path="/dashboard" element={<Dashboard />} />
          <Route path="/profile" element={<Profile />} />
        </Route>
        <Route element={<AdminGuard />}>
          <Route path="/admin" element={<AdminPanel />} />
        </Route>
        <Route path="*" element={<NotFound />} />
      </Routes>
    </BrowserRouter>
  )
}
```

## 常见问题

### Q1：Link 和 a 标签有什么区别？

**Link 是客户端导航，不刷新页面；a 标签会全页刷新：**

```tsx
<Link to="/about">关于</Link>  // ✅ 客户端导航
<a href="/about">关于</a>      // ❌ 全页刷新
```

### Q2：useNavigate 和 Link 选哪个？

**声明式用 Link，编程式用 useNavigate：**

```tsx
// 静态链接 → Link
<Link to="/about">关于</Link>

// 条件跳转/表单提交后 → useNavigate
const navigate = useNavigate()
if (success) navigate('/dashboard')
```

### Q3：如何实现 404 页面？

**用 `*` 通配符路由匹配所有未定义的路径：**

```tsx
<Routes>
  <Route path="/" element={<Home />} />
  <Route path="*" element={<NotFound />} />
</Routes>
```
