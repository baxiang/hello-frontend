## 3.1 什么是数据库？

### 3.1.1 数据库简介

```
┌─────────────────────────────────────────────────────────────┐
│  数据库 = 数据的"仓库"                                       │
│                                                             │
│  就像：                                                     │
│  - 图书馆：存储书籍，管理借阅                                 │
│  - 仓库：存储货物，管理库存                                  │
│  - 银行：存储资金，管理交易                                  │
│                                                             │
│  主要功能：                                                  │
│  - 存储数据                                                 │
│  - 查询数据                                                 │
│  - 更新数据                                                 │
│  - 删除数据                                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.1.2 数据库类型

```
┌─────────────────────────────────────────────────────────────┐
│                    数据库类型对比                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  关系型数据库（SQL）：                                       │
│  - MySQL、PostgreSQL、Oracle、SQL Server                    │
│  - 数据存储在表中                                            │
│  - 使用 SQL 查询                                            │
│  - 适合结构化数据                                            │
│  - 例子：用户表、订单表、商品表                              │
│                                                             │
│  非关系型数据库（NoSQL）：                                   │
│  - MongoDB、Redis、Cassandra                                │
│  - 数据存储在文档/键值对/图等形式                           │
│  - 灵活的数据模型                                            │
│  - 适合海量数据、高并发                                       │
│  - 例子：用户文档、日志、缓存                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 3.2 MongoDB 基础

### 3.2.1 MongoDB 简介

```
┌─────────────────────────────────────────────────────────────┐
│  MongoDB = 文档型数据库                                      │
│                                                             │
│  特点：                                                     │
│  - 存储 JSON 格式的文档（BSON）                             │
│  - 无需预先定义表结构                                        │
│  - 支持丰富的查询                                            │
│  - 易于扩展                                                 │
│                                                             │
│  与 MySQL 对比：                                            │
│  ┌─────────────┬─────────────┐                              │
│  │   MongoDB   │   MySQL     │                              │
│  ├─────────────┼─────────────┤                              │
│  │  数据库      │  数据库      │                              │
│  │  集合        │  表         │                              │
│  │  文档        │  行         │                              │
│  │  字段        │  列         │                              │
│  └─────────────┴─────────────┘                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2.2 文档结构示例

```javascript
// MongoDB 文档（类似 JSON）
{
    _id: ObjectId("507f1f77bcf86cd799439011"),
    name: "张三",
    age: 25,
    email: "zhang@example.com",
    hobbies: ["读书", "游泳", "编程"],
    address: {
        city: "北京",
        district: "朝阳"
    },
    createdAt: ISODate("2026-01-01T00:00:00Z")
}
```

// 使用原生驱动连接
const { MongoClient } = require('mongodb');

const uri = 'mongodb://localhost:27017';
const client = new MongoClient(uri);

async function run() {
    try {
        await client.connect();
        console.log('✅ MongoDB 连接成功');

    } finally {
        await client.close();
    }
}

run().catch(console.dir);

## 3.3 Mongoose ORM

### 3.3.1 什么是 Mongoose？

```
┌─────────────────────────────────────────────────────────────┐
│  Mongoose = MongoDB 的 ORM 框架                              │
│                                                             │
│  ORM = Object Relational Mapping                            │
│  对象关系映射                                                │
│                                                             │
│  就像：                                                     │
│  - 翻译员：把中文翻译成英文                                  │
│  - 适配器：把插头转换成插座                                  │
│  - 模板：按模板填写内容                                      │
│                                                             │
│  Mongoose 的作用：                                           │
│  - 定义数据模型（Schema）                                   │
│  - 提供 CRUD 操作方法                                        │
│  - 数据验证                                                  │
│  - 中间件支持                                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.3.2 连接数据库

```javascript
const mongoose = require('mongoose');

// 连接数据库
mongoose.connect('mongodb://localhost:27017/myapp', {
    useNewUrlParser: true,
    useUnifiedTopology: true
})
.then(() => console.log('✅ MongoDB 连接成功'))
.catch(err => console.error('❌ 连接失败:', err));

// 监听连接状态
mongoose.connection.on('connected', () => {
    console.log('Mongoose 已连接到 MongoDB');
});

mongoose.connection.on('error', (err) => {
    console.error('Mongoose 连接错误:', err);
});
```

### 3.3.3 定义 Schema

```javascript
const mongoose = require('mongoose');

// 定义用户 Schema
const userSchema = new mongoose.Schema({
    // 基础字段
    name: {
        type: String,
        required: true,      // 必填
        trim: true,          // 去除空格
        maxlength: 50        // 最大长度
    },
    email: {
        type: String,
        required: true,
        unique: true,        // 唯一索引
        lowercase: true      // 转小写
    },
    age: {
        type: Number,
        min: 0,              // 最小值
        max: 150             // 最大值
    },
    // 枚举类型
    role: {
        type: String,
        enum: ['user', 'admin', 'guest'],
        default: 'user'
    },
    // 嵌套对象
    address: {
        city: String,
        district: String,
        street: String
    },
    // 数组
    hobbies: [String],
    // 日期
    createdAt: {
        type: Date,
        default: Date.now
    }
}, {
    timestamps: true  // 自动添加 createdAt 和 updatedAt
});

// 创建模型
const User = mongoose.model('User', userSchema);
```

### 3.3.4 CRUD 操作

```javascript
// 创建文档
async function createUser() {
    const user = new User({
        name: '张三',
        email: 'zhang@example.com',
        age: 25,
        role: 'admin',
        hobbies: ['编程', '读书'],
        address: { city: '北京', district: '朝阳' }
    });
    
    const savedUser = await user.save();
    console.log('创建用户:', savedUser);
}

// 查询文档
async function queryUsers() {
    // 查询所有
    const users = await User.find();
    
    // 条件查询
    const admin = await User.find({ role: 'admin' });
    
    // 单个查询
    const user = await User.findOne({ email: 'zhang@example.com' });
    
    // 排序和分页
    const page = await User.find()
        .sort({ createdAt: -1 })  // 降序
        .skip(0)
        .limit(10);
    
    // 字段选择
    const names = await User.find({}, 'name email');
}

// 更新文档
async function updateUser(id) {
    // 方式 1：save 方法
    const user = await User.findById(id);
    user.age = 26;
    await user.save();
    
    // 方式 2：findByIdAndUpdate
    const updated = await User.findByIdAndUpdate(
        id,
        { age: 27 },
        { new: true }  // 返回更新后的文档
    );
    
    // 方式 3：updateOne
    await User.updateOne(
        { _id: id },
        { $set: { age: 28 } }
    );
}

// 删除文档
async function deleteUser(id) {
    // 删除单个
    await User.findByIdAndDelete(id);
    
    // 批量删除
    await User.deleteMany({ role: 'guest' });
}
```

## 3.4 数据验证

### 3.4.1 内置验证器

```javascript
const userSchema = new mongoose.Schema({
    // 字符串验证
    name: {
        type: String,
        required: [true, '用户名必须填写'],
        minlength: [2, '用户名至少2个字符'],
        maxlength: [20, '用户名最多20个字符'],
        trim: true
    },
    email: {
        type: String,
        required: true,
        match: [/^\w+@\w+\.\w+$/, '邮箱格式不正确']
    },
    
    // 数字验证
    age: {
        type: Number,
        min: [0, '年龄不能小于0'],
        max: [150, '年龄不能大于150']
    },
    
    // 枚举验证
    role: {
        type: String,
        enum: {
            values: ['user', 'admin', 'guest'],
            message: '角色必须是 user/admin/guest'
        }
    },
    
    // 自定义验证
    password: {
        type: String,
        validate: {
            validator: function(v) {
                return v.length >= 6;
            },
            message: '密码至少6个字符'
        }
    }
});
```

### 3.4.2 异步验证

```javascript
const userSchema = new mongoose.Schema({
    email: {
        type: String,
        validate: {
            validator: async function(email) {
                // 检查邮箱是否已存在
                const count = await this.constructor.countDocuments({ email });
                return count === 0;
            },
            message: '邮箱已被使用'
        }
    }
});
```

const articleSchema = new mongoose.Schema({

## 3.6 实践练习

### 练习 1：用户管理 API

```javascript
const express = require('express');
const mongoose = require('mongoose');
const app = express();
app.use(express.json());

// 连接数据库
mongoose.connect('mongodb://localhost:27017/userdb');

// 定义 User 模型
const userSchema = new mongoose.Schema({
    name: { type: String, required: true },
    email: { type: String, required: true, unique: true },
    age: Number,
    role: { type: String, enum: ['user', 'admin'], default: 'user' }
}, { timestamps: true });

const User = mongoose.model('User', userSchema);

// CRUD 路由
app.get('/users', async (req, res) => {
    const users = await User.find();
    res.json(users);
});

app.post('/users', async (req, res) => {
    try {
        const user = new User(req.body);
        await user.save();
        res.status(201).json(user);
    } catch (err) {
        res.status(400).json({ error: err.message });
    }
});

app.get('/users/:id', async (req, res) => {
    const user = await User.findById(req.params.id);
    if (!user) {
        return res.status(404).json({ error: '用户不存在' });
    }
    res.json(user);
});

app.put('/users/:id', async (req, res) => {
    const user = await User.findByIdAndUpdate(
        req.params.id,
        req.body,
        { new: true, runValidators: true }
    );
    res.json(user);
});

app.delete('/users/:id', async (req, res) => {
    await User.findByIdAndDelete(req.params.id);
    res.status(204).send();
});

app.listen(3000);
```

## 3.7 常见问答

### Q1: 什么时候用 MongoDB？什么时候用 MySQL？

```javascript
// MongoDB 适合：
// - 数据结构灵活多变
// - 大量非结构化数据
// - 需要快速开发
// - 日志、内容管理、用户画像

// MySQL 适合：
// - 数据结构固定
// - 需要复杂查询
// - 事务支持
// - 金融、订单、库存系统
```

### Q2: Mongoose 的 Schema 和 Model 有什么区别？

```javascript
// Schema = 模板/规则
// 定义数据的结构和验证规则
const userSchema = new mongoose.Schema({
    name: String  // 名字必须是字符串
});

// Model = 实际的"表格"
// 根据 Schema 创建的实际数据操作对象
const User = mongoose.model('User', userSchema);

// 使用
const user = new User({ name: '张三' });  // 创建
await user.save();                        // 保存
```

### Q3: 如何优化 MongoDB 查询性能？

```javascript
// 1. 创建索引
await User.createIndex({ email: 1 });  // 单字段索引
await User.createIndex({ name: 1, age: -1 });  // 复合索引

// 2. 选择性返回字段
await User.find({}, 'name email');  // 只返回 name 和 email

// 3. 分页查询
await User.find().skip(100).limit(10);

// 4. 使用 explain 分析查询
await User.find().explain();
```
