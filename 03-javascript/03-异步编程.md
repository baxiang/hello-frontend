## 3.1 同步与异步 - 有什么区别？

### 什么是同步？

**同步就是"排队"，一件事做完再做下一件。**

```
生活例子：
食堂打饭排队：
1. 打饭 → 等打完
2. 打菜 → 等打完
3. 结账 → 等结完
4. 找座位

必须按顺序，不能插队
```

```javascript
// 同步代码：按顺序执行
console.log('1. 打饭');
console.log('2. 打菜');
console.log('3. 结账');
// 输出：1 → 2 → 3（按顺序）
```

### 什么是异步？

**异步就是"拿号等叫"，不用一直等着。**

```
生活例子：
奶茶店点单：
1. 点单付款
2. 拿到号码牌
3. 去旁边玩手机（做其他事）
4. 听到叫号去取茶

不用一直站在柜台等
```

```javascript
// 异步代码：不等待，继续执行
console.log('1. 点单');

setTimeout(() => {
    console.log('2. 取茶（1 秒后）');
}, 1000);

console.log('3. 玩手机');
// 输出：1 → 3 → 2（2 虽然在前，但最后执行）
```

### 为什么需要异步？

```
场景 1：网络请求
- 如果同步：页面卡住，等数据回来才能操作
- 如果异步：可以先操作，数据回来再更新

场景 2：定时器
- 如果同步：无法实现倒计时、轮播图
- 如果异步：可以轻松实现各种定时功能

场景 3：用户交互
- 如果同步：点击按钮后要等处理完才能点下一个
- 如果异步：可以同时处理多个操作
```

## 3.2 事件循环 - JavaScript 的心脏

### 什么是事件循环？

**事件循环是 JavaScript 处理异步的机制。**

```
┌─────────────────────────────────────────┐
│           调用栈（Call Stack）           │
│  ┌─────────────────────────────────┐    │
│  │  同步代码在这里执行              │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
              ↓ 执行完
┌─────────────────────────────────────────┐
│     微任务队列    │    宏任务队列        │
│  Promise.then   │  setTimeout          │
│  MutationObserver│  setInterval         │
│                 │  I/O                 │
└─────────────────────────────────────────┘
```

### 执行顺序

```javascript
console.log('1. 同步代码');

setTimeout(() => {
    console.log('2. 宏任务');
}, 0);

Promise.resolve().then(() => {
    console.log('3. 微任务');
});

console.log('4. 同步代码');

// 输出顺序：1 → 4 → 3 → 2
// 原因：
// 1. 执行同步代码（1、4）
// 2. 执行微任务（3）
// 3. 执行宏任务（2）
```

### 记忆口诀

```
同步代码最先执行
微任务优先于宏任务
多个微任务按顺序执行
多个宏任务按顺序执行
```

## 3.3 回调函数 - 最早的异步方案

### 什么是回调函数？

**回调就是"完成后调用"的函数。**

```
生活例子：
- 你：老板，文件打印好了叫我（回调函数）
- 老板：好，打印完我叫你
- 老板：（打印完成后）文件好了！
```

```javascript
// 回调函数示例
function printFile(callback) {
    console.log('开始打印...');
    setTimeout(() => {
        console.log('打印完成');
        callback();  // 回调函数
    }, 1000);
}

printFile(() => {
    console.log('去取文件');
});
```

### 回调地狱

```javascript
// 场景：顺序加载 3 个文件
loadFile('file1.txt', (content1) => {
    console.log(content1);
    loadFile('file2.txt', (content2) => {
        console.log(content2);
        loadFile('file3.txt', (content3) => {
            console.log(content3);
            // 回调地狱：嵌套越来越深
            loadFile('file4.txt', (content4) => {
                // ...
            });
        });
    });
});

// 像金字塔，难以阅读和维护
```

### 回调的问题

```
1. 代码嵌套深，难以阅读
2. 错误处理困难
3. 无法使用 try/catch
4. 代码复用性差
```

## 3.4 Promise - 异步的新方案

### 什么是 Promise？

**Promise 是异步操作的替代品，让异步代码更像同步。**

```
生活例子：
Promise 就像"取餐凭证"：
- 点餐后给你凭证（Promise 对象）
- 凭证有 3 种状态：
  1. 制作中（pending）
  2. 做好了（fulfilled）
  3. 做不了（rejected）
```

### Promise 的三种状态

```javascript
// pending（进行中）→ fulfilled（成功）→ rejected（失败）

const promise = new Promise((resolve, reject) => {
    // 异步操作
    setTimeout(() => {
        const success = true;
        
        if (success) {
            resolve('成功了');  // 状态变为 fulfilled
        } else {
            reject('失败了');   // 状态变为 rejected
        }
    }, 1000);
});
```

### 使用 Promise

```javascript
promise
    .then((result) => {
        // 成功时的回调
        console.log(result);  // '成功了'
    })
    .catch((error) => {
        // 失败时的回调
        console.log(error);  // '失败了'
    })
    .finally(() => {
        // 无论成功失败都会执行
        console.log('结束了');
    });
```

### Promise 示例

```javascript
// 模拟加载数据
function loadData(url) {
    return new Promise((resolve, reject) => {
        console.log(`正在加载 ${url}...`);
        
        setTimeout(() => {
            if (url) {
                resolve({ data: '数据内容', url });
            } else {
                reject(new Error('URL 不能为空'));
            }
        }, 1000);
    });
}

// 使用
loadData('/api/users')
    .then(result => console.log(result))
    .catch(error => console.error(error));
```

## 3.5 Promise 链式调用 - 解决回调地狱

### 链式调用

```javascript
// 回调地狱
loadFile('file1.txt', (content1) => {
    loadFile('file2.txt', (content2) => {
        loadFile('file3.txt', (content3) => {
            console.log(content1, content2, content3);
        });
    });
});

// Promise 链式
loadFile('file1.txt')
    .then(content1 => {
        return loadFile('file2.txt');
    })
    .then(content2 => {
        return loadFile('file3.txt');
    })
    .then(content3 => {
        console.log('全部加载完成');
    })
    .catch(error => {
        console.error('出错了:', error);
    });
```

### 链式调用规则

```javascript
// 1. .then() 返回新的 Promise
Promise.resolve(1)
    .then(n => n + 1)    // 返回 2
    .then(n => n * 2)    // 返回 4
    .then(n => console.log(n));  // 4

// 2. 可以返回普通值或 Promise
Promise.resolve(1)
    .then(n => n + 1)        // 返回普通值 2
    .then(n => Promise.resolve(n * 2))  // 返回 Promise
    .then(promise => console.log(promise));  // 4

// 3. 错误会一直传递到 .catch()
Promise.resolve()
    .then(() => {
        throw new Error('错误');
    })
    .catch(err => console.error(err));
```

## 3.6 Promise 组合方法

### Promise.all - 全部成功

```javascript
// 等待所有 Promise 都成功
Promise.all([
    fetch('/api/users'),
    fetch('/api/posts'),
    fetch('/api/comments')
])
.then(([users, posts, comments]) => {
    console.log('全部加载完成');
    // 可以同时访问三个结果
})
.catch(error => {
    // 只要有一个失败就会执行
    console.error('有请求失败了');
});

// 生活例子：
// 就像等公交，要等所有乘客都上车才发车
// 只要有一个人没到，就等着
```

### Promise.race - 竞赛

```javascript
// 哪个 Promise 先完成就用哪个
Promise.race([
    fetch('/api/fast'),
    fetch('/api/slow')
])
.then(response => {
    console.log('先完成的请求:', response);
});

// 生活例子：
// 就像打车，谁先到就用谁
// 不管其他车
```

### Promise.allSettled - 不管成功失败

```javascript
// 等待所有 Promise 完成（不管成功失败）
Promise.allSettled([
    fetch('/api/users'),
    fetch('/api/error'),
    fetch('/api/posts')
])
.then(results => {
    results.forEach(result => {
        if (result.status === 'fulfilled') {
            console.log('成功:', result.value);
        } else {
            console.log('失败:', result.reason);
        }
    });
});

// 生活例子：
// 就像发快递给多个人
// 不管成功失败，最后都告诉你结果
```

### Promise.any - 有一个成功就行

```javascript
// 只要有一个 Promise 成功就成功
Promise.any([
    fetch('/api/server1'),
    fetch('/api/server2'),
    fetch('/api/server3')
])
.then(response => {
    console.log('成功的请求:', response);
})
.catch(error => {
    // 所有都失败才会执行
    console.error('所有请求都失败了');
});

// 生活例子：
// 就像问路，只要有一个人告诉你正确路线就行
```

## 3.7 async/await - 异步的终极方案

### 什么是 async/await？

**async/await 让异步代码写起来像同步代码。**

```
生活例子：
Promise 链式：
- 点餐 → 等叫号 → 取餐 → 吃
- 需要一步步 .then()

async/await：
- 点餐
- 等叫号（这里等着）
- 取餐
- 吃
- 代码更直观
```

### async 函数

```javascript
// async 函数总是返回 Promise
async function greet() {
    return 'Hello';  // 相当于 Promise.resolve('Hello')
}

greet().then(msg => console.log(msg));  // 'Hello'

// 箭头函数
const greet2 = async () => 'Hello';
```

### await 表达式

```javascript
// await 只能在 async 函数中使用
// await 让 Promise 等待完成

async function loadData() {
    console.log('开始加载');
    
    const response = await fetch('/api/users');
    // 代码在这里等待，直到 fetch 完成
    
    const data = await response.json();
    // 又在这里等待
    
    console.log('加载完成:', data);
    return data;
}

// 使用
loadData();
```

### async/await 错误处理

```javascript
// 用 try/catch 处理错误
async function loadData() {
    try {
        const response = await fetch('/api/users');
        
        if (!response.ok) {
            throw new Error('请求失败');
        }
        
        const data = await response.json();
        return data;
    } catch (error) {
        console.error('出错了:', error);
        // 可以在这里处理错误
        // 也可以重新抛出
        throw error;
    }
}
```

### Promise 链式 vs async/await

```javascript
// Promise 链式
function loadData1() {
    return fetch('/api/users')
        .then(res => res.json())
        .then(data => {
            console.log(data);
            return data;
        })
        .catch(err => console.error(err));
}

// async/await（更清晰）
async function loadData2() {
    try {
        const response = await fetch('/api/users');
        const data = await response.json();
        console.log(data);
        return data;
    } catch (err) {
        console.error(err);
    }
}
```

## 3.8 实际应用场景

### 场景 1：顺序请求

```javascript
// 需要先获取用户，再获取用户的帖子
async function getUserPosts(userId) {
    // 1. 获取用户信息
    const userRes = await fetch(`/api/users/${userId}`);
    const user = await userRes.json();
    
    // 2. 获取用户帖子
    const postsRes = await fetch(`/api/posts?userId=${userId}`);
    const posts = await postsRes.json();
    
    return { user, posts };
}
```

### 场景 2：并行请求

```javascript
// 同时请求多个数据
async function getDashboardData() {
    // 三个请求同时发起
    const [users, posts, comments] = await Promise.all([
        fetch('/api/users').then(res => res.json()),
        fetch('/api/posts').then(res => res.json()),
        fetch('/api/comments').then(res => res.json())
    ]);
    
    return { users, posts, comments };
}

// 比顺序请求快
// 顺序：1s + 1s + 1s = 3s
// 并行：max(1s, 1s, 1s) = 1s
```

### 场景 3：重试机制

```javascript
// 失败自动重试
async function fetchWithRetry(url, maxRetries = 3) {
    for (let i = 0; i < maxRetries; i++) {
        try {
            const response = await fetch(url);
            if (response.ok) {
                return await response.json();
            }
        } catch (error) {
            console.log(`第${i + 1}次请求失败`);
            if (i === maxRetries - 1) throw error;
            
            // 等待 1 秒再重试
            await new Promise(resolve => setTimeout(resolve, 1000));
        }
    }
}
```

### 场景 4：超时处理

```javascript
// 请求超时
async function fetchWithTimeout(url, timeout = 5000) {
    const controller = new AbortController();
    const timeoutId = setTimeout(() => controller.abort(), timeout);
    
    try {
        const response = await fetch(url, {
            signal: controller.signal
        });
        clearTimeout(timeoutId);
        return await response.json();
    } catch (error) {
        clearTimeout(timeoutId);
        if (error.name === 'AbortError') {
            throw new Error('请求超时');
        }
        throw error;
    }
}
```

## 3.10 实践练习

### 练习 1：图片加载器

```javascript
// 要求：
// 1. 加载多张图片
// 2. 显示加载进度
// 3. 所有图片加载完成后显示

class ImageLoader {
    constructor(containerId) {
        this.container = document.getElementById(containerId);
        this.loaded = 0;
        this.total = 0;
    }
    
    async load(images) {
        this.total = images.length;
        this.loaded = 0;
        
        // 显示进度
        this.updateProgress();
        
        // 并行加载所有图片
        const promises = images.map(src => this.loadImage(src));
        
        try {
            await Promise.all(promises);
            console.log('所有图片加载完成');
        } catch (error) {
            console.error('有图片加载失败:', error);
        }
    }
    
    loadImage(src) {
        return new Promise((resolve, reject) => {
            const img = new Image();
            
            img.onload = () => {
                this.loaded++;
                this.updateProgress();
                this.container.appendChild(img);
                resolve(img);
            };
            
            img.onerror = () => {
                this.loaded++;
                this.updateProgress();
                reject(new Error(`加载失败：${src}`));
            };
            
            img.src = src;
        });
    }
    
    updateProgress() {
        console.log(`进度：${this.loaded}/${this.total}`);
    }
}

// 使用
const loader = new ImageLoader('gallery');
loader.load([
    'image1.jpg',
    'image2.jpg',
    'image3.jpg'
]);
```

### 练习 2：API 请求队列

```javascript
// 要求：
// 1. 限制并发请求数
// 2. 按顺序处理结果

class RequestQueue {
    constructor(concurrency = 3) {
        this.concurrency = concurrency;  // 最大并发数
        this.queue = [];
        this.running = 0;
    }
    
    add(requestFn) {
        return new Promise((resolve, reject) => {
            this.queue.push({ requestFn, resolve, reject });
            this.next();
        });
    }
    
    async next() {
        if (this.running >= this.concurrency) return;
        if (this.queue.length === 0) return;
        
        const { requestFn, resolve, reject } = this.queue.shift();
        this.running++;
        
        try {
            const result = await requestFn();
            resolve(result);
        } catch (error) {
            reject(error);
        } finally {
            this.running--;
            this.next();
        }
    }
}

// 使用
const queue = new RequestQueue(3);

// 添加 10 个请求
for (let i = 0; i < 10; i++) {
    queue.add(() => 
        fetch(`/api/data/${i}`).then(res => res.json())
    ).then(data => {
        console.log(`请求${i}完成`, data);
    });
}
```

### 练习 3：批量上传

```javascript
// 要求：
// 1. 支持暂停/继续
// 2. 显示上传进度
// 3. 失败自动重试

class BatchUploader {
    constructor() {
        this.paused = false;
        this.cancelled = false;
    }
    
    async upload(files, batchSize = 5) {
        this.cancelled = false;
        this.paused = false;
        
        const batches = this.chunk(files, batchSize);
        const results = [];
        
        for (const batch of batches) {
            if (this.cancelled) {
                throw new Error('已取消');
            }
            
            while (this.paused) {
                await new Promise(r => setTimeout(r, 100));
            }
            
            const batchResults = await this.uploadBatch(batch);
            results.push(...batchResults);
        }
        
        return results;
    }
    
    async uploadBatch(files) {
        const promises = files.map(file => 
            this.uploadFile(file).catch(err => ({ error: err, file }))
        );
        
        return await Promise.all(promises);
    }
    
    async uploadFile(file) {
        const formData = new FormData();
        formData.append('file', file);
        
        const response = await fetch('/api/upload', {
            method: 'POST',
            body: formData
        });
        
        return await response.json();
    }
    
    chunk(array, size) {
        const chunks = [];
        for (let i = 0; i < array.length; i += size) {
            chunks.push(array.slice(i, i + size));
        }
        return chunks;
    }
    
    pause() {
        this.paused = true;
    }
    
    resume() {
        this.paused = false;
    }
    
    cancel() {
        this.cancelled = true;
    }
}

// 使用
const uploader = new BatchUploader();

const files = [/* 文件列表 */];

uploader.upload(files, 5)
    .then(results => console.log('上传完成', results))
    .catch(err => console.error('上传失败', err));

// 暂停
// uploader.pause();

// 继续
// uploader.resume();

// 取消
// uploader.cancel();
```

## 3.11 常见问答

### Q1: setTimeout 和 Promise 有什么区别？

**答：**
- setTimeout 是宏任务
- Promise.then 是微任务
- Promise.then 比 setTimeout 先执行

```javascript
setTimeout(() => console.log('timeout'), 0);
Promise.resolve().then(() => console.log('promise'));
// 输出：promise → timeout
```

### Q2: async/await 和 Promise 有什么关系？

**答：** async/await 基于 Promise。

```javascript
// async 函数返回 Promise
async function fn() {
    return 'hello';
}
// 等价于
function fn() {
    return Promise.resolve('hello');
}

// await 等待 Promise
const result = await promise;
// 等价于
promise.then(result => { /* ... */ });
```

### Q3: 如何处理多个异步操作？

**答：** 根据场景选择：

```javascript
// 顺序执行（一个完成后执行下一个）
await loadUser();
await loadPosts();

// 并行执行（同时执行）
await Promise.all([loadUser(), loadPosts()]);

// 竞赛（哪个快用哪个）
await Promise.race([fetch1(), fetch2()]);

// 不管成功失败
await Promise.allSettled([task1(), task2()]);
```
