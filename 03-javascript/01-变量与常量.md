# 01-变量与常量 - 存储数据的容器

> 用生活中的例子理解 JavaScript 变量的基本概念

## 学习目标

- ✅ 理解什么是变量和常量
- ✅ 掌握 var、let、const 三种声明方式的区别
- ✅ 理解变量提升和作用域的概念
- ✅ 学会正确的命名规范

---

## 1.1 什么是变量？

### 概念讲解

**变量就像一个带标签的盒子**，用来存储数据。

```
┌─────────────────────────────┐
│      变量：age              │
│  ┌─────────────────────┐    │
│  │         25          │    │
│  └─────────────────────┘    │
└─────────────────────────────┘
   标签（变量名）   盒子里的值
```

### 生活例子

想象你在搬家，需要给箱子贴标签：

```
箱子 → 装衣服
盒子 → 装书
```

在 JavaScript 中，变量就是这些"箱子"，标签就是变量名，箱子里的东西就是值。

## 1.2 三种声明方式

### var - 老式声明（不推荐）

```javascript
// var 是 JavaScript 早期的声明方式
var name = '张三';
var name = '李四';  // 可以重复声明，不会报错

// 问题：容易意外覆盖
var age = 25;
if (true) {
    var age = 30;  // 覆盖了外面的 age
}
console.log(age);  // 30（可能不是你想要的）
```

**var 的问题：**
- 可以重复声明，容易出错
- 没有块级作用域，只有函数作用域
- 存在变量提升，可能导致困惑

### let - 现代声明（推荐）

```javascript
// let 声明的变量可以修改
let age = 25;
age = 26;  // ✅ 可以重新赋值

// 不能重复声明
let name = '张三';
// let name = '李四';  // ❌ 报错：Identifier 'name' has already been declared

// 块级作用域 - 只在 {} 内有效
let outside = '外面';
if (true) {
    let inside = '里面';
    console.log(outside);  // ✅ 可以访问
}
console.log(inside);  // ❌ 报错：inside is not defined
```

**let 的特点：**
- 不能重复声明
- 有块级作用域（只在 `{}` 内有效）
- 不存在变量提升

### const - 常量（推荐用于不变的值）

```javascript
// const 声明后不能重新赋值
const PI = 3.14159;
// PI = 3;  // ❌ 报错：Assignment to constant variable

// 但对象的属性可以修改
const user = { name: '张三' };
user.name = '李四';  // ✅ 可以修改对象属性
// user = {};  // ❌ 不能重新赋值

// 数组也一样
const arr = [1, 2, 3];
arr.push(4);  // ✅ 可以往数组里加元素
// arr = [];  // ❌ 不能重新赋值
```

**const 的特点：**
- 声明后不能重新赋值
- 但对象/数组的内容可以修改
- 必须初始化（声明时就赋值）

## 1.3 三种声明方式对比

| 特性 | var | let | const |
|------|-----|-----|-------|
| 作用域 | 函数作用域 | 块级作用域 | 块级作用域 |
| 变量提升 | ✅ 是 | ❌ 否 | ❌ 否 |
| 重复声明 | ✅ 允许 | ❌ 不允许 | ❌ 不允许 |
| 重新赋值 | ✅ 允许 | ✅ 允许 | ❌ 不允许 |
| 必须初始化 | ❌ 否 | ❌ 否 | ✅ 是 |

### 使用建议

```javascript
// 优先使用 const
const MAX_COUNT = 100;
const API_URL = 'https://api.example.com';

// 需要修改时用 let
let count = 0;
count++;

// 避免使用 var
// var name = '张三';  // ❌ 不推荐
```

## 1.4 变量提升（Hoisting）

### 什么是变量提升？

**变量提升**是 JavaScript 的一种机制，变量声明会被"提升"到作用域顶部。

```javascript
// 代码写在这里
console.log(name);  // undefined
var name = '张三';

// 实际执行的是这样（提升后）：
var name;           // 声明被提升到顶部
console.log(name);  // undefined
name = '张三';      // 赋值留在原地
```

### let/const 没有变量提升

```javascript
console.log(age);  // ❌ 报错：Cannot access 'age' before initialization
let age = 25;

// 在声明前访问 let/const 声明的变量会报错
// 这叫"暂时性死区"（TDZ）
```

## 1.5 命名规范

### 合法 vs 非法

```javascript
// ✅ 合法的变量名
let userName;      // 驼峰命名（推荐）
let $element;      // 可以包含 $
let _private;      // 可以包含下划线
let 中文;          // 可以用中文（但不推荐）

// ❌ 非法的变量名
// let 123abc;     // ❌ 不能数字开头
// let user-name;  // ❌ 不能包含连字符
// let class;      // ❌ 不能用保留字
```

### 命名建议

```javascript
// 变量名：小驼峰
let userName = '张三';
let totalCount = 100;

// 常量名：全大写 + 下划线
const MAX_SIZE = 100;
const API_KEY = 'abc123';

// 布尔值：常用前缀
let isLoggedIn = true;
let hasPermission = false;
let canEdit = true;

// 数组：复数形式
let users = [];
let productList = [];
```

### JavaScript 保留字

```javascript
// 以下单词不能用作变量名
// break, case, catch, class, const, continue
// debugger, default, delete, do, else, export
// extends, false, finally, for, function, if
// import, in, instanceof, new, null, return
// super, switch, this, throw, true, try, typeof
// var, void, while, with, yield
```

## 1.6 实践练习

### 练习 1：变量声明方式对比

```javascript
// 1. 用 var 声明一个变量并修改它
var score = 100;
score = 90;
console.log(score);  // 90

// 2. 用 let 声明一个变量并修改它
let age = 25;
age = 26;
console.log(age);  // 26

// 3. 用 const 声明一个常量
const BIRTHDAY = '2000-01-01';
console.log(BIRTHDAY);

// 4. 尝试修改 const 声明的对象属性
const user = { name: '张三' };
user.name = '李四';  // ✅ 可以
console.log(user.name);  // 李四

// 5. 尝试重新赋值 const 变量（会报错）
// const PI = 3.14;
// PI = 3;  // ❌ 报错
```

### 练习 2：变量提升实验

```javascript
// 1. 观察 var 的变量提升
console.log(message);  // undefined
var message = 'Hello';

// 2. 观察 let 的暂时性死区
// console.log(greeting);  // ❌ 报错
// let greeting = 'Hi';

// 3. 函数中的变量提升
function test() {
    console.log(x);  // undefined
    var x = 10;
}
test();
```

## 1.7 常见问答

### Q1: 什么时候用 let，什么时候用 const？

**原则：优先用 const，需要修改时用 let**

```javascript
// 值不会变 → const
const PI = 3.14159;
const API_URL = 'https://api.example.com';

// 值会变 → let
let count = 0;
let isLoggedIn = false;
```

### Q2: const 声明的对象为什么能修改属性？

```javascript
const user = { name: '张三' };
user.name = '李四';  // ✅ 可以
```

**解释：** const 保证的是"引用"不变，而不是"内容"不变。

```
const user → 0x001 (内存地址)
             ↓
         { name: '张三' }

// user.name = '李四' 修改的是地址 0x001 处的内容
// user = {} 会改变 user 的引用地址，所以不允许
```

### Q3: var 和 let 在循环中有什么区别？

```javascript
// var 的问题
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// 输出：3, 3, 3（i 是同一个变量）

// let 的解决方案
for (let i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// 输出：0, 1, 2（每次循环 i 都是新变量）
```
