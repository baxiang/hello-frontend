**运算符就像"工具箱里的各种工具"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                    工具箱                             │
├──────────────────────────────────────────────────────┤
│                                                      │
│    算术运算符 = 计算器                                │
│    ─────────────                                     │
│    加减乘除，算数用的                                 │
│                                                      │
│    比较运算符 = 天平                                  │
│    ─────────────                                     │
│    比较两边谁大谁小、是否相等                         │
│    === 是精密天平（连类型一起比）                    │
│    ==  是粗糙天平（会自动转换再比）                  │
│                                                      │
│    逻辑运算符 = 电路开关                              │
│    ─────────────                                     │
│    && = 串联开关（两个都开才通电）                   │
│    || = 并联开关（一个开就通电）                     │
│    !  = 反向开关（开的变关，关的变开）               │
│                                                      │
│    三元运算符 = 岔路口                                │
│    ─────────────                                     │
│    条件成立走左边，不成立走右边                       │
│                                                      │
│    可选链 ?. = 安全探测器                            │
│    ─────────────────                                 │
│    前面有路才继续走，没路就停下（返回 undefined）     │
│                                                      │
│    空值合并 ?? = 备用方案                            │
│    ─────────────                                     │
│    有值就用原值，没值（null/undefined）才用备用的     │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

### 算术与比较

**语法结构图：**

```
算术运算符：
+  -  *  /  %（取余）  **（幂）  ++  --

比较运算符：
===  !==  → 严格比较（推荐）
==   !=   → 宽松比较（不推荐，会隐式转换）
>   <   >=   <=

推荐：永远用 === 而不是 ==
```

**最简示例（1-3行）：**

```javascript
10 + 3;       // 13
10 % 3;       // 1（余数）
5 === '5';    // false（类型不同）
```

**详细示例：**

```javascript
// 算术
const a = 10, b = 3;
a + b;    // 13
a / b;    // 3.333...
a % b;    // 1（判断奇偶：n % 2 === 0 → 偶数）
a ** b;   // 1000（10 的 3 次方）

// 比较 — 严格 vs 宽松
5 === 5;       // true
5 === '5';     // false（类型不同）
5 == '5';      // true（隐式转换，不推荐）
null == undefined;  // true
null === undefined; // false

// 字符串比较（按 Unicode）
'a' < 'b';     // true
'A' < 'a';     // true（大写编码更小）
```

### 逻辑运算符

**最简示例：**

```javascript
true && true;    // true
true || false;   // true
!true;           // false
!!'hello';       // true（双重否定 = 转布尔）
```

**详细示例：**

```javascript
// &&（与）— 两个都真才真
const canDelete = isLoggedIn && isAdmin;

// ||（或）— 一个真就真
const canView = role === 'admin' || role === 'user';

// !（非）— 取反
!true;           // false
!!0;             // false（转布尔值技巧）

// 短路求值
0 && 'anything';     // 0（左边为假，直接返回左边）
'hello' || 'world';  // 'hello'（左边为真，直接返回左边）

// 设置默认值（老方法）
const name = userInput || '默认用户';
```

### 三元运算符、?? 和 ?.

**语法结构图：**

```
三元运算符：
条件 ? 真值 : 假值

空值合并 ??：
值 ?? 默认值    ← 只有 null/undefined 才用默认值

可选链 ?.：
obj?.prop?.nested   ← 任意一环为 null/undefined 就返回 undefined
```

**最简示例：**

```javascript
score >= 60 ? '及格' : '不及格';
user?.name ?? '匿名用户';
```

**详细示例：**

```javascript
// 三元运算符
const grade = score >= 90 ? 'A'
               : score >= 80 ? 'B'
               : score >= 60 ? 'C'
               : 'D';

// 空值合并 ?? — 比 || 更精确
'' || 'default';       // 'default'（空字符串被当作假值）
'' ?? 'default';       // ''（空字符串保留）
0 || 10;               // 10
0 ?? 10;               // 0

// 可选链 ?.
const city = user?.address?.city ?? '未知';
const first = items?.[0];
const result = obj?.method?.();
```

## L2 实践层：用好

### 运算符速查

| 运算符 | 用途 | 示例 |
|------|------|------|
| `===` | 严格相等 | `5 === 5` → true |
| `??` | 空值合并 | `null ?? 'default'` → 'default' |
| `?.` | 可选链 | `user?.name` |
| `&&` | 逻辑与 | `true && false` → false |
| `\|\|` | 逻辑或 | `false \|\| true` → true |
| `??` vs `\|\|` | `??` 只检查 null/undefined | `0 ?? 10` → 0 |

### 反模式：不要这样做

```javascript
// ❌ 错误：用 == 比较（隐式转换陷阱）
'' == 0;        // true
'0' == 0;       // true
[] == 0;        // true
[] == '';       // true
[] == ![];      // true（？？？）

// ✅ 正确：总是用 ===
'' === 0;       // false
'0' === 0;      // false
```

```javascript
// ❌ 错误：三元运算符嵌套过深
const result = a ? (b ? (c ? 'x' : 'y') : 'z') : 'w';

// ✅ 正确：用 if-else
let result;
if (a && b && c) result = 'x';
else if (a && b) result = 'y';
else if (a) result = 'z';
else result = 'w';
```

```javascript
// ❌ 错误：依赖运算符优先级
let result = a + b * c > d && e || f;

// ✅ 正确：用括号明确意图
let result = ((a + (b * c)) > d && e) || f;
```

## L3 专家层：深入

### 运算符优先级（从高到低）

```
1.  () [] ?.           → 括号、属性访问
2.  ++ -- ! typeof     → 一元运算符
3.  * / %              → 乘除取余
4.  + -                → 加减
5.  < <= > >=          → 比较
6.  === !== == !=      → 相等
7.  &&                 → 逻辑与
8.  ||                 → 逻辑或
9.  ??                 → 空值合并
10. ?:                 → 三元
11. = += -=            → 赋值
```

> **建议：不要依赖优先级记忆，多用括号。**

### 性能考量

| 操作 | 性能影响 | 说明 |
|------|---------|------|
| 基本运算 | 极低 | 引擎原生支持 |
| `??` vs `||` | 无差异 | 现代引擎都优化了 |
| `?.` vs `&&` | 无差异 | 可选链是语法糖 |
| 短路求值 | 极低 | 不会计算右侧表达式 |

### 设计动机

**为什么引入 ?? 和 ?.？**

| 设计选择 | 原因 | 对比之前 |
|----------|------|----------|
| `??` | `||` 会误判 0、'' 等有效值 | 只有 null/undefined 才触发默认值 |
| `?.` | 避免 `a && a.b && a.b.c` 冗长 | 简洁的安全访问 |
| `===` | `==` 的隐式转换规则复杂且反直觉 | 类型不同直接返回 false |

### 知识关联

```
运算符关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  算术运算   │────→│  比较运算   │────→│  逻辑运算   │
│  + - * / %  │     │  === !== >  │     │  && || !    │
└─────────────┘     └─────────────┘     └─────────────┘
                                              │
                                   ┌──────────┼──────────┐
                                   ↓          ↓          ↓
                            ┌──────────┐ ┌──────────┐ ┌──────────┐
                            │  三元    │ │  空值    │ │  可选    │
                            │  ?:      │ │  合并 ?? │ │  链 ?.   │
                            └──────────┘ └──────────┘ └──────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **严格相等** | 值和类型都相同才相等 | `5 === 5` → true |
| **隐式转换** | JavaScript 自动转换类型再比较 | `5 == '5'` → true |
| **短路求值** | 左边的结果已决定整体结果时，不计算右边 | `false && x` 不计算 x |
| **三元运算符** | 条件表达式的简写 | `cond ? a : b` |
| **空值合并** | 只有 null/undefined 时用默认值 | `value ?? 'default'` |
| **可选链** | 安全访问可能不存在的属性 | `obj?.prop` |
| **取余** | 除法后的余数 | `10 % 3` → 1 |
| **幂运算** | 求一个数的 N 次方 | `2 ** 3` → 8 |

## 实践练习

### 练习：运算符应用

```javascript
// 练习 1：判断奇偶
const isEven = n => n % 2 === 0;
console.log(isEven(7));   // false
console.log(isEven(8));   // true

// 练习 2：安全访问 + 默认值
const config = { theme: 'dark' };
const timeout = config.timeout ?? 5000;
const theme = config.theme ?? 'light';
console.log(timeout);  // 5000
console.log(theme);    // 'dark'

// 练习 3：闰年判断
const isLeap = year =>
    (year % 4 === 0 && year % 100 !== 0) || (year % 400 === 0);
console.log(isLeap(2024));  // true
console.log(isLeap(1900));  // false
```

## 常见问题

### Q1：== 和 === 应该用哪个？

**推荐总是用 ===**，除非你明确需要类型转换：

```javascript
// 推荐
if (count === 0) { }
if (name === null) { }

// 特殊情况：同时检查 null 和 undefined
if (value == null) { }  // value 为 null 或 undefined 时都成立
```

### Q2：|| 和 ?? 有什么区别？

```javascript
// || 把所有假值（0, '', false）都当空值
0 || 10;        // 10
'' || 'default'; // 'default'

// ?? 只把 null/undefined 当空值
0 ?? 10;        // 0
'' ?? 'default'; // ''
```

### Q3：如何记住运算符优先级？

**不要记，多用括号：**

```javascript
// 不推荐 — 依赖优先级
a + b * c > d && e || f;

// 推荐 — 明确意图
((a + (b * c)) > d && e) || f;
```
