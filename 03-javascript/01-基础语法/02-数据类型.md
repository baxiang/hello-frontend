# JavaScript 数据类型 ⭐

> 7 种基本类型、引用类型、类型检测与转换

---

## 学习目标

- 掌握 7 种基本数据类型（string、number、boolean、null、undefined、symbol、bigint）
- 理解基本类型与引用类型的区别
- 学会类型检测方法（typeof、Array.isArray、Object.prototype.toString）
- 掌握类型转换规则

## 生活化比喻

**数据类型就像"不同种类的容器"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                  容器与货物                           │
├──────────────────────────────────────────────────────┤
│                                                      │
│    基本类型 = 小件物品                                │
│    ─────────────                                     │
│    string（文字条）→ "Hello"                         │
│    number（数字卡）→ 42, 3.14                       │
│    boolean（开关）→ 开/关                             │
│    null（空箱子）→ 明确说"这里没东西"                │
│    undefined（没箱子）→ 连箱子都没有                │
│                                                      │
│    引用类型 = 大仓库                                  │
│    ─────────────                                     │
│    object（储物柜）→ { name: '张三', age: 25 }       │
│    array（一排柜子）→ [1, 2, 3]                      │
│    function（加工机器）→ (x) => x * 2               │
│                                                      │
│    区别：                                             │
│    基本类型 → 你直接拿着物品                          │
│    引用类型 → 你拿着仓库的钥匙（地址）               │
│    两个人拿同一把钥匙，改的是同一个仓库               │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

### 7 种基本类型

**语法结构图：**

```
JavaScript 基本类型：

string      → 'Hello', "World", `模板`
number      → 42, 3.14, NaN, Infinity
boolean     → true, false
null        → null（有意的空）
undefined   → undefined（未定义）
symbol      → Symbol('id')（唯一标识）
bigint      → 9007199254740991n（大整数）

引用类型：
object      → { key: value }
array       → [1, 2, 3]
function    → function() {}
```

**最简示例（1-3行）：**

```javascript
typeof 'hello';      // 'string'
typeof 42;           // 'number'
typeof true;         // 'boolean'
```

**详细示例：**

```javascript
// string — 文本
const name = '张三';
const greeting = `你好，${name}！`;

// number — 整数和小数
const age = 25;
const price = 9.99;
const bigNum = 1_000_000;  // 数字分隔符（ES2021）

// boolean — 真假
const isLoggedIn = true;
const hasPermission = false;

// null — 有意的空
let user = null;  // 将来会赋值

// undefined — 未定义
let value;  // 声明但没赋值 → undefined

// symbol — 唯一标识
const id = Symbol('id');
const id2 = Symbol('id');
id === id2;  // false（即使描述相同）

// bigint — 超大整数
const big = 9007199254740991n;
```

### 引用类型

**最简示例：**

```javascript
const user = { name: '张三', age: 25 };    // 对象
const nums = [1, 2, 3];                      // 数组
const greet = () => console.log('Hi');       // 函数
```

**详细示例 — 基本 vs 引用：**

```javascript
// 基本类型：值传递（独立副本）
let a = 5;
let b = a;
b = 10;
console.log(a);  // 5（不受影响）

// 引用类型：引用传递（共享同一对象）
let obj1 = { name: '张三' };
let obj2 = obj1;
obj2.name = '李四';
console.log(obj1.name);  // '李四'（被影响了）
```

### 类型检测

**语法结构图：**

```
类型检测方法：

typeof 值              → 返回类型字符串（null 有 bug）
Array.isArray(值)      → 准确判断数组
Object.prototype.toString.call(值) → 最准确

typeof 'hello'     → 'string'
typeof 42          → 'number'
typeof true        → 'boolean'
typeof null        → 'object'（bug！）
typeof undefined   → 'undefined'
typeof []          → 'object'（不准确）
typeof {}          → 'object'
typeof function()  → 'function'
typeof Symbol()    → 'symbol'
typeof 123n        → 'bigint'
```

**最简示例：**

```javascript
typeof 'hello';           // 'string'
Array.isArray([1,2,3]);   // true
Array.isArray({});        // false
```

### 类型转换

**最简示例：**

```javascript
Number('123');      // 123
String(123);        // '123'
Boolean(0);         // false
```

**详细示例：**

```javascript
// 转数字
Number('123');        // 123
Number('123px');      // NaN
parseInt('123px');    // 123
+'123';               // 123（一元加号）

// 转字符串
String(123);          // '123'
`${123}`;             // '123'
(123).toString();     // '123'

// 转布尔
Boolean('hello');     // true
Boolean(0);           // false
Boolean('');          // false
Boolean(null);        // false
Boolean(undefined);   // false
Boolean([]);          // true（空数组也是真值）
Boolean({});          // true（空对象也是真值）
```

## L2 实践层：用好

### 真假值速查

| 假值（6 个） | 说明 |
|:---:|------|
| `false` | 布尔假 |
| `0` / `-0` | 数字零 |
| `0n` | BigInt 零 |
| `''` | 空字符串 |
| `null` | 空值 |
| `undefined` | 未定义 |
| `NaN` | 不是数字 |

> 除了以上 7 个假值，其他**所有值都是真值**（包括 `[]`、`{}`、`'0'`）

### 反模式：不要这样做

```javascript
// ❌ 错误：用 == 比较（会隐式转换）
5 == '5';       // true
0 == false;     // true
'' == false;    // true

// ✅ 正确：用 === 严格比较
5 === '5';      // false
0 === false;    // false
```

```javascript
// ❌ 错误：用 typeof 检测数组
typeof [];      // 'object'（不准确）

// ✅ 正确：用 Array.isArray
Array.isArray([]);  // true
```

```javascript
// ❌ 错误：用 typeof 检测 null
typeof null;    // 'object'（历史 bug）

// ✅ 正确：直接比较
value === null;
```

### 适用场景

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| 检测字符串 | `typeof x === 'string'` | 准确 |
| 检测数字 | `typeof x === 'number' && !Number.isNaN(x)` | 排除 NaN |
| 检测数组 | `Array.isArray(x)` | 官方推荐 |
| 检测 null | `x === null` | 最准确 |
| 检测 undefined | `x === undefined` 或 `typeof x === 'undefined'` | 安全 |
| 通用检测 | `Object.prototype.toString.call(x)` | 返回 `'[object Type]'` |

## L3 专家层：深入

### 基本类型 vs 引用类型底层原理

```
内存模型：

基本类型（栈内存）:
  let a = 5;
  ┌─────────┐
  │  a: 5   │  ← 值直接存在变量中
  └─────────┘

引用类型（堆内存 + 栈引用）:
  let obj = { name: '张三' };
  栈内存             堆内存
  ┌─────────┐       ┌─────────────────┐
  │ obj: →  │ ────→ │ { name: '张三' } │
  │ 0x001   │       └─────────────────┘
  └─────────┘
  变量存的是地址，不是值本身
```

### 性能考量

| 操作 | 性能影响 | 说明 |
|------|---------|------|
| 基本类型赋值 | 极低 | 直接复制值 |
| 引用类型赋值 | 极低 | 只复制地址 |
| 深拷贝大对象 | 高 | 需要递归复制所有层级 |
| typeof | 极低 | 引擎原生操作 |
| Object.prototype.toString | 低 | 需要调用方法 |

### 设计动机

**为什么 JavaScript 有这些类型？**

| 设计选择 | 原因 | 对比其他语言 |
|----------|------|-------------|
| 动态类型 | 灵活，学习门槛低 | TypeScript 提供静态类型检查 |
| null 和 undefined 共存 | null 表示"有意的空"，undefined 表示"未赋值" | 很多语言只有一个 null |
| typeof null === 'object' | 历史 bug，为兼容性保留 | 其他语言不会有此问题 |
| 真值/假值 | 方便条件判断 | Python 也有类似概念 |

### 知识关联

```
数据类型关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  基本类型   │────→│  引用类型   │────→│  类型检测   │
│  7 种       │     │  object/    │     │  typeof/    │
│  值在栈中   │     │  array/     │     │  isArray    │
│             │     │  function   │     └─────────────┘
└─────────────┘     └─────────────┘          │
                           │                 ↓
                    ┌─────────────┐   ┌─────────────┐
                    │  类型转换   │   │  深浅拷贝   │
                    │  Number/    │   │  .../       │
                    │  String/    │   │  structured │
                    │  Boolean    │   │  Clone      │
                    └─────────────┘   └─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **基本类型** | 值直接存储在变量中的类型，共 7 种 | `string`, `number`, `boolean` |
| **引用类型** | 变量存储的是内存地址的类型 | `object`, `array`, `function` |
| **假值（falsy）** | 在布尔上下文中被视为 false 的值 | `0`, `''`, `null`, `undefined`, `NaN` |
| **真值（truthy）** | 除了假值以外的所有值 | `'0'`, `[]`, `{}` |
| **NaN** | Not a Number，表示不是一个有效数字 | `parseInt('abc')` → NaN |
| **BigInt** | 表示超出 Number 安全范围的整数 | `9007199254740991n` |
| **Symbol** | 创建唯一标识符 | `Symbol('id') !== Symbol('id')` |
| **模板字符串** | 用反引号包裹的字符串，支持插值 | `` `Hello ${name}` `` |
| **浅拷贝** | 只复制一层，嵌套对象共享引用 | `{ ...obj }` |
| **深拷贝** | 递归复制所有层级 | `structuredClone(obj)` |

## 实践练习

### 练习：类型判断 + 数据处理

```javascript
// 练习 1：类型判断函数
function getType(value) {
    if (value === null) return 'null';
    if (Array.isArray(value)) return 'array';
    return typeof value;
}

console.log(getType(null));       // 'null'
console.log(getType([1, 2]));     // 'array'
console.log(getType('hello'));    // 'string'
console.log(getType(42));         // 'number'

// 练习 2：安全的类型转换
function safeToNumber(value) {
    const num = Number(value);
    return Number.isNaN(num) ? 0 : num;
}

console.log(safeToNumber('123'));   // 123
console.log(safeToNumber('abc'));   // 0
console.log(safeToNumber(''));      // 0

// 练习 3：真假值判断
function isTruthy(value) {
    return Boolean(value);
}

console.log(isTruthy('0'));     // true
console.log(isTruthy(0));       // false
console.log(isTruthy([]));      // true
console.log(isTruthy({}));      // true
```

## 常见问题

### Q1：为什么 typeof null 返回 'object'？

这是 JavaScript 早期的一个 bug，为了兼容性一直保留。正确检测 null 的方法：

```javascript
value === null;  // ✅
```

### Q2：0、'0'、[]、{} 的真假值是什么？

```javascript
Boolean(0);     // false（数字零是假值）
Boolean('0');   // true（非空字符串是真值）
Boolean([]);    // true（空数组是真值）
Boolean({});    // true（空对象是真值）
```

### Q3：基本类型和引用类型有什么本质区别？

```javascript
// 基本类型：值传递，互不影响
let a = 5, b = a;
b = 10;
console.log(a);  // 5

// 引用类型：引用传递，共享同一对象
let obj1 = { x: 1 }, obj2 = obj1;
obj2.x = 2;
console.log(obj1.x);  // 2
```
