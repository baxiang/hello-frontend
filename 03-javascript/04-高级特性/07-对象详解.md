## 生活化比喻

**对象就像"一张身份证"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                    身份证                             │
├──────────────────────────────────────────────────────┤
│                                                      │
│    属性 = 身份证上的信息                              │
│    ─────────────                                     │
│    姓名、号码、地址 → 每个字段都是一个属性            │
│    obj.name = "张三" → 姓名：张三                     │
│                                                      │
│    方法 = 身份证能做的事                              │
│    ─────────────                                     │
│    验证身份、读取信息 → 对象上的函数                  │
│    obj.verify() → 验证身份                            │
│                                                      │
│    this = "这张"身份证                                │
│    ─────────────                                     │
│    this.name → 这张身份证的名字                       │
│    谁调用 this 就指向谁                               │
│                                                      │
│    原型 = 身份证的模板                                │
│    ─────────────                                     │
│    所有身份证共享同一个模板格式                       │
│    每个实例都有模板的方法，但不重复存储               │
│                                                      │
│    原型链 = 模板的继承关系                            │
│    ─────────────────                                 │
│    身份证模板 ← 证件模板 ← 通用模板                   │
│    找不到就往上找，直到 null                          │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

### 创建对象

**语法结构图：**

```
创建对象的 4 种方式：

方式 1：字面量（最常用）
let obj = { key: value, method() {} };

方式 2：构造函数
function Person(name) { this.name = name; }
let p = new Person('张三');

方式 3：Object.create
let obj = Object.create(proto);

方式 4：class（ES6）
class Person { constructor(name) { this.name = name; } }
let p = new Person('张三');
```

**最简示例（1-3行）：**

```javascript
let user = { name: '张三', age: 25 };
let empty = {};
```

**详细示例：**

```javascript
// 字面量（推荐）
const user = {
    name: '张三',
    age: 25,
    greet() { console.log(`Hi, I'm ${this.name}`); }
};

// class 语法
class User {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
    greet() { console.log(`Hi, I'm ${this.name}`); }
}
const u = new User('李四', 30);
u.greet();  // 'Hi, I'm 李四'
```

### 访问和修改属性

**最简示例：**

```javascript
let obj = { name: '张三' };
obj.name;        // '张三'（点表示法）
obj['name'];     // '张三'（方括号表示法）
obj.age = 25;    // 添加属性
delete obj.age;  // 删除属性
```

**详细示例：**

```javascript
const user = { name: '张三', age: 25 };

// 点表示法 — 属性名固定时
console.log(user.name);  // '张三'

// 方括号表示法 — 属性名是变量或含特殊字符
const key = 'name';
console.log(user[key]);  // '张三'

const obj = { 'first-name': '李四' };
console.log(obj['first-name']);  // '李四'

// 检查属性
'name' in user;                // true
user.hasOwnProperty('name');   // true
Object.hasOwn(user, 'name');   // true（现代推荐）

// 遍历对象
for (const key of Object.keys(user)) console.log(key, user[key]);
for (const [key, value] of Object.entries(user)) console.log(key, value);
```

### this 绑定规则

**语法结构图：**

```
this 指向规则（按优先级）：

1. new 绑定      → new Fn() 中 this 指向新对象
2. 显式绑定      → call/apply/bind 指定的对象
3. 隐式绑定      → obj.fn() 中 this 指向 obj
4. 默认绑定      → fn() 中 this 指向全局（严格模式为 undefined）

箭头函数：没有自己的 this，继承定义时的外层 this
```

**最简示例：**

```javascript
const obj = { name: '张三', greet() { console.log(this.name); } };
obj.greet();  // '张三'（this → obj）

const fn = obj.greet;
fn();  // undefined（this → 全局）
```

**详细示例：**

```javascript
const person = {
    name: '张三',
    // ✅ 普通方法：this 指向调用者
    greet() { console.log(this.name); },

    // ❌ 箭头函数方法：this 指向定义时上下文（不是 person）
    arrowGreet: () => { console.log(this.name); }
};

person.greet();       // '张三'
person.arrowGreet();  // undefined

// 解决 this 丢失：bind
const greet = person.greet;
greet();                    // undefined
const boundGreet = greet.bind(person);
boundGreet();               // '张三'
```

## L2 实践层：用好

### 对象操作 API

| API | 用途 | 示例 |
|------|------|------|
| `Object.keys()` | 获取所有键 | `['name', 'age']` |
| `Object.values()` | 获取所有值 | `['张三', 25]` |
| `Object.entries()` | 获取键值对 | `[['name', '张三'], ...]` |
| `{ ...obj1, ...obj2 }` | 合并对象 | 后覆盖前 |
| `structuredClone()` | 深拷贝 | 现代浏览器推荐 |
| `Object.freeze()` | 冻结对象 | 不可修改 |

**对象拷贝 — 浅 vs 深：**

```javascript
const user = { name: '张三', address: { city: '北京' } };

// 浅拷贝：只复制一层，嵌套对象仍共享引用
const shallow = { ...user };
shallow.address.city = '上海';
console.log(user.address.city);  // '上海'（原对象被改）

// 深拷贝：完全独立
const deep = structuredClone(user);
deep.address.city = '上海';
console.log(user.address.city);  // '北京'（原对象不受影响）

// JSON 深拷贝（有局限：丢失函数、undefined、Date 等）
const deepJSON = JSON.parse(JSON.stringify(user));
```

### 反模式：不要这样做

```javascript
// ❌ 错误：对象方法用箭头函数（this 不对）
const obj = {
    name: '张三',
    greet: () => console.log(this.name)  // undefined
};

// ✅ 正确：用普通方法
const obj = {
    name: '张三',
    greet() { console.log(this.name); }  // '张三'
};
```

```javascript
// ❌ 错误：直接修改函数参数中的对象
function updateUser(user) {
    user.name = '新名字';  // 外部对象被意外修改
}

// ✅ 正确：返回新对象
function updateUser(user, newName) {
    return { ...user, name: newName };
}
```

```javascript
// ❌ 错误：在原型上直接赋值实例属性
function Person(name) {
    Person.prototype.name = name;  // 所有实例共享
}

// ✅ 正确：用 this 赋值
function Person(name) {
    this.name = name;  // 每个实例独立
}
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 简单数据容器 | 字面量 `{}` | 最简洁 |
| 创建多个同类实例 | class / 构造函数 | 代码复用 |
| 基于已有对象扩展 | `Object.create(proto)` | 原型继承 |
| 不可变数据更新 | `{ ...obj, key: newVal }` | 不修改原对象 |
| 配置合并 | `{ ...defaults, ...userPrefs }` | 用户配置覆盖默认 |

## L3 专家层：深入

### 原型链原理

```
原型链查找流程：

const dog = new Dog();
dog.speak();

查找路径：
dog 实例 ──[[Prototype]]──→ Dog.prototype
                              │
                              ──[[Prototype]]──→ Animal.prototype
                                                    │
                                                    ──[[Prototype]]──→ Object.prototype
                                                                          │
                                                                          ──[[Prototype]]──→ null

dog.speak() 查找：
1. dog 自身有 speak 吗？  → 有，调用
2. 没有 → 找 Dog.prototype
3. 没有 → 找 Animal.prototype
4. 没有 → 找 Object.prototype
5. 没有 → 到 null，抛出 TypeError
```

```javascript
// 验证原型链
function Animal(name) { this.name = name; }
Animal.prototype.speak = function() { console.log(`${this.name} speaks`); };

function Dog(name) { Animal.call(this, name); }
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog;
Dog.prototype.bark = function() { console.log(`${this.name} barks`); };

const dog = new Dog('旺财');
dog.bark();   // '旺财 barks'（Dog.prototype）
dog.speak();  // '旺财 speaks'（Animal.prototype）
```

### 属性描述符

```javascript
const obj = { name: '张三' };

// 查看属性描述符
Object.getOwnPropertyDescriptor(obj, 'name');
// { value: '张三', writable: true, enumerable: true, configurable: true }

// writable: false → 不可修改
// enumerable: false → for...in / Object.keys 不可见
// configurable: false → 不可删除，描述符不可修改

// getter / setter
const person = {
    _age: 0,
    get age() { return this._age; },
    set age(val) {
        if (val < 0) throw new Error('年龄不能为负');
        this._age = val;
    }
};
person.age = 25;
console.log(person.age);  // 25
```

### 性能考量

| 操作 | 性能影响 | 说明 |
|------|---------|------|
| 字面量创建 | 极低 | 引擎高度优化 |
| 原型方法调用 | 低 | 查找原型链有微小开销 |
| `Object.keys/values/entries` | 低 | 遍历所有属性 |
| 深拷贝大对象 | 高 | 递归遍历所有层级 |
| `delete` 属性 | 中等 | 可能破坏引擎优化 |

### 设计动机

**为什么 JavaScript 用原型而非类？**

| 设计选择 | 原因 | 替代方案对比 |
|----------|------|-------------|
| 原型继承 | 更灵活，运行时可修改 | Java 类继承在编译时确定 |
| `class` 语法糖 | 降低学习门槛 | 底层仍是原型 |
| 动态属性 | 对象可以随时添加属性 | 静态类型语言需要预定义 |
| `this` 动态绑定 | 方法可被不同对象复用 | Python 显式 self 更明确 |

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **对象** | 键值对的集合，用于表示现实事物 | `{ name: '张三', age: 25 }` |
| **属性** | 对象中的键值对 | `name: '张三'` |
| **方法** | 对象中的函数属性 | `greet() { ... }` |
| **this** | 指向方法调用者的关键字 | `obj.fn()` 中 this → obj |
| **原型** | 对象共享方法和属性的模板 | `Person.prototype` |
| **原型链** | 对象通过 [[Prototype]] 链接形成的查找链 | obj → Object.prototype → null |
| **构造函数** | 用 new 调用创建对象的函数 | `function Person() {}` |
| **浅拷贝** | 只复制对象第一层，嵌套对象共享引用 | `{ ...obj }` |
| **深拷贝** | 递归复制所有层级，完全独立 | `structuredClone(obj)` |
| **属性描述符** | 控制属性行为的元数据 | writable、enumerable、configurable |
| **getter/setter** | 访问/赋值属性时自动调用的方法 | `get age() { }` |

## 实践练习

### 练习：不可变对象工具 + 模块模式

```javascript
// 练习 1：不可变对象操作工具
const Immutable = {
    set(obj, key, value) {
        return { ...obj, [key]: value };
    },

    remove(obj, key) {
        const copy = { ...obj };
        delete copy[key];
        return copy;
    },

    merge(...objects) {
        return Object.assign({}, ...objects);
    },

    pick(obj, keys) {
        return keys.reduce((acc, key) => {
            if (key in obj) acc[key] = obj[key];
            return acc;
        }, {});
    }
};

const user = { name: '张三', age: 25, email: 'zhang@example.com' };
const updated = Immutable.set(user, 'age', 26);
const partial = Immutable.pick(user, ['name', 'email']);
console.log(updated);   // { name: '张三', age: 26, email: '...' }
console.log(partial);   // { name: '张三', email: '...' }

// 练习 2：用 class 实现栈
class Stack {
    #items = [];  // ES2022 私有字段

    push(item) { this.#items.push(item); }
    pop() { return this.#items.pop(); }
    peek() { return this.#items[this.#items.length - 1]; }
    get size() { return this.#items.length; }
    isEmpty() { return this.#items.length === 0; }
}

const stack = new Stack();
stack.push(1); stack.push(2); stack.push(3);
console.log(stack.pop());    // 3
console.log(stack.peek());   // 2
console.log(stack.size);     // 2
```

## 常见问题

### Q1：点表示法和方括号表示法有什么区别？

**点表示法**要求属性名是合法标识符且固定；**方括号**支持变量和特殊字符：

```javascript
const obj = { name: '张三' };
obj.name;           // ✅ 点表示法
obj['name'];        // ✅ 方括号表示法

const key = 'name';
obj[key];           // ✅ 只有方括号支持变量

const obj2 = { 'first-name': '李四' };
obj2['first-name']; // ✅ 只有方括号支持特殊字符
```

### Q2：箭头函数和普通方法的 this 有什么不同？

```javascript
const obj = {
    name: '张三',
    fn() { console.log(this.name); },       // '张三'（this → obj）
    arrow: () => console.log(this.name)     // undefined（this → 外层）
};
```
**规则：对象方法永远不要用箭头函数。**

### Q3：浅拷贝和深拷贝的区别？

```javascript
const obj = { a: 1, b: { c: 2 } };
const shallow = { ...obj };
shallow.b.c = 3;
console.log(obj.b.c);  // 3（嵌套对象被改了）

const deep = structuredClone(obj);
deep.b.c = 4;
console.log(obj.b.c);  // 3（原对象不受影响）
```
