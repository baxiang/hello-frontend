## L1 理解层：会用

### 错误类型

**语法结构图：**

```
JavaScript 错误类型：

Error           ← 基础错误
├── SyntaxError ← 语法错误（括号不匹配、缺少引号）
├── ReferenceError ← 引用错误（变量未定义）
├── TypeError   ← 类型错误（调用 null 的方法）
├── RangeError  ← 范围错误（数组负数长度）
├── URIError    ← URI 错误（无效编码）
└── EvalError   ← eval 错误（现代 JS 很少见）

错误对象属性：
error.name     → 错误类型名（如 'TypeError'）
error.message  → 错误描述
error.stack    → 堆栈跟踪（调试用）
```

**最简示例（1-3行）：**

```javascript
throw new Error('出错了');
try { risky(); } catch (e) { console.error(e.message); }
```

**详细示例：**

```javascript
// 常见错误场景
// ReferenceError：变量未定义
console.log(notDefined);  // ❌ ReferenceError: notDefined is not defined

// TypeError：调用不存在的方法
null.toString();          // ❌ TypeError: Cannot read properties of null

// SyntaxError：语法错误（代码根本不会运行）
// eval('if (true { }');  // ❌ SyntaxError

// RangeError：超出范围
new Array(-1);            // ❌ RangeError: Invalid array length
```

### try-catch-finally

**语法结构图：**

```
try-catch-finally 流程：

try {
    // 可能出错的代码
} catch (error) {        ← 只在出错时执行
    // 处理错误
} finally {              ← 无论是否出错都执行
    // 清理工作
}

执行路径：
正常：try → finally
出错：try → catch → finally
```

**最简示例：**

```javascript
try { JSON.parse('invalid'); } catch (e) { console.error(e.message); }
try { riskyOp(); } finally { cleanup(); }
```

**详细示例：**

```javascript
// 完整结构
async function fetchData(url) {
    try {
        const response = await fetch(url);
        if (!response.ok) throw new Error(`HTTP ${response.status}`);
        return await response.json();
    } catch (error) {
        console.error('请求失败:', error.message);
        throw error;  // 重新抛出，让调用者处理
    }
}

// 安全的 JSON 解析
function safeJSON(str, defaultValue = null) {
    try {
        return JSON.parse(str);
    } catch (e) {
        return defaultValue;
    }
}
safeJSON('{"a": 1}');  // { a: 1 }
safeJSON('invalid');    // null
```

### 抛出错误

**最简示例：**

```javascript
throw new Error('出错了');
throw new TypeError('类型错误');
```

**详细示例 — 自定义错误：**

```javascript
// 自定义错误类
class ValidationError extends Error {
    constructor(message) {
        super(message);
        this.name = 'ValidationError';
    }
}

function validateAge(age) {
    if (age < 0 || age > 150) {
        throw new ValidationError('年龄必须在 0-150 之间');
    }
}

try {
    validateAge(-1);
} catch (error) {
    if (error instanceof ValidationError) {
        console.error('验证失败:', error.message);
    }
}
```

## L2 实践层：用好

### 错误处理策略

| 策略 | 适用场景 | 示例 |
|------|---------|------|
| 捕获并返回默认值 | 非关键操作 | JSON 解析失败返回 null |
| 捕获并重新抛出 | 需要记录日志但不处理 | API 请求中间层 |
| 包装错误 | 添加上下文信息 | `throw new Error('获取用户失败: ' + e.message)` |
| 条件处理 | 不同错误不同处理 | `instanceof` 判断错误类型 |
| 吞掉错误（慎用） | 确定可以忽略 | localStorage 不可用时 |

**反模式：不要这样做**

```javascript
// ❌ 错误：空 catch 吞掉所有错误
try { doSomething(); } catch (e) {}
// 问题：错误被完全忽略，难以调试

// ✅ 正确：至少记录日志
try { doSomething(); } catch (e) {
    console.warn('操作失败:', e.message);
}
```

```javascript
// ❌ 错误：用 try-catch 替代条件判断
try {
    let result = user.profile.name;
} catch (e) {}

// ✅ 正确：用可选链
let name = user?.profile?.name ?? '匿名用户';
```

```javascript
// ❌ 错误：在循环中滥用 try-catch
for (let i = 0; i < 1000; i++) {
    try { process(arr[i]); } catch (e) {}
}

// ✅ 正确：在循环外包裹
try {
    for (let i = 0; i < 1000; i++) { process(arr[i]); }
} catch (e) { console.error(e); }
```

### 常见错误场景与预防

| 场景 | 错误类型 | 预防方案 |
|------|---------|---------|
| 空值访问 | TypeError | 可选链 `?.` |
| JSON 解析 | SyntaxError | try-catch 包裹 |
| 异步请求未 catch | 未处理 Promise | `.catch()` 或 try-catch |
| 数组越界 | undefined | 检查索引或 `?.[]` |
| 类型转换 | NaN | `Number.isNaN()` 检查 |

## L3 专家层：深入

### 错误处理底层原理

```
try-catch 执行模型：

1. 进入 try 块 → 建立错误处理上下文
2. 执行 try 中的代码
3. 如果抛出错误：
   a. 暂停当前执行
   b. 查找最近的 catch 块
   c. 将错误对象传给 catch
   d. 执行 catch 中的代码
4. 如果存在 finally，无论有无错误都执行
5. finally 中的 return 会覆盖 try/catch 的 return
```

```javascript
// finally 的 return 会覆盖 try 的 return
function test() {
    try {
        return 'from try';
    } finally {
        return 'from finally';  // 最终返回这个
    }
}
test();  // 'from finally'
```

### 性能考量

| 操作 | 性能影响 | 说明 |
|------|---------|------|
| try-catch 声明 | 极低 | 现代引擎已优化 |
| 实际抛出错误 | 高 | 创建 Error 对象 + 堆栈收集 |
| 循环中 try-catch | 中等 | 不推荐，但不影响不抛错的场景 |
| async/await + try-catch | 低 | 与普通 try-catch 相同 |

### 设计动机

**为什么 JavaScript 用 try-catch？**

| 设计选择 | 原因 | 替代方案对比 |
|----------|------|-------------|
| try-catch | 集中处理异常，代码清晰 | C 风格错误码需要处处检查返回值 |
| Error 对象 | 携带 name、message、stack | 纯字符串丢失堆栈信息 |
| finally | 确保资源释放 | 容易遗漏清理代码 |
| instanceof 判断 | 区分错误类型 | 错误码难以扩展 |

### 全局错误捕获

```javascript
// 浏览器：未捕获的同步错误
window.addEventListener('error', (event) => {
    console.error('全局错误:', event.error);
});

// 浏览器：未捕获的 Promise 错误
window.addEventListener('unhandledrejection', (event) => {
    console.error('未处理的 Promise:', event.reason);
});

// Node.js
process.on('uncaughtException', (error) => {
    console.error('未捕获的异常:', error);
});
process.on('unhandledRejection', (reason) => {
    console.error('未处理的 Promise:', reason);
});
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **异常** | 程序运行时出现的非预期情况 | `null.toString()` 抛出 TypeError |
| **try-catch** | 尝试执行代码，出错时捕获处理的语句 | `try { } catch (e) { }` |
| **finally** | 无论是否出错都执行的代码块 | 用于清理资源 |
| **throw** | 主动抛出错误 | `throw new Error('msg')` |
| **错误堆栈** | 记录错误发生时的调用链 | `error.stack` |
| **自定义错误** | 继承 Error 类的特定错误类型 | `class ValidationError extends Error` |
| **重新抛出** | 捕获错误后再抛出，让上层处理 | `catch (e) { log(e); throw e; }` |
| **全局错误** | 未被任何 try-catch 捕获的错误 | 通过 `window.onerror` 监听 |

## 实践练习

### 练习：安全的数据加载器

```javascript
// 自定义错误类
class NetworkError extends Error {
    constructor(url, status) {
        super(`网络请求失败: ${url} (${status})`);
        this.name = 'NetworkError';
        this.url = url;
        this.status = status;
    }
}

class ParseError extends Error {
    constructor(message) {
        super(`数据解析失败: ${message}`);
        this.name = 'ParseError';
    }
}

// 安全的数据加载器
async function loadJSON(url) {
    try {
        const response = await fetch(url);

        if (!response.ok) {
            throw new NetworkError(url, response.status);
        }

        const text = await response.text();

        try {
            return JSON.parse(text);
        } catch (e) {
            throw new ParseError(e.message);
        }
    } catch (error) {
        // 区分错误类型
        if (error instanceof NetworkError) {
            console.error('网络错误:', error.message);
        } else if (error instanceof ParseError) {
            console.error('解析错误:', error.message);
        } else {
            console.error('未知错误:', error.message);
        }
        throw error;
    }
}

// 使用
try {
    const data = await loadJSON('/api/data');
    console.log(data);
} catch (error) {
    // 根据错误类型做不同处理
    if (error instanceof NetworkError && error.status === 404) {
        console.log('数据不存在，使用默认值');
    }
}
```

## 常见问题

### Q1：try-catch 会影响性能吗？

**影响极小。** 现代 JS 引擎优化得很好。但实际抛出错误时开销较大（创建 Error 对象 + 收集堆栈）。

```javascript
// ❌ 不推荐：在循环中 try-catch
for (let i = 0; i < 1000; i++) {
    try { process(arr[i]); } catch (e) {}
}

// ✅ 推荐：在循环外包裹
try {
    for (let i = 0; i < 1000; i++) { process(arr[i]); }
} catch (e) { console.error(e); }
```

### Q2：什么时候用 try-catch，什么时候用条件判断？

```javascript
// 用 try-catch：处理真正的异常
try { JSON.parse(str); } catch (e) { /* 解析失败 */ }

// 用条件判断：处理预期内的情况
if (user?.name) { console.log(user.name); }
else { console.log('匿名用户'); }
```

### Q3：如何捕获全局未处理错误？

```javascript
// 浏览器同步错误
window.addEventListener('error', (e) => console.error(e.error));

// 浏览器 Promise 错误
window.addEventListener('unhandledrejection', (e) => console.error(e.reason));
```
