## 7.0 通俗理解模块化

### 什么是模块化？

```
模块化 = 把代码拆分成多个文件

就像：
- 一个项目有很多文件
- 每个文件负责不同功能
- 文件之间可以互相使用
```

**为什么需要模块化？**

```
不用模块化：
- 所有代码写在一个文件
- 很难找、很难维护

用模块化：
- 代码分散到多个文件
- 每个文件功能单一
- 方便维护和复用
```

**简单比喻：**

```
项目 = 一个公司
模块 = 不同的部门
- 财务部（math.js）
- 人事部（user.js）
- 技术部（api.js）

部门之间需要合作
- 财务部的东西可以给技术部用
- 这就是"导入导出"
```

## 7.1 导出（export）

### 什么是导出？

```
导出 = 把东西给别人用

就像：
- 财务部对外开放一些服务
- 其他部门可以使用这些服务
```

### 具名导出（Named Export）

```javascript
// math.js 文件

// 方式1：声明时直接导出
export const PI = 3.14159;
export function add(a, b) {
    return a + b;
}

// 方式2：先声明，后导出
function subtract(a, b) {
    return a - b;
}
export { subtract };

// 方式3：批量导出
function multiply(a, b) { return a * b; }
function divide(a, b) { return a / b; }
export { multiply, divide };
```

**具名导出的特点：**
- 可以导出多个
- 导入时需要用 `{ name }` 精确指定

### 默认导出（Default Export）

```javascript
// calculator.js 文件

// 方式1：导出已存在的
function Calculator() {
    this.add = (a, b) => a + b;
}
export default Calculator;

// 方式2：直接导出匿名
export default class {
    sayHi() {
        console.log('Hi');
    }
}

// 方式3：导出表达式
export default {
    name: '默认导出',
    version: '1.0.0'
};
```

**默认导出的特点：**
- 一个文件只能有一个
- 导入时可以随便起名字

### 对比

```javascript
// 一个文件只能有一个默认导出
export default class A {}  // ✅ OK
// export default class B {}  // ❌ 错误

// 但可以有多个具名导出
export class C {}  // ✅ OK
export class D {}  // ✅ OK
```

## 7.2 导入（import）

### 什么是导入？

```
导入 = 使用别人的东西

就像：
- 技术部需要用财务部的计算器
- 需要先"导入"进来
```

### 基本导入

```javascript
// 导入具名导出（需要用大括号）
import { add, subtract } from './math.js';

// 导入默认导出（不用大括号）
import Calculator from './calculator.js';

// 同时导入默认和具名
import Utils, { format, parse } from './utils.js';
//        │                    │
//        │                    └── 具名导出
//        └── 默认导出
```

### 重命名导入

```javascript
// 导入时重命名（避免名字冲突）
import { add as plus, subtract as minus } from './math.js';

// 全部导入（作为对象）
import * as Math from './math.js';
Math.add(1, 2);  // 使用 Math.add
Math.PI;         // 使用 Math.PI
```

### 只导入不执行

```javascript
// 只执行模块，不导入任何内容
import './polyfill.js';  // 执行一下
import './styles.css';   // 加载样式
```

## 7.3 动态导入

### 什么是动态导入？

```
动态导入 = 懒加载 = 需要时再加载

就像：
- 页面刚打开时不需要某个功能
- 等用户点击时再加载
- 可以提高性能
```

### 基本用法

```javascript
// 点击按钮时才加载模块
button.addEventListener('click', async () => {
    // 动态导入，返回 Promise
    const module = await import('./module.js');
    
    // 使用
    module.default();           // 默认导出
    module.namedFunction();     // 具名导出
});
```

### 条件加载

```javascript
// 根据条件加载不同模块
async function loadModule() {
    if (isMobile) {
        return import('./mobile.js');
    } else {
        return import('./desktop.js');
    }
}
```

### 按需加载（路由懒加载）

```javascript
// Vue/React 路由懒加载
const routes = [
    {
        path: '/home',
        component: () => import('./views/Home.vue')
    },
    {
        path: '/about',
        component: () => import('./views/About.vue')
    }
];
```

## 7.4 完整例子

### 文件结构

```
project/
├── index.html
├── main.js
├── math.js
└── calculator.js
```

### math.js（导出）

```javascript
// math.js
export const PI = 3.14159;

export function add(a, b) {
    return a + b;
}

export function subtract(a, b) {
    return a - b;
}
```

### calculator.js（导出）

```javascript
// calculator.js
export default class Calculator {
    constructor() {
        this.name = '计算器';
    }
    
    add(a, b) {
        return a + b;
    }
}
```

### main.js（导入使用）

```javascript
// main.js

// 导入具名导出
import { PI, add, subtract } from './math.js';

console.log(PI);           // 3.14159
console.log(add(1, 2));    // 3
console.log(subtract(5, 3)); // 2

// 导入默认导出
import Calculator from './calculator.js';

const calc = new Calculator();
console.log(calc.add(10, 20)); // 30
```

## 7.5 .mjs 是什么？

### 为什么有 .mjs？

```
.mjs = "JavaScript Module" = ES6 模块文件

因为：
- .js 文件可以用 CommonJS 也可以用 ES6 Module
- 为了区分，Node.js 规定用 .mjs 表示"这是 ES6 模块"
```

### .js vs .mjs

```javascript
// math.js - 不确定是什么模块系统
const add = (a, b) => a + b;
module.exports = { add };

// math.mjs - 明确是 ES6 模块
export const add = (a, b) => a + b;
```

### 实际使用

```javascript
// math.mjs（导出）
export const PI = 3.14159;
export function add(a, b) { return a + b; }

// main.mjs（导入）
import { PI, add } from './math.mjs';

console.log(add(1, 2));  // 3
```

### package.json 配置

```json
{
    "type": "module"
}
```

**如果设置了 `"type": "module"`，.js 文件也会被当作 ES6 模块**

```json
// package.json
{
    "type": "module"
}
```

```javascript
// math.js（现在会被当作 ES6 模块！）
export const add = (a, b) => a + b;
```

### 一句话总结

```
.mjs = 明确告诉 Node.js "这是 ES6 模块"
.js = 默认当作 CommonJS（除非 package.json 设置 type: module）
```

## 7.6 CommonJS vs ES6 Module

### 12-模块化系统4                                         4种模块系统

```
JavaScript 有两套模块系统：
 2. ES6 Module - 现代前端用
```

### 语法对比

```javascript
// ====== CommonJS (Node.js) ======
// 导出
module.exports = { add, subtract };
// 或
exports.add = add;

// 导入
const { add, subtract } = require('./math.js');


// ====== ES6 Module (现代前端) ======
// 导出
export { add, subtract };

// 导入
import { add, subtract } from './math.js';
```

### 核心区别

| 特性 | CommonJS | ES6 Module |
|------|----------|------------|
| 语法 | require/exports | import/export |
| 加载时机 | 运行时 | 编译时 |
| 输出 | 值的拷贝 | 值的引用 |
| 浏览器 | ❌ 不支持 | ✅ 支持 |
| 循环依赖 | 有问题 | ✅ 支持 |

### 通俗理解区别

```javascript
// CommonJS - 值的拷贝
// 导出的是值的"复制"，导出后修改不影响原值

// ES6 Module - 值的引用
// 导出的是值的"引用"，类似指针
```

## 7.6 一句话总结

```
模块化 = 把代码拆分成多个文件

导出 = 把自己东西给别人用
import = 用别人的东西

具名导出 = 精确指定要哪个
默认导出 = 随便起名字

CommonJS = Node.js 用
ES6 Module = 前端用
```
