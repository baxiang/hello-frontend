> ES6+ 为数组和对象添加了大量实用方法，让代码更简洁高效

## 6.0 为什么要学这些新方法？

### 6.0.1 故事背景

在 ES6 之前，操作数组和对象需要写很多重复代码：

```javascript
// 查找元素
const arr = [1, 2, 3, 4, 5];
let found = null;
for (let i = 0; i < arr.length; i++) {
    if (arr[i] > 3) {
        found = arr[i];
        break;
    }
}

// 判断数组是否包含某个值
const includes = arr.indexOf(3) !== -1;

// 合并对象
const merged = {};
Object.keys(obj1).forEach(key => merged[key] = obj1[key]);
Object.keys(obj2).forEach(key => merged[key] = obj2[key]);
```

ES6+ 提供了更简洁的方法来解决这些问题。

### 6.0.2 本章学习路径

```
第一步：Array.from 和 Array.of（创建数组的新方式）
    ↓
第二步：find/findIndex（查找元素）
    ↓
第三步：includes（判断包含）
    ↓
第四步：flat/flatMap（数组扁平化）
    ↓
第五步：Object 新方法（对象操作）
    ↓
第六步：可选链和空值合并（安全访问）
```

## 6.1 Array.from 与 Array.of

### 6.1.1 Array.from()

将类数组对象或可迭代对象转换为真正的数组。

**什么是类数组对象？**

```javascript
// 类数组对象：有 length 属性，但不是数组
const arrayLike = {
    0: 'a',
    1: 'b',
    2: 'c',
    length: 3
};

console.log(arrayLike[0]);  // 'a'
console.log(arrayLike.length);  // 3
// arrayLike.map is not a function（不是数组，没有 map 方法）
```

**转换方法：**

```javascript
// 使用 Array.from 转换为真数组
const arr = Array.from(arrayLike);
console.log(arr);  // ['a', 'b', 'c']
console.log(arr.map(x => x.toUpperCase()));  // ['A', 'B', 'C']
```

**常见用途：**

```javascript
// 1. 从字符串创建数组
const str = 'hello';
console.log(Array.from(str));  // ['h', 'e', 'l', 'l', 'o']

// 2. 从 Set 创建数组
const set = new Set([1, 2, 3]);
console.log(Array.from(set));  // [1, 2, 3]

// 3. 从 Map 创建数组
const map = new Map([['a', 1], ['b', 2]]);
console.log(Array.from(map));  // [['a', 1], ['b', 2]]

// 4. 生成数字范围数组
const range = Array.from({ length: 5 }, (_, i) => i + 1);
console.log(range);  // [1, 2, 3, 4, 5]
```

### 6.1.2 Array.of()

将一组值转换为数组，弥补 `Array()` 构造函数的不足。

**Array() 的问题：**

```javascript
// Array(3) 创建一个长度为 3 的空数组
console.log(Array(3));  // [empty × 3]（不是 [3]！）

// Array(1, 2) 创建包含两个元素的数组
console.log(Array(1, 2));  // [1, 2]
```

**Array.of() 的行为：**

```javascript
// Array.of(3) 创建一个包含单个元素 3 的数组
console.log(Array.of(3));  // [3]

// 行为一致
console.log(Array.of(1, 2, 3));  // [1, 2, 3]
console.log(Array.of());        // []
```

## 6.2 find 与 findIndex

### 6.2.1 find()

返回数组中第一个满足条件的元素。

```javascript
const users = [
    { id: 1, name: '张三', age: 25 },
    { id: 2, name: '李四', age: 30 },
    { id: 3, name: '王五', age: 35 }
];

// 查找年龄大于 28 的第一个用户
const user = users.find(u => u.age > 28);
console.log(user);  // { id: 2, name: '李四', age: 30 }

// 查找不存在的元素
const notFound = users.find(u => u.age > 100);
console.log(notFound);  // undefined
```

### 6.2.2 findIndex()

返回数组中第一个满足条件的元素索引。

```javascript
const numbers = [1, 5, 10, 15, 20];

// 查找第一个大于 10 的元素索引
const index = numbers.findIndex(n => n > 10);
console.log(index);  // 3（对应元素 15）

// 查找不存在的元素
const notFound = numbers.findIndex(n => n > 100);
console.log(notFound);  // -1
```

### 6.2.3 findLast 与 findLastIndex (ES2023)

从后向前查找。

```javascript
const numbers = [1, 5, 10, 15, 10, 20];

// 从后向前查找第一个大于 10 的元素
const last = numbers.findLast(n => n > 10);
console.log(last);  // 20

// 从后向前查找的索引
const lastIndex = numbers.findLastIndex(n => n === 10);
console.log(lastIndex);  // 4
```

## 6.3 includes()

判断数组是否包含某个值，比 `indexOf()` 更直观。

### 6.3.1 基本用法

```javascript
const arr = [1, 2, 3, NaN, Infinity];

// 判断是否包含
console.log(arr.includes(2));   // true
console.log(arr.includes(100)); // false
```

### 6.3.2 includes vs indexOf

```javascript
const arr = [1, 2, 3, NaN];

// includes 能正确判断 NaN
console.log(arr.includes(NaN));  // true

// indexOf 无法判断 NaN
console.log(arr.indexOf(NaN));   // -1（错误！）
```

### 6.3.3 第二个参数

```javascript
const arr = [1, 2, 3, 4, 5];

// 从指定索引开始搜索
console.log(arr.includes(3, 2));   // true（从索引 2 开始）
console.log(arr.includes(1, 2));   // false

// 负数索引
console.log(arr.includes(3, -2));  // false（从倒数第 2 个开始）
console.log(arr.includes(5, -1));  // true（从最后一个开始）
```

## 6.4 flat() 与 flatMap()

### 6.4.1 什么是数组扁平化？

```
┌─────────────────────────────────────────────────────────────┐
│  数组扁平化 = 把嵌套数组"铺平"成一层                       │
│                                                             │
│  就像：                                                     │
│  - 折叠的纸盒打开铺平                                       │
│  - 俄罗斯套娃全部打开平放                                  │
│                                                             │
│  例子：                                                     │
│  [1, [2, [3, [4]]]] → [1, 2, 3, 4]                       │
└─────────────────────────────────────────────────────────────┘
```

### 6.4.2 flat() - 扁平化数组

**基本语法：**

```javascript
// 嵌套数组
const nested = [1, [2, 3], [4, [5, 6]]];

console.log(nested);  // [1, [2, 3], [4, [5, 6]]]
```

**flat() 的作用：**

```javascript
// 默认展平一层（只展开一层）
console.log(nested.flat());   // [1, 2, 3, 4, [5, 6]]
//                  ↑
//                  只有这一层被展开了
//                  [5, 6] 还在里面
```

**展平多层：**

```javascript
// 展平指定层数
console.log(nested.flat(2));  // [1, 2, 3, 4, 5, 6]
//                    ↑
//                    展平 2 层

// 展平所有层级（不确定嵌套多深时用）
console.log(nested.flat(Infinity));  // [1, 2, 3, 4, 5, 6]
```

**生活例子：**

```
假设有一个文件夹结构：
- 文件夹1
  - 文件夹1-1
    - 文件1-1-1
  - 文件夹1-2
- 文件夹2

flat(1) = 只打开第一层文件夹
          结果：文件夹1-1, 文件1-1-1, 文件夹1-2, 文件夹2

flat(2) = 打开两层文件夹
          结果：文件1-1-1, 文件夹1-2, 文件夹2

flat(Infinity) = 全部打开
          结果：所有文件
```

### 6.4.3 flatMap() - 先映射后扁平

**为什么需要 flatMap？**

```
┌─────────────────────────────────────────────────────────────┐
│  flatMap = map + flat 的组合操作                            │
│                                                             │
│  适用场景：                                                 │
│  1. 先对数组元素进行转换（map）                            │
│  2. 然后把转换结果展平（flat）                            │
│                                                             │
│  优点：                                                    │
│  - 代码更简洁                                              │
│  - 性能更好（只遍历一次）                                  │
└─────────────────────────────────────────────────────────────┘
```

**示例：拆分字符串**

```javascript
const words = ['hello world', 'foo bar'];

// 传统方式：map + flat
const result1 = words.map(s => s.split(' '));
// ['hello', 'world'].flat() = ['hello', 'world', 'foo', 'bar']

// flatMap 方式（更简洁）
const result = words.flatMap(s => s.split(' '));
console.log(result);  // ['hello', 'world', 'foo', 'bar']
```

**执行过程图解：**

```
words = ['hello world', 'foo bar']

第 1 步：map 转换
  'hello world' → ['hello', 'world']
  'foo bar' → ['foo', 'bar']

第 2 步：flat 展平
  [['hello', 'world'], ['foo', 'bar']]
  → ['hello', 'world', 'foo', 'bar']
```

**实际应用：过滤并转换**

```javascript
const numbers = [1, 2, 3, 4, 5];

// 找出偶数并乘以 2
const result = numbers
    .filter(n => n % 2 === 0)
    .map(n => n * 2);

// flatMap 一步完成
const result2 = numbers.flatMap(n => 
    n % 2 === 0 ? [n * 2] : []
);

console.log(result);   // [4, 8]
console.log(result2); // [4, 8]
```

## 6.5 Object 新方法

### 6.5.1 Object.assign()

合并对象。

```javascript
const defaults = { theme: 'light', lang: 'zh' };
const userPrefs = { theme: 'dark' };

// 合并到新对象
const config = Object.assign({}, defaults, userPrefs);
console.log(config);  // { theme: 'dark', lang: 'zh' }

// 合并到现有对象
Object.assign(defaults, userPrefs);
console.log(defaults);  // { theme: 'dark', lang: 'zh' }
```

### 6.5.2 Object.entries() 与 Object.values()

```javascript
const obj = { name: '张三', age: 25, city: '北京' };

// 获取所有键值对
console.log(Object.entries(obj));
// [['name', '张三'], ['age', 25], ['city', '北京']]

// 获取所有值
console.log(Object.values(obj));
// ['张三', 25, '北京']

// 遍历对象
for (const [key, value] of Object.entries(obj)) {
    console.log(`${key}: ${value}`);
}
```

### 6.5.3 Object.fromEntries()

将键值对数组转换为对象。

```javascript
// 键值对数组
const entries = [['name', '张三'], ['age', 25]];

// 转换为对象
const obj = Object.fromEntries(entries);
console.log(obj);  // { name: '张三', age: 25 }

// 实用：URL 查询参数转对象
const params = new URLSearchParams('name=张三&age=25');
const paramsObj = Object.fromEntries(params);
console.log(paramsObj);  // { name: '张三', age: '25' }
```

## 6.6 可选链（?.）与空值合并（??）

### 6.6.1 可选链（Optional Chaining）

安全地访问深层嵌套属性。

**问题：访问深层属性可能出错**

```javascript
const user = {
    profile: {
        name: '张三'
    }
};

// 传统方式：层层检查
const city1 = user && user.profile && user.profile.address && user.profile.address.city;

// 可选链：安全访问
const city2 = user?.profile?.address?.city;
console.log(city2);  // undefined（不会报错）
```

**使用场景：**

```javascript
// 1. 访问对象属性
const user = { profile: { name: '张三' } };
console.log(user?.profile?.name);  // '张三'
console.log(user?.address?.city);  // undefined

// 2. 访问数组元素
const arr = [1, 2, 3];
console.log(arr?.[0]);    // 1
console.log(arr?.[10]);   // undefined

// 3. 调用可能不存在的方法
const obj = {
    fn: () => 'hello'
};
console.log(obj.fn?.());     // 'hello'
console.log(obj.notExist?.());  // undefined（不会报错）
```

### 6.6.2 空值合并（Nullish Coalescing）

当值为 null 或 undefined 时使用默认值。

**问题：|| 会把 0 和空字符串当作假值**

```javascript
const value = 0;

// || 的问题
console.log(value || 'default');  // 'default'（0 被当作假值！）

// ?? 的解决方案
console.log(value ?? 'default');  // 0（只有 null/undefined 才用默认值）
```

**对比：**

```javascript
// || vs ??
'' || 'default';    // 'default'
'' ?? 'default';    // ''（空字符串保留）

0 || 'default';     // 'default'
0 ?? 'default';     // 0（0 保留）

null || 'default';  // 'default'
null ?? 'default';  // 'default'

undefined || 'default';  // 'default'
undefined ?? 'default'; // 'default'
```

## 6.8 常见问答

### Q1: map 和 flatMap 有什么区别？

```javascript
const arr = [1, 2, 3];

// map：返回数组的数组
const mapped = arr.map(x => [x, x * 2]);
console.log(mapped);  // [[1, 2], [2, 4], [3, 6]]

// flatMap：先 map 再 flat
const flatMapped = arr.flatMap(x => [x, x * 2]);
console.log(flatMapped);  // [1, 2, 2, 4, 3, 6]
```

### Q2: includes 和 indexOf 哪个好？

```javascript
const arr = [1, 2, 3, NaN];

// includes 更直观，返回布尔值
arr.includes(NaN);  // true

// indexOf 返回索引，需要比较 -1
arr.indexOf(NaN);  // -1（无法判断 NaN）

// 推荐使用 includes
```

### Q3: 可选链可以替代 if 判断吗？

```javascript
// 可选链用于读取属性
const value = obj?.property;

// 不能用于赋值
// obj?.property = 'value';  // 报错

// 赋值仍需要 if 判断
if (obj) {
    obj.property = 'value';
}
```
