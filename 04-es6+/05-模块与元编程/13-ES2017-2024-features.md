## L1 理解层：会用

### 近年新特性速查

| 版本 | 特性 | 一句话说明 | 示例 |
|------|------|-----------|------|
| ES2017 | async/await | 异步代码像同步一样写 | `const data = await fetch(url)` |
| ES2017 | Object.values/entries | 获取对象的值和键值对 | `Object.entries(obj)` |
| ES2018 | 对象展开 | 合并对象更简洁 | `{ ...obj1, ...obj2 }` |
| ES2018 | 异步迭代 | `for await...of` | 异步数据流 |
| ES2019 | flat/flatMap | 数组扁平化 | `arr.flat(Infinity)` |
| ES2019 | trimStart/End | 清理首尾空白 | `'  hi  '.trimStart()` |
| ES2019 | Object.fromEntries | 键值对转对象 | `Object.fromEntries(entries)` |
| ES2020 | 可选链 ?. | 安全访问嵌套属性 | `obj?.a?.b` |
| ES2020 | 空值合并 ?? | 精确空值判断 | `value ?? 'default'` |
| ES2020 | BigInt | 超大整数 | `9007199254740991n` |
| ES2021 | replaceAll | 全局替换 | `'aa'.replaceAll('a','A')` |
| ES2021 | 逻辑赋值 | `\|\|=`, `&&=`, `??=` | `x ??= 5` |
| ES2022 | 私有字段 # | 语言级私有 | `class X { #field; }` |
| ES2022 | 顶层 await | 模块顶层用 await | `export const data = await fetch()` |
| ES2023 | findLast | 从后往前查找 | `arr.findLast(fn)` |
| ES2023 | toSorted 等 | 不修改原数组 | `arr.toSorted(fn)` |

## L2 实践层：用好

### 逻辑赋值

```javascript
// ||= — 假值时赋值
x ||= 5;  // 等价于 x || (x = 5)

// &&= — 真值时赋值
x &&= 5;  // 等价于 x && (x = 5)

// ??= — null/undefined 时赋值（最常用）
config.timeout ??= 5000;  // 只有未设置时才赋默认值
```

### 反模式：不要这样做

```javascript
// ❌ 错误：用 replace 替换全部（容易漏 g 标志）
str.replace('old', 'new');  // 只替换第一个

// ✅ 正确：用 replaceAll
str.replaceAll('old', 'new');
```

```javascript
// ❌ 错误：sort 修改原数组
const sorted = arr.sort((a, b) => a - b);  // arr 也被改了

// ✅ 正确：用 toSorted（ES2023）
const sorted = arr.toSorted((a, b) => a - b);
```

### 适用场景

| 场景 | 推荐特性 | 示例 |
|------|---------|------|
| 安全读取嵌套 | `?.` | `user?.address?.city` |
| 默认值配置 | `??=` | `options.timeout ??= 5000` |
| 不可变排序 | `toSorted` | `arr.toSorted()` |
| 反向查找 | `findLast` | `arr.findLast(x => x > 0)` |
| 模块初始化 | 顶层 await | `export const config = await loadConfig()` |
| 全局字符串替换 | `replaceAll` | `text.replaceAll('<', '&lt;')` |

## L3 专家层：深入

### ES2023 不可变数组方法

```javascript
const arr = [3, 1, 2];

// 传统方法 — 修改原数组
arr.sort((a, b) => a - b);  // arr = [1, 2, 3]

// ES2023 — 返回新数组
arr.toSorted((a, b) => a - b);  // 返回 [1, 2, 3]，arr 不变
arr.toReversed();               // [3, 2, 1]
arr.toSpliced(1, 1, 99);        // [3, 99, 2]
arr.with(0, 99);                // [99, 1, 2]
```

### BigInt 注意事项

```javascript
// BigInt 不能与 Number 混用
// 10n + 5;  // ❌ TypeError

// 正确做法
10n + BigInt(5);  // 15n
Number(10n) + 5;  // 15

// BigInt 比较
10n > 5;    // true（可以比较）
10n === 10; // false（类型不同）
10n == 10;  // true（宽松相等）
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **可选链** | 安全访问可能不存在的属性 | `obj?.prop` |
| **空值合并** | 只有 null/undefined 时用默认值 | `value ?? 'default'` |
| **逻辑赋值** | 逻辑运算 + 赋值的简写 | `x ??= 5` |
| **顶层 await** | 模块顶层直接使用 await | `const data = await fetch()` |
| **不可变方法** | 返回新数组而不修改原数组 | `arr.toSorted()` |

## 常见问题

### Q1：`??` 和 `||` 用哪个好？

**`??` 更精确，只把 null/undefined 当空值：**

```javascript
0 || 'default';    // 'default'（0 被当作假值）
0 ?? 'default';    // 0（保留有效值）
'' ?? 'default';   // ''（保留有效值）
```

### Q2：BigInt 有什么用？

**处理超出 Number 安全范围的整数：**

```javascript
Number.MAX_SAFE_INTEGER;  // 9007199254740991
9007199254740992n + 1n;   // 9007199254740993n（精确）
9007199254740992 + 1;     // 9007199254740992（精度丢失！）
```
