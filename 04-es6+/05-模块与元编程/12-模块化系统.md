## 生活化比喻

**模块就像"图书馆的书架"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                  图书馆                               │
├──────────────────────────────────────────────────────┤
│                                                      │
│    export = 把书放上书架                              │
│    ─────────────                                     │
│    export const PI = 3.14  → 把 PI 这本书放上架       │
│    export default App      → 把 App 放在推荐位        │
│                                                      │
│    import = 从书架借书                               │
│    ─────────────                                     │
│    import { PI } from './math'  → 借 PI 这本书       │
│    import App from './App'      → 借推荐位的书       │
│                                                      │
│    具名导出 = 普通书架                                │
│    ─────────────                                     │
│    每本书有固定位置，必须知道书名才能借               │
│    export { add, sub } → 借时必须用原名              │
│                                                      │
│    默认导出 = 推荐书架                               │
│    ─────────────                                     │
│    每个书架只有一个推荐位，借的时候可以自己起名字     │
│    export default App → import MyApp from './App'    │
│                                                      │
│    动态导入 = 临时借书                                │
│    ─────────────                                     │
│    需要的时候才去借，不用一开始全借回来              │
│    const mod = await import('./heavy.js')            │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

### 导出与导入

**语法结构图：**

```
导出（export）：

具名导出：
  export const PI = 3.14;              ← 声明时导出
  export function add(a, b) { }        ← 声明时导出
  export { foo, bar as renamed };      ← 统一导出

默认导出（每个文件只能有一个）：
  export default function App() { }    ← 默认导出
  export default class User { }

导入（import）：

具名导入：
  import { PI, add } from './math.js';
  import { add as sum } from './math.js';  ← 重命名
  import * as math from './math.js';       ← 全部导入

默认导入：
  import App from './App.js';              ← 名字可自定

混合导入：
  import React, { useState, useEffect } from 'react';
```

**最简示例：**

```javascript
// math.js
export const PI = 3.14;
export function add(a, b) { return a + b; }

// main.js
import { PI, add } from './math.js';
```

### 动态导入

**最简示例：**

```javascript
const mod = await import('./utils.js');
mod.helper();
```

**详细示例：**

```javascript
// 路由懒加载
const Home = () => import('./pages/Home.js');
const About = () => import('./pages/About.js');

// 条件加载
if (user.isAdmin) {
    const { AdminPanel } = await import('./admin.js');
    render(AdminPanel);
}

// 错误处理
try {
    const mod = await import('./optional-feature.js');
    mod.init();
} catch {
    console.log('可选功能加载失败，使用降级方案');
}
```

## L2 实践层：用好

### 导出方式选择

| 场景 | 推荐方式 | 示例 |
|------|---------|------|
| 工具函数库 | 具名导出 | `export { add, sub, mul }` |
| React/Vue 组件 | 默认导出 | `export default function App()` |
| 类定义 | 默认导出 | `export default class User {}` |
| 常量配置 | 具名导出 | `export const API_URL = '...'` |
| 一个文件多个功能 | 具名导出 | `export { fn1, fn2, fn3 }` |

### 反模式：不要这样做

```javascript
// ❌ 错误：一个文件多个默认导出
export default function A() {}
export default function B() {}  // SyntaxError

// ✅ 正确：用具名导出
export function A() {}
export function B() {}
```

```javascript
// ❌ 错误：在 import 前使用模块内容
console.log(PI);  // ❌ ReferenceError
import { PI } from './math.js';

// ✅ 正确：import 必须在顶部
import { PI } from './math.js';
console.log(PI);
```

### ESM vs CommonJS

| 特性 | ESM (import/export) | CommonJS (require) |
|------|:---:|:---:|
| 加载时机 | 编译时（静态分析） | 运行时（动态） |
| 语法 | `import`/`export` | `require()`/`module.exports` |
| 默认导出 | 支持 | 无（`module.exports = {}`） |
| 动态加载 | `import()` | `require()` 原生支持 |
| Tree Shaking | ✅ 支持 | ❌ 不支持 |
| 顶层 await | ✅ 支持 | ❌ 不支持 |
| 文件扩展 | `.mjs` 或 `"type":"module"` | `.cjs` 或默认 |

## L3 专家层：深入

### 循环依赖处理

```
模块 A ←→ 模块 B 的循环依赖：

A.js:
  import { fnB } from './B.js';
  export function fnA() { fnB(); }

B.js:
  import { fnA } from './A.js';
  export function fnB() { fnA(); }

ESM 处理：编译时建立引用关系，运行时可能拿到未初始化的值
解决：提取共享依赖到第三个模块 C
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **ESM** | ES6 模块系统，使用 import/export | `import { x } from './m.js'` |
| **具名导出** | 带名称的导出，导入时必须用同名 | `export const PI = 3.14` |
| **默认导出** | 每个文件一个，导入时可自定名 | `export default App` |
| **动态导入** | 运行时按需加载模块 | `await import('./m.js')` |
| **Tree Shaking** | 构建工具移除未使用的导出 | Webpack/Vite 自动优化 |
| **CommonJS** | Node.js 传统模块系统 | `require('./m.js')` |

## 实践练习

### 练习：工具库模块化

```javascript
// utils/math.js
export const PI = 3.14159;
export function add(a, b) { return a + b; }
export function mul(a, b) { return a * b; }
export default { PI, add, mul };

// utils/string.js
export function capitalize(str) { return str.charAt(0).toUpperCase() + str.slice(1); }
export function reverse(str) { return str.split('').reverse().join(''); }

// main.js（使用）
import { add, mul } from './utils/math.js';
import { capitalize } from './utils/string.js';
import math from './utils/math.js';  // 默认导出

console.log(add(1, 2));        // 3
console.log(capitalize('hi')); // 'Hi'
```

## 常见问题

### Q1：默认导出和具名导出用哪个好？

**推荐优先用具名导出，默认导出只用于单一主功能：**

```javascript
// 工具库 — 具名导出
export { add, sub, mul };

// 组件 — 默认导出
export default function App() { }
```

### Q2：ESM 和 CommonJS 能混用吗？

**不推荐。如果必须混用：**

```javascript
// ESM 导入 CommonJS
import cjsModule from './cjs-file.js';  // ✅ 默认导入整个 module.exports

// CommonJS 导入 ESM（Node.js 需要 async）
const esmModule = await import('./esm-file.mjs');  // ✅
```
