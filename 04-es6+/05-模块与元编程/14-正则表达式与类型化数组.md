## 生活化比喻

**正则增强就像"给放大镜加新功能"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                  放大镜                               │
├──────────────────────────────────────────────────────┤
│                                                      │
│    dotAll (s) = 放大镜能穿透换行                      │
│    ─────────────────                                 │
│    之前点号遇到换行就看不到后面                       │
│    加上 s 标志后，换行也能穿透                       │
│                                                      │
│    命名捕获组 = 给找到的东西贴标签                    │
│    ─────────────                                     │
│    之前：找到第1个、第2个...容易搞混                 │
│    现在：找到年份、月份、日期，一目了然              │
│                                                      │
│    后行断言 = 回头看                                 │
│    ─────────────                                     │
│    "找前面是 $ 的数字"                              │
│    只取数字部分，不包含 $                            │
│                                                      │
│    类型化数组 = 专用储物盒                            │
│    ─────────────                                     │
│    普通数组 = 万能盒子（什么都能装，但占空间）       │
│    类型化数组 = 专用格子（只装一种，紧凑高效）       │
│                                                      │
└──────────────────────────────────────────────────────┘
```

## L1 理解层：会用

### 正则增强

**最简示例：**

```javascript
// dotAll — 点号匹配换行
/hello.world/s.test('hello\nworld');  // true

// 命名捕获组 — 给匹配结果起名字
'2024-01-15'.match(/(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/).groups;
// { year: '2024', month: '01', day: '15' }

// 后行断言 — 查找前面是某物的内容
'￥100 $200'.match(/(?<=\$)\d+/g);  // ['200']
```

**详细示例：**

```javascript
// 命名捕获组 + replace
const date = '2024-01-15';
const reformatted = date.replace(
    /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/,
    '$<month>/$<day>/$<year>'
);
// '01/15/2024'

// matchAll — 获取所有匹配及捕获组
const html = '<div class="a">1</div><div class="b">2</div>';
const matches = [...html.matchAll(/<div class="(?<cls>\w+)">(?<num>\d+)<\/div>/g)];
matches.map(m => m.groups);
// [{ cls: 'a', num: '1' }, { cls: 'b', num: '2' }]
```

### 类型化数组

**最简示例：**

```javascript
new Uint8Array([1, 2, 3]);       // 8位无符号整数
new Float32Array([1.5, 2.5]);    // 32位浮点数
```

**详细示例：**

```javascript
// ArrayBuffer — 原始二进制缓冲区
const buffer = new ArrayBuffer(8);  // 8 字节

// 通过不同视图读写
const u8 = new Uint8Array(buffer);       // 8位视图，8个元素
const u16 = new Uint16Array(buffer);     // 16位视图，4个元素
const view = new DataView(buffer);       // 灵活视图

view.setUint8(0, 255);
view.setFloat64(1, 3.14);
view.getFloat64(1);  // 3.14
```

## L2 实践层：用好

### 正则新特性速查

| 特性 | 语法 | 用途 | 示例 |
|------|------|------|------|
| dotAll | `/./s` | 点号匹配换行 | `/<div>.*?<\/div>/s` |
| 命名捕获 | `(?<name>...)` | 给捕获组命名 | `/(?<year>\d{4})/` |
| 后行肯定断言 | `(?<=...)` | 前面必须是 | `/(?<=\$)\d+/` |
| 后行否定断言 | `(?<!...)` | 前面不能是 | `/(?<!\$)\d+/` |
| matchAll | `str.matchAll(re)` | 获取所有匹配+组 | `[...str.matchAll(re)]` |

### 反模式：不要这样做

```javascript
// ❌ 错误：用数字索引引用捕获组（易混淆）
date.match(/(\d{4})-(\d{2})-(\d{2})/);
console.log(match[1], match[2], match[3]);  // 哪个是月？哪个是日？

// ✅ 正确：用命名捕获组
date.match(/(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/);
console.log(match.groups.month);  // 一目了然
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 解析日期/日志 | 命名捕获组 | 字段名清晰 |
| 提取 HTML 属性 | matchAll | 一次获取所有匹配 |
| 多行文本匹配 | dotAll (`s`) | 点号穿透换行 |
| 图片/音频处理 | ArrayBuffer + TypedArray | 直接操作二进制 |
| WebSocket 二进制消息 | DataView | 灵活读取不同字节类型 |

## L3 专家层：深入

### matchAll vs match

```javascript
const text = 'a1b2c3';

// match — 无 g 标志时返回捕获组，有 g 标志时不返回捕获组
text.match(/(\w)(\d)/);     // ['a1', 'a', '1', index: 0, ...]
text.match(/(\w)(\d)/g);    // ['a1', 'b2', 'c3']（无捕获组！）

// matchAll — 始终返回捕获组
[...text.matchAll(/(\w)(\d)/g)];
// [['a1','a','1'], ['b2','b','2'], ['c3','c','3']]
```

### 类型化数组内存模型

```
ArrayBuffer 与视图的关系：

ArrayBuffer(8) — 8 字节原始缓冲区
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ 0 │ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │  字节偏移
└───┴───┴───┴───┴───┴───┴───┴───┘
 ↑              ↑              ↑
Uint8Array     Uint16Array    Float64Array
(8 个元素)     (4 个元素)     (1 个元素)

同一块内存，不同视角解读
```

### 性能考量

| 特性 | 性能影响 | 说明 |
|------|---------|------|
| 命名捕获组 | 极低 | 编译时处理，运行时无额外开销 |
| 后行断言 | 低 | 比先行断言稍慢，需回看 |
| dotAll (`s`) | 极低 | 只是改变 `.` 的匹配范围 |
| matchAll | 低 | 返回迭代器，惰性求值 |
| TypedArray | 极低 | 连续内存，比 Array 快 2-10 倍 |
| DataView | 中等 | 灵活但有类型转换开销 |

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **dotAll** | 让 `.` 匹配包括换行符在内的所有字符 | `/hello.world/s` |
| **命名捕获组** | 给正则捕获组命名 | `(?<name>pattern)` |
| **后行断言** | 匹配前面是/不是某模式的内容 | `(?<=\$)\d+` |
| **matchAll** | 返回所有匹配及其捕获组的迭代器 | `str.matchAll(/pattern/g)` |
| **ArrayBuffer** | 固定长度的原始二进制数据缓冲区 | `new ArrayBuffer(16)` |
| **TypedArray** | 以特定类型读写 ArrayBuffer 的视图 | `new Uint8Array(buffer)` |
| **DataView** | 灵活读写 ArrayBuffer 中不同类型数据 | `view.setUint8(0, 255)` |

## 实践练习

```javascript
// 练习 1：解析日志行
const log = '[2024-01-15 10:30:45] ERROR: Connection timeout';
const regex = /\[(?<date>\d{4}-\d{2}-\d{2}) (?<time>\d{2}:\d{2}:\d{2})\] (?<level>\w+): (?<message>.+)/;
const match = log.match(regex);
console.log(match.groups);
// { date: '2024-01-15', time: '10:30:45', level: 'ERROR', message: 'Connection timeout' }

// 练习 2：解析 HEX 颜色
function hexToRgb(hex) {
    const match = hex.match(/^#(?<r>[a-f\d]{2})(?<g>[a-f\d]{2})(?<b>[a-f\d]{2})$/i);
    if (!match) return null;
    const { r, g, b } = match.groups;
    return { r: parseInt(r, 16), g: parseInt(g, 16), b: parseInt(b, 16) };
}
hexToRgb('#ff0000');  // { r: 255, g: 0, b: 0 }

// 练习 3：二进制文件头读取
function readPNGHeader(buffer) {
    const view = new DataView(buffer);
    const signature = new Uint8Array(buffer, 0, 8);
    const width = view.getUint32(16);   // IHDR 块的宽度
    const height = view.getUint32(20);  // IHDR 块的高度
    return { width, height };
}
```

## 常见问题

### Q1：dotAll 和 `[\s\S]` 有什么区别？

**功能相同，dotAll 更简洁：**

```javascript
// 旧方式
/hello[\s\S]*?world/.test('hello\nworld');  // true

// ES2018
/hello.*?world/s.test('hello\nworld');      // true
```

### Q2：类型化数组和普通数组有什么区别？

```javascript
// 普通数组：可存任意类型，不连续内存
const arr = [1, 'a', true, { x: 1 }];

// 类型化数组：固定类型，连续内存，高效
const uint8 = new Uint8Array([1, 2, 3]);  // 只能存 0-255 的整数
```
