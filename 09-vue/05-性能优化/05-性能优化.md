## L1 理解层：会用

### 响应式优化

**最简示例（1-3行）：**

```vue
<script setup>
import { shallowRef, markRaw } from 'vue'
const config = shallowRef({ settings: { theme: 'dark' } })
const largeData = markRaw(fetchLargeData())
</script>
```

**详细示例：**

```vue
<script setup>
import { ref, shallowRef, markRaw } from 'vue'

// shallowRef — 只追踪 .value 变化，不追踪内部属性
const config = shallowRef({ settings: { theme: 'dark' } })
config.value = { settings: { theme: 'light' } }  // ✅ 触发更新
config.value.settings.theme = 'light'            // ❌ 不触发更新

// markRaw — 标记为非响应式
import { Chart } from 'chart.js'
const chartInstance = markRaw(new Chart(ctx, { ... }))  // 不需要响应式的对象
</script>
```

### 组件懒加载

```vue
<!-- 路由懒加载 -->
const router = createRouter({
  routes: [
    { path: '/about', component: () => import('./views/About.vue') },
    { path: '/dashboard', component: () => import('./views/Dashboard.vue') }
  ]
})

<!-- 条件懒加载 -->
<script setup>
import { defineAsyncComponent } from 'vue'

const HeavyComponent = defineAsyncComponent({
  loader: () => import('./HeavyComponent.vue'),
  loadingComponent: LoadingSpinner,
  delay: 200,
  timeout: 3000
})
</script>
```

---

### v-if vs v-show

```vue
<!-- v-if：销毁/创建 DOM（切换不频繁时使用） -->
<div v-if="show">内容</div>

<!-- v-show：CSS display 切换（频繁切换时使用） -->
<div v-show="show">内容</div>
```

## L2 实践层：用好

### 优化策略速查

| 策略 | 方法 | 何时用 | 效果 |
|------|------|--------|------|
| shallowRef | `shallowRef(obj)` | 大型对象只需替换 | 减少响应式开销 |
| markRaw | `markRaw(obj)` | 第三方实例、大型数据 | 不创建响应式代理 |
| 组件懒加载 | `() => import()` | 非首屏组件 | 减少首屏体积 |
| computed 缓存 | `computed(fn)` | 复杂计算、频繁读取 | 避免重复计算 |
| v-if | `v-if="show"` | 不频繁切换 | 销毁/创建 DOM |
| v-show | `v-show="show"` | 频繁切换 | CSS 切换 |
| v-for key | `:key="item.id"` | 列表渲染 | 减少 DOM 操作 |
| 虚拟列表 | `vue-virtual-scroller` | 长列表（1000+ 项） | 只渲染可见区域 |

### 反模式：不要这样做

```vue
<!-- ❌ 错误：用 index 作为 v-for 的 key -->
<li v-for="(item, index) in items" :key="index">{{ item.name }}</li>

<!-- ✅ 正确：用唯一 ID -->
<li v-for="item in items" :key="item.id">{{ item.name }}</li>
```

```vue
<!-- ❌ 错误：v-if 和 v-for 在同一元素上 -->
<li v-for="item in items" v-if="item.active">{{ item.name }}</li>

<!-- ✅ 正确：用 computed 过滤 -->
<template v-for="item in activeItems" :key="item.id">
  <li>{{ item.name }}</li>
</template>
```

```vue
<!-- ❌ 错误：大型数据直接响应式 -->
<script setup>
const largeData = ref(fetchLargeData())  // 创建大量响应式代理
</script>

<!-- ✅ 正确：用 markRaw -->
<script setup>
const largeData = markRaw(fetchLargeData())
</script>
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 首屏优化 | 路由懒加载 | 减少初始加载体积 |
| 长列表渲染 | 虚拟列表 | 只渲染可见区域 |
| 频繁切换 | v-show | CSS 切换开销低 |
| 不频繁切换 | v-if | 销毁节省内存 |
| 大型第三方对象 | markRaw | 避免响应式开销 |
| 只需替换的对象 | shallowRef | 浅层响应式 |

## L3 专家层：深入

### 响应式原理与性能

```
Vue 响应式工作流程：

ref(obj) → 递归遍历对象所有属性
    ↓
为每个属性创建 getter/setter 代理
    ↓
读取属性 → 收集依赖
    ↓
修改属性 → 触发依赖更新

性能开销：
- 对象越深，代理创建越慢
- 属性越多，依赖追踪越多
- shallowRef 只代理 .value，不递归
- markRaw 完全不创建代理
```

### 知识关联

```
性能优化关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  响应式优化 │────→│  渲染优化   │────→│  加载优化   │
│  shallowRef/│     │  computed/  │     │  懒加载/    │
│  markRaw    │     │  v-if/v-show│     │  虚拟列表   │
└─────────────┘     └─────────────┘     └─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **shallowRef** | 浅层响应式，只追踪 .value | `shallowRef({ nested: data })` |
| **markRaw** | 标记为非响应式对象 | `markRaw(largeData)` |
| **懒加载** | 按需加载组件代码 | `() => import('./About.vue')` |
| **虚拟列表** | 只渲染可见区域的列表项 | `vue-virtual-scroller` |
| **computed 缓存** | 依赖不变时不重新计算 | `computed(() => expensive())` |

<script setup>
import { shallowRef } from 'vue'

// 10000 条数据，不用 markRaw 会慢

      {{ item.name }}
    </div>
  </div>

import { ref, computed } from 'vue'

const items = ref([{ id: 1, name: 'Apple' }, { id: 2, name: 'Banana' }])
const query = ref('')

// 使用 computed 缓存过滤结果
const filteredItems = computed(() =>
  items.value.filter(item => item.name.toLowerCase().includes(query.value.toLowerCase()))
)
</script>

## 常见问题

### Q1：shallowRef 和 ref 有什么区别？

**ref 递归响应式（深），shallowRef 只响应 .value 变化（浅）：**

```vue
const obj = ref({ nested: { value: 1 } })
obj.value.nested.value = 2  // ✅ 触发更新

const obj = shallowRef({ nested: { value: 1 } })
obj.value.nested.value = 2  // ❌ 不触发更新
obj.value = { nested: { value: 2 } }  // ✅ 触发更新
```

### Q2：v-if 和 v-show 什么时候用哪个？

**频繁切换用 v-show（CSS 切换），不频繁用 v-if（销毁/创建）：**

```vue
<!-- 频繁切换 → v-show -->
<div v-show="showModal">弹窗内容</div>

<!-- 不频繁切换 → v-if -->
<div v-if="isLoggedIn">欢迎回来</div>
```

### Q3：什么时候用 markRaw？

**第三方库实例、大型数据、不需要响应式的对象：**

```vue
// ✅ 图表实例
import { Chart } from 'chart.js'
const chart = markRaw(new Chart(ctx, { ... }))

// ✅ 大型只读数据
const largeData = markRaw(fetchLargeDataset())
```
