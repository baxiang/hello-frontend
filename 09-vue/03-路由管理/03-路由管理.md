## L1 理解层：会用

### 路由配置

**最简示例（1-3行）：**

```typescript
import { createRouter, createWebHistory } from 'vue-router'
const router = createRouter({
  history: createWebHistory(),
  routes: [{ path: '/', component: () => import('./Home.vue') }]
})
```

**详细示例：**

```typescript
// router/index.ts — 路由配置
import { createRouter, createWebHistory } from 'vue-router'
import Home from '../views/Home.vue'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    { path: '/', name: 'Home', component: Home },
    { path: '/about', name: 'About', component: () => import('../views/About.vue') },
    { path: '/posts/:id', name: 'Post', component: () => import('../views/PostDetail.vue') },
    { path: '/:pathMatch(.*)*', name: 'NotFound', component: () => import('../views/NotFound.vue') }
  ]
})

export default router
```

```vue
<!-- App.vue — 路由入口 -->
<script setup>
import { useRoute, useRouter } from 'vue-router'
const route = useRoute()
const router = useRouter()
</script>

<template>
  <nav>
    <router-link to="/">首页</router-link>
    <router-link to="/about">关于</router-link>
    <router-link :to="{ name: 'Post', params: { id: 1 } }">文章</router-link>
  </nav>
  <router-view />
</template>
```

### 动态路由与嵌套路由

```vue
<!-- views/PostDetail.vue — 动态路由 -->
<script setup>
import { useRoute } from 'vue-router'
const route = useRoute()
const id = route.params.id  // 获取 URL 中的 :id

// 查询参数
const page = route.query.page  // ?page=2
</script>

<template>
  <article>
    <h1>文章 {{ id }}</h1>
    <p>第 {{ page }} 页</p>
  </article>
</template>
```

```typescript
// 嵌套路由配置
const routes = [
  {
    path: '/dashboard',
    component: Dashboard,
    children: [
      { path: 'overview', component: Overview },    // /dashboard/overview
      { path: 'settings', component: Settings },     // /dashboard/settings
      { path: '', redirect: 'overview' }             // /dashboard → /dashboard/overview
    ]
  }
]
```

## L2 实践层：用好

### 路由 API 速查

| API | 用途 | 示例 |
|------|------|------|
| `<router-link>` | 声明式导航 | `<router-link to="/about">` |
| `router.push()` | 编程式导航 | `router.push('/about')` |
| `router.replace()` | 替换当前页（不留历史） | `router.replace('/login')` |
| `router.back()` | 后退 | `router.back()` |
| `useRoute()` | 获取路由信息 | `route.params.id` |
| `useRouter()` | 获取路由实例 | `router.push('/')` |

### 导航守卫

```typescript
// router/index.ts

// 全局前置守卫 — 所有路由访问前执行
router.beforeEach((to, from) => {
  const token = localStorage.getItem('token')
  if (to.meta.requiresAuth && !token) {
    return { name: 'Login', query: { redirect: to.fullPath } }
  }
})

// 路由独享守卫
const routes = [
  {
    path: '/dashboard',
    component: Dashboard,
    meta: { requiresAuth: true },
    beforeEnter: (to, from) => {
      if (!isAuthenticated()) return { name: 'Login' }
    }
  }
]
```

### 反模式：不要这样做

```vue
<!-- ❌ 错误：用 a 标签导航（会全页刷新） -->
<a href="/about">关于</a>

<!-- ✅ 正确：用 router-link（客户端导航） -->
<router-link to="/about">关于</router-link>
```

```vue
<!-- ❌ 错误：在路由配置中同步导入所有组件 -->
import Home from './views/Home.vue'
import About from './views/About.vue'
import Dashboard from './views/Dashboard.vue'
// 首屏加载所有组件代码

<!-- ✅ 正确：路由懒加载 -->
{ path: '/', component: () => import('./views/Home.vue') }
{ path: '/about', component: () => import('./views/About.vue') }
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 页面跳转 | `<router-link>` | 客户端导航 |
| 表单提交后跳转 | `router.push()` | 编程式控制 |
| 权限拦截 | `beforeEach` 守卫 | 统一处理 |
| 长列表页面 | 路由懒加载 | 减少首屏体积 |
| 面包屑导航 | `route.matched` | 获取路由层级 |

## L3 专家层：深入

### 路由渲染原理

```
Vue Router 渲染流程：

用户点击 router-link → 阻止默认行为
    ↓
更新 history 状态（pushState）
    ↓
触发 URL 变化
    ↓
匹配路由配置
    ↓
渲染 router-view 中的组件
    ↓
不刷新页面，只更新变化的组件
```

### 知识关联

```
路由管理关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  路由配置   │────→│  导航与参数 │────→│  导航守卫   │
│  routes/    │     │  push/      │     │  beforeEach/│
│  嵌套/      │     │  replace/   │     │  beforeEnter│
│  懒加载     │     │  params     │     │  权限控制   │
└─────────────┘     └─────────────┘     └─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **Vue Router** | Vue 官方路由管理库 | `createRouter` |
| **动态路由** | 路径中带参数的路由 | `path: '/posts/:id'` |
| **嵌套路由** | 路由内部再嵌套子路由 | `children: [...]` |
| **导航守卫** | 拦截路由访问的钩子 | `beforeEach`, `beforeEnter` |
| **路由懒加载** | 按需加载组件代码 | `() => import('./About.vue')` |

## 实践练习

```typescript
// 练习：实现带权限的博客路由系统
import { createRouter, createWebHistory } from 'vue-router'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    { path: '/', name: 'Home', component: () => import('../views/Home.vue') },
    { path: '/login', name: 'Login', component: () => import('../views/Login.vue') },
    {
      path: '/dashboard',
      name: 'Dashboard',
      component: () => import('../views/Dashboard.vue'),
      meta: { requiresAuth: true },
      children: [
        { path: '', redirect: 'overview' },
        { path: 'overview', component: () => import('../views/Overview.vue') },
        { path: 'settings', component: () => import('../views/Settings.vue') }
      ]
    },
    { path: '/:pathMatch(.*)*', name: 'NotFound', component: () => import('../views/NotFound.vue') }
  ]
})

// 全局守卫
router.beforeEach((to) => {
  const token = localStorage.getItem('token')
  if (to.meta.requiresAuth && !token) {
    return { name: 'Login', query: { redirect: to.fullPath } }
  }
})

export default router
```

## 常见问题

### Q1：router-link 和 a 标签有什么区别？

**router-link 是客户端导航，不刷新页面；a 标签会全页刷新：**

```vue
<router-link to="/about">关于</router-link>  <!-- ✅ 客户端导航 -->
<a href="/about">关于</a>                    <!-- ❌ 全页刷新 -->
```

### Q2：router.push 和 router.replace 有什么区别？

**push 会保留历史记录（可后退），replace 替换当前页（不留历史）：**

```typescript
router.push('/about')    // 可以 router.back() 回来
router.replace('/login') // 不能 back 到之前的页面
```

### Q3：如何实现 404 页面？

**用通配符路由匹配所有未定义的路径：**

```typescript
{ path: '/:pathMatch(.*)*', name: 'NotFound', component: NotFound }
```
