**Pinia 就像"公司的共享信息中心"**：

```
比喻对应：

┌──────────────────────────────────────────────────────┐
│                  共享信息中心                         │
├──────────────────────────────────────────────────────┤
│                                                      │
│    state = 公告板内容                                │
│    ─────────────                                     │
│    公告板上贴的数据：用户信息、购物车、设置          │
```

## L1 理解层：会用

### Pinia 基础

**最简示例（1-3行）：**

```typescript
import { defineStore } from 'pinia'
export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0 }),
  actions: { increment() { this.count++ } }
})
```

**详细示例：**

```typescript
// stores/user.ts
import { defineStore } from 'pinia'

interface User { id: number; name: string }

export const useUserStore = defineStore('user', {
  // state — 数据（就像组件的 reactive）
  state: () => ({
    user: null as User | null,
    token: localStorage.getItem('token') || ''
  }),

  // getters — 计算属性（就像 computed）
  getters: {
    isLoggedIn: (state) => !!state.token,
    userName: (state) => state.user?.name || '访客'
  },

  // actions — 方法（就像 methods，支持 async）
  actions: {
    async login(email: string, password: string) {
      const res = await fetch('/api/login', {
        method: 'POST',
        body: JSON.stringify({ email, password })
      })
      const data = await res.json()
      this.token = data.token
      this.user = data.user
      localStorage.setItem('token', data.token)
    },

    logout() {
      this.user = null
      this.token = ''
      localStorage.removeItem('token')
    }
  }
})
```

```vue
<!-- 组件中使用 -->
<script setup>
import { useUserStore } from '@/stores/user'
const userStore = useUserStore()
</script>

<template>
  <div>
    <p v-if="userStore.isLoggedIn">欢迎，{{ userStore.userName }}</p>
    <button v-else @click="userStore.login('test@test.com', '123')">登录</button>
    <button @click="userStore.logout">退出</button>
  </div>
</template>
```

## L2 实践层：用好

### Pinia vs Vuex

| 特性 | Pinia | Vuex |
|------|-------|------|
| TypeScript | ✅ 原生支持 | ⚠️ 需要额外配置 |
| 模块化 | 每个 store 独立 | 单一 store + modules |
| DevTools | ✅ | ✅ |
| 体积 | 更小（1KB） | 较大 |
| 官方推荐 | ✅ Vue 3 默认 | ❌ 已不再维护 |

### Store 组织

```
stores/
├── user.ts       # 用户状态
├── cart.ts       # 购物车状态
└── settings.ts   # 设置状态
```

### 反模式：不要这样做

```typescript
// ❌ 错误：直接修改 state 而不通过 action
store.user = { id: 1, name: '张三' }  // 绕过业务逻辑

// ✅ 正确：通过 action 修改
await store.login('user@example.com', 'password')
```

```typescript
// ❌ 错误：把所有状态放在一个 store 里
export const useGlobalStore = defineStore('global', {
  state: () => ({ user, cart, settings, theme, lang, ... })
})

// ✅ 正确：按功能拆分 store
export const useUserStore = defineStore('user', { ... })
export const useCartStore = defineStore('cart', { ... })
```

### 适用场景

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 组件内部状态 | `ref`/`reactive` | 无需共享 |
| 父子组件通信 | `props`/`emit` | 直接传递 |
| 全局用户信息 | Pinia | 多组件共享 |
| 购物车状态 | Pinia | 跨页面共享 |
| 主题/语言设置 | Pinia | 全局生效 |

## L3 专家层：深入

### Pinia 响应式原理

```
Pinia 工作流程：

组件调用 useUserStore()
    ↓
创建 reactive 对象（state）
    ↓
组件读取 state → 建立依赖关系
    ↓
action 修改 state
    ↓
所有依赖的组件自动重新渲染
    ↓
getters 自动重新计算
```

### 知识关联

```
状态管理关联：

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  组件状态   │────→│  全局状态   │────→│  状态持久化 │
│  ref/       │     │  Pinia/     │     │  localStorage/│
│  reactive   │     │  state/     │     │  IndexedDB   │
│             │     │  actions    │     └─────────────┘
└─────────────┘     └─────────────┘
```

## 术语定义

| 术语 | 定义 | 示例 |
|------|------|------|
| **Store** | Pinia 的状态容器 | `defineStore('user', {...})` |
| **state** | 响应式数据 | `state: () => ({ count: 0 })` |
| **getters** | 计算属性 | `getters: { double: s => s.count * 2 }` |
| **actions** | 状态变更方法 | `actions: { increment() { this.count++ } }` |
| **defineStore** | 定义 store 的函数 | `defineStore('name', options)` |

## 实践练习

```typescript
// 练习 1：实现购物车 store
import { defineStore } from 'pinia'

interface CartItem { id: number; name: string; price: number; quantity: number }

export const useCartStore = defineStore('cart', {
  state: () => ({ items: [] as CartItem[] }),

  getters: {
    totalItems: (state) => state.items.reduce((sum, item) => sum + item.quantity, 0),
    totalPrice: (state) => state.items.reduce((sum, item) => sum + item.price * item.quantity, 0)
  },

  actions: {
    addItem(item: Omit<CartItem, 'quantity'>) {
      const existing = this.items.find(i => i.id === item.id)
      if (existing) existing.quantity++
      else this.items.push({ ...item, quantity: 1 })
    },

    removeItem(id: number) {
      this.items = this.items.filter(i => i.id !== id)
    }
  }
})
```

## 常见问题

### Q1：Pinia 和 Vuex 选哪个？

**新项目一律用 Pinia（Vue 3 默认），老项目迁移时再考虑 Vuex：**

```typescript
// Pinia — 推荐
export const useUserStore = defineStore('user', { ... })

// Vuex — 不再维护
export default new Vuex.Store({ state: { ... } })
```

### Q2：Pinia 的状态刷新后会丢失吗？

**会。需要手动持久化到 localStorage：**

```typescript
// 在 action 中持久化
actions: {
  login(user: User) {
    this.user = user
    localStorage.setItem('user', JSON.stringify(user))
  }
}

// 或使用 pinia-plugin-persistedstate 插件
```

### Q3：多个 store 之间怎么通信？

**直接导入其他 store，在 action 中使用：**

```typescript
// stores/order.ts
import { useCartStore } from './cart'

export const useOrderStore = defineStore('order', {
  actions: {
    checkout() {
      const cart = useCartStore()
      // 使用 cart.items 创建订单
      cart.items = []  // 清空购物车
    }
  }
})
```
